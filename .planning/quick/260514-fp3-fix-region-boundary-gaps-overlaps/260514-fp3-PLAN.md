# Plan: Fix region boundary gaps and overlaps (#14)

## Problem

User reports visible gaps and overlaps between adjacent county and ecoregion
polygons at zoom levels as low as 8. Specific examples: three-way intersection
of Okanogan/Chelan/Skagit counties; ecoregion borders near McClure Mountain.

## Root cause (after investigation pivot)

The original hypothesis — "per-feature `ST_SimplifyPreserveTopology` in dbt is
cutting shared borders inconsistently" — was *wrong*. The user-visible
overlaps come from two distinct upstream-source issues:

1. **TIGER 2024 `tl_2024_us_county.zip` has 192 km² of overlap polygons across
   71 WA county-pair adjacencies.** Worst offenders: Chelan ∩ Skagit (4.7 km²
   in 6 sliver polygons), Okanogan ∩ Chelan (3.5 km²), and many similar. Not a
   simplification artifact — measurable directly in the raw shapefile via
   `ST_Intersection`.
2. **EPA Level III ecoregions** has 160 km² of overlap across 16 ecoregion-pair
   adjacencies in WA. Columbia Plateau ∩ Blue Mountains alone is 33 km².
   Source-data issue, identical to (1).

The pre-fix dbt pipeline's `ST_SimplifyPreserveTopology(geom, 0.001)` was
incidental: for counties at this tolerance it removed zero vertices, and for
ecoregions it removed only ~14 vertices total. The simplification step was
neither causing nor hiding the bug.

## Fix

### Counties: switch the data source

`tl_2024_us_county.zip` (TIGER/Line) → `cb_2024_us_county_5m.zip` (Census
Cartographic Boundary 1:5M). Same authority, built from the topology DB rather
than the per-state-export pipeline that produces the overlaps in tl_. Result:
0 km² overlap (verified end-to-end via `ST_Intersection` on the final output).
3.4× more vertices than tl_ (4,490 vs 1,305), still tiny in absolute terms.

### Ecoregions: post-hoc topology cleanup

No cleaner authoritative source exists for EPA L3 ecoregions. `mapshaper
-clean` resolves the source's overlap polygons by absorbing slivers below a
configurable area threshold; `-simplify` then reduces vertex count
topology-aware (shared arcs simplified once, no re-introduction of gaps).
Verified: 0 km² overlap in final output, all 8 distinct WA L3 ecoregions
preserved. Two sub-1-hectare uninhabited islands in the 54-polygon "Strait of
Georgia/Puget Lowland" group are dropped — confirmed acceptable with user.

Additional change: clip ecoregion polygons to WA in the dbt mart (the EPA
features extend to BC/OR — Cascades alone spans 10° of latitude). Before clip,
raw ecoregions GeoJSON is 6 MB; after clip + mapshaper it's 193 KB (matches
pre-fix size, lazy-loaded so no UX impact).

### Drop the per-feature dbt simplification

`ST_SimplifyPreserveTopology(geom, 0.001)` retired from
`data/dbt/macros/emit_feature_collection.sql`. mapshaper now owns all
simplification, and CB 5m counties are already at cartographic scale.

## Changes

- `data/geographies_pipeline.py`: switch `SOURCES["us_counties"]` URL +
  shapefile basename to CB 5m. Update docstring.
- `data/dbt/macros/emit_feature_collection.sql`: drop ST_SimplifyPreserveTopology.
- `data/dbt/models/marts/counties_geo.sql`: comment updates only.
- `data/dbt/models/marts/ecoregions_geo.sql`: clip via `ST_Intersection(e.geom,
  wa.geom)` against `stg_geo__us_states WHERE abbreviation='WA'`. Comment
  updates.
- `data/topology_postprocess.py` (new): runs `mapshaper -clean
  gap-fill-area=0.01km2 -simplify percentage=N% planar keep-shapes` on both
  GeoJSONs in `public/data/`. Per-layer simplify percentage (counties 10%,
  ecoregions 3%) tuned to match pre-fix file sizes.
- `data/run.py`: new STEPS entry `("topology-postprocess", ...)` after dbt-build.
- `data/tests/test_dbt_diff.py`: update county spatial-diff baseline from 84
  (TIGER nondeterminism) to 0 (CB 5m clean topology).
- `package.json`: add `mapshaper` devDependency.

## Verification

- `bash data/dbt/run.sh build`: PASS=44.
- `uv run pytest tests/test_dbt_scaffold.py tests/test_dbt_diff.py`: 22 passed.
- Programmatic topology check (DuckDB `ST_Intersection` → polygon-area sum)
  on final output: 0 km² overlap on counties, 0 km² on ecoregions.
- Visual check via Playwright at the user's two reported locations:
  - Counties at -120.74, 48.59, zoom 10 — three-way Okanogan/Chelan/Skagit
    meeting reads as clean line junction.
  - Ecoregions at -120.75, 48.45, zoom 10 — McClure Mountain area boundary
    runs continuously, no visible gaps.

## Out of scope

- Switching public data format to TopoJSON (option C from discussion).
- Optimizing region GeoJSON file size further (current 22 KB / 193 KB matches
  the pre-fix sizes; lazy-loaded so no UX cost).
- The 54 "Strait of Georgia/Puget Lowland" sub-polygons (separate Puget Sound
  islands) keeping the same NA_L3NAME — different bug, separate UX concern.

## Concerns about the change

- Adds Node.js + mapshaper to the data pipeline (was Python-only). Node
  toolchain already exists at repo root for frontend, so not a new *kind* of
  dependency, but it does mean `npm install` is now required to run the data
  pipeline.
- Spatial assignment of bee occurrences to counties is now stable (zero
  boundary-nondeterminism rows). Pre-fix, 84 occurrences flipped county
  arbitrarily between runs depending on which TIGER overlap polygon ST_Within
  returned first.
