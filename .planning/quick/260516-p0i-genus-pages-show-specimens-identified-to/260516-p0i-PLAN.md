# Quick Task 260516-p0i

## Goal

Add a "Genus sp." entry (grey swatch, no link) to the species key on genus and subgenus pages when genus-level occurrence records exist.

## Root Cause

`_data/species.js` builds `species` arrays filtering to `specific_epithet !== null`, so genus-level records appear as grey dots in the SVG map but have no corresponding key entry.

## Tasks

### T1: Add grey "sp." entry to genusList and subgenusList in _data/species.js

**File:** `_data/species.js`

In `genusList.map(...)` (around line 125-128):
- Compute `unresolvedMembers` and `unresolvedOccurrences` from `withOcc` members where `specific_epithet === null`
- Keep `speciesCount` = count of resolved species only
- If `unresolvedOccurrences > 0`, append `{ scientificName: '${g.genus} sp.', hexColor: '#aaaaaa', occurrence_count: unresolvedOccurrences, slug: null }` to `species`
- Fix `totalOccurrences` to include unresolved occurrences (currently excludes them, inconsistent with subgenusList)

Same change for `subgenusList.map(...)` (around line 175-178).

### T2: Conditionally render link vs plain text in genus.njk and subgenus.njk

**Files:** `_pages/genus.njk`, `_pages/subgenus.njk`

Change:
```html
<a href="/species/{{ sp.slug }}/index.html"><em>{{ sp.scientificName }}</em></a>
```

To:
```html
{%- if sp.slug %}<a href="/species/{{ sp.slug }}/index.html"><em>{{ sp.scientificName }}</em></a>{%- else %}<em>{{ sp.scientificName }}</em>{%- endif %}
```
