# Quick Task 260422-sc1: Fix Specimen Count Mismatch

## Root Cause

`queryVisibleIds` adds both `ecdysis:N` and `inat:N` for each occurrence row that has both IDs.
This makes `_visibleIds.size` ≥ 2× the row count for specimens with both IDs.

The filter panel uses `_visibleIds?.size` as `specimenCount`.
The table uses `COUNT(*)` from `queryTablePage`.

Result: map filter panel count > table count for the same filter.

## Tasks

### Task 1: Add rowCount to queryVisibleIds return value

**File:** `frontend/src/filter.ts`
**Action:** Change return type of `queryVisibleIds` from `Set<string> | null` to
`{ ids: Set<string>; rowCount: number } | null`. Track row count by incrementing
a counter in the row callback.

**Verify:** TypeScript compiles; return type updated.
**Done:** `queryVisibleIds` returns `{ ids, rowCount }` or `null`.

### Task 2: Thread rowCount through bee-atlas.ts

**File:** `frontend/src/bee-atlas.ts`
**Action:**
1. Add `@state() private _filteredRowCount: number | null = null;`
2. In `_runFilterQuery`: destructure `result?.ids`/`result?.rowCount`; assign both states.
3. Wherever `_visibleIds = null` is set (lines ~312, ~527), also set `_filteredRowCount = null`.
4. Change line 199 specimenCount binding to use `_filteredRowCount` instead of `_visibleIds?.size`.

**Verify:** Filter panel shows same count as table view when filtering by county/ecoregion.
**Done:** `specimenCount` prop sourced from `_filteredRowCount`.
