# Plan: Fix mobile sidebar close button obscured by Regions button

## Problem

GitHub issue #12: On narrow viewports (`max-aspect-ratio: 1`), when the sidebar is
open, the Regions button (`.region-control` inside `bee-map`, absolutely positioned
at `top: 0.5em; right: 0.5em`) visually overlaps the sidebar's close button.

Root cause: in the `@media (max-aspect-ratio: 1)` block (`src/bee-atlas.ts:132-146`),
both `bee-map` and `bee-sidebar` have `flex-grow: 1`. Default `min-height: auto` for
flex items lets the sidebar's min-content height push `bee-map` down to near-zero
height. The Regions button then lands on top of the sidebar's `.sidebar-header`
(close button).

## Fix

Preserve a "sliver of map" when the sidebar is open on narrow viewports so the
Regions button keeps its own real estate. When the sidebar is closed (mobile,
no selection), `bee-map` should still fill the screen.

### Changes â€” `src/bee-atlas.ts`

1. In `render()`: when `_viewMode === 'map' && _sidebarOpen`, add a
   `sidebar-open` modifier to the `.content` div class string.

2. In the `static styles` `@media (max-aspect-ratio: 1)` block:
   - Add `min-height: 0` to `bee-sidebar` so its min-content doesn't push the
     map below the desired sliver.
   - Add a `.content.sidebar-open bee-map` rule that pins the map to a small
     fixed height (the sliver) and disables flex-grow / flex-shrink. Use the
     same shape as the existing `.content.table-mode bee-map` rule for
     consistency.

The sliver height should comfortably contain the Regions button (~2rem total
including padding). Use `3.5rem` to match the existing `top: calc(0.5em +
2.5rem + 2.5rem + 0.5em)` rhythm used for the absolute sidebar layout.
