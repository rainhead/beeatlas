<task type="auto">
  <name>Task 1: Add modified column to ecdysis parquet export and schema validation</name>
  <files>data/export.py, scripts/validate-schema.mjs</files>
  <action>
In `data/export.py`, in `export_ecdysis_parquet()`:

1. Add a CTE to compute the max identification modified date per occurrence:
```sql
id_modified AS (
    SELECT coreid, MAX(modified) AS max_id_modified
    FROM ecdysis_data.identifications
    GROUP BY coreid
)
```

2. LEFT JOIN this CTE in the final SELECT:
```sql
LEFT JOIN id_modified im ON im.coreid = o.id
```

3. Add to the SELECT list:
```sql
strftime(GREATEST(o.modified, im.max_id_modified), '%Y-%m-%d') AS modified
```
Use `strftime` to format as a date string (consistent with the `date` column pattern). `GREATEST` handles NULL gracefully in DuckDB -- if one side is NULL it returns the other.

Place the new column at the end of the SELECT list (after `inat_quality_grade`).

In `scripts/validate-schema.mjs`:

Add `'modified'` to the end of the `EXPECTED['ecdysis.parquet']` array.
  </action>
