# Requirements: Washington Bee Atlas v4.0

**Defined:** 2026-05-23
**Core Value:** Tighten learning cycles for volunteer collectors — surface existing data in ways difficult to achieve without the site; convey liveness and togetherness among participants.

## v4.0 Requirements

### TAX — Offline Taxonomy

- [x] **TAX-01**: Pipeline downloads `taxa.csv.gz` from iNat AWS Open Data with ETag/Last-Modified caching; no re-download when archive is unchanged — Phase 110 ✓
- [x] **TAX-02**: DuckDB ancestry walk via `unnest(string_split(ancestry,'/'))` produces `taxon_lineage_extended` with identical schema to current (family, subfamily, tribe, genus, subgenus per taxon_id) — Phase 110 ✓
- [x] **TAX-03**: Live `/v2/taxa` enricher functions removed from pipeline; `dbt build` and `npm test` pass after deletion — Phase 110 ✓
- [x] **TAX-04**: Taxa archive cached at `data/raw/taxa.csv.gz`; synced to/from S3 by `nightly.sh` to persist across nightly runs — Phase 110 ✓

### CHECK — Checklist Pipeline

- [x] **CHECK-01**: Pipeline reads committed checklist CSV, parses specific epithet from Scientific Name (strips author+year), normalizes date formats, applies TRIM() to varchar fields, spatial-joins county and ecoregion_l3 — Phase 111 ✓
- [x] **CHECK-02**: `checklist.parquet` produced with columns: `canonical_name, scientificName, genus, specific_epithet, family, lat (nullable), lon (nullable), year (nullable), month (nullable), county, ecoregion_l3, source='checklist'` — Phase 111 ✓
- [x] **CHECK-03**: `checklist.parquet` uploaded to S3/CloudFront as part of nightly pipeline export — Phase 111 ✓
- [x] **CHECK-04**: Pytest assertions pass: row count ≥ 2000, no null `canonical_name`, no null `specific_epithet`, `TRIM(family) = family` — Phase 111 ✓

### MAP — Checklist Map Layer

- [x] **MAP-01**: "Checklist records" toggle appears in filter panel alongside Specimens and Samples toggles — Phase 112 ✓
- [x] **MAP-02**: When enabled, checklist records render as a county-fill overlay (green fill on counties GeoJSON source), visually distinct from WABA specimen points; county presence derived from checklist.parquet coordinates — Phase 112 ✓
- [x] **MAP-03**: Checklist layer responds to taxon filter only; year, month, and collector filters have no effect on the checklist layer — Phase 112 ✓ (UAT confirmed year filter also applies — desired behavior)
- [x] **MAP-04**: `cl=1` URL param encodes checklist layer visibility; restored on page load — Phase 112 ✓

### SPEC — Species Page Expansion

- [x] **SPEC-01**: All 565 checklist species appear in the species index and have dedicated taxon pages, including species with zero WABA records — Phase 113 ✓
- [x] **SPEC-02**: Checklist-only species appear on genus and subgenus pages alongside WABA species — Phase 113 ✓
- [x] **SPEC-03**: Species pages show occurrence map including checklist record points (visually distinct from WABA occurrence points) — Phase 113 ✓
- [x] **SPEC-04**: Species pages show attribution for checklist records: "N checklist records · Bartholomew et al. 2024" — Phase 113 ✓
- [x] **SPEC-05**: Seasonality histogram draws from all sources (WABA + checklist); suppressed only when the species has zero records from any source — Phase 113 ✓

### EXT — Extensibility

- [x] **EXT-01**: `source='checklist'` column present in `checklist.parquet`; pipeline architecture documented to support future sources (other Bee Atlas programs, GBIF) as additional parquet files with the same `source` field convention — Phase 111 ✓
