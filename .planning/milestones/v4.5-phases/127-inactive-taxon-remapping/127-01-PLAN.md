<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| iNat API -> Python | `GET /v1/taxa/{id}` returns `current_synonymous_taxon_ids` and successor records — untrusted external input flowing into a SQL UPSERT against the bridge and into a CSV seed |
| taxa.csv.gz -> Python | Monthly iNat open-data dump (gitignored, S3-synced) — successor name lookup source; untrusted text flows into auto_synonyms.csv |
| inactive_unresolved.csv -> gate | The gate's authority to block the build depends on this file being authored only by generate_inactive_remaps() |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-127-01 | Tampering | iNat successor name -> bridge UPSERT and auto_synonyms.csv | mitigate | All DuckDB writes use parameterized queries (`?` placeholders, never string interpolation of API values) per the _resolve_one shape; csv.writer/DictWriter escapes field content. Successor names are lower().strip() normalized, never concatenated into SQL. |
| T-127-02 | Tampering | Malformed/unexpected `/v1/taxa/{id}` response shape | mitigate | Normalize `current_synonymous_taxon_ids` with `or []` (handles None and []); guard empty `results` list; wrap fetch in try/except requests.HTTPError routing to api_error triage — a malformed response degrades to a blocking triage row, never a crash or silent skip (RD-02, Pitfall 3). |
| T-127-03 | Elevation of Privilege (bypass) | inactive-gate silently bypassed | mitigate | check_inactive_gate() sys.exit(non-zero) on ANY row in inactive_unresolved.csv — no acknowledged-exclusion set (D-07); generate_inactive_remaps() always overwrites the triage CSV each run so a stale empty file cannot mask new offenders; gate runs as a dedicated STEP before dbt-build. |
| T-127-04 | Denial of Service | iNat rate-limit / hang on per-inactive fetch | accept | Reuse `_inat_get_with_retry` + `_INAT_PACE_SECONDS` (handles 429/5xx + Retry-After) with timeout=30; inactive taxa are rare (0 today) so fetch volume is negligible. Low-value dormant path. |
| T-127-05 | Tampering | SQL injection via taxa_path f-string in read_csv | accept | taxa_path is a code-derived `Path(__file__).parent` join, not user/API input; matches the existing verified enumeration-query pattern. No external value reaches the f-string. |
| T-127-SC | Tampering | npm/pip/cargo installs | mitigate | No new packages introduced (RESEARCH Package Legitimacy Audit: not applicable). No install tasks in this plan. |
</threat_model>
