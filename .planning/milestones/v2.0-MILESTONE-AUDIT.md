## Critical Gaps

### TABLE-05: Column Sort — Completely Unimplemented

The sort-by-column-header feature (TABLE-05) was planned in Phase 40 but **never implemented**:

- `bee-table.ts`: No `sortColumn`/`sortDir` `@property` declarations; no clickable column headers; no `sort-changed` CustomEvent
- `bee-atlas.ts`: No `_sortColumn`/`_sortDir` `@state` fields; no `_onSortChanged` handler
- `url-state.ts`: No sort parameter encoding or decoding
- `filter.ts` `queryTablePage`: No sort arguments; uses fixed `SPECIMEN_ORDER`/`SAMPLE_ORDER` constants

The Phase 40 VERIFICATION.md claimed SATISFIED based on code that does not exist — a false positive. The Phase 40 SUMMARY (40-02-SUMMARY.md) describes the planned sort implementation in detail but the code was never written.

**Impact:** Users cannot change the sort order of the table. All rows appear in a fixed date-descending order.

### TABLE-06: Date Column vs. Separate Year/Month Columns

Requirement specifies specimen columns include separate **year** and **month** columns. Shipped implementation uses a single **Date** column (pre-formatted `YYYY-MM-DD` string from the `date` field).

The `SpecimenRow` type in `filter.ts` includes `year: number` and `month: number` as separate fields — the data is available but `SPECIMEN_COLUMN_DEFS` in `bee-table.ts` exposes only the `date` string.

**Impact:** Users see a formatted date string rather than distinct numeric year and month values. Filtering by year/month still works correctly (server-side). This is a display divergence from the spec.
