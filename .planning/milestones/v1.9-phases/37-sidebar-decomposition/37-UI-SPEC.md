## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px (1rem) | 400 (regular) | 1.5 |
| Label / small content | 14px (0.85–0.9rem) | 400 (regular) | 1.5 |
| Section heading | 15px (0.95rem) | 600 (semibold) | 1.2 |
| Micro / metadata | 13px (0.8rem) | 400 (regular) | 1.5 |

Source: Existing `bee-sidebar.ts` static styles (verified). The four sizes map to: dd elements at 1rem, filter inputs and back-btn at 0.9rem, filter section headings and toggle-btn at 0.95rem, event-observer / event-count / month-grid labels at 0.8rem.

Rules:
- Sub-components inherit these sizes verbatim from the existing sidebar CSS.
- Each sub-component's `static styles` uses the same relative rem values already in use.
- No new font sizes are introduced in Phase 37.
- Font weight is either 400 (body/label) or 600 (semibold headings, active tab, sample-header). These are the only two weights used throughout.

## Color

All colors are CSS custom properties defined in `frontend/src/index.css`. Sub-components reference these tokens directly in their `static styles` blocks.

| Role | Token | Hex | Usage |
|------|-------|-----|-------|
| Dominant (60%) | `--surface` | #fafafa | Component host backgrounds, input backgrounds |
| Secondary (30%) | `--surface-subtle` / `--surface-muted` | #f5f5f5 / #f0f0f0 | Section headers (event-date-heading), chip backgrounds, hover states |
| Accent (10%) | `--accent` | #2c7a2c | Active layer-toggle tab underline and text only |
| Destructive | `--error` | #bb0000 | No destructive actions in this phase — token available if needed |

Accent reserved for: the active state of the layer-mode toggle buttons (`.toggle-btn.active`). No other elements use `--accent` in these sub-components.

Text tokens in use:
- `--text-body` (#213547) — primary content
- `--text-secondary` (#444) — section headings, filter labels
- `--text-muted` (#666) — metadata, hint text, button labels
- `--text-hint` (#767676) — inactive toggle tabs, event counts, sample meta; WCAG AA compliant on `--surface`
- `--text-tertiary` (#555) — date-group headings in recent events list

Border tokens in use:
- `--border` (#ddd) — section dividers
- `--border-subtle` (#eee) — list item dividers within detail panels
- `--border-input` (#ccc) — input borders, button borders, chip borders

Source: `frontend/src/index.css` (verified).

## Component Inventory

Three new Lit custom elements are created. Each gets its own file and shadow DOM. CSS is co-located with the component that renders the markup it styles.

### bee-filter-controls

**File:** `frontend/src/bee-filter-controls.ts`
**Custom element tag:** `<bee-filter-controls>`
**Renders:** Taxon input + datalist, year-from/year-to inputs, month checkboxes (4-column grid), county chip list + text input, ecoregion chip list + text input, boundary mode toggle, "Clear all filters" button.
**Interactions:**
- Taxon text input: fires on exact datalist match only; mid-keystroke does not emit
- Year inputs: fire on `change` event
- Month checkboxes: fire on `change` event
- County/ecoregion text inputs: fire on exact option match only
- Chip remove (×) button: removes the item and emits full filter-changed
- "Clear all" button: resets all filter fields to null/empty, emits filter-changed
- Boundary mode radio/toggle: emits filter-changed with updated boundaryMode
**Emits:** `filter-changed` (CustomEvent, bubbles: true, composed: true) with full `FilterChangedEvent` payload on every interaction
**Internal state allowed:** `_taxonInputText`, `_countyInputText`, `_ecoregionInputText` (transient keystroke text only — not filter state)
**CSS to migrate from bee-sidebar:** `.filter-controls`, `.filter-row`, `.year-row`, `.month-grid`, `.taxon-row`, `.taxon-clear-btn`, `.clear-btn`, `.region-chips`, `.chip`, `.chip-type`, `.chip-remove`, boundary toggle styles

### bee-specimen-detail

**File:** `frontend/src/bee-specimen-detail.ts`
**Custom element tag:** `<bee-specimen-detail>`
**Renders:** Back button, list of sample blocks (each with date/collector/field-number header and italic species list with optional iNat links).
**Interactions:**
- "Back" button: emits `close` (CustomEvent, bubbles: true, composed: true, no detail)
**Emits:** `close`
**Internal state:** none
**CSS to migrate from bee-sidebar:** `.back-btn`, `.sample`, `.sample-header`, `.sample-meta`, `.species-list`, `.inat-missing`

### bee-sample-detail

**File:** `frontend/src/bee-sample-detail.ts`
**Custom element tag:** `<bee-sample-detail>`
**Renders:** Back button, sample event detail panel (date, observer, specimen count, iNat link).
**Interactions:**
- "Back" button: emits `close` (CustomEvent, bubbles: true, composed: true, no detail)
**Emits:** `close`
**Internal state:** none
**CSS to migrate from bee-sidebar:** `.back-btn`, `.panel-content`, `.sample-dot-detail`, `.sample-dot-detail-header`, `.event-inat`, `dt`, `dd`, `.hint`

### bee-sidebar (after decomposition)

**Retains:** Layer-mode toggle (`.layer-toggle`, `.toggle-btn`), recent sample events list (`.recent-events`, `.recent-events-header`, `.event-row`, `.event-date`, `.event-date-heading`, `.event-observer`, `.event-count`), summary panel, layout host styles.
**Removes:** All filter input markup, all specimen/sample detail markup, all `restored*` properties, the `updated()` restore logic, and `_dispatchFilterChanged()` handler methods.

## Accessibility Notes

- Chip remove buttons maintain `min-width: 24px; min-height: 24px` touch targets (existing constraint, carry forward to bee-filter-controls).
- `--text-hint` (#767676) on `--surface` (#fafafa) achieves 4.54:1 contrast ratio — WCAG AA minimum for normal text (documented in index.css comment; preserve).
- Month checkbox labels use `display: flex; align-items: center; gap: 0.25rem` for adequate click target area.
- Back button and Clear buttons have no explicit focus style beyond browser default — acceptable for existing component visual bar; do not add or remove focus styles in this phase.
- Icon-only "×" chip remove buttons must carry `aria-label="Remove {item}"` (e.g., `aria-label="Remove Apis mellifera"`) so screen readers announce the action and target. Taxon clear (×) button must carry `aria-label="Clear taxon filter"`. These labels apply to all "×" buttons rendered in bee-filter-controls.
