### Integration Warnings (not blockers)

**W1 — `firstUpdated` bounds branch: sidebar open not synchronous (SEL-06)**
`firstUpdated` for `ids`/`cluster` cases sets `_sidebarOpen=true` synchronously before awaiting data. The bounds case does not — sidebar appears after both `tablesReady` and `queryOccurrencesByBounds` resolve. `_onPopState` bounds path does set `_sidebarOpen=true` synchronously (parity gap). Human smoke-test confirmed sidebar does open correctly; this is a visual latency inconsistency, not a functional failure.

**W2 — Zero-result redraw does not clear stale `sel=` from URL (SEL-05/SEL-07 edge case)**
When a second rectangle yields zero results, `_onSelectionDrawn` sets `_selectionBounds=null` and returns without calling `_pushUrlState()`. A prior `sel=` value in the URL bar remains until the next successful selection or dismiss. Pasting the stale URL would re-run `_restoreBoundsSelection` for the previous (non-empty) area. The explicit requirement text for SEL-07 covers only the dismiss path, not the zero-results redraw path — this is a UX edge case.
