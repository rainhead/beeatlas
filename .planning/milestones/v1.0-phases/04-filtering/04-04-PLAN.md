**Fix 2 — Taxon datalist dropdown selection bug (Issue 2):**

The root cause: when the user clicks an item in the native datalist dropdown, the browser fires an `input` event (then sometimes `change`, sometimes not — browser-dependent). The current `_onTaxonInput` handler only clears the filter if the value is empty; it does NOT apply the filter when the value exactly matches an option label. The `change` event is unreliable for datalist selections in some browsers.

Fix: In `_onTaxonInput`, after setting `this._taxonInput`, check whether the current value exactly matches any `TaxonOption.label` in `this.taxaOptions`. If it does, immediately apply that taxon filter (set `_taxonName`, `_taxonRank`, dispatch). This ensures dropdown selection via `input` event succeeds regardless of whether `change` also fires.

Updated `_onTaxonInput`:
```typescript
private _onTaxonInput(e: Event) {
  const input = e.target as HTMLInputElement;
  const value = input.value;
  this._taxonInput = value;
  if (value === '') {
    this._taxonName = null;
    this._taxonRank = null;
    this._dispatchFilterChanged();
    return;
  }
  // Also resolve on exact label match — catches native datalist dropdown selection
  // (browser fires 'input' reliably; 'change' is unreliable for datalist picks)
  const option = this.taxaOptions.find(o => o.label === value);
  if (option) {
    this._taxonName = option.name;
    this._taxonRank = option.rank;
    this._dispatchFilterChanged();
  }
  // If no exact match, user is mid-keystroke — do not apply a partial filter
}
```

The `_onTaxonChange` handler remains as-is (handles manual completion via keyboard Enter/Tab); it's now redundant for dropdown selection but harmless.
