Two-level skip logic (from research — implement exactly):
```python
# Level 1: already in links.parquet → skip entirely (no cache check, no HTTP)
# Level 2: HTML on disk → parse without HTTP, no rate limit sleep
# Otherwise: enforce rate limit, fetch, cache to disk, parse
```

Rate limit — only before actual HTTP requests:
```python
elapsed = time.monotonic() - last_fetch_time
if elapsed < RATE_LIMIT_SECONDS:
    time.sleep(RATE_LIMIT_SECONDS - elapsed)
# fetch here
last_fetch_time = time.monotonic()
```

Merge pattern (new results win):
```python
combined = (
    pd.concat([existing, new_df], ignore_index=True)
    .drop_duplicates(subset=['occurrenceID'], keep='last')
    .reset_index(drop=True)
)
```

Critical pitfalls (all verified in research):
- WRONG: `occid=occurrence_uuid` — UUID param returns short page with no association-div
- CORRECT: `occid=ecdysis_id` where ecdysis_id is the integer 'id' column from ecdysis.parquet
- WRONG: sleep before every record — wastes 38+ minutes on cache hits
- CORRECT: sleep only immediately before `requests.get()`
- WRONG: no User-Agent header — returns HTTP 403
- CORRECT: always set User-Agent
- WRONG: overwrite links.parquet mid-run on error — lose previously fetched data
- CORRECT: accumulate all results in memory list, write once at end after loop completes

HTML cache key: use integer ecdysis_id as filename (e.g. "5594056.html"), NOT the UUID occurrenceID.
