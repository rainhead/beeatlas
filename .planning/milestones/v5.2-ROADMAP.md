## Phases

- [x] **Phase 160: Overlap-capable place model (many-to-many membership)** — Make a bee occurrence able to belong to *multiple* places. Today `marts/occurrences.sql` assigns a single `place_slug` via `ST_Within` + `DISTINCT ON` (no tiebreak), and `places_validation.py` rejects partially-overlapping place polygons (`ST_Overlaps`) to keep that assignment deterministic — an implementation artifact, not a domain requirement (land management genuinely nests/overlaps). Per the locked 160-CONTEXT decisions (D-01/D-02 supersede the earlier `place_slugs VARCHAR[]` sketch): introduced a normalized `occurrence_places` **bridge mart** (one row per occurrence↔place membership, keyed on a synthetic `occ_id` mirroring `occIdFromRow`), **dropped** the scalar `place_slug` from the occurrences mart (dbt contract 37→36 cols), dropped the overlap-rejection guard, recompute per-place counts (`places_export.py`) + maps (`places_maps.py`) via the bridge (double-count per D-05), and rewrote the frontend place filter (`filter.ts`) to an `EXISTS` membership test + list all member places in occurrence detail (D-04). **Depends on:** v3.7 place data model. **Plans:** 4 plans (4 waves). ## Milestone Summary

**Decisions:**
- One-place-per-occurrence was an implementation artifact, not a requirement → place membership is many-to-many via the `occurrence_places` bridge (occ_id ↔ place_slug); multi-place occurrences double-count per-place by design (160 D-05).
- Reusable place-source curation pattern: fetch authoritative geometry → DuckDB-spatial transform (dissolve / metric-CRS buffer) → `ST_SimplifyPreserveTopology` for browser weight → emit `[[places]]` TOML via the shared `toml_block()` writer. Zero new Python dependencies across both 161 and 162.
- Linear hike features are represented as corridor-buffer polygons (ST_Within can never match a bare LineString); 250 m buffer computed in a metric CRS (UTM 10N) with `always_xy=true`.
- `places.geojson` browser-weight budget (~1 MB) is the binding constraint on geometry fidelity; tolerances ratified per source (161: 0.0005°; 162: 0.0002°). Final size 919,929 bytes after both sources.

**Issues / tech debt:**
- `snoqualmie-pass-to-olallie-meadow-trail` deferred (OSM only has the full PCT Section J) — re-add with a hand-traced GPX in a future pass.
- Phase 162 code-review residual (INFO): the OSM relation/way geometry-assembly paths still lack a direct regression test (WR-02/WR-03 fixes verified by reasoning, not an executing test). Add a fixture test before expanding the hike set.
- Backlog grew: 999.10 (sidebar list ignores `src=` source filter), 999.11 (federal wilderness areas as regions).

Production is unaffected (the nightly pipeline rebuilds it).
