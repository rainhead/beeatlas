## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | none — this phase is display-only, no actions |
| Elevation label | `Elevation` |
| Elevation value format | `{integer} m` — e.g. `1219 m` (no decimal, no thousands separator, space before unit) |
| Null elevation (omission) | Row is absent entirely — no placeholder, no dash, no "—" |
| Empty state heading | not applicable — elevation row absence is silent |
| Error state | not applicable — null elevation silently omits the row |
| Destructive confirmation | none — no destructive actions in this phase |

**Format rule (from ELEV-05/ELEV-06):** `Math.round(elevation_m)` rendered as a plain integer
string, then the literal string " m" appended. Do NOT use `toFixed()`, do NOT display decimals.

## Component Inventory

### bee-specimen-detail

**Current pattern:** Each sample renders a `.sample` div containing `.sample-header`,
`.sample-meta`, and a `<ul class="species-list">`. The sample-level metadata (date, collector,
field number) displays before the species list.

**Elevation placement:** Add a `.sample-meta` line immediately after the existing `.sample-meta`
line, only when `sample.elevation_m` is non-null. This keeps elevation adjacent to the other
sample-level facts (date, collector).

**Markup pattern:**
```html
<!-- only rendered when elevation_m !== null -->
<div class="sample-meta">
  <span class="detail-label">Elevation</span>
  ${sample.elevation_m} m
</div>
```

The `.detail-label` class uses `color: var(--text-hint); font-size: 0.75rem` to distinguish
the label from the value within the same line — matching the `.host-label` pattern already
present in the component.

### bee-sample-detail

**Current pattern:** Uses a flex column with `.sample-dot-detail`. Fields are flat `<div>`
elements with distinct class names. The component has `dt`/`dd` styles defined but uses divs
in practice.

**Elevation placement:** Add a new `<div class="event-elevation">` after `.event-count`,
only when `sampleEvent.elevation_m` is non-null.

**Markup pattern:**
```html
<!-- only rendered when sampleEvent.elevation_m !== null -->
<div class="event-elevation">${sampleEvent.elevation_m} m</div>
```

Style: `font-size: 0.8rem; color: var(--text-muted)` — matches `.event-count` and `.event-observer`
weight and tone, as elevation is a contextual fact of the same category as specimen count.

## Data Contract

Both components must accept `elevation_m: number | null` on their data objects.

**`Sample` interface** (in `bee-sidebar.ts`): add `elevation_m: number | null`

**`SampleEvent` interface** (in `bee-sidebar.ts`): add `elevation_m: number | null`

Rendering condition: `elevation_m !== null` (not `elevation_m != null` — use strict equality
to avoid rendering if value is `undefined`).

## Interaction Contract

This phase introduces no new interactions. Elevation is a static read-only display field.

- No click target
- No tooltip
- No expand/collapse
- No loading state (data arrives with the existing parquet payload)

## Accessibility Contract

- Elevation label text "Elevation" must be present in the DOM (not CSS-only) for screen reader traversal
- Color contrast: `--text-muted (#666)` on `--surface (#fafafa)` = 5.74:1 — passes WCAG AA
- Color contrast: `--text-hint (#767676)` on `--surface (#fafafa)` = 4.54:1 — passes WCAG AA minimum (documented in index.css)
- No ARIA roles required — plain text within an existing list/panel structure
