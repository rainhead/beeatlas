**1. bee-sidebar.ts — extend interfaces**

Add `elevation_m: number | null;` as the last field before the closing brace of both interfaces:

- `Sample` interface (after `species: Specimen[];`): add `elevation_m: number | null;`
- `SampleEvent` interface (after `coordinate: number[];`): add `elevation_m: number | null;`

After this change, TypeScript will flag every incomplete object literal constructing Sample or SampleEvent. Use those errors as a checklist for the remaining edits.

**2. features.ts — add elevation_m to DuckDB SELECTs and setProperties**

EcdysisSource loader: add `elevation_m` to the end of the SELECT column list (after `specimen_observation_id`). In the `setProperties()` call, add:
```typescript
elevation_m: obj.elevation_m != null ? Number(obj.elevation_m) : null,
```
This follows the exact pattern used for `host_observation_id` and `specimen_observation_id` (loose `!= null` to guard against both null and undefined from DuckDB row, `Number()` to coerce potential BigInt).

SampleSource loader: add `elevation_m` to the end of the SELECT column list (after `ecoregion_l3`). In the `setProperties()` call, add the same coercion pattern:
```typescript
elevation_m: obj.elevation_m != null ? Number(obj.elevation_m) : null,
```

**3. bee-map.ts — propagate through buildSamples(), _buildRecentSampleEvents(), map-click-sample**

In `buildSamples()`, add to the `map.set(key, {...})` literal (after `species: []`):
```typescript
elevation_m: f.get('elevation_m') != null ? Number(f.get('elevation_m')) : null,
```
Note: elevation_m belongs on Sample (the grouping key), NOT on Specimen. All specimens in one sample share the same coordinates and thus the same elevation.

In `_buildRecentSampleEvents()`, add to the returned `SampleEvent` literal (after `coordinate`):
```typescript
elevation_m: f.get('elevation_m') != null ? Number(f.get('elevation_m')) : null,
```

In the `map-click-sample` emit block, add to the event detail literal (after `coordinate`):
```typescript
elevation_m: f.get('elevation_m') != null ? Number(f.get('elevation_m')) : null,
```

**4. bee-atlas.ts — update _restoreSelectionSamples()**

In `_restoreSelectionSamples()`, add `elevation_m` to the DuckDB SELECT statement (after `specimen_observation_id`).

In the `map.set(key, {...})` literal within the same method, add (after `species: []`):
```typescript
elevation_m: obj.elevation_m != null ? Number(obj.elevation_m) : null,
```

This prevents the pitfall where elevation displays on first click but disappears after browser back/forward navigation (URL-restore path rebuilds Sample objects independently of buildSamples()).
