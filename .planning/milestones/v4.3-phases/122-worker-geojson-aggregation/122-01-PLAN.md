<objective>
Replace the 92,802-callback SQL geo query with a single json_group_array aggregate query, transfer the result as a transferable ArrayBuffer, and decode + build GeoJSON on the main thread. This eliminates both the per-row WASM→JS overhead (~560 ms) and the structured-clone transfer cost (~100 ms).

Purpose: Cut worker tablesReady from ~930 ms to an estimated ~50–150 ms for the geo step.
Output: Three modified source files + one new test file. The external API of features.ts (loadOccurrenceGeoJSON) is unchanged.
</objective>

    Make the following changes:

    1. Replace `GEO_SQL` (the multi-row SELECT) with `GEO_AGG_SQL` — the single-row json_group_array aggregate:
       ```
       SELECT COALESCE(json_group_array(json_object(
         'lat', CAST(lat AS REAL),
         'lon', CAST(lon AS REAL),
         'ecdysis_id', ecdysis_id,
         'observation_id', observation_id,
         'specimen_observation_id', specimen_observation_id,
         'year', year,
         'scientificName', scientificName,
         'genus', genus,
         'family', family,
         'source', source
       )), '[]') AS rows_json
       FROM occurrences WHERE lat IS NOT NULL AND lon IS NOT NULL
       ```
       COALESCE handles the edge case where no rows match (json_group_array returns NULL → coalesce to '[]').
       CAST(lat/lon AS REAL) ensures JSON numbers, not strings, regardless of SQLite affinity.

    2. Replace the `geoRows: unknown[][]` accumulation pattern with a single `let jsonStr = '[]'` variable:

       ```typescript
       let jsonStr = '[]';

         jsonStr = (row[0] as string | null) ?? '[]';
       });
       ```

       ```typescript
       const encoded = new TextEncoder().encode(jsonStr);
       ```
