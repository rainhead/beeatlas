## Overview

Bee species exploration page — hierarchical taxonomic nav, image-forward content, occurrence maps, seasonality viz; volunteer-learning oriented. Driven by WA state checklist + iNat-derived taxonomy; per-species deep-link into the SPA pre-filtered. Static SVG occurrence maps generated in Python; photo manifest authored as TOML (no build-time iNat queries). New `/species/` page ships in its own Vite chunk (gated <100 KB gzipped) with strict architectural isolation from the SPA's mapbox-gl + wa-sqlite stack.

## Milestone Summary

**Decimal Phases:**

- Phase 77: Lineage Coverage Expansion (INSERTED 2026-05-03 between Phase 76 and the originally-planned Phase 77 outputs phase, after Phase 78 research surfaced ~70% NULL family/subfamily/tribe coverage; downstream phase numbers bumped 78→ pipeline outputs, 79→ photo manifest, 80→ page scaffolding, 81→ filter UX & nav, 82→ hardening)

**Key Decisions (selected — full set in PROJECT.md Key Decisions):**

- `canonical_name` (lowercase, single-spaced, authority-stripped, subgenus-paren-stripped) is THE join key shared between checklist and occurrences — used everywhere, computed identically in Python and DuckDB
- Bridge table `inaturalist_data.canonical_to_taxon_id` is the iNat resolution cache — re-running pipeline produces zero new API calls; `--refresh-lineage` flag forces re-resolution; rate-limited to ≤1 req/sec, D-02 filter ladder (matched_term → name → is_active → Insecta → rank), D-03 rank fallback (1-token → genus, 2-token → species → genus(tokens[0]))
- DuckDB 1.4.x lacks COALESCE on INTEGER[12] — Python-side `[0]*12` backfill in `species_export.py` for checklist-only rows (workaround, not defect)
- Static SVG occurrence maps written via xml.etree with single embedded `<style>` block (NOT literal inline fill/stroke per element) — functionally equivalent under `<img src=.svg>`; viewBox `0 0 600 320`; off-WA-bbox coordinates clipped silently with logged count
- SVG byte-stability via sorted attrib dicts before `ET.tostring` — sha256 byte-equality across consecutive runs proven on 556 SVGs at host scale (idempotency verified)
- Slug generation shared with `data/feeds.py::_slugify` (path-traversal-safe per v2.1) — SVG filename / parquet `slug` column / URL slug agree byte-for-byte across all 735 species
- Photo manifest authored as TOML, NOT fetched at build — `content/species-photos.toml` carries `url` resolved at fill time; `scripts/seed-species-photos.mjs` is the one-shot helper (NOT in `npm run build`)
- iNat enforces tighter effective burst limit than documented 1 req/sec — `--rate-ms=1500` cleared 231 HTTP 429s on a 735-species sweep; production seed pacing recommended at 1500ms
- Photo seed fill-only recovery loop established — programmatically delete bare entries after a 429 burst, rerun at slower pacing; existing photo-bearing entries preserved; reusable for future incremental data-fetch repair
- `<bee-species-page>` is the page coordinator (mirrors v1.9 `<bee-atlas>` ARCH-03 invariant); `<bee-taxon-nav>`, `<bee-species-card>`, `<bee-species-filter>`, `<seasonality-viz>` are pure presenters with light-DOM rendering and `createRenderRoot() → this` invariant for SSR preservation
- ARCH-04 source-analysis test in `src/tests/arch.test.ts` enforces species chunk isolation — no `mapbox-gl` / `wa-sqlite` / `../sqlite.ts` / `../filter.ts` / `../bee-map.ts` / `../bee-atlas.ts` imports under `src/species/**`; mitigates one accidental import ballooning the chunk from ~50 KB to ~2 MB
- Per-card "View N occurrences →" navigates via shared `buildSpaTaxonLink()` helper; SPA URL contract (`taxon` + `taxonRank` BOTH required) documented as a stable interface in `src/url-state.ts` header comment
- Filter cards use mute-not-hide (opacity 0.35) rather than `display: none` — preserves orientation; descendant selector targets card hosts (NOT `:host` — fix in commit 195232d)
- Server-rendered `<details>`/`<ul>` taxon tree remains navigable without the Lit bundle (pnwmoths pattern); aria-expanded synced on toggle for keyboard accessibility
- Seasonality viz: inline `<svg>` via Lit svg template; bars when `n ≥ 5`, text fallback ("3 records, May–June") when `n < 5`; D-08 dropped ambiguous single-letter month suffix (T7)
- Bundle gate: species chunk 5.4 KB / 100.0 KB cap = 94.6 KB headroom (PERF-01)
- Lighthouse mobile LCP on Andrena page = 1312 ms / 3000 ms budget (PERF-02); `scripts/measure-lcp.sh` aliased as `npm run measure-lcp` (NOT in build chain per D-06)
- Weekly photo-availability cron (NOT every build per PITFALLS #1) — `scripts/check-photo-availability.mjs` HEADs each manifest URL; informational report, does not block deploys per D-10
