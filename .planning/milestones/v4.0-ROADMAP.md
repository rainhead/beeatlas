## Overview

Added the Bartholomew et al. 2024 annotated checklist as a second curated data source alongside WABA occurrence records. Eliminated iNat API rate-limit risk by switching to the AWS Open Data taxa.csv.gz offline archive. All 565 checklist species now have taxon pages, county-presence SVG maps, attribution, and seasonality histograms. A new "Checklist records" toggle-able county-fill layer appears on the map alongside Specimens and Samples.

## Milestone Summary

**Key Decisions:**

- iNat taxonomy source is AWS Open Data `taxa.csv.gz` (not the DwC-A zip archive) — has `ancestry` column; DwC-A disqualified (URL-form IDs, no subfamily/tribe, no ancestry column)
- Checklist records are county-range assertions, NOT point occurrences — they must NOT enter occurrences.parquet or int_combined; checklist.parquet is a separate dbt mart
- Checklist map layer uses Mapbox county-fill on the existing counties GeoJSON source, not a new point cluster layer; `addLayer` with `beforeId: 'ghost-points'` ensures specimen dots render on top
- Checklist county-fill responds to taxon AND year filters (not taxon-only as originally planned); UAT confirmed year filter narrowing is desired behavior
- `_checklistAllRows` cached in bee-map after first parquet fetch; subsequent taxon/year changes re-filter in JS without re-fetching
- `checklist-only` badge on species index uses three-branch count slot: occurrence_count > 0 / on_checklist / else; genus/subgenus pages show "N checklist records" for checklist-only entries
- Atlas link on species-detail.njk wrapped in `occurrence_count > 0` guard — checklist-only species don't link to the atlas map
- ETag/304 caching: taxa.csv.gz download sends If-None-Match + If-Modified-Since; skips 37MB download on 304

**Issues Resolved:**

- Rate-limit risk for iNat taxonomy eliminated: 5 API calls/minute was a bottleneck for large checklists; offline DuckDB ancestry walk processes all active Anthophila in ~1s
- eco_fallback CTE required for Island County and Kitsap County (centroids fall in Puget Sound); ST_Distance nearest-neighbor fallback wired in checklist.sql
- ghost-points beforeId: layer wasn't yet added at checklist layer init; fixed by removing beforeId and relying on insertion order

**Issues Deferred:**

- TAX-04 S3 runtime verification: full round-trip exercisable only on maderas nightly cron run; code is complete and wired
- CHECK-03 CloudFront verification: requires live nightly + real AWS credentials; code wired in nightly.sh
- Pre-existing pytest failures in test_dbt_diff.py and test_places_maps.py (unrelated to v4.0 work)

**Technical Debt Incurred:**

- 5 pre-existing pytest failures in test_dbt_diff.py (stale sandbox vs public/data artifacts) — require full nightly pipeline run with current data; predated v4.0
- Phase 112 has no VERIFICATION.md (UAT was the gate; all MAP-* requirements verified in practice)
