## Milestone Summary

**Key Decisions:**

- `occIdFromRow` returns `string | null` (not `string`) — matches bee-table.ts `rowOccId` contract; avoids silent `inat:0` bug when both ecdysis_id and observation_id are null
- `isSampleOnly` defined as `ecdysis_id == null && !is_provisional` — deliberately excludes provisional rows; `!isSpecimenBacked` is the correct non-specimen partition in bee-occurrence-detail.ts
- `isSpecimenId(occId: string)` exported as sixth helper — allows features.ts and bee-map.ts to replace `startsWith('ecdysis:')` guards without restating the prefix literal
- filter.ts uses partial-row construction for `occIdFromRow` — safe because occurrence.ts uses `import type` only, erased before bundle resolution
- `isSpecimenBacked` is canonical — `places_export.py` now uses `ecdysis_id IS NOT NULL`, matching the TypeScript predicate; `!is_provisional` was an incorrect synonym
- dbt macros centralize OFV field IDs `8338`, `9963`, `18116`, `1718` and the Plantae CASE expression into `data/dbt/macros/inat_field_ids.sql`

**Issues Resolved:**

- places_export.py was overcounting specimens — sample-only iNat rows counted because the predicate was `is_provisional IS NOT TRUE` rather than `ecdysis_id IS NOT NULL` (SEM-01)
- `BEE_FAMILIES` constant in species_export.py was dead code — the SQL gate in int_species_universe.sql was already authoritative
- 4 anonymous OFV integer literals scattered across 4 intermediate dbt models — now named macros

**Issues Deferred:**

- `OccurrenceRow` / `OCCURRENCE_COLUMNS` / `sqlite.ts` DDL / dbt schema.yml full alignment (full schema governance project)
- TypeScript `slugify` in `filter.ts` not aligned with Python `domain.slugify` (different contract: CSV filename, truncated) — documented and deferred
- `canonical_name` column absent from `OCCURRENCE_COLUMNS` — under investigation

**Technical Debt Incurred:**

- `speicmenLayer` typo preserved in `bee-map.ts` per CLAUDE.md constraint (consistent, intentionally deferred)
- Pre-existing test failures in `src/tests/build-output.test.ts` and `src/tests/data-species.test.ts` (missing pipeline-generated `public/data/species.json`) unchanged
