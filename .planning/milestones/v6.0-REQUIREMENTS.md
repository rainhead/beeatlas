# Requirements: Washington Bee Atlas — v6.0 My Work: Progress & Provenance

**Defined:** 2026-06-24
**Core Value:** Tighten learning cycles for volunteer collectors (close the gap between collection and identification appearing on the map) and convey liveness and togetherness among participants.

**Milestone goal:** Stand up the first "work" surface — a bookmarkable, no-auth, public **per-collector page** showing the collection→ID lifecycle as an event stream and accomplishments as coverage/breadth — on a rebuilt occurrence model that replaces `source` with orthogonal facets and a temporal status lifecycle.

> **Temporal approach (operator decision 2026-06-24):** the event timeline is read from the **intrinsic dates already present in the source data** (collection/event date, iNat `created_at`, identification dates) — **not** by diffing nightly snapshots. This dissolves the research's Option A/B/C snapshot-history fork and the first-run-flood concern. An append-only history table remains deferred to the community-feed milestone. The one residual gap: precise per-identification dates are weak in the current data (Ecdysis `date_identified` is mostly year-only / `s.d.`; the rich iNat per-identification `created_at` needs an added nightly fetch) — that enrichment is a **v2** item.

## v1 Requirements

### Collector Identity (IDENT)

- [x] **IDENT-01**: Each occurrence carries a unified `collector_inat_login` (COALESCE `specimen_inat_login` > `host_inat_login` > `user_login`), projected through the `occurrences` mart (dbt contract bump 36→37, shipped data-before-code). WABA collectors always have an iNat handle (they appear in iNaturalist before Ecdysis), so no free-text `recordedBy` reconciliation seed is needed.

### Provenance & Facets (PROV)

- [x] **PROV-01**: The occurrence model exposes orthogonal facets — collector, place, taxon, time, and provenance-tier — replacing the mutually-exclusive `source` enum as the UI's organizing primitive.
- [x] **PROV-02**: Map filter, map symbology, and the detail card are driven by **provenance tier** (not `source`); tier filtering round-trips via a `tier=` URL param with legacy `src=` back-compat.
- [x] **PROV-03**: The `occ_id` positional coupling across `src/occurrence.ts`, `src/filter.ts`, and `data/dbt/models/marts/occurrence_places.sql` is preserved and asserted by a test; the facets rebuild ships as one atomic commit across all three consumers.

### Temporal Lifecycle (TEMP)

- [x] **TEMP-01**: The occurrence model surfaces the intrinsic lifecycle dates already present in the source data — collection/event date, iNat posting date (`created_at`), and best-available identification date — projected into the mart so a per-record timeline can be built **without** snapshot-diffing (dbt contract bump, shipped data-before-code).
- [x] **TEMP-02**: A `waba_specimen → ecdysis` transition reads as a single specimen carrying both its iNat posting date and its Ecdysis cataloguing date (linked via `specimen_observation_id`) — cataloguing is one event in that specimen's timeline, not a separate delete + create.

### Per-Collector Page (PAGE)

- [x] **PAGE-01**: Every collector with a resolved iNat handle has a bookmarkable, public page at `/collectors/{inat_login}/`, generated by the existing Eleventy static-page pattern.
- [x] **PAGE-02**: The page shows headline contribution stats (specimens, samples, species count).
- [x] **PAGE-03**: The page shows a **pending-vs-identified** status split ("did my bees get IDed?").
- [x] **PAGE-04**: The page links to the main map filtered to that collector.

### Event Stream (STREAM)

- [x] **STREAM-01**: The collector page shows a reverse-chronological **collection→identification event feed**.
- [x] **STREAM-02**: A specimen being catalogued (`waba_specimen → ecdysis`) appears as an event in the feed.
- [x] **STREAM-03**: The feed paginates / bounds its length for high-volume collectors (500+ records).

### Accomplishment View (ACCOM)

- [x] **ACCOM-01**: The page shows a **county coverage SVG map** (reusing the taxon/place SVG pattern).
- [x] **ACCOM-02**: The page shows a **taxonomic-breadth** list of contributed species, linked to taxon pages.
- [x] **ACCOM-03**: The page shows **ecoregion breadth**.
- [x] **ACCOM-04**: The page shows an **"Active since YYYY (N seasons)"** badge (no streaks).

## Out of Scope

Explicitly excluded for v6.0.

| Feature | Reason |
|---------|--------|
| Authentication / self-identification flow | Unnecessary — per-collector pages are bookmarkable over already-public data; no "who are you" step. |
| Privacy gating (opt-out seed, min-activity threshold, `noindex`, coordinate suppression) | **Operator decision 2026-06-24:** pages are public with no gating; the underlying data is already public on iNaturalist. Revisit if a collector objects. (Residual risks acknowledged: SEO-surfacing a named page, easier scraping.) |
| Collector pages for checklist-only contributors | Not WABA volunteers and have no iNat handle (only free-text `recordedBy`); page infra gates on `collector_inat_login IS NOT NULL`. |
| "New county record!" milestone events in the stream | Descoped from MVP by operator 2026-06-24; the event stream stays the plain collection→ID feed. Candidate for a later pass. |
| Leaderboards / rankings | Demotivates the bottom 90%; counter to "togetherness" Core Value. |
| Generic point totals | Disconnected from scientific meaning; use counts that matter (species, counties, seasons). |
| Streak tracking | Wrong model for seasonal (Mar–Oct) collecting; punishes winter. Use "years/seasons active." |
| Push / email notifications | Requires server infrastructure; project is static-hosting-only. |
| Append-only history table (temporal Option A) | Deferred to the community-feed milestone; v6.0 reads intrinsic lifecycle dates from the source data instead. |

**Honored-scope reconciliations (intentional, documented):**

- **TEMP-01 — scope narrowed during 168-discuss (CONTEXT D-02).** The requirement text names "iNat posting date (`created_at`)" as a surfaced lifecycle date, but the operator dropped `posted_date`/`created_at` as a column and an event ("posting is not an event"). What shipped: a single new `id_date VARCHAR` column (Ecdysis `date_identified`, dirty-parse: year-only + full dates verbatim, garbage NULLed); "Collected" is served by the pre-existing `date` column (no separate `collection_date`). dbt contract 37→38. The absence of `posted_date` is **correct per the narrowed scope**, not a gap.
- **TEMP-02** satisfied structurally by the existing ARM-3 de-dup (waba_specimen→ecdysis reads as one continuous row) — no new plumbing; no fake catalogue event in the feed (171 D-EVENT-01).
- **PROV-02** shipped as `tier` (atlas/other) + `record_type` rather than a single "provenance tier"; `inat_obs`→`inat_expert` rename; `tier=` URL param with legacy `src=` back-compat (parse-only, lossy by design).
