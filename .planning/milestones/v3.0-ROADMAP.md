## Overview

Replace OpenLayers with Mapbox GL JS v3 to gain WebGL rendering for 250K+ points, drop ~400 KB of legacy mapping JS from the application bundle (excluding mapbox-gl itself), and unlock 3D terrain, vector tiles, and geocoding for future milestones. Three phases: rebuild the base map and clustered occurrence layer, port boundary layers and the full click interaction chain, then remove all OpenLayers dependencies and verify feature parity.

## Milestone Summary

**Decimal Phases:** None.

**Key Decisions:**

| Decision | Rationale | Outcome |
|----------|-----------|---------|
| Mapbox `accessToken` assignment requires explicit cast | `verbatimModuleSyntax` + `nodenext` resolves `mapbox-gl` default import to module namespace type; runtime property exists but TS cannot see it | ✓ Good — cast localized to one line in bee-map.ts |
| `features.ts` outputs `[lon, lat]` WGS84 (no EPSG:3857 projection) | Mapbox GL JS expects WGS84 natively; OL's projected coordinate convention no longer applicable | ✓ Good — also simplified GeoJSON construction |
| Filter-based selection highlighting (`setFilter` on `selected-ring` layer) instead of `feature-state` | `feature-state` requires `promoteId`, which conflicts with cluster auto-IDs in Mapbox v3; `setFilter` on a dedicated unclustered layer is conflict-free | ✓ Good — selection works on points; cluster blobs intentionally show no ring (UX follow-up filed) |
| County/ecoregion filter options loaded from SQLite DISTINCT queries in `bee-atlas._loadCountyEcoregionOptions` | Decoupled from map source events; coordinator owns option loading regardless of map state | ✓ Good — D-02; preserves pure-presenter invariant for `bee-map` |
| Boundary layers render BELOW occurrence layers (ecoregion-fill/line, county-fill/line, then ghost-points, clusters, points) | Dots remain clickable and visually prominent over polygon fills | ✓ Good — verified in human UAT |
| `_clickConsumed` flag pattern for robust empty-click fallback | Mapbox `addInteraction` with `preventDefault` may or may not propagate to generic `map.on('click')`; flag pattern is correct in either case | ✓ Good — replaces manual `queryRenderedFeatures` hit testing |
| Cluster click emits all leaves (D-01) without auto-zoom | Preserves v2.7 selection semantics — sidebar opens with the full leaf list; user can still zoom manually | ✓ Good — verified in human UAT; auto-zoom rejected during research |
| Use source/layer existence checks, NOT `isStyleLoaded()`, in `_applyVisibleIds`/`_applySelection`/`_applyBoundaryMode`/`_applyBoundarySelection` | Mapbox v3 reports `isStyleLoaded()` = false during async clustered GeoJSON processing, blocking URL-restored filters and selection | ✓ Good — discovered during Phase 73 human UAT and Phase 71 human UAT; uniform pattern across all four guards |
| `mapbox-gl` v3 main chunk ~1,700 KB, not tree-shakeable | App code minus mapbox-gl is ~318 KB (lit + wa-sqlite + hyparquet); ROADMAP "<200 KB" target reframed as "app code excluding mapbox-gl" | ⚠️ Revisit — bundle target was unrealistic given vendor library size; future ideas: defer mapbox-gl import, or reconsider WebGL for lower point counts |
| `npm workspaces` uses single root `package-lock.json` (deleted `frontend/package-lock.json`) | Stale per-workspace lockfile shadowed root resolution; clean dep tree requires single source of truth | ✓ Good — discovered while removing OL packages |
| Source-analysis (`readFileSync`) tests verify architectural invariants in `bee-map.ts` | Avoids Mapbox GL JS runtime in happy-dom; reliably checks API usage and import graph contracts | ✓ Good — pattern continued from v1.9 |
