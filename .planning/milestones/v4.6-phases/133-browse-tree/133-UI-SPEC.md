# Phase 133 — UI Design Contract: Browse Tree

> Visual and interaction contract for the `/species` expandable taxonomy tree.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — plain HTML/CSS/TypeScript, no component library |
| Preset | not applicable |
| Component library | none (native `<details>/<summary>`) |
| Icon library | Unicode / emoji characters only (existing site pattern) |
| Font | system-ui, Avenir, Helvetica, Arial, sans-serif (from `src/index.css`) |

**Design tokens source:** `src/index.css` CSS custom properties. All new styles go
in `src/styles/taxon-pages.css` as `.species-index` modifier rules, extending the
existing `.species-index` block already present there. Do NOT introduce a parallel
token system.

**shadcn gate:** Not applicable. No React/Next.js/Vite SPA on the `/species` page.
The page is an Eleventy-rendered static HTML document with one TypeScript entry point.

## Spacing Scale

Declared values follow the existing codebase's rem-based 8-point scale anchored to
the browser default 16px base:

| Token | Value | Rem | Usage in this phase |
|-------|-------|-----|---------------------|
| xs | 4px | 0.25rem | Icon gaps within `<summary>` (between name and count, count and 🗺) |
| sm | 8px | 0.5rem | Genus/species row vertical padding; control bar gap; existing `.genus-row` mb |
| md | 16px | 1rem | Page content padding (existing `.taxon-page` padding: 1rem) |
| lg | 24px | 1.5rem | Family section margin-bottom (existing `.family-section` mb: 1.5rem) |
| xl | 32px | 2rem | Not used in this phase |
| 2xl | 48px | 3rem | Not used in this phase |
| 3xl | 64px | 4rem | Not used in this phase |

Exceptions:
- Tree indentation: each nesting depth adds `1.5rem` (24px) left padding on the
  child `<details>` or `<ul>` — this is a half-`lg` step consistent with the
  existing `.genus-row` and `.species-list` visual rhythm.
- 🗺 touch target: minimum 44×44px tap area via `padding: 0.5rem; margin: -0.5rem`
  on the `<a>` element — meets WCAG 2.5.5 (AAA target size) without changing layout.
- Control bar: `gap: 0.5rem` (8px) between filter input and "Show all ranks" toggle.

## Typography

All sizes are relative to the root 16px base, matching existing `.taxon-page` conventions.
No new font sizes are introduced; every role maps to an existing size in the codebase.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body / node name | 1rem (16px) | 400 (regular) | 1.5 | `<summary>` text, species `<li>` text |
| Label / count | 0.85rem (13.6px) | 400 (regular) | 1.5 | Count spans (`.count`), attribution; matches existing `.species-list .count` |
| Attribution / small | 0.75rem (12px) | 400 (regular) | 1.5 | "Show all ranks" label caption if needed; matches `.attribution` |
| Control label | 0.875rem (14px) | 400 (regular) | 1.5 | Filter input text (matches existing `.filter-input` in `bee-pane.ts`) |

Font weights declared for this phase: regular (400) only. Existing site uses 500 for
links (`a { font-weight: 500 }`) — node name links inherit this automatically.

is always present as a structural separator, not only as a color cue.

## Component Inventory

No new component library entries. The following native and existing elements are used:

### `<details>/<summary>` — Expandable tree node

Source: D-02 (locked decision, 133-CONTEXT.md)

```html
<!-- Page-backed node (genus, subgenus, tribe, subfamily, species) -->
<details class="tree-node tree-node--genus">
  <summary>
    <a href="/species/Bombus/" class="node-name"><em>Bombus</em></a>
    <span class="node-counts">430 · 210</span>
    <a href="/?taxon=Bombus&amp;taxonRank=genus" class="node-map"
       aria-label="Map: Bombus occurrences">🗺</a>
  </summary>
  <!-- children -->
</details>

<!-- Family node — no page link, plain text name -->
<details class="tree-node tree-node--family">
  <summary>
    <span class="node-name">Apidae</span>
    <span class="node-counts">1,240 · 800</span>
    <a href="/?taxon=Apidae&amp;taxonRank=family" class="node-map"
       aria-label="Map: Apidae occurrences">🗺</a>
  </summary>
  <!-- children -->
</details>
```

The `<summary>` is a flex row: `display: flex; align-items: baseline; gap: 0.5rem`.
The `.node-name` takes `flex: 1 1 auto` and is left-aligned. `.node-counts` is
`flex: 0 0 auto`. `.node-map` is `flex: 0 0 auto` and is the last item.

### Control bar

Sits above the tree, below the `<h1>`. Contains two controls on one row:

```html
<div class="species-index-controls">
  <input type="search" id="species-filter"
         aria-label="Filter taxa"
         placeholder="Filter taxa…"
         autocomplete="off">
  <label class="rank-toggle-label">
    <input type="checkbox" id="show-all-ranks">
    Show all ranks
  </label>
</div>
```

Layout: `display: flex; align-items: center; gap: 0.5rem; margin-bottom: 0.5rem`.
On mobile (< 480px): `flex-wrap: wrap` so the toggle drops below the filter input.
The filter input has `flex: 1 1 auto; min-width: 0`. The toggle label has
`flex: 0 0 auto; white-space: nowrap`.

## Visual Specifications

**Primary focal point:** the tree node **name column** — it is the widest flex
element (`flex: 1 1 auto`) and the only link-colored, italicized text in each row,
so the eye lands on taxon names first and reads left-to-right into counts then the
🗺 affordance. The control bar (filter + toggle) is the secondary anchor at the top;
counts and the map glyph are deliberately `--text-muted` so they never out-compete
the name for attention.

### Node layout within `<summary>`

```
[disclosure triangle]  [name link or plain text]  [count]  [🗺]
```

- Disclosure triangle: browser-native `::marker` — do NOT replace with custom SVG.
  No CSS reset of `::marker` is needed (native behavior matches D-02 intent).
- Name: page-backed ranks use `<a class="node-name"><em>…</em></a>` (link + italic
  for scientific names). Family uses `<span class="node-name">…</span>` (plain text,
  no italic, no link — per D-07).
- Count: `<span class="node-counts">430 · 210</span>` — the `·` is U+00B7 (middle
  dot). No units suffix on the tree node. Color: `--text-muted`. Font-size: 0.85rem.
  This matches the existing `.count` convention on `.species-list`.
- 🗺 map affordance: the glyph is U+1F5FA (world map). Rendered as an `<a>` element.
  Font-size: 0.85rem (same visual weight as the count). Color: `--text-muted` when
  unfocused/unhovered; `--text-body` on `:hover/:focus-visible`. Minimum tap target
  44×44px via `padding: 0.5rem; margin: -0.5rem`. Accessible name:
  `aria-label="Map: {TaxonName} occurrences"` — this MUST differ from the node name
  link's accessible name to disambiguate for screen readers.

### Tree indentation

Each additional depth level adds `padding-left: 1.5rem` to the child `<details>` or
`<ul>`. Maximum visible depth in default view (family→genus→species): 3 levels.
With "Show all ranks" ON: up to 6 levels (family→subfamily→tribe→genus→subgenus→species).
Total max indentation at species level with all ranks: `6 × 1.5rem = 9rem` on desktop.
On narrow viewports (< 480px): reduce indent to `padding-left: 1rem` per level.

### Visual distinction: page-backed name vs. family plain text

| Node type | Name element | Color | Style |
|-----------|-------------|-------|-------|
| Family (no page) | `<span>` | `--text-body` (#213547) | Not italic, not underlined |
| Genus/subgenus/tribe/subfamily (page exists) | `<a><em>` | `--link` (#646cff) | Italic (scientific name), underline on hover (browser default) |
| Species (page exists) | `<a><em>` | `--link` (#646cff) | Italic, underline on hover |

### "Show all ranks" toggle

- Rendered as a native `<input type="checkbox">` wrapped in a `<label>`.
- When checked (all ranks visible): the label receives a visual indicator — either
  `border: 1px solid var(--accent)` or a background of `rgba(44,122,44,0.08)` on the
  label. Do not use color alone; the checkbox itself provides the non-color signal.
- Label text: "Show all ranks" (6 words, fits on one line at any viewport > 200px).
- `aria-describedby` or an adjacent `<p id="rank-toggle-hint">` may add:
  "Includes subfamily, tribe, and subgenus groupings."
- localStorage key: `beeatlas.speciesTree.showAllRanks` (string `"1"` / `"0"` or
  absent = default OFF). D-04: one key, boolean only.

### Intermediate rank nodes (shown when "Show all ranks" is ON)

These are hidden (`hidden` attribute or `display: none`) when the toggle is OFF.
They are `<details class="tree-node tree-node--subfamily">` (or `--tribe`,
`--subgenus`) elements. Graceful degradation (D-05): when a branch has no intermediate
rank, the child `<details>` attaches directly to its nearest present ancestor — no
empty wrapper or "Other" bucket is inserted.

### Focus-visible styling

All interactive elements must show a clearly visible focus ring. Use:

```css
:focus-visible {
  outline: 2px solid var(--accent);
  outline-offset: 2px;
  border-radius: 2px;
}
```

This applies to:
- `<summary>` (expandable node): receives `:focus-visible` ring around the whole row.
- `#show-all-ranks` checkbox: browser default focus ring is acceptable; supplement
  with `outline: 2px solid var(--accent)` if browser default is invisible.
- `<a class="node-map">` (🗺 link): focus ring must be visible even when the element
  has `padding: 0.5rem; margin: -0.5rem` compensation.
- `#species-filter` input: `border-color: var(--accent)` on `:focus` (existing
  pattern from `bee-pane.ts`; replicate for this light-DOM context — `outline: none;
  border-color: var(--accent)`).

## Accessibility Contract

| Element | ARIA / Keyboard requirement |
|---------|----------------------------|
| `<details>/<summary>` | Native; Enter/Space toggles; arrow keys do NOT need custom handling (browser handles tree navigation within disclosure widgets) |
| `#show-all-ranks` | Native checkbox; Enter/Space toggles; `aria-describedby` pointing to hint text |
| `.node-map` (🗺 link) | `aria-label="Map: {TaxonName} occurrences"` — must not duplicate the adjacent node name link's accessible name |
| `#species-filter` | `aria-label="Filter taxa"` (updated from old "Filter genera and species" — now covers all ranks) |
| `#filter-empty` | `role="status"` or `aria-live="polite"` so screen readers announce when matches change |
| Count text | Separator `·` does not convey meaning alone; counts are numbers that stand on their own — no color-only indication |
| Intermediate rank nodes (hidden) | Use `hidden` attribute (not only `display:none`) or `aria-hidden="true"` when ranks are collapsed via the toggle, so hidden nodes are not announced |

## No-JS Fallback

Per D-02: `<details>/<summary>` works without JavaScript. Without JS:

- All `<details>` elements render in their default closed state (no `open` attribute
  set by JS). Users can expand any node manually.
- Intermediate rank nodes (subfamily, tribe, subgenus) are visible because there is
  no JS to hide them — they appear as collapsible `<details>` nodes at all times.
  This is acceptable (fuller view is the no-JS fallback, not a degraded one).
- The filter input renders but produces no DOM filtering (no event listener).
- The "Show all ranks" toggle renders but has no effect.
- `#filter-empty` remains `hidden` (its `hidden` attribute is set in HTML, only
  removed by JS).
- Tree links (taxon pages and 🗺 map links) all work without JS.

No wording changes are needed for the no-JS state. The page is usable and complete.
