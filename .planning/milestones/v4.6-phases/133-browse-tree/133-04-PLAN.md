<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| localStorage → app | The rank-toggle value is read from a client-controlled store; a corrupt/forged value must not execute code or break the page. |
| filter query → DOM | The user-typed query is echoed into the empty-state message. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-133-07 | Tampering (XSS) | `#filter-query` empty-state echo | mitigate | Write the query via `.textContent`, never `.innerHTML`; acceptance criterion + test assert the textContent path. |
| T-133-08 | Tampering | localStorage rank-toggle value | mitigate | Read with strict `=== '1'` coercion (no eval/JSON.parse of the value); try/catch around access; any non-`"1"` value → default OFF. |
| T-133-09 | Denial of service | localStorage unavailable (private mode/quota) | mitigate | try/catch around getItem/setItem; failure degrades to default-OFF, page stays functional. |
| T-133-SC | Tampering | npm/pip/cargo installs | accept | No package installs (plain DOM, existing imports only); nothing to gate. |
</threat_model>
