## Component Inventory

### `<bee-occurrence-detail>` ‚Äî NEW (replaces `<bee-specimen-detail>` and `<bee-sample-detail>`)

Receives: flat array of raw occurrence row objects (OL feature properties).

Render logic:
1. **Specimen-backed rows** (`ecdysis_id` non-null): render existing grouped-by-sample display ‚Äî `.sample` block with `.sample-header` (month + year), `.sample-meta` (recordedBy ¬∑ fieldNumber), optional elevation `.sample-meta`, and `.species-list` with ecdysis + iNat links. Copy styles verbatim from `bee-specimen-detail`.
2. **Sample-only rows** (`ecdysis_id` null): render compact `.sample-entry` block ‚Äî date (formatted via `_formatSampleDate`), observer, specimen count, optional elevation, iNat link. Style using same `.sample` padding and `.sample-meta` text rules from `bee-sample-detail`.
3. **Mixed cluster**: specimen groups first, then a `<hr class="separator">` (1px solid `var(--border-subtle)`, no margin), then sample-only entries.
4. **Null-omit rule**: omit any field that is null ‚Äî no placeholder dashes.

CSS class reuse plan:
- `.sample` ‚Äî padding block (carry from specimen detail)
- `.sample-header` ‚Äî month/year or date heading (carry from specimen detail)
- `.sample-meta` ‚Äî secondary metadata text (carry from both)
- `.species-list` ‚Äî italic species list (carry from specimen detail)
- `.quality-badge`, `.quality-badge.research`, `.quality-badge.needs_id`, `.quality-badge.casual` ‚Äî carry from specimen detail; rendered at 12.8px (label/meta size)
- `.inat-missing` ‚Äî carry from specimen detail
- `.no-determination` ‚Äî carry from specimen detail
- `.sample-entry` ‚Äî NEW class for sample-only row block (same padding as `.sample`, no species list)

## Interaction Contracts

### Sidebar detail panel

- `bee-sidebar` shows `<bee-occurrence-detail>` when any occurrence is selected (specimen-backed or sample-only)
- Single `occurrences` property (array of OccurrenceRow objects) replaces `samples` + `selectedSampleEvent`
- Close button behavior unchanged
- Scroll behavior unchanged (`:host { overflow-y: auto }`)

### Map click

- Clicking a specimen-backed dot: sidebar shows occurrence detail with specimen groups
- Clicking a sample-only dot: sidebar shows occurrence detail with compact sample entry
- Clicking a cluster: sidebar shows all occurrences in cluster (specimen groups first, then sample-only)
- Filter highlight: sample-only dots are now correctly highlighted when filter is active (via `visibleIds` unification ‚Äî no visual change needed, just correctness fix)

### Table sorting

- `date` column is sortable (desc default)
- `modified` column is sortable
- All other columns: not sortable
- Sort indicator (‚ñº) shown on active sort column header only
- Sort applies across all occurrence types uniformly

### URL state

- `layerMode` key removed from url-state params
- All other URL params unchanged
