**1. buildFilterSQL (line 225):**

Change signature to return `{ occurrenceWhere: string }`.

Replace the dual `ecdysisClauses`/`samplesClauses` arrays with a single `occurrenceClauses: string[]` array.

Filter-by-filter changes:
- **Taxon (D-04):** Push `family = '${escaped}'` (or genus/scientificName) to `occurrenceClauses`. Do NOT push any ghost clause. SQL null comparison semantics naturally exclude sample-only rows (where family/genus/scientificName are null).
- **Year (D-01):** Push `year >= ${f.yearFrom}` and/or `year <= ${f.yearTo}` to `occurrenceClauses`. No strftime. Null year for sample-only rows naturally excludes them.
- **Month (D-01):** Push `month IN (${monthList})` to `occurrenceClauses`. No strftime. Null month for sample-only rows naturally excludes them.
- **County:** Push `county IN (${counties})` to `occurrenceClauses` (same SQL, single clause).
- **Ecoregion:** Push `ecoregion_l3 IN (${ecors})` to `occurrenceClauses` (same SQL, single clause).
- **Collector (D-05):** Build parts array:
  ```typescript
  const parts: string[] = [];
  if (recordedBys.length > 0) parts.push(`recordedBy IN (${recordedBys.join(',')})`);
  if (observers.length > 0) parts.push(`observer IN (${observers.join(',')})`);
  if (parts.length > 0) occurrenceClauses.push(`(${parts.join(' OR ')})`);
  ```
- **Elevation:** Same logic as before but pushed once to `occurrenceClauses` (not twice).

Return:
```typescript
const occurrenceWhere = occurrenceClauses.length > 0 ? occurrenceClauses.join(' AND ') : '1 = 1';
return { occurrenceWhere };
```
