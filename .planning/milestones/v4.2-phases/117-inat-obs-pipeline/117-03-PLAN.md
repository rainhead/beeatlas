Existing pattern (do not modify):
- `_upload_hashed` shell function: takes (src_path, basename, [extra aws args]); computes sha256, prepends 12 chars to filename, uploads with `Cache-Control: public, max-age=31536000, immutable`; echoes the hashed filename.

- CloudFront invalidation block at end of nightly.sh covers `/data/manifest.json` plus per-asset paths — the parquet itself is content-hashed and immutable, so no per-parquet invalidation is needed.

Manifest key choice: `"inat_obs"` (lowercase snake_case matching the parquet basename), consistent with `"checklist"`, `"occurrences"`, `"places_meta"`, `"species"`. Confirmed by CONTEXT.md `Claude's Discretion` → nightly.sh / manifest section.
