## Component Inventory

### View Mode Toggle Row (new)

A second row of two segmented buttons inside `bee-sidebar`, rendered immediately below the existing Specimens/Samples layer-toggle row and above `<bee-filter-controls>`.

**Structure:**

```
<div class="view-toggle">
  <button class="toggle-btn [active]">Map</button>
  <button class="toggle-btn [active]">Table</button>
</div>
```

**CSS class reuse:** Use the existing `.layer-toggle` container class and `.toggle-btn` / `.toggle-btn.active` classes from `bee-sidebar.ts` static styles. No new CSS rules needed for the toggle itself — the pattern is fully established.

**Button labels:** "Map" and "Table" (no emoji — CONTEXT.md D-02 included map emoji in mockup, but final form defers icon choice to Claude's discretion per the decisions section; text-only is safer and consistent with the existing "Specimens"/"Samples" tab which uses no icons).

**Active state:** Active button: `color: var(--accent)`, `border-bottom-color: var(--accent)`, `font-weight: 600`. Inactive: `color: var(--text-hint)`, no border indicator.

**Interaction:** Click on inactive button emits `view-changed` CustomEvent with `detail: 'map' | 'table'`, bubbles + composed. Click on already-active button is a no-op.

### Table Slot Placeholder (new)

In table view, `bee-atlas` renders a `<div class="table-slot">` in the flex row where `<bee-map>` normally appears. This div takes `flex-grow: 1` and has no visual content in Phase 39.

```css
.table-slot {
  flex-grow: 1;
  background: var(--surface);
}
```

No loading indicator, no empty state message — per CONTEXT.md D-07, this is a blank container that Phase 40 populates with `<bee-table>`.

## Interaction States

| State | Visual |
|-------|--------|
| Map view (default) | "Map" button active (green underline + green text), `<bee-map>` rendered |
| Table view | "Table" button active (green underline + green text), `<div class="table-slot">` rendered, `<bee-map>` absent from DOM |
| Toggle button hover | Background `var(--surface-subtle)`, text `var(--text-secondary)` |
| Toggle button pressed (active) | No change (button is already active, no separate pressed state needed) |

---

## Layout

**Map view (default):** `bee-atlas` flex row: `<bee-map flex-grow:1>` + `<bee-sidebar width:25rem>`. Unchanged from current.

**Table view:** `bee-atlas` flex row: `<div class="table-slot" flex-grow:1>` + `<bee-sidebar width:25rem>`. Sidebar stays at full width; table slot occupies the left/main area.

**Mobile (portrait, `max-aspect-ratio: 1`):** `bee-atlas` flex column. Table slot takes `flex-grow: 1` (same as map). Sidebar is below at `width: 100%`.
