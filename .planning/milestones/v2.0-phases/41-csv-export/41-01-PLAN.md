       ```typescript
       private async _onDownloadCsv() {
         try {
           const rows = await queryAllFiltered(this._filterState, this._layerMode);
           if (rows.length === 0) return;
           const headers = Object.keys(rows[0]);
           const csvLines = [
             headers.join(','),
             ...rows.map(row =>
               headers.map(h => {
                 const val = row[h];
                 const str = val == null ? '' : String(val);
                 return str.includes(',') || str.includes('"') || str.includes('\n')
                   ? '"' + str.replace(/"/g, '""') + '"'
                   : str;
               }).join(',')
             ),
           ];
           const csvContent = csvLines.join('\n');
           const blob = new Blob([csvContent], { type: 'text/csv;charset=utf-8;' });
           const url = URL.createObjectURL(blob);
           const filename = buildCsvFilename(this._filterState, this._layerMode);
           const a = document.createElement('a');
           a.href = url;
           a.download = filename;
           a.style.display = 'none';
           document.body.appendChild(a);
           a.click();
           document.body.removeChild(a);
           URL.revokeObjectURL(url);
         } catch (err) {
           console.error('CSV download failed:', err);
         }
       }
       ```
       This follows the standard browser download pattern per Claude's Discretion in CONTEXT.md.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| DuckDB SQL queries | FilterState values are interpolated into SQL strings (existing pattern from buildFilterSQL) |
| CSV content | User-generated data (collector names, species names) rendered into downloadable file |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-41-01 | Injection | queryAllFiltered SQL | accept | Uses existing buildFilterSQL which escapes single quotes. Column list is hardcoded (no user input). Same pattern as queryTablePage already in production. |
| T-41-02 | Information Disclosure | CSV export | accept | CSV exports the same data already visible in the table UI. No additional data exposure beyond what the user can already see in the browser. All columns are from public parquet files served via static hosting. |
| T-41-03 | Tampering | CSV filename | mitigate | slugify function strips all characters except a-z, 0-9, and hyphens. Segment length capped at 20 chars. Prevents path traversal or special chars in filenames. |
| T-41-04 | Denial of Service | Large CSV generation | accept | Data volume is bounded by the parquet file size (already fully loaded into DuckDB WASM in browser). No amplification vector. Worst case: browser tab uses more memory briefly during CSV string construction. |
</threat_model>
