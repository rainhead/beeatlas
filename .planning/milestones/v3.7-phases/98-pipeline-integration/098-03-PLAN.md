<objective>
Generate per-place SVG occurrence maps following the established `species_maps.py` pattern.
For every slug in `content/places.toml`, write `public/data/place-maps/{slug}.svg` with the
WA county backdrop and an `<circle class="occ">` per occurrence whose `place_slug` matches.
Reuse the SVG helpers (`_load_county_geojsons`, `_build_county_backdrop`, `_write_species_svg`)
from species_maps.py rather than duplicating them.

Purpose: PPAGE-03 — the per-place static map that Phase 99 embeds on each `/places/{slug}.html`
page. Byte-stability matters because these SVGs may end up in git or in CloudFront caches.

Output: data/places_maps.py + run.py STEPS wiring + test_places_maps.py + public/data/place-maps/
populated on the next pipeline run. SVGs themselves are NOT committed to git (they live under the
blanket `/public/data/` ignore — only places.geojson + places.json have negation rules).
</objective>
