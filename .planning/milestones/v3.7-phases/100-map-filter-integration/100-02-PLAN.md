## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GeoJSON data → Mapbox source | `places.geojson` feature.properties.slug consumed by click handler |
| places.json → DOM | `places.json` records' `name` field rendered as chip label text |
| Click event → CustomEvent | Map click translates to a `place-selected` event whose `detail.slug` will be SQL-interpolated by Plan 03 (via FilterState → buildFilterSQL) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-100-04 | Injection (XSS) | bee-filter-panel chip label renders `_placeNameBySlug.get(slug)` from places.json | mitigate | Lit's `html` template literal escapes interpolated text automatically — `${displayName}` is rendered as a text node, not parsed as HTML. The chip uses `${...}` interpolation, NOT `unsafeHTML(...)`. Any malicious content in places.json would render as literal text. Defense-in-depth: place names are sourced from `content/places.toml`, which is maintainer-controlled per Phase 97 governance — no untrusted input path. |
| T-100-05 | Injection (SQL) | Place click → slug → FilterState.selectedPlace → buildFilterSQL | mitigate | Quote-escaping is enforced in Plan 01's buildFilterSQL (single-quote doubled, asserted by filter.test.ts). Slug values come from `places.geojson`, which is generated from validated TOML where slug regex `[a-z0-9-]` is enforced at pipeline time (Phase 97 PLC-03). No SQL metacharacter can reach the WHERE clause from a click. |
| T-100-06 | Spoofing | Untrusted click producing a slug not in the actual places table | accept | wa-sqlite is a read-only in-memory store. A spoofed slug yields zero matching rows — the worst outcome is an empty filter result, identical to any non-matching filter. No data exposure or modification possible. |
| T-100-07 | Tampering | Mapbox feature-state mutation via developer console | accept | Feature-state is client-side display only. Mutating it cannot affect SQL filtering, which reads `filterState.selectedPlace` not feature-state. No privileged surface exposed. |
| T-100-SC | Tampering (supply chain) | npm/pip/cargo installs | mitigate | No new package installs in this plan. All work extends existing source files. No package legitimacy audit required. |
</threat_model>
