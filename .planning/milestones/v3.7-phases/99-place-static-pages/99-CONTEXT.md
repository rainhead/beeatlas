# Phase 99: Place Static Pages - Context

**Gathered:** 2026-05-17
**Status:** Ready for planning

<domain>
## Phase Boundary

Generate two Eleventy static page groups from `places.json`:
1. `/places.html` — index listing all collecting locations (name, land owner, specimen count)
2. `/places/{slug}.html` — per-place detail page (name, land owner, specimen count, SVG occurrence map, deep-link to filtered SPA map)

No permit table — permits are deferred out of this milestone entirely. The per-place page scope is name + owner + count + SVG map + deep-link only.

</domain>

<decisions>
## Implementation Decisions

### Scope Change
- **D-01:** Permit table is **out of scope** for Phase 99 (and for the v3.7 milestone). Do not implement any permit display. Requirements PPAGE-02's "permit table with active/inactive/no-expiry status" clause is superseded by this decision. Update REQUIREMENTS.md and ROADMAP.md success criteria to remove the permit table before or during planning.

### URL Format
- **D-02:** Direct-path URLs throughout — `/places.html` for the index, `/places/{slug}.html` for per-place pages. Do NOT use trailing-slash directories (`/places/{slug}/`). CloudFront does not redirect `/foo/` to `/foo/index.html`.

### Data Module
- **D-03:** `_data/places.js` reads `public/data/places.json` at build time, following the `_data/species.js` pattern exactly. Exposes a `places` global to Nunjucks templates (array of place objects with slug, name, land_owner, specimen_count, sample_count fields).

### Templates
- **D-04:** Index page (`_pages/places.njk`) follows the species index pattern — a compact list, no JS entry. Per-place pages (`_pages/place-detail.njk`) use Eleventy pagination over `places.placesArray` (size: 1), `permalink: "/places/{{ place.slug }}.html"`.

### Detail Page Layout
- **D-05:** Follow the species-detail.njk structure: SVG map in a `media-grid` div at top, then a metadata line (specimen count, land owner), then the deep-link. No photo section (places have no photos).

### Deep-Link
- **D-06:** Deep-link anchor: `<a href="/?place={{ place.slug }}">View occurrences on the atlas →</a>`. Phase 100 implements the JS side (PMAP-04). For now this is a valid URL that Phase 100 will activate.

### SVG Map Path
- **D-07:** SVG maps served from `/data/places-maps/{{ place.slug }}.svg` (generated by Phase 98 pipeline). Only render the `<img>` if the place has occurrences (match the `if sp.occurrence_count > 0` guard in species-detail.njk).

### CSS
- **D-08:** New `src/styles/places.css` following the `src/styles/species.css` pattern. Import in the place page templates, not globally.

### No JS Entry
- **D-09:** Both places pages are fully static Nunjucks — no TypeScript entry point, no Vite bundle. No `<script type="module">` tag on either page.

### Claude's Discretion
- Index page: exact HTML structure within the compact-list approach (ul vs table, column order within the row)
- Whether to add `<title>` computed per-place (should, following `eleventyComputed` pattern in genus.njk)

</decisions>
