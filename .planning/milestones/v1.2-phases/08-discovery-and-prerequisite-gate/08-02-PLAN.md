Modify the `build` job in `.github/workflows/deploy.yml`:

1. Add a `permissions` block to the `build` job (same as the `deploy` job):
```yaml

      id-token: write   # Required for OIDC JWT request — must be on THIS job
      contents: read
```

2. Insert a `Configure AWS credentials via OIDC` step AFTER `Install dependencies` and BEFORE `Build`:
```yaml
      - name: Configure AWS credentials via OIDC
        uses: aws-actions/configure-aws-credentials@v4
        with:
          role-to-assume: ${{ vars.AWS_DEPLOYER_ROLE_ARN }}
          aws-region: us-west-2
```

The `build` job should look like this when done:
```yaml
  build:
    name: Build frontend
    runs-on: ubuntu-latest
    permissions:
      id-token: write   # Required for OIDC JWT request — must be on THIS job
      contents: read
    steps:
      - uses: actions/checkout@v4

      - uses: astral-sh/setup-uv@v5

      - uses: actions/setup-node@v4
        with:
          node-version: '22'
          cache: 'npm'

      - name: Install dependencies
        run: npm ci

      - name: Configure AWS credentials via OIDC
        uses: aws-actions/configure-aws-credentials@v4
        with:
          role-to-assume: ${{ vars.AWS_DEPLOYER_ROLE_ARN }}
          aws-region: us-west-2

      - name: Build
        run: npm run build
```

Do NOT change the `deploy` job — it already has the correct credentials setup.
