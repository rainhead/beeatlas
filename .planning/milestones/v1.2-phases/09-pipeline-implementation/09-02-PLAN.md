Create `data/inat/download.py`. The script must be run from the `data/` directory (working directory matters — file paths are relative).

Key implementation notes:

**pyinaturalist returns model objects, not raw dicts.** Access fields via model attributes:
- `obs.id` → int
- `obs.user.login` → str
- `obs.observed_on` → date object; call `str()` to get ISO string
- `obs.location` → tuple `(lat, lon)` — already correct order (NOT GeoJSON order)
- `obs.ofvs` — list of ObservationFieldValue objects; call `obs.to_dict().get('ofvs', [])` to get raw dict list for extract_specimen_count

**Do NOT** use `obs.get('geojson')` — that is raw dict API, not model attribute access. Use `obs.location` (already lat/lon tuple).

**Incremental fallback:** If incremental fetch throws any exception, catch it, print a warning, and fall back to full fetch. Do NOT let a failed incremental fetch break the build.

**last_fetch.txt timing:** Write ONLY after `merged.to_parquet()` returns without error. Use `datetime.now(timezone.utc).isoformat()` format.

**Progress logging format (matches success criterion 5):**
```
[inat] Full fetch / Incremental fetch since {timestamp}
[inat] Fetched {N} observations (~{pages} pages)
[inat] Total in parquet: {total}
[inat] specimen_count null rate: {rate:.1%}
[inat] Wrote samples.parquet and last_fetch.txt ({timestamp})
```

**File paths:** Use `pathlib.Path` with relative paths: `SAMPLES_PATH = Path("samples.parquet")`, `LAST_FETCH_PATH = Path("last_fetch.txt")`. The script is always run from `data/` (via `npm run fetch-inat` which does `cd data && uv run python inat/download.py`).

**Export these names** so tests can import them: `fetch_all`, `fetch_since`, `obs_to_row`, `build_dataframe`, `merge_delta`, `main`.
