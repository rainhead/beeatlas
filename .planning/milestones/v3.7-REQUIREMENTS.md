# Requirements Archive: Washington Bee Atlas v3.7 Places

**Archived:** 2026-05-18 (milestone complete)
**Original defined:** 2026-05-17
**Core Value:** Tighten learning cycles for volunteer collectors; surface existing data in ways difficult to achieve without the site; convey liveness and togetherness among participants.

## v3.7 Requirements — All Complete

### PLC — Place Data Model

- [x] **PLC-01**: Coordinator can define a place via a `content/places.toml` entry with slug, name, land_owner, geometry_wkt (WGS84), and a permits array — Phase 97 ✓
- [x] **PLC-02**: Each permit record carries: issuing_authority, permit_number (optional), expiry_date (nullable ISO date), and type (project-level vs site-level) — Phase 97 ✓ (W-02: not runtime-enforced; see deferred items)
- [x] **PLC-03**: Build fails if any place has an invalid or non-WGS84 geometry, a duplicate slug, or a slug with characters outside `[a-z0-9-]` — Phase 97 ✓
- [x] **PLC-04**: Build fails if any two place polygons overlap (ST_Intersects check at validation time) — Phase 97 ✓

### PPIPE — Pipeline Integration

- [x] **PPIPE-01**: Pipeline loads places.toml into a `geographies.places` DuckDB table before dbt runs — Phase 98 ✓
- [x] **PPIPE-02**: occurrences.parquet gains a `place_slug` column via ST_Within LEFT JOIN; occurrences outside all places get NULL (no nearest-polygon fallback) — Phase 98 ✓
- [x] **PPIPE-03**: dbt `schema.yml` updated to 31 columns atomically with the `place_slug` addition — Phase 98 ✓
- [x] **PPIPE-04**: Pipeline exports `public/data/places.geojson` (slug + geometry only, for Mapbox) and `public/data/places.json` (all metadata + specimen/sample counts, no geometry, for Eleventy) — Phase 98 ✓
- [x] **PPIPE-05**: `places.geojson` and `places.json` committed to git so CI frontend-only builds succeed without running the pipeline — Phase 98 ✓

### PMAP — Map and Filter Integration

- [x] **PMAP-01**: Boundary mode toggle extended to Off / Counties / Ecoregions / Places; mutually exclusive; place boundaries shown in a visually distinct color — Phase 100 ✓
- [x] **PMAP-02**: Clicking a place boundary polygon applies that place as the active filter — Phase 100 ✓
- [x] **PMAP-03**: Place filter chip appears in the filter panel when a place is active; removable; ghosts occurrences outside the polygon — Phase 100 ✓
- [x] **PMAP-04**: `place=` URL param encodes the active place slug; restored on page load; deep-link from place pages opens the map with that place pre-filtered — Phase 100 ✓

### PPAGE — Place Static Pages

- [x] **PPAGE-01**: `/places.html` index page lists all places with name, land owner, and specimen count — Phase 99 ✓
- [x] **PPAGE-02**: Per-place page at a direct-path URL shows name, owner, specimen count, SVG occurrence map, and a link that opens the main map with that place's filter applied — Phase 99 ✓
- [x] **PPAGE-03**: Per-place SVG occurrence map generated at pipeline time, following the `species_maps.py` pattern — Phase 98 ✓; B-01 blocker (S3 upload missing) fixed in Phase 100.1 ✓

## Out of Scope

| Feature | Reason |
|---------|--------|
| Community-editable place metadata | Static hosting + legal sensitivity of permit data; maintainer-curated TOML with git history is the governance model |
| Real-time permit status from agency APIs | No permit APIs exist; nightly pipeline constraint precludes runtime checks |
| All-WA public lands layer | Scope and dataset size; would pull in thousands of polygons beyond curated collecting sites |
| Annual specimen trend charts per place | HIGH pipeline complexity; defer to v3.8+ |
| Nearest-polygon fallback for unmatched occurrences | Semantically wrong for places — most occurrences are not at any named site |
| Permit display (table or summary) | Removed from v3.7 milestone per Phase 99 decision D-01; revisit in v3.8+ when permit tracking resurfaces |
