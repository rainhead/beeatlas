# Requirements: Washington Bee Atlas v2.0

**Defined:** 2026-04-07
**Core Value:** Collectors can see where bees have been collected and where target host plants grow, enabling informed planning of future collecting events.

## v2.0 Requirements

### View Mode

- [x] **VIEW-01**: User can toggle between map view and table view via a control in the main UI — *Shipped Phase 39*
- [x] **VIEW-02**: In table view, the map is not rendered, giving the table full layout space — *Shipped Phase 39*
- [x] **VIEW-03**: View mode (map/table) is encoded in the URL so the table view is bookmarkable and shareable — *Shipped Phase 39*

### Table Component

- [x] **TABLE-01**: Table displays specimens when layer mode is "specimens", samples when layer mode is "samples" — *Shipped Phase 40*
- [x] **TABLE-02**: Table reflects the active filter state — rows shown match `visibleIds` (or all rows if no filter active) — *Shipped Phase 40*
- [x] **TABLE-03**: Table shows a row count indicator (e.g. "showing 1–100 of 3,847 specimens") — *Shipped Phase 40*
- [x] **TABLE-04**: Table is paginated at 100 rows/page with previous/next controls and current page display — *Shipped Phase 40*
- ~~**TABLE-05**: User can sort by clicking a column header; clicking again reverses sort direction~~ *(deferred — implemented then removed in UI refactor; hardcoded date DESC order ships in v2.0)*
- [x] **TABLE-06**: Specimen table columns: species, collector (recordedBy), year, month, county, ecoregion, field number — *Shipped Phase 40*
- [x] **TABLE-07**: Sample table columns: observer, date, specimen count, county, ecoregion — *Shipped Phase 40*

### CSV Export

- [x] **CSV-01**: User can download the full filtered result set as a CSV via a "Download CSV" button (no pagination limit) — *Shipped Phase 41*
- [x] **CSV-02**: Downloaded filename reflects the active filter (e.g. `specimens-bombus-2023.csv`, `samples-all.csv`) — *Shipped Phase 41*

## Future Requirements

### Enhanced Table

- **TABLE-08**: User can choose which columns to display (column visibility toggle)
- **TABLE-09**: Virtual scrolling replaces pagination for seamless infinite-scroll browsing

### Advanced Export

- **CSV-03**: User can export only the current page (not full result set)
- **CSV-04**: Export includes all filter metadata as a header comment in the CSV

## Out of Scope

| Feature | Reason |
|---------|--------|
| Virtualized scrolling (this milestone) | Pagination sufficient for v2.0; add later if UX demands it |
| Column chooser (this milestone) | Deferred — fixed column set is simpler and covers all use cases |
| In-table filtering | Filter controls in sidebar already handle this |
| Row selection / bulk actions | No clear use case beyond CSV export which is already covered |
| Server-side pagination | Static hosting constraint — all queries run in DuckDB WASM |
