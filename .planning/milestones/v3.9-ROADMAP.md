## Overview

Collapsed the separate filter panel, occurrence sidebar, and full-screen table into a single unified `bee-pane` component with three states (collapsed/list/table) on desktop. The pane is always visible via a floating toggle button; a selection+filter unified occurrence query drives the list view; the table renders as a split-screen (40% map / 60% table). Legacy `bee-filter-panel.ts` and `bee-sidebar.ts` deleted.

## Milestone Summary

**Key Decisions:**

- URL-state uses `paneState: 'collapsed' | 'list' | 'table'` (not `viewMode: 'map' | 'table'`); legacy `?view=table` preserved via Option A precedence chain
- MAP-01 satisfied by overlay architecture: `bee-pane` is `position:absolute`; `bee-map` element dimensions never change across pane transitions; no explicit `map.resize()` call needed
- Three-flag view state (`_viewMode + _sidebarOpen + _tableFilterOpen`) replaced with single discriminated-union `@state() private _paneState` — cleaner state machine
- `queryListPage` uses WHERE intersection for selection + filter (not priority sort) — unified occurrence model
- Table-mode collapse goes to `'collapsed'` not `'list'` (D-08 from v2.9 preserved)
- `_open` field retained in bee-pane for suggestion dropdown management even though it no longer gates the filter panel itself

**Issues Resolved:**

- URL-01/URL-02: pane state round-trip through URL (previously tracked as pending; Phase 105 delivered)
- MAP-01: map resize on pane transition (overlay architecture; Phase 108 delivered)
- List refresh on filter change when pane open (gap closed in Phase 109-06)
- Pane-close overlap with expand button (gap closed in Phase 109-05)
- Scroll containment in occurrence list (gap closed in Phase 109-05)

**Technical Debt Incurred:**

- Phase 108 UAT found 5 UI regressions (Mapbox attribution z-index, sidebar button order, table close button, row-pan auto-shrink, map header icon) — all fixed before milestone close
