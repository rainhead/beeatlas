## Cross-Phase Integration — Clean

Integration checker (gsd-integration-checker) traced the full pipeline and found
**0 broken flows, 0 missing connections, 18 wired connections**.

**dbt DAG (unbroken):** `stg_checklist__records_full` → `int_checklist_collapsed` →
`int_dedup_candidates` / `int_checklist_dedup_status` → `int_combined` ARM 4 →
`occurrences` mart. The three columns promoted in Phase 138
(`verbatim_name`, `locality`, `collapsed_count`) exist in `int_checklist_collapsed`,
pass through `int_checklist_dedup_status`, and survive into ARM 4 and the mart.

**Contract consistency:** schema.yml occurrences model = 37 columns =
occurrences.sql SELECT; all 4 int_combined ARMs emit aligned columns/types
(ARMs 1–3 emit typed NULLs for the 4 checklist-only columns). No 34-vs-37 mismatch
left behind between Phase 137 and 138.

**geo_blob positional contract:** `sqlite_export._GEO_COLS` index 7 = `checklist_id`
matches `src/features.ts` `row[7]` decode → `occId = 'checklist:<N>'`. The export
SELECT addresses columns by name, so order drift is harmless.

**run.py STEPS ordering:** `resolve-checklist-names` (no-op nightly) →
`checklist-resolution-gate` → `dbt-build` → `dedup-candidates` → `dedup-gate` →
`generate-sqlite`. No step reads an artifact a later step produces.

**E2E flow (5 flows complete):** checklist CSV row → synonym applied → dedup filter →
promoted as `source='checklist'` → green point → clickable → detail card →
source toggle hides → URL `src=checklist` round-trips → selection survives reload.
