## v3.6 Requirements

Replace the monolithic /species/ all-cards page with a proper per-taxon page architecture: a searchable family→genus index as entry point, plus individual pages for tribes, genera, subgenera, and species.

### URL Routing

- [x] **URL-01**: Each species has a dedicated page at `/species/{Genus}/{specificEpithet}/` where Genus preserves original capitalization and specificEpithet is lowercase (hyphens permitted in epithet) — Phase 94 ✓
- [x] **URL-02**: Each genus has a dedicated page at `/species/{Genus}/` — Phase 94 ✓
- [x] **URL-03**: Each subgenus has a dedicated page at `/species/{Genus}/{Subgenus}/` (both capitalized, distinguishing it from lowercase specific epithets) — Phase 95 ✓
- [x] **URL-04**: Each tribe has a dedicated page at `/species/tribe/{TribeName}/` — Phase 95 ✓
- [x] **URL-05**: The existing `/species/` all-cards single-page layout (tree-nav + per-species card wall) is replaced entirely by the new index page — Phase 96 ✓

### Index Page

- [x] **IDX-01**: `/species/` lists all species grouped by family, then by genus within each family — Phase 96 ✓
- [x] **IDX-02**: A type-to-filter text input narrows the displayed genera and species as the user types — Phase 96 ✓
- [x] **IDX-03**: Clicking a genus name navigates to `/species/{Genus}/` — Phase 96 ✓
- [x] **IDX-04**: Clicking a species name navigates to `/species/{Genus}/{specificEpithet}/` — Phase 96 ✓

### Genus Pages

- [x] **GEN-01**: Genus page lists all species belonging to that genus with specimen counts — Phase 94 ✓
- [x] **GEN-02**: Genus page displays a static SVG occurrence map with each species rendered in a distinct color — Phase 94 ✓ (BLOCKER-01 closed; S3 upload added)
- [x] **GEN-03**: Each species entry on the genus page links to its individual species page — Phase 94 ✓

### Subgenus Pages

- [x] **SUBG-01**: Subgenus page lists species belonging to that subgenus with specimen counts — Phase 95 ✓
- [x] **SUBG-02**: Subgenus page displays a multi-color static SVG occurrence map for species in the subgenus — Phase 95 ✓ (BLOCKER-01 closed)
- [x] **SUBG-03**: Each species entry links to its individual species page — Phase 95 ✓

### Tribe Pages

- [x] **TRIBE-01**: Tribe page lists all genera belonging to that tribe — Phase 95 ✓
- [x] **TRIBE-02**: Tribe page displays a multi-color static SVG occurrence map for all species in the tribe — Phase 95 ✓ (BLOCKER-01 closed)
- [x] **TRIBE-03**: Each genus entry links to its genus page — Phase 95 ✓

### Species Pages

- [x] **SPE-01**: Each species in the WA checklist has a dedicated static page at `/species/{Genus}/{specificEpithet}/` — Phase 94 ✓
- [x] **SPE-02**: Species page displays photo(s) from `content/species-photos.toml` (fallback when none) — Phase 94 ✓
- [x] **SPE-03**: Species page displays a static SVG occurrence map for that species alone — Phase 94 ✓ (BLOCKER-01 closed)
- [x] **SPE-04**: Species page displays a seasonality visualization — Phase 94 ✓

### Pipeline & Build

- [x] **PIPE-01**: Eleventy generates one static page per species, genus, subgenus, and tribe from `species.json` data — Phase 94/95 ✓
- [x] **PIPE-02**: `species_maps.py` generates multi-color SVG occurrence maps for genus, subgenus, and tribe pages — Phase 93 ✓
- [x] **PIPE-03**: `species_export.py` updates the `slug` field to the new hierarchical path format; `content/species-photos.toml` keys migrated to match — Phase 92 ✓
