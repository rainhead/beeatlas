# Milestone v2.5: Elevation Data

**Status:** ✅ SHIPPED 2026-04-16
**Phases:** 55–58
**Total Plans:** 7

## Overview

Annotate specimens and samples with inferred elevation (meters) from the USGS 3DEP DEM, surface elevation in sidebar detail panels and a filter toolbar range input.

## Phases

### Phase 55: DEM Acquisition Module

**Goal**: A tested Python module can download the USGS 3DEP DEM for Washington and sample elevation at arbitrary coordinates
**Depends on**: Phase 54
**Requirements**: ELEV-01
**Plans**: 1 plan

Plans:
- [x] 55-01-PLAN.md — Add dependencies, create dem_pipeline.py with ensure_dem and sample_elevation, unit tests with synthetic fixture

**Details:**
`dem_pipeline.py` with `ensure_dem(cache_dir)` (download+cache USGS 3DEP 10m WA DEM via seamless-3dep) and `sample_elevation(lons, lats, dem_path)` (rasterio sampling with nodata from `dataset.nodata`). 5 unit tests using synthetic 2x2 GeoTIFF fixture, no network access required.

### Phase 56: Export Integration

**Goal**: Both parquet export files contain a nullable `elevation_m` INT16 column populated from the DEM, and CI enforces its presence
**Depends on**: Phase 55
**Requirements**: ELEV-02, ELEV-03, ELEV-04
**Plans**: 2 plans

Plans:
- [x] 56-01-PLAN.md — Wire elevation sampling into export.py, update schema gate, add pyarrow dep
- [x] 56-02-PLAN.md — Add elevation integration tests to test_export.py

**Details:**
`export.py` gains `_add_elevation()` helper using pyarrow post-processing (read parquet → sample elevation → append INT16 column → rewrite). `validate-schema.mjs` enforces `elevation_m` in both parquet files. Fixed geometry_wkt schema mismatch in test fixtures that had blocked the full pytest suite since Phase 47.

**Note:** `dem_pipeline.py` was built (Phase 55) then immediately dropped in Phase 56 — `ecdysis_data.occurrences` already has `minimum_elevation_in_meters` (DarwinCore field, ~96% coverage). Ecdysis `elevation_m` now comes from source SQL inline; iNat sample `elevation_m` is always null (no source field).

### Phase 58: Elevation Filter

**Goal**: Users can filter the map and table to specimens and samples within an elevation range, with the range bookmarkable in the URL
**Depends on**: Phase 57
**Requirements**: ELEV-07, ELEV-08, ELEV-09
**Plans**: 2 plans

Plans:
- [x] 58-01-PLAN.md — Extend FilterState, buildFilterSQL, isFilterActive, URL state, and tests
- [x] 58-02-PLAN.md — Add elevation number inputs to bee-filter-controls with state sync and CSS

**Details:**
`FilterState` extended with `elevMin`/`elevMax` (number | null). `buildFilterSQL` applies D-06 conditional null semantics: both bounds → `BETWEEN + IS NOT NULL`; one bound → `IS NULL OR >=/<= `; neither → no clause. `elev_min`/`elev_max` URL params with `parseInt || null` guard. Two `<input type="number">` in `.elev-inputs` sibling div outside `.search-section` (z-index scoping). `filterStatesEqual` extended so `updated()` guard ignores own emissions. 19 new tests (SQL clauses, URL round-trips).

## Milestone Summary

**Key Decisions:**

- DEM pipeline built then dropped — Ecdysis source already has `minimum_elevation_in_meters` at ~96% coverage; no need for rasterio sampling for specimens. iNat samples have no elevation source, `elevation_m` is always null for them.
- pyarrow post-processing for parquet column append: read → append INT16 → rewrite. Avoids DuckDB-rasterio integration complexity.
- D-06 conditional null semantics: single-bound passes null-elevation records; both bounds required to exclude nulls (~4% of specimens have null elevation)
- Elevation inputs placed as sibling div outside `.search-section` — dropdown z-index scoping on `.search-section` would clip token dropdown if inputs were inside
- `filterStatesEqual` extended with elevMin/elevMax before dispatching filter-changed — prevents `updated()` loop from component re-receiving own emission

**Issues Resolved:**

- geometry_wkt / geom GEOMETRY schema mismatch in conftest.py and test_feeds.py (Phase 47 partial revert artifact) — fixed in Phase 56-02, unblocking the full pytest suite
- Pre-existing bee-table.test.ts failures (TABLE-01, TABLE-08 sort indicator tests) — out of scope, carried as tech debt

**Technical Debt Incurred:**
