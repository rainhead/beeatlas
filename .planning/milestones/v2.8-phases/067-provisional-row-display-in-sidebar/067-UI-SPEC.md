## Spacing Scale

All spacing in `bee-occurrence-detail` uses em/rem units that map to the existing scale.
Declared values align with the 8-point base (component uses rem units internally):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (0.25rem) | Inline badge margin (`margin-left: 0.4em` ≈ 6-7px — acceptable variance for inline text) |
| sm | 8px (0.5rem) | Gap between stacked event fields in `.sample-dot-detail` (`gap: 0.4rem`) |
| md | 12px (0.75rem) | `.sample` vertical padding (`padding: 0.75rem 1rem`) |
| lg | 16px (1rem) | `.panel-content` padding, `.sample` horizontal padding |
| xl | 20px (1.25rem) | Species list left padding |

Exceptions: The `.sample-dot-detail` gap is `0.4rem` (≈6.4px) — existing value, do not change.
No new spacing values are introduced in Phase 67.

## Typography

All sizes are existing CSS classes in `bee-occurrence-detail.ts`. Phase 67 adds no new
type roles — new content slots into existing roles.

This phase governs exactly 2 font weights:

| Role | Size | Weight | Line Height | CSS Class / Element |
|------|------|--------|-------------|---------------------|
| Event date / section header | 0.85rem (≈13.6px) | 600 semibold | 1.5 (inherited) | `.event-date`, `.sample-header` |
| Body / label text | 0.8rem (≈12.8px) | 400 regular | 1.5 (inherited) | `.event-observer`, `.event-count`, `.event-elevation`, `.sample-meta` |
| iNat ID label | 0.8rem (≈12.8px) | 400 regular | 1.5 (inherited) | new `.inat-id-label` — inherits body size |
| Taxon name (italic) | 0.8rem (≈12.8px) | 400 regular, italic | 1.5 (inherited) | `<em>` inside `.inat-id-label` |
| Quality badge | 0.7rem (≈11.2px) | 400 regular | n/a (inline-block) | `.quality-badge` (existing) |

**Link weight:** `<a>` elements inherit `font-weight: 500` from the project-level global
`a` rule in `index.css`. Phase 67 does not change link styling and does not own this
weight. It is out of scope for this phase's typography contract.

**Taxon name styling:** Render as `<em>` (italic) inside the iNat ID label. Italic conveys
scientific name convention, matching the existing `.species-list` italic treatment for
specimen scientific names.

## Component Inventory

Phase 67 modifies one existing component. No new components are created.

### Modified: `bee-occurrence-detail` (`frontend/src/bee-occurrence-detail.ts`)

#### New rendering branch: Sample-only rows (SID-01)

**Condition:** `ecdysis_id == null && !row.is_provisional`

**Change from current:** Add "N specimens collected, identification pending" as a labeled
text element above the existing sample context fields. The `observer` field reference
is updated to `host_inat_login` (Phase 66 rename, D-07).

**Layout (top to bottom in `.panel-content.sample-dot-detail`):**

1. `.event-date` — formatted date (existing, unchanged)
2. `.event-observer` — `host_inat_login` value (field rename from `observer`)
3. `.event-count` — "N specimens collected, identification pending" (exact copy — see Copywriting)
4. `.event-elevation` — elevation in metres if present (existing, unchanged)
5. iNat host observation link if `observation_id` present (existing, unchanged)

**Copy change for `.event-count`:** The existing copy is "N specimens" or "not recorded".
Phase 67 replaces this with the locked success criteria copy (see Copywriting Contract below).

#### New rendering branch: Provisional rows (SID-02)

**Condition:** `row.is_provisional === true`

**Layout (top to bottom in `.panel-content.sample-dot-detail`):**

1. `.inat-id-label` — "iNat ID: [taxon name (italic)] [quality badge]" (new element)
2. `.event-date` — formatted date (reuse `_formatSampleDate`)
3. `.event-observer` — `host_inat_login` value
4. `.event-count` — "N specimens collected" (count only, no "identification pending" suffix — the iNat ID line already communicates status)
5. `.event-elevation` — elevation in metres if present
6. `.event-inat` — "View WABA observation" link to `https://www.inaturalist.org/observations/${row.specimen_observation_id}` (always present for provisional rows)

**iNat ID label structure (pseudo-HTML):**
```
<div class="inat-id-label">
  iNat ID: <em>${row.specimen_inat_taxon_name}</em>
  <span class="quality-badge ${row.specimen_inat_quality_grade}">
    ${grade abbreviation}
  </span>
</div>
```

Grade abbreviations (match existing `_renderHostInfo` pattern):
- `research` → "RG"
- `needs_id` → "NID"
- `casual` → "casual"

**Fallback if `specimen_inat_taxon_name` is null:** Render `<span class="hint">identification pending</span>` in place of the `<em>` taxon name.

**Fallback if `specimen_inat_quality_grade` is null:** Omit the badge entirely.

#### New CSS class: `.inat-id-label`

```css
.inat-id-label {
  font-size: 0.8rem;
  color: var(--text-body);
  font-weight: 400;
}
```

No other new CSS classes. No new CSS custom properties.

#### Render method dispatch (updated logic)

Current split: `specimenBacked` (ecdysis_id non-null) vs `sampleOnly` (ecdysis_id null).

Phase 67 splits `sampleOnly` into two sub-groups:

```
specimenBacked  → _renderSpecimenGroup (unchanged)
sampleOnly where is_provisional === true  → _renderProvisional (new method)
sampleOnly where !is_provisional          → _renderSampleOnly (updated)
```

#### Optional helper: `_renderQualityBadge(grade: string | null)`

At Claude's discretion: extract a shared helper used by both `_renderHostInfo` (existing)
and `_renderProvisional` (new). This eliminates duplication of the badge conditional and
abbreviation map. Recommended — implement if the executor judges it cleaner.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Sample-only count label (SID-01) | "N specimens collected, identification pending" where N is the numeric count. If count is 1: "1 specimen collected, identification pending". If count is null/NaN: "identification pending" (omit count phrase entirely). |
| Provisional iNat ID label prefix (SID-02) | "iNat ID:" followed by the taxon name and quality badge. Example: "iNat ID: Bombus sp. [NID]". |
| WABA observation link text (SID-02) | "View WABA observation" |
| WABA observation link aria-label | `aria-label="View WABA observation on iNaturalist"` |
| Provisional taxon name null fallback | "identification pending" (`.hint` class — italic, muted) |
| Empty state (no occurrences) | Not in scope for this phase — handled upstream by `bee-sidebar`. |
| Error state | Not in scope — no new async operations introduced. |
| Destructive actions | None in this phase. |

**Copy rationale:**

- "iNat ID:" is not described as "provisional" per D-03 — the quality grade badge communicates certainty. A research-grade WABA observation is a real identification.
- "WABA observation" is used rather than "iNaturalist observation" to distinguish the WABA observation (which carries the ID) from the host observation (already linked as "View on iNaturalist" in sample-only rows). This avoids two links with identical text pointing to different observations.
- "identification pending" on sample-only rows replaces the blank species name that currently appears.

## Accessibility Contract

| Element | Requirement |
|---------|-------------|
| WABA observation link | `aria-label="View WABA observation on iNaturalist"` — link text alone is sufficient but aria-label adds context when read in isolation |
| Taxon name `<em>` | No aria requirement — italic is purely presentational; screen readers pronounce the text correctly |
| Quality badge `<span>` | Add `aria-label` equal to the full grade string: `aria-label="research grade"`, `aria-label="needs ID"`, or `aria-label="casual"` — the abbreviation alone ("RG", "NID") is ambiguous |
| Existing specimen photo link (📷) | Already has `aria-label="View photo on iNaturalist"` — no change needed |
