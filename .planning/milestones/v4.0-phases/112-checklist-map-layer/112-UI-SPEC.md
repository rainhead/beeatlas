## Component Inventory

### New: Checklist Layer Toggle

**Location:** Inside `_renderWhen()` section of `bee-pane.ts`, after the year-bucket checkboxes.

**Pattern to follow:** Exactly mirrors the `.year-label` / `<input type="checkbox">` pattern already in `_renderWhen()`:

```
<label class="year-label">
  <input type="checkbox" .checked=${this._showChecklist}
    @change=${...}
    accent-color: var(--accent)
  />
  Checklist records
</label>
```

**Placement:** Below the year checkboxes (This year / Last year / Earlier), separated by the same `gap: 0.75rem` flex layout. No new visual section; it belongs in the "When" row only if taxon is the real filter surface. However â€” per STATE.md locked decision â€” checklist responds only to taxon filter, NOT year/month. Therefore place the toggle in a new "Show" section (fifth filter row) below the When row, using the same `.filter-row` pattern, with a layers/eye icon (see Icon section).

**Visual states:**
- Unchecked (default): checkbox unchecked, label in `--text-body`, county overlay absent from map
- Checked: checkbox checked with `accent-color: var(--accent)`, label in `--text-body`, county fill overlay active on map

**State propagation:** `bee-pane` owns `_showChecklist: boolean` reactive state. On change, emits a new custom event `checklist-layer-changed` with `detail: { visible: boolean }` (bubbles, composed) to `bee-atlas`, which owns `_checklistVisible: boolean` and passes it to `bee-map` as a property.

### New: Mapbox County Fill Layer (in bee-map)

**Layer type:** Mapbox GL JS `fill` layer on the existing `counties-source` GeoJSON.

**Layer ID:** `checklist-county-fill`

**Visibility:** Controlled by `layout: { visibility: this.showChecklist ? 'visible' : 'none' }` â€” toggled via `bee-map.updated()` when `showChecklist` property changes.

**Filter:** `['in', 'name', ...checklist_counties]` â€” an array of county names from checklist.parquet that match the current taxon filter. Counties without checklist records in the filtered set are not highlighted.

**Paint:**
```
fill-color: rgba(44, 122, 44, 0.25)
fill-outline-color: rgba(44, 122, 44, 0.7)
```

**Z-ordering:** Add the layer below `occurrences` cluster layer so specimen points remain visible on top of checklist county shading.

**Records without coordinates:** Already excluded at the data level â€” checklist.parquet has `lat (nullable)` but the county fill approach does not use lat/lon at all. County-fill approach inherently handles NULL coordinates: a record with a county but no lat/lon still contributes to county presence.

## Interaction Contract

### Toggle Behavior

| Action | Result |
|--------|--------|
| User checks "Checklist records" | County fill layer becomes visible; `cl=1` written to URL |
| User unchecks | County fill layer hidden; `cl` param removed from URL |
| Page load with `cl=1` | Toggle pre-checked; county fill layer initialized visible |
| Page load without `cl` | Toggle unchecked; layer hidden (default) |
| Taxon filter applied while toggle checked | County fill updates to show only counties with matching checklist records |
| Taxon filter cleared while toggle checked | County fill shows all counties with any checklist record |
| Year/month filter applied | No effect on checklist layer (locked decision from STATE.md) |

### URL State

| Param | Value | Meaning |
|-------|-------|---------|
| `cl` | `1` | Checklist layer is visible |
| (absent) | â€” | Checklist layer is hidden (default) |

**Integration point:** `buildParams()` in `url-state.ts` adds `params.set('cl', '1')` when checklist is visible. `parseParams()` reads `const showChecklist = p.get('cl') === '1'` and includes it in the returned `UiState`.

**UiState extension:** Add `checklistVisible: boolean` to the `UiState` interface in `url-state.ts`.

### Data Loading

Checklist county presence data is loaded from `checklist.parquet` via a separate hyparquet fetch (not via SQLite). The query returns distinct county names, optionally filtered by `canonical_name` when a taxon filter is active. This is an async fetch that resolves after the checklist parquet is fetched.

**Loading state:** While checklist data is loading, show the county fill layer as absent (no visual indication needed â€” the layer simply has no counties to highlight yet). No spinner required; the filter panel toggle remains interactive.

**Data URL:** Resolved via `resolveDataUrl('checklist')` from `manifest.ts` â€” same pattern as `occurrences`.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Toggle label | `Checklist records` |
| Toggle aria-label | `Show checklist county records on map` |
| Filter panel section | No new section heading â€” toggle placed in existing "When" row area as a standalone row with a layers icon |

| Error state (checklist parquet fails to load) | Silent fail â€” checklist layer stays hidden, toggle remains checked. Do not surface a toast; this is a secondary data source. Log to console: `console.warn('checklist data unavailable:', err)` |

**Toggle label rationale:** "Checklist records" matches the MAP-01 requirement exactly ("'Checklist records' toggle appears in filter panel"). It parallels the existing "Specimens" and "Samples" terminology in the codebase (those are WABA-specific layer labels in bee-pane). Do not use "Bartholomew checklist" or "Historical records" â€” these would introduce jargon.
