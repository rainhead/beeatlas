### Gaps Summary

**Gap 1: Parquet artifacts still lack the new columns (blocks PIPE-05, PIPE-06)**

The implementation is complete and correct — both pipelines are properly wired to add county and ecoregion_l3 columns. However, the cached parquet files in `data/` and `frontend/src/assets/` are pre-Phase-16 artifacts that were restored from the S3 cache. As a result, `node scripts/validate-schema.mjs` currently exits 1, CI deploy will fail on schema validation, and the frontend build is blocked.

This gap resolves automatically once the fetch-data workflow runs (regenerating parquet files with the new columns and uploading them to S3 cache). However, that workflow has not yet been triggered — it's a post-implementation data refresh step that is expected.

**Gap 2: GeoJSON files do not exist and have no path to exist in CI (blocks PIPE-07)**

`frontend/src/assets/wa_counties.geojson` and `frontend/src/assets/epa_l3_ecoregions_wa.geojson` do not exist anywhere:
- Not committed to git
- Not uploaded to S3 cache (cache_upload.sh only handles parquet)
- Not restored in cache_restore.sh
- Not generated in fetch-data.yml or deploy.yml

The `build-data.sh` script correctly calls `build_geojson.py`, but `build-data.sh` is a local development script — it's not used in any CI workflow. The fetch-data.yml workflow calls individual Python scripts directly (not via build-data.sh). Until either (a) the GeoJSON files are committed to git, (b) they are added to the S3 cache pipeline, or (c) build_geojson.py is called from fetch-data.yml or deploy.yml, these files will never be present during frontend builds.

PIPE-07 states the files are "bundled with the frontend build, and available at runtime." That contract cannot be met with the current CI configuration.

**Resolution options for Gap 2 (in order of simplicity):**
1. Commit the generated GeoJSON files to git (simplest — they are static WA boundary files that rarely change)
2. Add `uv run python scripts/build_geojson.py` to `.github/workflows/fetch-data.yml` and add the GeoJSON files to the S3 cache upload/restore scripts
3. Add GeoJSON generation to the deploy workflow before the build step
