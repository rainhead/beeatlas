<objective>
Regenerate the frontend parquet assets to include county and ecoregion_l3 columns via spatial join.

Purpose: Gap 1 root cause. frontend/src/assets/ecdysis.parquet and samples.parquet were committed before the Phase 16 spatial-join pipeline ran — they have no county or ecoregion_l3 columns. feature.get('county') returns undefined for every feature, so matchesFilter() returns false for all features when any county filter is active, ghosting everything instead of only features outside the selected region. Gap 2 (sidebar list unaffected) is a downstream symptom of the same bug.

Output: Both frontend parquet files with county (string, e.g. "King") and ecoregion_l3 (string, e.g. "North Cascades") columns that match the GeoJSON NAME / NA_L3NAME property values stored in filterState.
</objective>
