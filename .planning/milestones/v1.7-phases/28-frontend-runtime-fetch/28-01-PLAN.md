   - `minTtl: cdk.Duration.seconds(0)`

   - `corsBehavior.accessControlAllowCredentials: false`
   - `corsBehavior.accessControlAllowHeaders: ['*']`
   - `corsBehavior.accessControlAllowMethods: ['GET', 'HEAD']`
   - `corsBehavior.accessControlAllowOrigins: ['*']`
   - `corsBehavior.accessControlExposeHeaders: ['Content-Range', 'Content-Length', 'ETag']`
   - `corsBehavior.originOverride: true`

3. **addBehavior** call:
   ```typescript
   distribution.addBehavior('/data/*',
     origins.S3BucketOrigin.withOriginAccessControl(siteBucket),
     {
       viewerProtocolPolicy: cloudfront.ViewerProtocolPolicy.REDIRECT_TO_HTTPS,
       cachePolicy: dataCachePolicy,
       responseHeadersPolicy: dataCorsPolicy,
     }
   );
   ```

Do NOT modify the defaultBehavior. Do NOT use `CachePolicy.CACHING_OPTIMIZED` (it does not include Origin in cache key). Do NOT add S3 bucket CORS config (not needed with OAC + ResponseHeadersPolicy).
