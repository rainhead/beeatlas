**data/stub_handler.py:**
```python
import os
import boto3
from botocore.exceptions import ClientError

BUCKET = os.environ['BUCKET_NAME']
DB_KEY = 'db/beeatlas.duckdb'
TMP_PATH = '/tmp/beeatlas.duckdb'
SENTINEL_KEY = 'db/stub-sentinel.txt'


def handler(event, context):
    s3 = boto3.client('s3')

    # Download DuckDB file — graceful miss on first run
    print(f"Attempting download: s3://{BUCKET}/{DB_KEY}")
    try:
        s3.download_file(BUCKET, DB_KEY, TMP_PATH)
        size = os.path.getsize(TMP_PATH)
        print(f"Downloaded {DB_KEY}: {size} bytes")
    except ClientError as e:
        if e.response['Error']['Code'] in ('NoSuchKey', '404'):
            print(f"File not found (first run): {DB_KEY}")
            with open(TMP_PATH, 'w') as f:
                f.write('stub-placeholder')
        else:
            raise

    # Verify /tmp write access and DLT_DATA_DIR
    sentinel = '/tmp/dlt/.sentinel'
    os.makedirs('/tmp/dlt', exist_ok=True)
    with open(sentinel, 'w') as f:
        f.write('ok')
    print(f"/tmp write confirmed: {sentinel}")

    # Upload sentinel to S3 (proves write permission on db/* prefix)
    s3.upload_file(TMP_PATH, BUCKET, SENTINEL_KEY)
    print(f"Uploaded sentinel to s3://{BUCKET}/{SENTINEL_KEY}")

    print("S3 round-trip complete")
    return {
        'statusCode': 200,
        'body': 'S3 round-trip complete',
    }
```

Key details:
- Uses `os.environ['BUCKET_NAME']` (injected by CDK as Lambda env var per D-06)
- Catches `ClientError` and checks `e.response['Error']['Code']` for NoSuchKey (not a separate exception class per Pitfall 6)
- Uploads to `db/stub-sentinel.txt` rather than overwriting `db/beeatlas.duckdb` (per research Pattern 6 note)
- Creates `/tmp/dlt/` to verify `DLT_DATA_DIR` path is writable
- Prints "S3 round-trip complete" for CloudWatch log verification (per D-07)

**3. EventBridge Scheduler rules (LAMBDA-04, per D-09):**
```typescript
new Schedule(this, 'NightlyInatSchedule', {
  schedule: ScheduleExpression.cron({
    minute: '0',
    hour: '8',
    timeZone: TimeZone.ETC_UTC,
  }),
  target: new LambdaInvoke(pipelineFn, {
    input: ScheduleTargetInput.fromObject({ pipeline: 'inat' }),
  }),
  description: 'Nightly iNat pipeline refresh',
});

new Schedule(this, 'WeeklyFullSchedule', {
  schedule: ScheduleExpression.cron({
    minute: '0',
    hour: '10',
    weekDay: 'SUN',
    timeZone: TimeZone.ETC_UTC,
  }),
  target: new LambdaInvoke(pipelineFn, {
    input: ScheduleTargetInput.fromObject({ pipeline: 'full' }),
  }),
  description: 'Weekly full pipeline run (all 5 pipelines)',
});
```

Property name is `schedule:` (NOT `expression:`) per Pitfall 2. Cron field is `weekDay:` (NOT `dayOfWeek:`). LambdaInvoke auto-creates the IAM role — no manual policy needed.

**4. Lambda Function URL (LAMBDA-05, per D-11):**
```typescript
const fnUrl = pipelineFn.addFunctionUrl({
  authType: lambda.FunctionUrlAuthType.NONE,
});
```

Using `NONE` auth — volunteer project, manual invocation only, no sensitive data. Property is `fnUrl.url` (NOT `fnUrl.functionUrl` per Pitfall 3).

**5. CfnOutputs (matching existing output style):**
```typescript
new cdk.CfnOutput(this, 'PipelineFunctionUrl', {
  value: fnUrl.url,
  description: 'Lambda Function URL for manual pipeline invocation',
});

  value: pipelineFn.functionArn,

```

Key anti-patterns to avoid:
- Do NOT use `@aws-cdk/aws-scheduler-targets-alpha` (superseded)
- Do NOT use `events.Rule` for scheduling (legacy)
- Do NOT use `bucket.grantReadWrite(fn)` without prefix (would grant access to all objects)
- Do NOT hardcode bucket name (use `siteBucket.bucketName` env var)
- Do NOT omit `platform: Platform.LINUX_AMD64` (ARM Mac builds ARM64 images that fail on x86_64 Lambda)
