## v3.9 Requirements

### Pane Layout

- [x] **PANE-01**: User sees a persistent toggle button at the pane edge in all three pane states (collapsed, list, table) — SATISFIED Phase 107
- [x] **PANE-02**: User can open the pane to list state from collapsed, and collapse it from list or table state, via the toggle button — SATISFIED Phase 107
- [x] **PANE-03**: User can expand the pane to table state via an expand button visible in the pane's list state on desktop — SATISFIED Phase 107
- [x] **PANE-04**: User can return to list state from table state via a shrink button visible in the table state header — SATISFIED Phase 107
- [x] **PANE-05**: Pane's list state shows all filter controls (taxon, date, region, collector, place) and occurrence detail when a cluster is selected — SATISFIED Phase 107
- [x] **PANE-06**: Expand-to-table button is hidden on mobile; pane on mobile behaves as open/close only (no three-state treatment) — SATISFIED Phase 107

### Table in Pane

- [x] **TABLE-01**: Table view in the pane's table state retains all existing functionality (DuckDB-backed pagination, CSV export, filter state integration) — SATISFIED Phase 107
- [x] **TABLE-02**: Full-screen `viewMode='table'` that replaces the map is removed; table is accessible only as a pane sub-state — SATISFIED Phase 109

### URL State

- [x] **URL-01**: Pane state is encoded in the URL and restored on page load (collapsed state omitted from URL; list and table states encoded) — SATISFIED Phase 105
- [x] **URL-02**: Legacy `?view=table` URLs are parsed as pane table state for backward compatibility — SATISFIED Phase 105

### Map Resize

- [x] **MAP-01**: Mapbox canvas resizes correctly after any pane state transition (collapsed↔list, list↔table) — SATISFIED Phase 108 (overlay architecture; no explicit resize call needed)
