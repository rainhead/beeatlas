# Requirements Archive: v2.5 Elevation Data

**Archived:** 2026-04-16
**Milestone status:** ✅ SHIPPED

All requirements validated and shipped in v2.5.

---

## Pipeline & Export

- [x] **ELEV-01**: `dem_pipeline.py` downloads the USGS 3DEP 1/3 arc-second GeoTIFF for Washington bounding box using `seamless-3dep` and caches it locally; subsequent runs skip download if cache exists
  - _Outcome: Built and validated (Phase 55). Module was subsequently superseded — ecdysis elevation sourced from `minimum_elevation_in_meters` DarwinCore field inline; dem_pipeline.py unused in production but retained in repo._
- [x] **ELEV-02**: `export.py` samples elevation at each specimen's lat/lon; nodata sentinel read from `dataset.nodata` (not hardcoded) and converted to NULL; `elevation_m` (INT16, nullable) added to `ecdysis.parquet`
  - _Outcome: Validated (Phase 56). Source is `minimum_elevation_in_meters` SQL column, not rasterio sampling._
- [x] **ELEV-03**: `export.py` samples elevation at each sample's lat/lon using same approach; `elevation_m` (INT16, nullable) added to `samples.parquet`
  - _Outcome: Validated (Phase 56). Column present; always null (no elevation source for iNat samples)._
- [x] **ELEV-04**: `validate-schema.mjs` schema gate enforces `elevation_m` column presence in both parquet files; ships in same commit as `export.py` changes
  - _Outcome: Validated (Phase 56). Schema gate enforced in CI._

## Sidebar Display

- [x] **ELEV-05**: `bee-specimen-detail` shows elevation as "1219 m" (integer, no decimal) when `elevation_m` is non-null; row omitted entirely when null
  - _Outcome: Validated (Phase 57). Strict `!== null` check, `Math.round()` formatting._
- [x] **ELEV-06**: `bee-sample-detail` shows elevation in the same format and null-omit behavior
  - _Outcome: Validated (Phase 57). Identical pattern._

## Filter Toolbar

- [x] **ELEV-07**: Elevation range filter (min/max number inputs) appears in `bee-filter-controls`; encoded as `elev_min=` / `elev_max=` URL params; URL round-trip preserved
  - _Outcome: Validated (Phase 58)._
- [x] **ELEV-08**: `buildFilterSQL` in `filter.ts` applies D-06 conditional null semantics — null rows excluded only when both bounds are set
  - _Outcome: Validated (Phase 58). `BETWEEN + IS NOT NULL` when both set; `IS NULL OR >=/<= ` when one set._
- [x] **ELEV-09**: "Clear filters" resets elevation min/max inputs in addition to existing filter fields
  - _Outcome: Validated (Phase 58). Reset via external `filterState` sync in `updated()`._

## Out of Scope

| Feature | Reason |
|---------|--------|
| Browser-side elevation lookup | Static hosting constraint — DEM too large to ship to browser |
| Feet toggle | Darwin Core standard is meters; no user need identified |
| DEM tiles for outside WA | Specimens outside WA extent are assigned NULL elevation |
| Elevation histogram / distribution chart | Map is the analytical surface; charts are scope creep |
