## Integration Findings

### Fixed: `asyncBufferFromUrl` not imported (was LINK-05 blocker)

**Found by:** integration checker
**Impact:** `loadLinksMap()` always threw `ReferenceError: asyncBufferFromUrl is not defined`. The `.catch(() => new Map())` fallback silently produced an empty links map — all specimen rows showed `iNat: —`, iNat links never appeared.
**Root cause:** Eager-load refactor in this session replaced `asyncBufferFromUrl({url})` (no spaces) with `asyncBufferFromUrlEager(url)` but missed the `asyncBufferFromUrl({ url })` call (with spaces) in `loadLinksMap`.
**Fix applied:** Replaced `asyncBufferFromUrl({ url })` on line 21 with `asyncBufferFromUrlEager(url)`. Build confirmed passing.

### Minor: Back Button Mutates `@property` Directly (MAP-05 concern)

**File:** `frontend/src/bee-sidebar.ts` line 624
**Pattern:** `@click=${() => { this.selectedSampleEvent = null; }}` — mutates an externally-owned `@property`
**Risk:** Works for simple cases. If BeeMap re-renders (map pan, feature load) before the user navigates, `_selectedSampleEvent` is pushed back down and the detail panel reappears.
**Correct fix:** Dispatch `sample-event-back` CustomEvent and handle in BeeMap. Not a blocker for v1.4 — the race window is narrow and the consequence is cosmetic.

### Orphaned Export: `SAMPLE_RECENCY_COLORS` (INFO)

**File:** `frontend/src/style.ts`
**Impact:** None — `sampleDotStyle` renders correctly. Export exists for future legend component.
