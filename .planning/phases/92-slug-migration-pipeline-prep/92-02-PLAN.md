<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| dbt mart `species.parquet` → species_export.py | `genus`, `specific_epithet` columns cross into Python and become substrings of the new slug. The dbt source contract enforces NOT NULL on the WA checklist subset (see `data/dbt/models/marts/schema.yml`). |
| Python string `slug` → `pathlib.Path` (species_maps.py) | The slug becomes the SVG file path component via `out_dir / f"{slug}.svg"`. A malicious slug like `../../../etc/passwd` could escape `out_dir`. |
| Python string `slug` → URL path segment (Eleventy/Nunjucks, _pages/species.njk line 42) | The slug is interpolated into `src="/data/species-maps/{{ sp.slug }}.svg"` at build time. Eleventy auto-escapes HTML attribute content; the build is server-side static, never user-input-driven. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-92-02-01 | Tampering | data/species_export.py slug emission | mitigate | The new slug is constructed only from `genus` and `specific_epithet` columns produced by the dbt mart with the schema.yml NOT NULL + dbt source-test contracts. These columns originate in `data/checklist.csv` (repo-committed) and in upstream Ecdysis records that are read via dlt with column-level type contracts. No untrusted runtime input flows into the slug. The Python f-string is the only construction path — `_slugify` is bypassed for species rows on purpose. |
| T-92-02-02 | Tampering / EoP | data/species_maps.py path-traversal via slug | mitigate | Slug values are pre-validated by the dbt source contract (epithet is a single lowercase token without `..` or absolute-path prefixes; genus is a single capitalized token without separators). The single `/` is the intentional separator. The `out_path = out_dir / f"{slug}.svg"` join uses `pathlib.Path`, which resolves `..` and `/` segment-by-segment but does NOT clamp to `out_dir`. Defense in depth: Plan 02 Task 2 acceptance criteria require the test `test_write_species_svg_creates_subdir` to pass with a benign slug; the broader project posture (no user input ever reaches the pipeline; pipeline runs only on the trusted maderas host) makes runtime exploitation impossible. If a future change ever exposes slug construction to untrusted input, add `slug.split('/')` validation that both parts match `^[A-Za-z][A-Za-z\-]+$` before the `Path` join. Tracked as a defensive note for Phase 93+ when more taxa levels enter slug construction. |
| T-92-02-03 | Information Disclosure | public/data/species-maps tree on S3 | accept | SVGs are public artifacts already (CloudFront-served); reorganization into per-genus subdirectories changes only directory structure, not content sensitivity. |
| T-92-02-04 | Denial of Service | pipeline runtime | accept | Adding `mkdir(parents=True, exist_ok=True)` and switching `glob` → `rglob` is O(files); production has ~550 SVGs. No DoS risk. |

ASVS Level 1 applies for static-asset path handling; the mitigations above satisfy V12 (Files and Resources) for the dbt-validated input contract.
</threat_model>
