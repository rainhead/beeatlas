<objective>
Audit and clean `content/species-photos.toml` so every `[species."..."]` (or bare-word `[species.foo]`) section key matches a `scientificName` value in `public/data/species.json`. Today there are 106 bare-word keys (e.g., `[species.agapostemon]`) and an unknown number of mis-cased or mis-spaced quoted keys that do not match any scientificName — these surface as warnings from `npm run validate-species`. PIPE-03's "TOML keys migrated to match" requirement is satisfied when this validate run reports zero warnings.

Per 92-RESEARCH.md Pitfall 5 + Assumption A1: TOML keys are scientificName strings, NOT slugs. The "hierarchical slug format" language in PIPE-03 means "no orphans, every key matches a known scientificName"; it does NOT mean rekeying TOML sections to `Andrena/milwaukeensis` form (which would break the `photos[sp.scientificName]` lookup in `_pages/species.njk`).

Purpose: Phase 94 will render `/species/{Genus}/{epithet}/` pages that look up photos via `photos[sp.scientificName]`. Orphan TOML entries either represent lost photos (if the key is a typo or case-mismatch for a real species) or noise (if the bee no longer appears in the WA checklist). Both need disposition before Phase 94 ships.

Output:
- `content/species-photos.toml` with every key matching a known scientificName
- `.planning/phases/92-slug-migration-pipeline-prep/92-03-toml-audit.json` recording disposition per original key
- `npm run validate-species` exits 0 with zero `warn:` lines
</objective>

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-92-03-01 | Tampering | content/species-photos.toml mutation | mitigate | Mutations are driven by an audit JSON that is itself reviewed by the user (Task 2 checkpoint). The mutation script uses `tomlkit` (parser-validated round-trip) so syntactic corruption is detected before write. Post-write validation via `@iarna/toml` parse confirms validity. |
| T-92-03-02 | Information Disclosure | photo loss via incorrect `remove` disposition | mitigate | Task 2 is a blocking human-verify checkpoint specifically because the auditor's heuristic could mis-classify a real species as orphaned.
