<objective>
Resolve CLEAN-01 per the locked decision (D-03): keep the existing `FORMAT CSV, DELIMITER '',
QUOTE '', HEADER false` workaround in `data/dbt/macros/emit_feature_collection.sql` and replace
its inline documentation with a fuller rationale block.

Purpose: REQUIREMENTS.md states the macro should be "replaced"; ROADMAP.md SC#2 says it should "no
longer use FORMAT CSV". Both are overridden by the locked decision after the researcher verified
that the only documented alternative (`FORMAT GDAL, DRIVER 'GeoJSON'`) is NOT byte-comparable: it
adds a `"name"` key to the FeatureCollection root, breaking the existing 2-key
`{type, features}` structure used by `public/data/counties.geojson` and
`public/data/ecoregions.geojson`. A Python post-hook was not tested and is NOT being pursued — the
project has no Python dbt models and the cost of greenfielding one for a single macro does not pay
off when the CSV workaround already works.

Output: A comment-only edit to one file. No functional change; the macro body remains identical so
`counties.geojson` and `ecoregions.geojson` continue to be byte-comparable to the current public
files. The "is replaced" language in REQUIREMENTS.md becomes "is documented with rationale" for
Phase 85 purposes — record this re-interpretation in the SUMMARY.
</objective>
