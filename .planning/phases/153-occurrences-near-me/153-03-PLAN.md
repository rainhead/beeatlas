<objective>
Wire near-me end-to-end in `<bee-atlas>`, the state owner, reusing the EXISTING shift-drag bounds machinery so a near-me box is indistinguishable from a shift-drag box (D-01) — same `_selectionBounds`, same `boundsClause` query (D-02, D-09), same `sel=west,south,east,north` URL round-trip (D-03). This is the crux of the redesign: produce a bounds selection, then let the existing paths fire.

Five wirings:
1. **Trigger (D-06):** handle `near-me-requested` from `<bee-pane>` by calling the new `<bee-map>` public `requestUserLocation()` (plan 153-01) so the blue dot + accuracy ring appear and the control relays `user-location-changed`.
2. **Box compute (D-02):** when a `user-location-changed` SUCCESS arrives AND a near-me request is pending, compute a ±10 km box (`dLat=10/111.32`, `dLon=10/(111.32·cos(lat·π/180))`, `box={west:lon−dLon, east:lon+dLon, south:lat−dLat, north:lat+dLat}`, with an `isFinite` guard on all four edges), set `_selectionBounds` to it, and run the EXISTING selection→query+URL path (mirror `_onSelectionDrawn`).
3. **Freeze/re-tap (D-07):** falls out for free — `_selectionBounds` is an explicit snapshot; the moving dot does not re-filter. Re-tapping the button issues a fresh request and recomputes from the current position.
4. **Clear (D-05):** handle `near-me-cleared` by clearing `_selectionBounds` exactly like the existing clear paths.
5. **Toast fix (D-08):** on geolocation denial/unavailable, surface the EXISTING Phase 152 toast (`_locationError`/`_locationErrorKind`) and apply NO bounds — and FIX why the toast didn't fire in UAT.

Also bind `<bee-pane>` `.selectionBoundsActive=${this._selectionBounds !== null}` so the chip reflects active bounds (D-05).

NO haversine, NO `?near=1`, NO `nearMeCenter`, NO new query function, NO new URL param — strictly reuse.

Purpose: deliver the user-visible near-me behavior on top of the existing, tested bounds substrate.
Output: bee-atlas event bindings, a box-compute helper, the `_userLocation`→`_selectionBounds` wiring, the toast fix, the `selectionBoundsActive` binding, and bee-atlas tests.
