<task type="auto">
  <name>Task 1: Switch singleclick to click with dragging guard</name>
  <files>frontend/src/bee-map.ts</files>
  <action>
In `firstUpdated()`, change the click handler registration from:
```
this.map.on('singleclick', async (event: MapBrowserEvent) => {
```
to:
```
this.map.on('click', async (event: MapBrowserEvent) => {
  if (event.dragging) return;
```

This eliminates the 250ms delay that OL's `singleclick` introduces (it waits to disambiguate from double-click). The `event.dragging` guard prevents accidental clicks during map pans.

The entire handler body stays the same — only the event name and the early-return guard change.
  </action>
  <verify>
    <automated>cd /Users/rainhead/dev/beeatlas && grep -n "map.on('click'" frontend/src/bee-map.ts && ! grep -n "map.on('singleclick'" frontend/src/bee-map.ts && grep -n "event.dragging" frontend/src/bee-map.ts</automated>
  </verify>
  <done>Click handler uses 'click' event with dragging guard. No 'singleclick' remains.</done>
</task>

<task type="auto">
  <name>Task 2: Fix bare-genus labels in taxon filter options</name>
  <files>frontend/src/bee-map.ts</files>
  <action>
In `buildTaxaOptions()` (lines ~78-94), the `species` Set contains raw `scientificName` values. When a specimen is identified only to genus level, `scientificName` equals the bare genus name (e.g. "Bombus"), which creates a confusing duplicate alongside "Bombus (genus)".

Fix: When building the species entries at the end of buildTaxaOptions, filter out any scientificName that is a single word (no space) AND already exists in the genera Set. These are genus-only identifications that are already represented by the "(genus)" entry.

Change the species mapping line from:
```typescript
...[...species].sort().map(v => ({ label: v, name: v, rank: 'species' as const })),
```
to:
```typescript
...[...species].filter(v => !(genera.has(v) && !v.includes(' '))).sort().map(v => ({ label: v, name: v, rank: 'species' as const })),
```

This suppresses bare-genus scientificNames from the species list since they are already covered by the genus filter entry. Single-word species names that are NOT in the genera Set (if any exist) are preserved.
  </action>
  <verify>
    <automated>cd /Users/rainhead/dev/beeatlas && grep -A2 "species.*filter.*genera" frontend/src/bee-map.ts</automated>
  </verify>
  <done>Taxon filter dropdown shows only "(genus)" entries for genus-level taxa, no confusing bare duplicates.</done>
</task>
