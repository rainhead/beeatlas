  <action>
The problem: `_visibleEcdysisIds` and `_visibleSampleIds` default to `null`, which style functions interpret as "show all". When a URL has an active filter, features paint as "show all" until the async filter query completes.

Fix in `firstUpdated()`, after restoring `_filterState` from URL params (after line 190 where `this._filterState` is assigned), add:

```typescript
// If URL restores an active filter, initialize visible ID sets to empty (hide-all)
// so no dots flash before the async filter query completes.
if (isFilterActive(this._filterState)) {
  this._visibleEcdysisIds = new Set();
  this._visibleSampleIds = new Set();
}
```

This ensures the style functions see an empty Set (hide all dots / ghost all clusters) rather than null (show all) during the DuckDB query window.

Also, call `_runFilterQuery()` earlier — right after the filter state + visible ID initialization, BEFORE the DuckDB init block. `_runFilterQuery` calls `queryVisibleIds` which already internally awaits `tablesReady`, so it is safe to start it in parallel with DuckDB table loading. Move it from `_onDataLoaded` to `firstUpdated`:

After the `if (isFilterActive)` block above and before the `getDuckDB()` call, add:
```typescript
// Start filter query early — queryVisibleIds awaits tablesReady internally,
// so this runs in parallel with DuckDB init and resolves as soon as tables load.
if (isFilterActive(this._filterState)) {
  this._runFilterQuery();
}
```

Keep the existing `_runFilterQuery()` call in `_onDataLoaded` as-is — it handles the case where a filter is active and specimen features have just loaded (the filter query may have already resolved by then, and the generation guard will deduplicate).
