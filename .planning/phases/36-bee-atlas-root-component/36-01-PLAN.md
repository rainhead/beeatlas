**_getRestoredTaxonInput() helper:**
Returns the display label for the current taxon filter. Looks up the taxon in `_taxaOptions` to get the label (e.g., "Bombus (genus)"). Falls back to the taxonName if no match found:
```typescript
private _getRestoredTaxonInput(): string {
  if (!this._filterState.taxonName) return '';
  const opt = this._taxaOptions.find(
    o => o.name === this._filterState.taxonName && o.rank === this._filterState.taxonRank
  );
  return opt?.label ?? this._filterState.taxonName;
}
```

Note about data-loaded event: The current bee-map computes `buildSamples` and `computeSummary` from OL features. After this refactor, bee-map will still have access to specimenSource features. The data-loaded event should include `summary` and `taxaOptions` computed by bee-map (since it has direct access to OL features). Additionally, for occurrence restore, the event should include the raw features or bee-atlas should pass the occIds to bee-map as a property and bee-map resolves them. The simplest approach: bee-map computes summary/taxaOptions/buildSamples and emits them. Bee-atlas stores them.

For filteredSummary computation, bee-atlas needs access to individual feature data. Two approaches:
(a) bee-map emits all specimen features in the data-loaded event (expensive for large datasets)
(b) bee-map provides a method or event to resolve filteredSummary given visibleIds

Best approach: have bee-map emit `data-loaded` with `{summary, taxaOptions}` computed from features. For filteredSummary, bee-atlas calls `queryVisibleIds` to get the ID sets, then emits a `compute-filtered-summary` property/signal to bee-map. But this is circular.

SIMPLEST approach that avoids circular dependency: Move the `computeSummary`, `buildSamples`, `buildTaxaOptions` helper functions from bee-map.ts to a shared utility module, OR keep them in bee-map.ts and have bee-map emit richer events. Since these functions operate on OL Feature objects (which bee-atlas should NOT import), the cleanest path is:

1. bee-map emits `data-loaded` with `{summary, taxaOptions}` (computed internally)
2. bee-map emits `filtered-summary-computed` with `{filteredSummary}` after bee-atlas triggers a filter
3. When bee-atlas updates `visibleEcdysisIds` property on bee-map, bee-map's `updated()` computes the filteredSummary from its own features and emits it upward

This keeps OL-specific computation in bee-map while bee-atlas owns the state. Add this pattern to the render template:
- `@filtered-summary-computed=${this._onFilteredSummaryComputed}`

For occurrence restore from URL, bee-map handles it when it receives `selectedOccIds` as a property and resolves features internally, emitting back `map-click-specimen` with the resolved samples.

**_pushUrlState() method:**
Port from bee-map.ts lines 475-491. Uses buildParams with current state. Requires current view coordinates — stored from the last `view-moved` event. replaceState immediately, pushState after 500ms debounce.

```typescript
private _currentView: { lon: number; lat: number; zoom: number } = { lon: DEFAULT_LON, lat: DEFAULT_LAT, zoom: DEFAULT_ZOOM };

private _pushUrlState() {
  const params = buildParams(
    this._currentView,
    this._filterState,
    { occurrenceIds: this._selectedOccIds ?? [] },
    { layerMode: this._layerMode, boundaryMode: this._boundaryMode }
  );
  window.history.replaceState({}, '', '?' + params.toString());
  if (this._mapMoveDebounce) clearTimeout(this._mapMoveDebounce);
  this._mapMoveDebounce = setTimeout(() => {
    window.history.pushState({}, '', '?' + params.toString());
    this._mapMoveDebounce = null;
  }, 500);
}
```

**_onPopState handler:**
Port from bee-map.ts lines 257-301. Read URL params, update all state properties (filterState, layerMode, boundaryMode, selections). Set `_viewState` to trigger bee-map to update its OL view. Set `_isRestoringFromHistory = true` and reset after view-moved event.

```typescript
private _onPopState = () => {
  this._isRestoringFromHistory = true;
  if (this._mapMoveDebounce) {
    clearTimeout(this._mapMoveDebounce);
    this._mapMoveDebounce = null;
  }
  const parsed = parseParams(window.location.search);
  const lon = parsed.view?.lon ?? DEFAULT_LON;
  const lat = parsed.view?.lat ?? DEFAULT_LAT;
  const zoom = parsed.view?.zoom ?? DEFAULT_ZOOM;
  // Update view state — bee-map's updated() will apply to OL map
  this._viewState = { lon, lat, zoom };
  this._currentView = { lon, lat, zoom };
  // Restore filter state
  this._filterState = {
    taxonName: parsed.filter?.taxonName ?? null,
    taxonRank: parsed.filter?.taxonRank ?? null,
    yearFrom: parsed.filter?.yearFrom ?? null,
    yearTo: parsed.filter?.yearTo ?? null,
    months: parsed.filter?.months ?? new Set(),
    selectedCounties: parsed.filter?.selectedCounties ?? new Set(),
    selectedEcoregions: parsed.filter?.selectedEcoregions ?? new Set(),
  };
  // Restore UI state
  const newLayerMode = parsed.ui?.layerMode ?? 'specimens';
  const newBoundaryMode = parsed.ui?.boundaryMode ?? 'off';
  this._layerMode = newLayerMode;
  this._boundaryMode = newBoundaryMode;
  // Restore selection
  const parsedOccIds = parsed.selection?.occurrenceIds ?? [];
  if (parsedOccIds.length > 0) {
    this._selectedOccIds = parsedOccIds;
    // bee-map will resolve these to features when it receives the property
  } else {
    this._selectedSamples = null;
    this._selectedOccIds = null;
    this._selectedSampleEvent = null;
  }
  // Run filter query for restored state
  if (isFilterActive(this._filterState)) {
    this._runFilterQuery();
  } else {
    this._visibleEcdysisIds = null;
    this._visibleSampleIds = null;
    this._filteredSummary = null;
  }
};
```
