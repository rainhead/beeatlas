## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| beforeId on non-existent layer | Used `'ghost-points'` as beforeId for `checklist-county-fill` addLayer — ghost-points didn't exist yet, causing entire load callback to throw and abort | advisory | Add checklist layer without beforeId; insertion order in load callback determines render order |
| updated() fires before map loads | Checklist property changes from URL restore trigger `_applyChecklistLayer()` before map layers exist; guard bails silently | advisory | Always call `_applyChecklistLayer()` inside `_map.on('load')` callback after layer is added when `showChecklist` is already true |
| Internal state not synced from parent | bee-pane's `_showChecklist` is local `@state()` — not updated when bee-atlas restores `checklistVisible` from URL | advisory | Add `@property checklistVisible` to bee-pane; sync to `_showChecklist` in `updated()` |
