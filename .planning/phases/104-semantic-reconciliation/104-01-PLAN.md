<objective>
Reconcile the divergent "confirmed (non-provisional) specimen" predicates across the
BeeAtlas stack so that all three layers — TypeScript frontend, Python places pipeline,
and dbt SQL — agree on a single canonical definition: a row is a confirmed specimen iff
`ecdysis_id IS NOT NULL`. Satisfies SEM-01 of the v3.8 Conceptual Tidying milestone.

Purpose: `places_export.py:_query_counts` currently overcounts by using
`is_provisional = false OR is_provisional IS NULL`, which is TRUE for both
Ecdysis-backed rows AND sample-only iNat rows (where `ecdysis_id IS NULL`,
`is_provisional = false`). The TypeScript `isSpecimenBacked` and the dbt
`int_species_occurrences_agg.specimen_count` are both already correct. The fix
direction is therefore: update `places_export.py` to match the frontend, document
the canonical layer in `src/occurrence.ts`, add a cross-reference comment in the dbt
model, and lock the semantics in with an extended pytest fixture.

Output: One predicate fix in a SQL string, one JSDoc block expansion, one parquet
fixture extension with a third row type, one SQL comment, and a `dbt build` to
confirm the SQL-comment edit is harmless. No schema changes, no new columns, no
behavioral change to anything other than `places.json` specimen counts (which
become semantically correct — sample-only rows no longer counted).
</objective>
