**Step 1: Add `tablesReady` promise to duckdb.ts (per D-03 — sources need to know when tables are loaded)**

Add a module-level deferred promise to `frontend/src/duckdb.ts`:

```typescript
// Add after existing _dbPromise declaration:
let _tablesReadyResolve: (() => void) | null = null;
export const tablesReady: Promise<void> = new Promise(resolve => {
  _tablesReadyResolve = resolve;
});
```

Then at the END of `loadAllTables()`, just before the final closing brace (after the debug log section), add:

```typescript
if (_tablesReadyResolve) _tablesReadyResolve();
```

This allows features.ts to `await tablesReady` before querying, solving the race condition where VectorSource loaders fire before `loadAllTables` completes.

**EcdysisSource class (FEAT-01):**
- `constructor({ onError }: { onError?: (err: Error) => void })` — no `url` parameter
- Loader function: `await tablesReady`, then `getDuckDB()`, then `db.connect()`, then `conn.query()` with this exact SQL:
  ```sql
  SELECT ecdysis_id, occurrenceID, longitude, latitude, year, month,
         scientificName, recordedBy, fieldNumber, genus, family,
         floralHost, county, ecoregion_l3, inat_observation_id
  FROM ecdysis
  ```
- Row iteration: `table.toArray()` → `.flatMap(row => ...)` with `row.toJSON()` to get plain object
- Null check: skip rows where `obj.longitude == null || obj.latitude == null` (return `[]`)
- Feature construction — exactly matching parquet.ts:
  - `feature.setGeometry(new Point(fromLonLat([obj.longitude, obj.latitude])))`
  - `feature.setId(\`ecdysis:\${obj.ecdysis_id}\`)`
  - `feature.setProperties({...})` with these exact property names and coercions:
    - `occurrenceID: obj.occurrenceID`
    - `year: Number(obj.year)` (BigInt coercion)
    - `month: Number(obj.month)` (BigInt coercion)
    - `scientificName: obj.scientificName`
    - `recordedBy: obj.recordedBy`
    - `fieldNumber: obj.fieldNumber`
    - `genus: obj.genus`
    - `family: obj.family`
    - `floralHost: obj.floralHost ?? null`
    - `county: obj.county ?? null` (no `as string` cast needed — DuckDB returns string)
    - `ecoregion_l3: obj.ecoregion_l3 ?? null` (no `as string` cast needed)
    - `inat_observation_id: obj.inat_observation_id != null ? Number(obj.inat_observation_id) : null`
- `console.debug(\`Adding \${features.length} ecdysis features from DuckDB\`)`
- `this.addFeatures(features); if (success) success(features);`
- Connection closed in `finally` block: `await conn.close()`
- Error path: `.catch((err: Error) => { if (onError) onError(err); failure(); })`
- `super({ loader: load, strategy: all })`
