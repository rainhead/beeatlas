## Copywriting Contract

No new copy is introduced in Phase 31. Existing copy that must be preserved:

| Element | Copy | Source |
|---------|------|--------|
| Loading overlay | "Loading…" (U+2026 ellipsis character) | bee-map.ts line 627 |
| Error overlay | Dynamic: propagated from DuckDB error message | bee-map.ts line 626 |
| Error fatality | Any DuckDB error in Phase 31 is fatal — replaces previous non-fatal Phase 30 pattern | CONTEXT.md D-05 |

**Error state contract (Phase 31 change):** DuckDB errors now propagate via `onError` → `dataErrorHandler` → `_dataError`, rendering the `.error-overlay` with the error message. Previously (Phase 30), DuckDB errors were non-fatal and logged to console only. The overlay copy itself does not change — the propagation path does.

**Empty state:** Not applicable — the map renders with or without features. If DuckDB returns 0 rows, the map renders with an empty feature set (no explicit empty-state UI).

**Destructive actions:** None in this phase.
