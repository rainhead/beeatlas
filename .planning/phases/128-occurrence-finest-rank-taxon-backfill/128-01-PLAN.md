<objective>
Backfill `occurrences.taxon_id` for genus-level (single-token `canonical_name`) occurrence rows
with the genus self-row taxon_id, closing the re-scoped TID-02 gap. After this phase, every
occurrence row with any identification (species OR genus) carries the taxon_id of its finest
identified rank; truly-unidentified specimens and non-bee bycatch genera legitimately stay NULL.

Purpose: Final blocker for the v4.5 milestone close. Phase 126 made `occurrences.taxon_id`
non-null only for species-level (two-token) rows. TID-02 was re-scoped (human decision 2026-06-01)
to "every identified row"; this phase surfaces the genus taxon_ids that already exist in the iNat
taxonomy dump.

Output: A regenerated `occurrences.parquet` where ~17k single-token genus rows go from NULL to a
genus taxon_id, with the 37-column contract intact and re-scoped tests green.

Scope guard: This is a surfacing change only — NO new taxonomy/ID work, NO iNat API calls, NO
column add/remove, NO change to species.parquet/species.taxon_id. The genus source is the existing
`data/raw/taxa.csv.gz` (already downloaded by the pipeline). Subgenus/tribe/family rungs backfill 0
rows today and are OMITTED (per RESEARCH A2); the ladder is effectively species → genus.
</objective>
