<objective>
Add Workbox 7.4.1 runtime caching to `src/sw.ts` so `/data/*.db` and `/data/*.geojson` requests
are served `CacheFirst` from a `data-artifacts` cache. Attach `ExpirationPlugin({maxEntries:1,
purgeOnQuotaError:true})` to the DB route only — this collapses content-hash churn (every nightly
pipeline ships a new `occurrences_<hash>.db` URL; without an entry cap, old hashes accumulate
toward the iOS ~50 MB quota) and provides the partial-write cleanup path for `QuotaExceededError`
(SC-6). The GeoJSON route uses `CacheFirst` with no entry cap (three stable URLs, <5 MB total).

Also adds the three new Workbox subpackages (`workbox-strategies`, `workbox-expiration`,
`workbox-cacheable-response`) to `package.json` devDependencies and extends
`src/tests/build-output.test.ts` with assertions that prove the runtime routes are bundled into
`_site/app/sw.js` and that the no-skipWaiting invariant from Phase 147/148 is not regressed.

Purpose: This is the structural foundation for OFF-02 (runtime cache) and OFF-03 (no
skipWaiting). Plans 02 and 03 attach behaviour around it (cold-start probe, online/offline UI)
but cannot start until this cache name and these routes exist in source.

Output: A `src/sw.ts` that compiles to a `_site/app/sw.js` containing two new `registerRoute`
calls, a `package.json` that `npm ci` will install cleanly, and build-output tests that fail loudly
if any of the above regress.

Implements requirements OFF-02 (per ROADMAP SC-1/SC-2) and OFF-03 (per SC-7 carry-forward).
Quota handling follows D-09 (console warn only, no new UI in Phase 149) — the partial-write
cleanup is delivered by `purgeOnQuotaError: true` on the ExpirationPlugin (Task 2); no banner,
toast, or postMessage plumbing is added.
The user-facing UX for OFF-03 (prompt-to-reload banner) is deferred to Phase 150 — this plan
only enforces the structural lifecycle invariant.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → SW | Page-issued `fetch()` for `/data/*` crosses into the SW's `fetch` event handler |
| SW → CloudFront | SW issues network fetch on cache miss; same-origin HTTPS |
| SW → Cache Storage | SW writes Response bodies into the `data-artifacts` named cache |
| npm registry → repo | Three new devDependencies installed from npm during `npm install -D` |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-149-01 | Tampering | SW cache contents | mitigate | `CacheableResponsePlugin({ statuses: [200] })` on both routes blocks caching of 3xx/4xx/5xx responses — a 403 from an expired CloudFront access policy or a 502 from origin failure can never poison the `data-artifacts` cache (RESEARCH §"Security Domain") |
| T-149-02 | Tampering | route predicate scope | mitigate | Route predicates use BOTH `url.pathname.startsWith('/data/')` AND extension check (`.db`/`.geojson`); cross-origin URLs and `/api/*` style paths cannot match because `url.pathname` is parsed from the Request URL by Workbox before the predicate runs; the SW scope itself is `/app/` so cross-origin requests are not even routed through it (RESEARCH P-D: NavigationRoute allowlist is `/^\/app\//`, no broad `/data/*` route exists) |
| T-149-03 | Information Disclosure | stale cache entry served after rotation | mitigate | Content-hashed URLs (`occurrences_<hash>.db`) mean a new pipeline run = new URL = guaranteed cache miss = re-fetch; `ExpirationPlugin({ maxEntries: 1 })` evicts the previous DB on cache write so stale entries cannot accumulate beyond the freshest one (RESEARCH Q8) |
| T-149-04 | Denial of Service | quota exhaustion via hash churn | mitigate | `ExpirationPlugin({ maxEntries: 1, purgeOnQuotaError: true })`: the cap collapses the steady-state DB cache to ~23 MB (single entry); purgeOnQuotaError clears the entire `data-artifacts` cache on quota failure as the recovery path (D-04, RESEARCH Q8) |
| T-149-05 | Elevation of Privilege | version-skew code execution via SW auto-update | mitigate | `src/sw.ts` contains zero `skipWaiting`/`clients.claim`; the no-skipWaiting invariant from Phase 147 D-06 / Phase 148 D-04 is now also gated by Task 3's build-output assertion against `_site/app/sw.js` (structural + CI enforcement) |
| T-149-06 | Tampering | cache name collision | accept | The `data-artifacts` cache name is namespaced enough for this single-app deployment; Workbox's own `workbox-precache-v2` cache uses a versioned suffix that cannot collide; no third-party SW shares the `/app/` scope |
| T-149-SC | Tampering | npm install of workbox-strategies / -expiration / -cacheable-response | mitigate | All three packages are in the official `googlechrome/workbox` monorepo (verified by `npm view <pkg> repository` in RESEARCH §"Package Legitimacy Audit"); slopcheck OK; ~8 years old with 8–12M weekly downloads; same version (^7.4.1) and same publisher as `workbox-precaching`/`workbox-routing` already in package.json. No `[ASSUMED]`/`[SUS]` checkpoints required per RESEARCH; pinned to `^7.4.1` to match the existing Workbox surface |
</threat_model>
