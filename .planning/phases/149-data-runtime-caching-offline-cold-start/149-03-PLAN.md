<objective>
Wire OFF-05 (online/offline indicator) and OFF-04 (blank-basemap honest label) end-to-end.
`<bee-atlas>` (the sole reactive-state owner per CLAUDE.md) gains a single `_offline` `@state`
field driven by `online`/`offline` window events; `<bee-header>` renders a small "Offline" pill
only when offline (quiet UI per D-10 — no editorializing when online); `<bee-map>` renders a
bottom-left explanation overlay only when offline (per D-11). Both components receive `offline`
as a `@property` input — neither owns the state, neither listens for events — preserving the
pure-presenter invariant for `<bee-map>` and applying the same discipline to `<bee-header>` so
Phase 150's "ready for offline" indicator can read the same single source of truth without
adding another listener pair.

This plan is parallel-safe with Plan 02 (no file overlap; no shared symbols; Plan 02's
page-entry-scope online listener is independent of Plan 03's component-scope listener — there is
no shared mutable state between them, both just call `navigator.onLine` and respond to the same
browser event, which the spec guarantees is broadcast to every listener).

Purpose: Field collectors need an honest signal that the device is offline (so they don't
mistake "no new dots" for "no bees here") and an honest explanation of why the basemap is blank
(so they don't think the app is broken). Both are non-blocking — the map stays fully usable in
either state, satisfying OFF-05 SC-4.

Output: Three component edits + three test edits; the rendered DOM gains a pill in `<bee-header>`'s
shadow root and an overlay in `<bee-map>`'s shadow root, both conditional on `offline === true`.

Implements requirements OFF-04 (per ROADMAP SC-3) and OFF-05 (per SC-4).
</objective>
