<objective>
Wire the page-side half of CACHE-05: a cold-start probe that detects when the current
`occurrences.db` is missing from the `data-artifacts` cache (e.g. after iOS Safari has evicted
it from a long-unused PWA) and silently triggers a background `fetch()` that the SW intercepts
and caches; an `online` event listener that re-runs the same probe when the device reconnects;
and a single `navigator.storage.persist()` call at first launch, gated by a `localStorage` flag
and result-logged only.

Per D-08 there is NO new transient UI in Phase 149 â€” the re-prime is intentionally silent. The
"ready for offline" indicator that surfaces what this code does ships in Phase 150 and will read
the same `data-artifacts` cache state.

Purpose: CACHE-05 has two halves â€” the SW-side cleanup (Plan 01's `purgeOnQuotaError`) and the
page-side re-prime trigger. This plan owns the page side. Without it, iOS Safari users whose DB
has been evicted would see an empty map until they manually pulled the file back in by acting on
a filter â€” the field-mode UX would silently degrade.

Output: A `src/app-entry.ts` that runs the probe on cold-start and re-runs it on `online`; a
`src/sw-registration.ts` that requests persistent storage once per browser-profile; a new
`src/tests/cache-probe.test.ts` with unit coverage for the four branches of the probe (online +
cache miss â†’ fetch; online + cache hit â†’ no fetch; offline â†’ no fetch; 'online' event â†’ probe
re-runs).

Implements requirement CACHE-05 (per ROADMAP SC-5).
</objective>
