runtime-caches Mapbox basemap requests (style JSON, vector tiles, glyphs,
sprites) from api.mapbox.com into a dedicated mapbox-basemap cache, so warm map

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| /app page -> Service Worker -> api.mapbox.com | Third-party CDN responses cross into on-device Cache Storage. Responses are read-only basemap assets (CORS / non-opaque); no user input is written to the cache key beyond Mapbox-controlled URLs. |
| Service Worker -> events.mapbox.com / api.mapbox.com/map-sessions | Mapbox billing/accounting path. This boundary is deliberately NOT crossed — the route excludes it. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-154-01 | Tampering | SW cache poisoning via non-200 / opaque responses | mitigate | CacheableResponsePlugin({ statuses: [200] }) — only status-200 responses are cached; suppresses SWR's default status-0 (opaque) allowance (RESEARCH Pitfall 2). |
| T-154-02 | Repudiation / Information disclosure | Interception of events.mapbox.com or /map-sessions/ billing telemetry | mitigate | matchCallback uses strict url.hostname === 'api.mapbox.com' AND !url.pathname.startsWith('/map-sessions/'); telemetry host is never matched (RESEARCH Pitfall 1; ToS sections 2.9.4 / 1.3.9). |
| T-154-03 | Denial of service | Unbounded cache growth exhausting the iOS ~50 MB origin quota | mitigate | ExpirationPlugin maxEntries: 150 + maxAgeSeconds: 604800 + purgeOnQuotaError: true; non-opaque CORS responses are stored at real size (RESEARCH section "CORS vs Opaque"). |
| T-154-SC | Tampering | npm/pip/cargo installs | accept | No new packages installed — workbox-strategies/expiration/cacheable-response are already direct deps at 7.4.1 (RESEARCH Package Legitimacy Audit, all Approved existing deps). No install task in this plan; no [ASSUMED]/[SUS] packages. |

**No high-severity threats.** This phase introduces no new auth, session,
access-control, or cryptography surface. The only security-adjacent items are ToS
compliance (token retention D-04, no telemetry interception D-07), both mitigated
above.
</threat_model>
