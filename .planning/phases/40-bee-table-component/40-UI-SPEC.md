## Component Specification

### bee-table component

**Registration:** `customElements.define('bee-table', BeeTable)`

**Properties (received from bee-atlas):**
- `rows: SpecimenRow[] | SampleRow[]` — current page of rows
- `rowCount: number` — total matching rows (from COUNT(*) query)
- `layerMode: 'specimens' | 'samples'` — determines column set
- `page: number` — current page (1-based)
- `sortColumn: string` — active sort column key
- `sortDir: 'asc' | 'desc'` — active sort direction

**Events emitted (up to bee-atlas):**
- `sort-changed` — detail: `{ column: string, dir: 'asc' | 'desc' }`
- `page-changed` — detail: `{ page: number }`

**Page size:** 100 rows fixed.

### Layout

The table fills the `.table-slot` area: `flex-grow: 1`, full viewport height minus header bar. Internal layout:

```
┌─────────────────────────────────────────────┐
│ COLUMN HEADERS (fixed, sticky top)          │
├─────────────────────────────────────────────┤
│                                             │
│ TABLE ROWS (scrollable, overflow-y: auto)   │
│                                             │
├─────────────────────────────────────────────┤
│ PAGINATION BAR (fixed, anchored bottom)     │
└─────────────────────────────────────────────┘
```

`:host` is `display: flex; flex-direction: column; height: 100%; overflow: hidden`.

### Cell Overflow

All cells: `overflow: hidden; text-overflow: ellipsis; white-space: nowrap`. Full value exposed via `title` attribute on hover (native browser tooltip). Row height: 36px fixed (single line).

### Column Header Sort Interaction

- Each column header is a `<button>` with `cursor: pointer`.
- Unsorted column: label + `↕` sort arrow in `var(--text-hint)` color.
- Active ascending column: label + `↑` in `var(--accent)` color + `border-bottom: 2px solid var(--accent)` on header cell.
- Active descending column: label + `↓` in `var(--accent)` color + `border-bottom: 2px solid var(--accent)` on header cell.
- Clicking an active column header reverses sort direction; clicking a different column sets it as active, ascending.
- Emit `sort-changed` with new `{ column, dir }`.

### Pagination Bar

Layout: `display: flex; align-items: center; justify-content: space-between; padding: 8px 16px; border-top: 1px solid var(--border); background: var(--surface-subtle)`.

Left: row count indicator string (see Copywriting Contract below).
Center: Previous button / page info / Next button.
Right: empty (reserved for Phase 41 CSV export button).

**Previous button:** `← Prev` — disabled and styled `opacity: 0.4; cursor: not-allowed` when `page === 1`.
**Next button:** `Next →` — disabled and styled `opacity: 0.4; cursor: not-allowed` when `page * 100 >= rowCount`.
**Page info:** `Page N of M` where M = `Math.ceil(rowCount / 100)`.

Clicking Previous emits `page-changed` with `{ page: page - 1 }`.
Clicking Next emits `page-changed` with `{ page: page + 1 }`.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | "Next →" / "← Prev" (pagination navigation) |
| Row count — unfiltered | "Showing 1–100 of N specimens" (or "samples") |
| Row count — range | "Showing {start}–{end} of N specimens" (or "samples") |
| Row count — last page | "Showing {start}–{N} of N specimens" (end = rowCount) |
| Empty state heading | "No specimens match the current filters." (or "No samples match the current filters.") |
| Empty state body | "Try adjusting the filters in the sidebar." |
| Loading state | "Loading…" (reuse existing `loading-overlay` pattern from `bee-atlas.ts`) |
| Error state | "Failed to load data. Reload the page to try again." |
| Destructive confirmation | none — no destructive actions in this phase |

Row count format notes:
- Start = `(page - 1) * 100 + 1`
- End = `Math.min(page * 100, rowCount)`
- Noun is "specimens" when `layerMode === 'specimens'`, "samples" when `layerMode === 'samples'`
- Comma-format N via `toLocaleString()` (e.g. "3,847")

Source: CONTEXT.md specifics section, ROADMAP.md success criteria item 3, REQUIREMENTS.md TABLE-03/TABLE-04.

## States and Transitions

| State | Trigger | Visual |
|-------|---------|--------|
| Loading | DuckDB query in flight | `loading-overlay` div over table area (reuse existing CSS class) |
| Empty | `rowCount === 0` | Empty state message centered in table body area |
| Populated | `rows.length > 0` | Normal table with rows, headers, pagination |
| Row hover | `mouseenter` on row | Background: `var(--surface-hover)` |
| Sort active | `sortColumn` matches column key | Accent border + accent arrow on header |
| Next disabled | `page * 100 >= rowCount` | Next button opacity 0.4, cursor not-allowed |
| Prev disabled | `page === 1` | Prev button opacity 0.4, cursor not-allowed |

Filter change (property update from `bee-atlas`): page resets to 1, loading overlay shown during query, then rows update. Handled in `bee-atlas` (`_onFilterChanged` resets `_tablePage = 1`).

## Accessibility

- Column header buttons: `aria-sort="ascending" | "descending" | "none"` attributes.
- Table element: `role="table"`, `<thead>`, `<tbody>`, `<th scope="col">`.
- Pagination buttons: `aria-label="Previous page"` / `aria-label="Next page"`.
- Disabled pagination buttons: `disabled` attribute (not just visual styling).
- Row count region: wrapped in `aria-live="polite"` `<span>` so screen readers announce changes.
