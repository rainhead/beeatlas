<objective>
Patch `data/ecdysis_pipeline.py` `_download_zip()` so the nightly pipeline can again
fetch Ecdysis dataset 44, which now requires an authenticated Symbiota session
(anonymous `publicsearch=1` POSTs return `401 {"error":"Unauthorized access"}` —
reproduced live 2026-06-24). Three layered changes, all locked by CONTEXT.md:

- **D-2 (auth):** Build a `requests.Session()`, POST the real credential login form,
  reuse the same session (carrying `PHPSESSID`) for the existing `downloadhandler.php`
  POST, and guard the response so a JSON/401 body raises loudly instead of being
  cached as a corrupt ZIP.
- **D-3 (resilience):** On any login/download/guard/network failure, fall back to a
  valid cached ZIP (warn + reuse) instead of aborting the nightly; hard-fail only
  when no usable cache exists.
- **D-1 (credentials, security prereq):** Add a gitignore rule covering
  `data/.dlt/secrets.toml` BEFORE any credential can land on disk. Read creds via
  `dlt.secrets["sources.ecdysis.username"/".password"]`.

Purpose: `ecdysis` is `run.py` STEP 1, so every nightly currently fails at the start
and production S3 data goes stale. This restores ingestion and hardens it against
future Ecdysis outages.

Output: a gitignore rule, the patched helper with four new symbols, a new fast-tier
test file, and one manual maderas integration verification.

CONTEXT.md and RESEARCH.md supersede 163-FINDINGS.md on the login form: there is
**NO CSRF token** (skip the pre-GET), the submit control is **`action=login`** (NOT
`loginButton` — that is a decoy header form with no credential inputs), and the
username field is **`login`** (NOT `username`). RESEARCH "Answers to the Open
Technical Unknowns" (Q1–Q6) and "Architecture Patterns" are the concrete blueprint
— read them before implementing; do not re-derive from FINDINGS.

**Goal (outcome):** The nightly Ecdysis ingestion fetches dataset 44 again, resists a
future outage, and stores credentials safely.

**Observable truths → required artifacts → key wiring:**

1. *Login authenticates before download (D-2)* → `_login_session` POSTs
   `{login, password, action=login, remember}` to `/profile/index.php` on a
   `requests.Session` → download POST reuses the SAME session object.
2. *Corrupt/JSON body never cached (D-2 guard)* → `_assert_zip_response` runs INSIDE
   the try, BEFORE the atomic write → magic-bytes `PK\x03\x04` is the authoritative gate.
3. *Outage degrades gracefully (D-3)* → `_is_valid_cached_zip` + try/except around the
   network+guard block → warn + return cached bytes on failure.
4. *No silent zeroing (D-3)* → re-raise when `_is_valid_cached_zip` is False.
5. *Credentials safe (D-1)* → gitignore rule lands FIRST; `git check-ignore` proves it.
6. *No credential leak (V7)* → password never interpolated into any error/print/traceback.
7. *Existing semantics preserved* → TTL fast-path, atomic tmp-write+replace, and
   `ECDYSIS_CACHE_TTL_SECONDS=0` force-refresh are LEFT UNCHANGED (RESEARCH Q5).

**Reachability check:** every truth maps to a concrete in-phase artifact and a
