## Layout Contract

### Toolbar position

The `<bee-filter-toolbar>` element is a new child of `<bee-atlas>`, inserted **between** `<bee-header>` and the `.content` div. It spans the full viewport width.

```
[ bee-header        (dark navy, 48px tall)      ]
[ bee-filter-toolbar (surface, ~48px tall)       ]
[ .content: bee-map | bee-table + bee-sidebar    ]
```

### Toolbar structure (left to right)

```
[ bee-filter-controls (flex-grow:1) ] [ "Download CSV" button ]
```

- `bee-filter-controls` takes all available horizontal space via `flex-grow: 1`.
- The CSV download button is right-aligned, separated by `gap: 8px` from the filter controls.
- The toolbar itself has `padding: 0 8px` (xs on sides) and `height: 48px` with `align-items: center`.

### Narrow viewport behavior

On viewports `max-width: 640px`, the toolbar row wraps: filter field takes full width on its own row, CSV button appears on the same row but visually right-aligned via `margin-left: auto`. This matches the deferred REQUIREMENTS.md note: "Toolbar filter controls on narrow viewports — deferred to implementation decision." The implementation decision is: no collapse, allow natural wrapping.

## Component Inventory

### bee-filter-toolbar (new component)

Lit `@customElement('bee-filter-toolbar')`.

**Properties (received from bee-atlas):**

| Property | Type | Notes |
|----------|------|-------|
| filterState | FilterState | Passed through to bee-filter-controls |
| taxaOptions | TaxonOption[] | Passed through to bee-filter-controls |
| countyOptions | string[] | Passed through to bee-filter-controls |
| ecoregionOptions | string[] | Passed through to bee-filter-controls |
| collectorOptions | CollectorEntry[] | Passed through to bee-filter-controls |
| summary | DataSummary \| null | Passed through to bee-filter-controls |
| layerMode | 'specimens' \| 'samples' | Used to label CSV download filename hint |

**Events dispatched (bubble + composed):**

| Event | Detail type | Trigger |
|-------|-------------|---------|
| `filter-changed` | FilterChangedEvent | Re-emitted from bee-filter-controls |
| `csv-download` | void | CSV download button click |

**Internal layout:**

```
:host {
  display: flex;
  align-items: center;
  width: 100%;
  padding: 0 8px;
  background: var(--surface);
  border-bottom: 1px solid var(--border);
  min-height: 48px;
  box-sizing: border-box;
}
bee-filter-controls { flex-grow: 1; }
.csv-btn { flex-shrink: 0; margin-left: 8px; }
```

### bee-filter-controls (existing — no changes)

`bee-filter-controls` is moved from `bee-sidebar` into `bee-filter-toolbar`. The component itself is unchanged. Its `.search-section` padding (`0.75rem`) provides internal spacing.

**The only structural change** inside bee-filter-controls: the `border-bottom` on `.search-section` becomes redundant once it sits in the toolbar (the toolbar border-bottom separates from content). Leave the border-bottom on `.search-section` in place — the visual overlap is negligible and avoids a breaking change to the existing component.

### CSV Download Button

Inline in bee-filter-toolbar (not a separate component).

**Label:** "Download CSV"

**Appearance:**
- `background: transparent`
- `border: 1px solid var(--border-input)` = `#ccc`
- `border-radius: 4px`
- `padding: 6px 12px` (sm vertical, 12px horizontal)
- `font-size: 0.875rem` (14px)
- `font-weight: 500`
- `color: var(--text-body)` = `#213547`
- `cursor: pointer`
- `min-height: 44px` (touch target)
- `white-space: nowrap`

**Hover state:** `background: var(--surface-muted)` = `#f0f0f0`, border-color unchanged.

**Active/pressed state:** `background: var(--surface-pressed)` = `#d0d0d0`.

**No loading/disabled state needed** in this phase (CSV export is synchronous in the existing implementation from Phase 41).

## Interactions Contract

### Filter controls behavior

All existing bee-filter-controls behaviors are preserved identically:
- Token typeahead with dropdown suggestions
- Chip removal via ✕ button or Backspace
- "Clear filters" link when tokens are present
- Recent filter suggestions on focus
- Keyboard navigation (ArrowUp/Down, Enter, Escape)
- `filter-changed` event bubbles through bee-filter-toolbar to bee-atlas

### CSV download button behavior

1. User clicks "Download CSV"
2. `bee-filter-toolbar` dispatches `csv-download` (bubbles, composed, no detail payload)
3. `bee-atlas` handles `csv-download` using the existing `queryAllFiltered` + `buildCsvFilename` logic from Phase 41 (currently wired to the sidebar's Download CSV button)

### Sidebar removal

`bee-sidebar` no longer renders `<bee-filter-controls>` or the CSV download button. The sidebar's `filter-changed` event handler in bee-atlas is removed from that pathway. All filter wiring moves to bee-filter-toolbar.

## Accessibility Contract

- Token field `role="combobox"` behavior inherited from existing bee-filter-controls (input with listbox suggestions).
- Suggestion list `role="listbox"`, each item `role="option"` — inherited from existing bee-filter-controls.
- CSV button: `aria-label` not required — visible text label "Download CSV" is sufficient.
- Toolbar landmark: use `<div role="toolbar" aria-label="Filter controls">` as the shadow root's root element.
- Minimum touch target: 44px height on CSV button (min-height: 44px).
- Color contrast: `var(--text-body)` (#213547) on `var(--surface)` (#fafafa) — meets WCAG AA (contrast ratio > 7:1).
