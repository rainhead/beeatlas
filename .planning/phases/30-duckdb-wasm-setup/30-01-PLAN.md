  <action>
1. Install the package:
   ```
   cd frontend && npm install @duckdb/duckdb-wasm
   ```

2. Create `frontend/src/duckdb.ts` with the following structure:

   **Imports:** Import `* as duckdb` from `@duckdb/duckdb-wasm`. Import WASM and worker files using Vite `?url` suffix -- all four files need `?url`:
   - `@duckdb/duckdb-wasm/dist/duckdb-mvp.wasm?url`
   - `@duckdb/duckdb-wasm/dist/duckdb-browser-mvp.worker.js?url`
   - `@duckdb/duckdb-wasm/dist/duckdb-eh.wasm?url`
   - `@duckdb/duckdb-wasm/dist/duckdb-browser-eh.worker.js?url`
   Also import `DuckDBDataProtocol` from `@duckdb/duckdb-wasm`.

   **MANUAL_BUNDLES constant:** Build a `duckdb.DuckDBBundles` object with `mvp` and `eh` entries pointing to the `?url` imports. This lets `selectBundle` pick the EH bundle when available (avoiding SharedArrayBuffer / COOP-COEP per DUCK-04).

   **`_init()` private async function:** Calls `duckdb.selectBundle(MANUAL_BUNDLES)`, creates a `new Worker(bundle.mainWorker!)`, creates `new duckdb.AsyncDuckDB(new duckdb.ConsoleLogger(), worker)`, calls `await db.instantiate(bundle.mainModule, bundle.pthreadWorker)`, returns `db`.

   **`getDuckDB()` exported function:** Singleton pattern -- module-level `let _dbPromise: Promise<duckdb.AsyncDuckDB> | null = null`. On first call, sets `_dbPromise = _init()`. Returns `_dbPromise`.

   **`loadAllTables(db, baseUrl)` exported async function:** Takes `AsyncDuckDB` and `baseUrl` string. Does the following in order:

   a. Load parquet tables -- for each of `['ecdysis', 'ecdysis.parquet']` and `['samples', 'samples.parquet']`:
      - Call `await db.registerFileURL(file, \`${baseUrl}/${file}\`, DuckDBDataProtocol.HTTP, false)`
      - Open connection: `const conn = await db.connect()`
      - Execute: `await conn.query(\`CREATE TABLE ${tableName} AS SELECT * FROM '${file}'\`)`
      - Close: `await conn.close()`

   b. Load GeoJSON tables via spatial extension:
      - Open a single connection
      - Execute `await conn.query('INSTALL spatial')` then `await conn.query('LOAD spatial')` (separate calls per Pitfall 5 -- do NOT combine into one string)
      - Execute `await conn.query(\`CREATE TABLE counties AS SELECT * FROM '${baseUrl}/counties.geojson'\`)`
      - Execute `await conn.query(\`CREATE TABLE ecoregions AS SELECT * FROM '${baseUrl}/ecoregions.geojson'\`)`
      - Close connection
      - Wrap spatial loading in try/catch. On failure, log `console.warn('Spatial extension failed, falling back to read_json_auto:', error)` and retry with:
        - `await conn.query(\`CREATE TABLE counties AS SELECT * FROM read_json_auto('${baseUrl}/counties.geojson')\`)`
        - `await conn.query(\`CREATE TABLE ecoregions AS SELECT * FROM read_json_auto('${baseUrl}/ecoregions.geojson')\`)`

   c. Log table counts for debugging:
      - Open connection, run `SELECT COUNT(*) as n FROM ecdysis`, `SELECT COUNT(*) as n FROM samples`, `SELECT COUNT(*) as n FROM counties`, `SELECT COUNT(*) as n FROM ecoregions`
      - Log each count with `console.debug('DuckDB table counts:', ...)`
      - Close connection

   All exports should use named exports (no default export).
