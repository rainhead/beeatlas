```sql
SELECT ib.ecdysis_id
FROM {{ ref('int_ecdysis_base') }} ib
WHERE ib.ecdysis_id IS NOT NULL
  AND CAST(ib.ecdysis_id AS VARCHAR) NOT IN (
    SELECT id FROM {{ ref('stg_ecdysis__occurrences') }}
  )
```

Why this is correct (from RESEARCH.md):
- `int_ecdysis_base.ecdysis_id` is `CAST(o.id AS INTEGER)` (e.g., 5594060).
- `stg_ecdysis__occurrences` is `SELECT * FROM source`, so it exposes `id VARCHAR` (e.g., '5594060').
- `CAST(ecdysis_id AS VARCHAR)` produces '5594060', which IS in the `id` set.
- VERIFIED: query returns 0 rows against live data.

Why catalog_number is WRONG (Pitfall 1 from RESEARCH.md):
- `catalog_number` is e.g. 'WSDA_2303966' (different namespace from numeric id).
- A NOT IN catalog_number test always returns ALL rows and always fails.
