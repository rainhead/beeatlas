<objective>
Move the occurrence-layer render DECISION into `<bee-map>` so it is a pure function of its inputs,
gated on `mapReady`, making an unfiltered flash structurally impossible rather than timed-around.

Today `<bee-atlas>` encodes hide-all by pushing empty `_filteredGeoJSON` + empty `_visibleIds` at
exactly the right moment, and `<bee-map>._applyVisibleIds` branches on `filteredGeoJSON !== null`.
Target: pass `intendedFilterActive` (the Plan 01 getter) as a new `@property` to `<bee-map>`; the
occurrence source data becomes:
  - intendedFilterActive true  → render `filteredGeoJSON ?? empty`  (hide-all until the filtered set arrives)
  - intendedFilterActive false → render the full set

The first render is gated on `mapReady` (the 'load' handler is where the sources exist). With this in
place, `<bee-atlas>` STOPS pre-seeding empty collections as the hide-all mechanism — "filter intended
but data not ready" renders empty by construction.

Purpose: close the last structural seam of the map-init race — the flash can no longer reopen because
it is no longer prevented by bee-atlas timing.
Output: a `<bee-map>` whose occurrence source is f(filteredGeoJSON, intendedFilterActive) gated on
mapReady, a `<bee-atlas>` that expresses hide-all via the gate (not empty pre-seeds), and bee-map tests.

DEPENDS ON Plan 01: `intendedFilterActive` / `_filterResolving` must already exist on `<bee-atlas>`.
</objective>
