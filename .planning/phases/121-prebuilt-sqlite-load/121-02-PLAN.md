<objective>
Replace the body of `src/sqlite-worker.ts` with the MemoryVFS-seeding approach validated in the spike: fetch `occurrences.db`, seed `MemoryVFS.mapNameToFile` with the ArrayBuffer before `open_v2`, then build the initial GeoJSON via a SQL query rather than parsing parquet. Delete the parquet loader, the in-memory INSERT loop, the SQL-value escaper, and the parquet-row GeoJSON builder.

Purpose: This is the worker half of the cutover and the source of the ~70% load-time win (~2125 ms → ~725 ms, per spike findings). The runtime cost of loading 92K rows via batched INSERTs is eliminated; in its place is a single fetch of a 3.9 MB gzipped binary.

Output: A rewritten `src/sqlite-worker.ts` that fetches the prebuilt SQLite DB and serves the same message protocol consumed by `src/sqlite.ts`.

The wire-protocol shape (`tables-ready` with `logs`, `exec` -> `exec-result` / `exec-error`, `build-geojson` -> `geojson-result`) MUST be preserved verbatim so no change to `src/sqlite.ts` is required.
</objective>
