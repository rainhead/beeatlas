The component decorates the existing DOM via `willUpdate` + `querySelectorAll`. To sync `aria-expanded` we use the same approach: listen for the native `toggle` event (fired by `<details>` whenever its open state changes) and mirror it onto the host's `aria-expanded` attribute.

Native keyboard contract for `<details>/<summary>`: focusing the `<summary>` and pressing Enter or Space toggles `open`. This works WITHOUT JavaScript. The existing `bee-taxon-nav._onClick` calls `e.preventDefault()` on click events but Enter/Space on `<summary>` fire as `keydown`+`click` — verify the click is the native one from the summary (in which case preventDefault stops the toggle!). Inspect carefully: the existing handler bails early if `target.closest('li[data-taxon]')` doesn't match, but `<summary>` IS inside `<li>` so it does match — meaning currently Enter/Space on a summary may dispatch `taxon-selected` AND have its native toggle prevented. **This is a bug surface for PERF-05 / D-11.**

Fix: in `_onClick`, detect when the clicked element is a `<summary>` (or descendant of one) and skip the `preventDefault()` so the native disclosure toggle still fires. Still dispatch `taxon-selected` so the filter behaviour is unchanged.

1. **Stop preventDefault on summary clicks** so native Enter/Space toggle on `<summary>` is preserved. In `_onClick`, after computing `li`, detect whether the click originated inside a `<summary>` and short-circuit the preventDefault:

   ```ts
   const target = e.target as HTMLElement | null;
   if (!target) return;
   const li = target.closest('li[data-taxon]') as HTMLElement | null;
   if (!li) return;
   const inSummary = !!target.closest('summary');
   // ... existing path / rank computation ...
   this.dispatchEvent(new CustomEvent('taxon-selected', { ... }));
   // D-11: only call preventDefault for non-summary clicks. Summary clicks
   // must fall through so the native <details> disclosure toggles, which
   // is what gives us free Enter/Space keyboard support.
   if (!inSummary) {
     e.preventDefault();
   }
   ```

2. **Mirror aria-expanded onto each `<li>` when its `<details>` toggles.** Add a `toggle` event listener in `connectedCallback` that catches the bubbling toggle event from any `<details>` descendant and writes `aria-expanded` onto the enclosing `<li[data-taxon]>`:

   ```ts
   connectedCallback(): void {
     super.connectedCallback();
     this.addEventListener('click', this._onClick);
     // D-11: <details> emits a non-bubbling 'toggle' event in the spec,
     // but addEventListener with capture=true catches it on descendants.
     this.addEventListener('toggle', this._onToggle, true);
     // Initial sync from SSR open state (in case any <details> ships open).
     this._syncAllAria();
   }
   disconnectedCallback(): void {
     super.disconnectedCallback();
     this.removeEventListener('click', this._onClick);
     this.removeEventListener('toggle', this._onToggle, true);
   }

   private _onToggle = (e: Event): void => {
     const det = e.target as HTMLDetailsElement | null;
     if (!det || det.tagName !== 'DETAILS') return;
     const li = det.closest('li[data-taxon]') as HTMLElement | null;
     if (!li) return;
     li.setAttribute('aria-expanded', det.open ? 'true' : 'false');
   };

   private _syncAllAria(): void {
     this.querySelectorAll<HTMLDetailsElement>('details').forEach(det => {
       const li = det.closest('li[data-taxon]');
       if (li) li.setAttribute('aria-expanded', det.open ? 'true' : 'false');
     });
   }
   ```
