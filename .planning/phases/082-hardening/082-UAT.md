## Pre-UAT findings (recorded for the audit trail)

Two issues were discovered while running the PERF spot checks before walking the use cases. Both were fixed in-flight; the values cited in the spot-check table reflect the post-fix build.

1. **`scripts/measure-lcp.sh` runtime errors** (commit `a450504`).
   - Lighthouse 13 rejects `--preset=desktop --form-factor=mobile` (screen-emulation/form-factor mismatch). Dropped `--preset=desktop`; added explicit `--screenEmulation.mobile=true`.
   - `node -e "require('${OUT_JSON}')"` failed because macOS `mktemp -t TEMPLATE` appends its random suffix *after* the template (so `lcp-XXXXXX.json` becomes `lcp-XXXXXX.json.<rand>` and `require()` can't infer JSON by extension). Switched to `fs.readFileSync` + `JSON.parse`.

2. **PERF-03 photo data not reaching Eleventy templates** (commit `86b53d8`).
   - Plan 082-03 added `deriveSrcset` as a named export of `_data/photos.js` alongside `export default result`. Eleventy 3 only auto-unwraps the default export of a `_data/*.js` module when there are no other named exports; with both present the template received the module namespace, so `photos[sp.scientificName]` was always `undefined`. The unit tests passed because they imported `deriveSrcset` via direct ESM, bypassing the data cascade.
   - Pre-fix: `grep -c 'srcset=' _site/species/index.html` → 0.
   - Fix: extract `deriveSrcset` to `lib/inat-srcset.js`; `_data/photos.js` now default-exports only its data table. Test imports updated.
   - Post-fix: `grep -c 'srcset=' _site/species/index.html` → 489.
