<objective>
Update `.github/workflows/deploy.yml` to match the new repo layout
landed in plan 074-01: drop `--workspace=frontend` from npm commands,
shift artifact paths from `frontend/dist/` to `_site/`, update both
`aws s3 sync` source paths, and rename the build job and its build
step from `Build frontend` to `Build site` so the workflow contains
no remaining references to the old layout. Cache-control rules and
the deploy job's OIDC + CloudFront invalidation steps remain
untouched.

Purpose: Close the CI gap left intentionally open by plan 074-01.
After this plan lands, pushing a branch triggers a green build job and
(on `main`) a deploy job that publishes the same site from the new
artifact location.

Output: One file changed (`.github/workflows/deploy.yml`), small
number of edits, no semantic change to what gets shipped (same site,
same cache headers, same S3 layout). Job/step renames are cosmetic
but enable a strict `! grep frontend` post-condition.
</objective>
