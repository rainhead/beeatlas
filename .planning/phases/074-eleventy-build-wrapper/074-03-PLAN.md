<task type="checkpoint:human-verify" gate="blocking">
  <name>Task 3: Manual UAT — npm run dev serves the SPA correctly</name>
  <what-built>
    Eleventy 3.x outer build wrapping the Vite SPA via
    `@11ty/eleventy-plugin-vite`. After `npm run dev`, Eleventy starts
    its dev server with Vite running as middleware (single port, HMR
    for `src/**` changes).
  </what-built>
  <how-to-verify>
    Run the dev server and verify the SPA loads with full functionality.

    **Setup:**
    ```bash
    npm run dev
    ```
    Expected console output:
    - Eleventy banner ("[11ty] Wrote 0 files in ...s ...")
    - A "Server at http://localhost:8080" (or similar) line from the
      Eleventy dev server. The exact port may differ; the URL is
      printed in the dev server output.

    **Verification steps in browser:**

    1. Open the dev URL in a browser. The SPA should render: header,
       map view, filter button overlay.

    2. **Map tiles load** — Mapbox tiles appear (not blank gray). This
       confirms `VITE_MAPBOX_TOKEN` is being read from the local
       `.env` file (which moved from `frontend/.env` to root `.env`
       in plan 01).
       - If tiles are blank: open browser devtools console; if you
         see 401 from `api.mapbox.com`, the `.env` move was missed
         or the token is invalid. Verify `grep VITE_MAPBOX_TOKEN
         .env` shows a populated token (the `.env` file is
         gitignored; the value is local-only).

    3. **Data layer loads** — small dots appear on the map after a
       brief load. This confirms `/data/occurrences.parquet` is
       reachable via the Eleventy dev server's static file serving
       (or Vite's middleware fallback).

    4. **Filter panel works** — click the magnifying-glass overlay
       button (top of map, left of "Regions"). The What/Who/Where/When
       panel opens. Apply any filter (e.g., type a genus in "What").
       The dot count updates and the map re-renders.

    5. **HMR works** (optional but valuable):
       - Edit `src/bee-header.ts` — change the visible title text.
       - Save the file.
       - The browser should hot-reload the updated text within
         ~1 second without a full page refresh.

    6. **Stop the dev server** (`Ctrl-C` in the terminal).

    **Known acceptable differences from pre-migration:**
    - The dev URL may differ from previous `npm run dev` (was
      `http://localhost:5173/` for vanilla Vite; Eleventy's dev
      server may use `http://localhost:8080/` or another port). This
      is expected; document the actual port in the SUMMARY for future
      reference.
    - Initial console output is more verbose (Eleventy + Vite both
      log). This is expected.

    **Do NOT consider the UAT passing if:**
    - Map tiles do not load (Mapbox 401 or blank gray)
    - Filter panel doesn't open or filters don't apply
    - Console shows uncaught errors related to `wa-sqlite` URL
      resolution (would indicate `optimizeDeps.exclude` broke)
    - HMR is broken (full page reload on every change) — this would
      suggest the plugin's middleware integration failed; investigate
      before approving
  </how-to-verify>
  <resume-signal>
    Type "approved" if the SPA loads correctly with map, data, and
