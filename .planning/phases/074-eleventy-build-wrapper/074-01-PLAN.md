```javascript
import EleventyVitePlugin from "@11ty/eleventy-plugin-vite";

export default function (eleventyConfig) {
  eleventyConfig.addPassthroughCopy({ "index.html": "index.html" });
  eleventyConfig.addPassthroughCopy({ "src": "src" });
  eleventyConfig.addPassthroughCopy({ "public": "/" });

  eleventyConfig.addPlugin(EleventyVitePlugin, {
    viteOptions: { appType: "mpa" },
  });

  return {
    dir: {
      input: "_pages",       // empty for v3.1; Phase 75 populates
      output: "_site",
      includes: "_includes",
      layouts: "_layouts",
      data: "_data",
    },
  };
}
```

Note: `dir.input` MUST be disjoint from `src/` (which holds the SPA's
`.ts` files) — using `_pages/` (or any underscore-prefixed name)
prevents Eleventy from trying to template TypeScript files. The dir is
empty at the end of this phase; Phase 75 populates it.

Plugin behavior (verified in research):
  1. Eleventy writes templates to `_site/` (with passthroughs)
  2. Plugin renames `_site/` -> `.11ty-vite/` (default tempFolderName)
  3. Vite runs with `root: .11ty-vite, outDir: _site`, discovers
     `index.html` and bundles `<script type="module">` references
  4. Plugin removes `.11ty-vite/`
This is why `vite.config.ts` does NOT need `root`/`outDir`/`emptyOutDir`
— the plugin overrides them at build time.

Pitfall (research §Common Pitfalls #2): Vite's default `publicDir` would
resolve to `<root>/public` (i.e. `.11ty-vite/public`) and double-copy
files. Disable Vite's publicDir handling so Eleventy passthrough is the
single owner. Set `publicDir: false` in vite.config.ts.
