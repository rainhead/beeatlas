<decisions>
## Implementation Decisions

### WA county GeoJSON source
- Use Census TIGER/Line shapefiles (authoritative, standard FIPS codes, freely available)
- Filter full US county file to WA state only (`STATEFP='53'`) — covers all 39 WA counties, no geometry clipping needed
- County name in Parquet output: name only (`"King"`, not `"King County"`) — clean for filter UI display in Phase 19

### GeoJSON generation strategy
- GeoJSON files regenerated by `build-data.sh` on every pipeline run (consistent with how other pipeline assets are produced)
- Source boundary files: download only if missing (skip 35MB ecoregion download if `NA_CEC_Eco_Level3.zip` already present; same pattern for TIGER)
- Simplification tool: geopandas `.simplify()` + `.to_file()` — already a project dependency, no new tools
- A reusable script is kept in the repo (documents how files were produced, enables regeneration)

### Shared spatial join utility
- Shared `data/spatial.py` module — single source of truth for join logic used by both pipelines
- Interface: caller loads boundary GeoDataFrames first, then passes them as arguments — `add_region_columns(df, counties_gdf, ecoregions_gdf)`
- Enables reuse without reloading files when processing multiple batches
- geopandas is already a project dependency; adding it to the iNat pipeline import is fine

### Claude's Discretion
- Exact signature and internal structure of `spatial.py`
- How CRS conversion (`to_crs('EPSG:4326')`) is handled inside the utility
- Nearest-polygon fallback implementation details (geopandas `sjoin_nearest`)
- Where TIGER download URL is defined/documented
- Exact step ordering in `build-data.sh` for GeoJSON generation vs. Parquet generation

</decisions>

- `scripts/build-data.sh`: Sequential pipeline — add GeoJSON generation step before the Ecdysis/iNat processing steps (boundaries needed for join)
- `scripts/validate-schema.mjs` `EXPECTED` dict: Add `county` and `ecoregion_l3` to both `ecdysis.parquet` and `samples.parquet` column lists

### Established Patterns
- CRS handling: EPA L3 ecoregion shapefile uses non-EPSG spherical Lambert AEA CRS — must call `.to_crs('EPSG:4326')` before `sjoin` (documented in STATE.md)
- Nearest-polygon fallback: ~408 WA specimens (~0.9%) fall outside polygon boundaries; `sjoin_nearest` fallback required for null rows after `'within'` sjoin
- Download-only-if-missing: mirrors iNat incremental cache pattern — check if file exists before downloading
- Parquet copy pattern in `build-data.sh`: GeoJSON copy to `frontend/src/assets/` follows same pattern as `cp ecdysis.parquet frontend/src/assets/ecdysis.parquet`

### Integration Points
- `data/ecdysis/occurrences.py` `to_parquet()`: Add `county` and `ecoregion_l3` via `spatial.add_region_columns()`; function signature may need to accept pre-loaded GDFs or paths
- `data/inat/`: Add spatial join step after `samples.parquet` is written (or before, inline in the iNat download module)
- `scripts/build-data.sh`: Add GeoJSON generation step; add GeoJSON copy to `frontend/src/assets/`
- `scripts/validate-schema.mjs`: Update `EXPECTED['ecdysis.parquet']` and `EXPECTED['samples.parquet']`
