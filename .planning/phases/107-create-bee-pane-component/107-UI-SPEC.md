## Spacing Scale

This project uses rem-based spacing matching an 8-point grid. The existing Lit components
use consistent rem steps. All values below are the canonical declarations for this phase.

| Token | rem | px equiv | Usage |
|-------|-----|----------|-------|
| xs | 0.25rem | 4px | Icon gap, inline padding (e.g. close-button left padding) |
| sm | 0.5rem | 8px | Compact element gaps, chip margins, header row padding |
| md | 1rem | 16px | Default section padding, content area padding |
| lg | 1.5rem | 24px | Section breaks, filter-row bottom margin |
| xl | 2rem | 32px | Major vertical gaps |
| 2xl | 3rem | 48px | (reserved; not used in this phase) |
| 3xl | 4rem | 64px | (reserved; not used in this phase) |

Exceptions:
- Toggle button width in collapsed state: 2.5rem (40px) — matches existing floating button
  size convention in bee-filter-panel (filter-btn uses 0.4rem 0.6rem padding; toggle strip
  needs fixed 2.5rem to hold a single icon cleanly)
- Pane list-state width: 25rem — matches existing bee-sidebar width (bee-atlas.ts line 108)
- Pane table-state width: 100% of content area — full-width drawer (matches Phase 69 behavior)
- Shadow DOM positioning: 0.5em insets for pane edge offset (matches existing bee-sidebar
  and bee-filter-panel positioning conventions in bee-atlas.ts)

## Typography

All sizes are rem-relative to root (16px browser default). Source: existing bee-sidebar.ts
and bee-filter-panel.ts measured values.

| Role | Size | Weight | Line Height | CSS |
|------|------|--------|-------------|-----|
| Body / filter input | 0.875rem (14px) | 400 (regular) | 1.5 | `font-size: 0.875rem; font-weight: 400` |
| Label / section heading | 0.85rem (13.6px) | 600 (semibold) | 1.2 | `font-size: 0.85rem; font-weight: 600` |
| Pane header / occurrence title | 1rem (16px) | 400 (regular) | 1.5 | `font-size: 1rem; font-weight: 400` |
| Toggle / icon button | 1.25rem (20px) | 400 (regular) | 1 | `font-size: 1.25rem; line-height: 1` |

Only two font weights are used: 400 (regular) and 600 (semibold). This matches the exact
pattern in bee-sidebar.ts (lines 70, 86) and bee-header.ts.

## Color

Source: `src/index.css :root` — all values below are the canonical tokens. Do NOT introduce
new hex literals in bee-pane.ts; use `var(--token)` for every color declaration.

| Role | Token | Fallback | Usage |
|------|-------|---------|-------|
| Dominant (60%) | `var(--surface)` | #fafafa | Pane background, filter panel background |
| Secondary (30%) | `var(--surface-subtle)` | #f5f5f5 | Hover rows, suggestion highlight, section backgrounds |
| Accent (10%) | `var(--accent)` | #2c7a2c | Reserved elements only (see below) |
| Destructive | `var(--error)` | #b00 | Not used in this phase — no destructive actions |

Accent `var(--accent)` is reserved for exactly these elements in this phase:
1. The toggle button when pane is open (active state indicator)
2. Filter input and elevation input focus ring/border-color
3. Checkbox `accent-color` in the year-filter rows
4. Active filter-section icon color when a section has a selection

Accent must NOT be applied to: the toggle button default state, general text, headings,
borders, or the pane background.

Text color tokens (use exactly as declared):
- Primary text: `var(--text-body)` — #213547
- Secondary text: `var(--text-secondary)` — #444
- Muted / labels: `var(--text-muted)` — #666
- Placeholder / hint: `var(--text-hint)` — #767676 (WCAG AA minimum on --surface)

Border tokens:
- Standard border: `var(--border)` — #ddd
- Subtle divider: `var(--border-subtle)` — #eee
- Input border: `var(--border-input)` — #ccc

Box shadow (existing convention, maintain parity):
- Pane panel: `box-shadow: 0 2px 8px rgba(0,0,0,0.15)` (from bee-sidebar.ts)
- Dropdown/suggestion overlay: `box-shadow: 0 2px 6px rgba(0,0,0,0.1)` (from bee-filter-panel.ts)

Border radius (existing convention):
- Buttons, inputs, filter panel: 4px
- Chips: 3px

## Layout Contract

### Pane States and Dimensions

| State | Width | Height | Position |
|-------|-------|--------|----------|
| collapsed | 2.5rem (toggle strip only) | from top offset to bottom: 0.5em | absolute, right: 0.5em |
| list | 25rem | from top offset to bottom: 0.5em | absolute, right: 0.5em |
| table | 100% of content area (or full right edge) | TBD in Phase 108 — pane provides flex column | absolute, right: 0 |

The `:host` element in `bee-pane.ts` declares only `position: absolute`. The specific
top/right/bottom values are set from `bee-atlas` CSS rules (Phase 108). `bee-pane` must not
hardcode its own top/right positioning — only `position: absolute` on `:host`.

### Toggle Button — Always-Visible Contract

The toggle button (`.toggle-btn`) MUST be outside all `paneState` conditional template
branches. It renders regardless of `paneState`. In collapsed state it is the ONLY visible
element. It sits at the left edge of the pane (which is the right edge of the map).

Toggle button dimensions: minimum 2.5rem × 2.5rem (40×40px), matching the floating button
size used elsewhere in the app. This meets the 44px touch-target guideline with minor
padding.

### Expand Button — Desktop Only

The `.expand-btn` class is rendered only when `paneState === 'list'`. The CSS hides it on
mobile using the existing project breakpoint:

```css
@media (max-aspect-ratio: 1) {
  .expand-btn { display: none; }
}
```

This is the exact breakpoint used in bee-atlas.ts for mobile layout changes.

### Shrink Button — Table State Only

The `.shrink-btn` is rendered only inside `_renderTableContent()` in the table state header
row. It is not visible in list or collapsed states.

### Filter Panel — List State

In list state, the filter content is always visible (the panel is permanently open — the
per-section collapsible behavior for What/Who/Where/When rows is preserved from
bee-filter-panel). The floating filter trigger button from the old bee-filter-panel is NOT
present in bee-pane's list state — the pane itself is the persistent filter surface.

### Occurrence Detail — List State, Selection Present

When `occurrences` is non-null and non-empty, `bee-occurrence-detail` renders below the
filter rows inside the list state content area, separated by a 1px `var(--border-subtle)`
divider.

## Interaction Contract

### Toggle Button
- Default icon: chevron pointing left (pane open states) / chevron pointing right (collapsed)
  — use Unicode `⟩` / `⟨` or inline SVG chevron consistent with bee-header pattern
- `aria-label`: "Open filter pane" (collapsed) / "Close filter pane" (list or table)
- Click dispatches: `pane-expand-list` (when collapsed) or `pane-collapse` (when list or table)
- No `@state` internal tracking — reads `this.paneState` (external property)

### Expand Button (list → table)
- Label: `aria-label="Expand to table view"`
- Icon: table grid icon (inline SVG, 4 cells)
- Click dispatches: `pane-expand-table`
- Hidden on mobile via CSS (no JS detection)

### Shrink Button (table → list)
- Label: `aria-label="Return to list view"`
- Icon: left-arrow or shrink icon (inline SVG)
- Click dispatches: `pane-shrink-list`

### Filter Rows (What / Who / Where / When)
- Behavior: copy from bee-filter-panel.ts `_renderWhat/Who/Where/When` methods exactly
- Section open/close toggle behavior preserved (`.filter-row` expand-on-click pattern)
- All filter-changed events dispatched upward with `bubbles: true, composed: true`

### bee-table Events
- No event listeners added to `<bee-table>` in bee-pane's template
- All bee-table events (`page-changed`, `sort-changed`, `row-pan`, `download-csv`,
  `toggle-filter`) have `bubbles: true, composed: true` and propagate naturally to bee-atlas

### Filter State Sync
- `updated(changed: PropertyValues)` must sync `_selectedTaxon`, `_selectedCollectors`,
  `_selectedCounties`, `_selectedEcoregions`, `_selectedPlace`, `_elevMin`, `_elevMax`,
  `_yearThisYear`, `_yearLastYear`, `_yearEarlier` from `this.filterState` on property change
- Pattern copied from bee-filter-panel.ts `updated()` method

### Keyboard / Accessibility
- Toggle, expand, and shrink buttons are `<button>` elements (not `<div>`)
- All `aria-label` values are non-empty strings (listed above)
- Focus ring: browser default `outline` — do not suppress

## Copywriting Contract

| Element | Copy |
|---------|------|
| Toggle button aria-label (collapsed) | "Open filter pane" |
| Toggle button aria-label (open) | "Close filter pane" |
| Expand button aria-label | "Expand to table view" |
| Shrink button aria-label | "Return to list view" |
| Pane header label (list state) | "Filters" — displayed as a text label next to the toggle; weight 600 |
| Pane header label (table state) | "Table view" — displayed in the table header row |
| Empty filter state | "No filters applied" (below section headers when all rows are default) |
| Empty occurrence detail | Not shown — bee-occurrence-detail handles its own empty state |
| Filter section headings | "What", "Who", "Where", "When" — uppercase, 0.75rem, var(--text-muted) |

No destructive actions in this phase. No error states are introduced (bee-pane is a pure
presenter; error states live in bee-occurrence-detail and bee-table).
