# Phase 172 — UI Design Contract

> per-collector static page. Generated by gsd-ui-researcher.

> established static page. Every pattern below is borrowed from existing
> templates and CSS. No new component patterns are introduced.

## Typography

Inherited from `src/index.css` and established page patterns. No new type roles
introduced. Both declared weights (400 and 700) are inherited from the existing
stylesheet — this phase introduces neither.

| Role | Size | Weight | Line Height | Color token |
|------|------|--------|-------------|-------------|
| Body | 16px (1rem) | 400 (inherited — `:root font-weight: 400`) | 1.5 | `--text-body` (#213547) |
| Metadata / label | ~13.6px (0.85rem) | 400 (inherited) | 1.5 (inherited) | `--text-muted` (#666) |
| Section heading (H2) | browser default ~24px (1.5rem) | 700 (browser default bold) | 1.2 (browser) | `--text-body` (#213547) |
| Genus heading (H3) | 16px (1rem) | 700 (bold heading, italic distinguishes from H2) | 1.2 | `--text-body` (#213547) |

**Declared weights: 400 and 700 only.** Both pre-exist in the stylesheet.

Note on links: `src/index.css` line 50 sets `a { font-weight: 500; }` globally.
Species list links inherit this 500 weight from the existing rule — this phase
does not introduce or change it, and 500 is therefore not a declared typography
weight of this phase.

Notes:
- Genus headings are rendered in `font-style: italic` (scientific naming convention).
  The combination of bold (700) + italic distinguishes the H3 genus heading
  visually from H2 (bold, not italic) and body (regular, not italic).
- All metadata lines (badge, map captions, list counts) use `0.85rem` at
  `--text-muted` — the same `.metadata` style already used on this page.
- No new font sizes are introduced beyond what the page already uses.

### SVG map color contract

Inherited from `data/species_maps.py` `STYLE_CSS`. The same `STYLE_CSS` block is
used for both the county coverage map and the ecoregion coverage map. No new SVG
fill colors.

| SVG class | Fill | Opacity | Stroke | Purpose |
|-----------|------|---------|--------|---------|
| `.county` | `#f4f4f0` | 1.0 | `#888` / `0.5px` | Blank (not-contributed) polygon |
| `.checklist-county` | `#b0cfe8` | 0.5 | `#888` / `0.5px` | Contributed polygon |

Both coverage map SVGs (county and ecoregion) use `.county` for blank polygons and
`.checklist-county` for contributed polygons. The class names are SVG-internal
implementation details — they never appear in the HTML page's DOM (the SVGs are
delivered as `<img>`). The visual contrast between blank (#f4f4f0) and contributed
(#b0cfe8 at 50% = ~#D2E2EC effective) is sufficient for a binary categorical map
where stroke borders (#888) reinforce polygon boundaries. WCAG contrast requirements
do not apply to decorative/map imagery (non-text); the quantitative count is also
conveyed in the text caption.

## Page Layout Contract

**The two coverage maps (county + ecoregion, side-by-side at ≥768px) are the primary
visual anchor of the accomplishment view; the species list is secondary.** Executors
must not visually deprioritize the maps relative to the text sections.

### Section order in `_pages/collector-detail.njk`

The four new elements are inserted between the existing atlas link and the event
feed. This places the "where/what/how long" accomplishment summary above the
chronological event history.

```
<article class="places-page place-detail">
  <h1>                        ← existing: display name
  <p class="metadata">        ← existing: N specimens · N samples · N species
  <p class="metadata">        ← NEW (#1): badge — "Active since YYYY (N seasons)"
  <p class="status-split">    ← existing: N identified to species, N awaiting ID
  <a>                         ← existing: View on the atlas →

  <section class="coverage-section">   ← NEW (#2 + #3): county map + ecoregion map
    <div class="coverage-maps">
      <div class="map-block">          ← county map
        <img loading="lazy" ...>
        <p class="metadata">           ← county count caption
      </div>
      <div class="map-block">          ← ecoregion map
        <img loading="lazy" ...>
        <p class="metadata">           ← ecoregion count caption
      </div>
    </div>
  </section>

  <section class="species-section">    ← NEW (#4): taxonomic breadth list

  </section>
```

## Element-by-Element Contract

### Element 1: Active-Seasons Badge (ACCOM-04)

**Pattern:** Second `<p class="metadata">` line — reuses the existing metadata
paragraph class already on this page. No new class or element type.

**Nunjucks:**
```html
<p class="metadata">Active since {{ collector.active_since }} ({{ collector.seasons_count | quantify("season") }})</p>
```

**Conditional:** Render only when `collector.active_since` is truthy (any collector
meeting the D-01 gate will have at least one season). If `seasons_count` is 0 or
`active_since` is null, omit the line rather than render "Active since None".

**Data source:** `collector.active_since` (int, MIN year) and `collector.seasons_count`
(int, COUNT DISTINCT year) from `collectors.json`, computed by `collectors_export.py`.

**Copy locked by D-05:** "Active since {YYYY} ({N} seasons)" — do not substitute
"years" for "seasons", do not add "(streak)" language.

### Elements 2 + 3: Coverage Maps (ACCOM-01, ACCOM-03)

**Pattern:** Mirrors `_pages/species-detail.njk` line 25:
```html
<img loading="lazy" src="/data/species-maps/{{ sp.slug }}.svg" alt="Occurrence map for {{ sp.scientificName }}">
```

**Nunjucks for the coverage section:**
```html
{%- if collector.county_count > 0 or collector.ecoregion_count > 0 -%}
<section class="coverage-section">
  <div class="coverage-maps">
    {%- if collector.county_count > 0 -%}
    <div class="map-block">
      <img loading="lazy"
           src="/data/collector-maps/{{ collector.login }}.svg"
           alt="County coverage map for {{ collector.display_name }} — {{ collector.county_count | quantify('county', 'counties') }}">
      <p class="metadata">{{ collector.county_count | quantify("county", "counties") }}</p>
    </div>
    {%- endif -%}
    {%- if collector.ecoregion_count > 0 -%}
    <div class="map-block">
      <img loading="lazy"
           src="/data/collector-maps/{{ collector.login }}-eco.svg"
           alt="Ecoregion coverage map for {{ collector.display_name }} — {{ collector.ecoregion_count | quantify('ecoregion') }}">
      <p class="metadata">{{ collector.ecoregion_count | quantify("ecoregion") }}</p>
    </div>
    {%- endif -%}
  </div>
</section>
{%- endif -%}
```

**Alt text format (locked):**
- County map: `"County coverage map for {display_name} — {N} counties"`
- Ecoregion map: `"Ecoregion coverage map for {display_name} — {N} ecoregions"`

The count is embedded in the alt text so screen reader users receive the same
quantitative information as sighted users reading the caption below the image.

**Image sizing (CSS, mirrors existing map rules):**
```css
.places-page img[src*="/collector-maps/"] {
  aspect-ratio: 15 / 8;   /* matches viewBox "0 0 600 320" */
  width: 100%;
  max-width: 600px;
}
```

This selector mirrors the existing rules for `/place-maps/` and `/species-maps/`
in `places.css` and `taxon-pages.css`.

**County count caption (recommended — discretion):** Render "N counties" under
the county map, symmetric with the required "N ecoregions" caption. Both captions
use `<p class="metadata">`. This is recommended to maintain visual symmetry
between the two map blocks — if the operator prefers to omit the county caption,
the ecoregion caption still satisfies D-03.

**Layout — two-column on wide screens:**
```css
.coverage-section {
  margin-top: 1.5rem;   /* 24px — matches .event-feed-section */
}

.coverage-maps {
  display: flex;
  flex-direction: column;
  gap: 1.5rem;           /* 24px */
}

@media (min-width: 768px) {
  .coverage-maps {
    display: grid;
    grid-template-columns: 1fr 1fr;
    gap: 1.5rem;
    align-items: start;
  }
}
```

On narrow screens the two maps stack vertically. On ≥768px they sit side by side,
matching the operator's intent: "ecoregions might as well be a map like the
counties" (symmetric visual treatment).

**SVG file paths:**
- County: `/data/collector-maps/{login}.svg`
- Ecoregion: `/data/collector-maps/{login}-eco.svg`

Both delivered via S3 stable URL, not committed to git, not in `manifest.json`
(same delivery mechanism as `species-maps/` and `place-maps/`).

### Element 4: Taxonomic-Breadth Species List (ACCOM-02)

**Pattern:** Genus-grouped list. Nearest existing analog: the genus-level grouping
in `_pages/species-detail.njk`'s breadcrumb and the `.species-list` in
`taxon-pages.css`. Rules for `.species-list` are duplicated into `places.css`
scoped to `.species-section` (avoids loading all of `taxon-pages.css` on the
collector page, which has many taxon-specific rules).

**Nunjucks:**
```html
{%- if collector.species_by_genus and collector.species_by_genus.length > 0 -%}
<section class="species-section">
  <h2>Species collected</h2>
  <div class="species-by-genus">
  {%- for genus_group in collector.species_by_genus -%}
    <div class="genus-section">
      <h3 class="genus-heading"><em>{{ genus_group.genus }}</em></h3>
      <ul class="species-list">
      {%- for sp in genus_group.species -%}
        <li>
          <a href="/species/{{ sp.slug }}/">{{ sp.canonical_name }}</a>
          <span class="count">({{ sp.count }})</span>
        </li>
      {%- endfor -%}
      </ul>
    </div>
  {%- endfor -%}
  </div>
</section>
{%- endif -%}
```

**Per-species count (recommended — discretion):** Show occurrence count as
`<span class="count">(N)</span>` after the species link. Uses the `.count` class
already in `taxon-pages.css` — muted, right-aligned, 0.85rem. Occurrence count is
consistent with `species_count` semantics already used on this page. The count
gives collectors context ("I've seen this species 8 times").

**H2 heading copy:** "Species collected" — factual, not "My species" or "Checklist."

**H3 genus heading:** Italic (scientific convention), 1rem body size, weight 700
(bold heading), `--text-body` color. Bold + italic together distinguish the H3
from H2 (bold, not italic) and body text (regular weight, upright).

**CSS additions to `places.css`:**
```css
/* Phase 172: species-by-genus section */
.species-section {
  margin-top: 1.5rem;   /* 24px — matches .event-feed-section, .coverage-section */
}

.genus-section {
  margin-bottom: 1rem;  /* 16px */
}

.genus-heading {
  font-size: 1rem;
  font-weight: 700;
  font-style: italic;   /* scientific naming convention */
  color: var(--text-body, #213547);
  margin: 0.5rem 0 0.25rem; /* sm top, xs bottom */
}

/*
 * .species-list pattern: duplicated from taxon-pages.css to avoid loading
 * the entire taxon-pages.css stylesheet on collector pages.
 */
.species-section .species-list {
  list-style: none;
  margin: 0;
  padding: 0;
}

.species-section .species-list li {
  display: flex;
  align-items: baseline;
  gap: 0.5rem;          /* 8px */
  padding: 0.25rem 0;   /* 4px */
  border-bottom: 1px solid var(--border, #ddd);
}

.species-section .species-list .count {
  margin-left: auto;
  font-size: 0.85rem;
  color: var(--text-muted, #666);
}
```

**Data shape** (from `collectors.json` `species_by_genus` array — per RESEARCH.md):
```json
{
  "genus": "Andrena",
  "species": [
    {"canonical_name": "Andrena milwaukeensis", "slug": "Andrena/milwaukeensis", "count": 8},
    {"canonical_name": "Andrena prunorum",       "slug": "Andrena/prunorum",       "count": 3}
  ]
}
```

Genera ordered alphabetically. Species ordered alphabetically within each genus.

## SVG Visual Contract

### County coverage map

- ViewBox: `"0 0 600 320"` (inherits from `species_maps.py` `VIEWBOX`)
- Projection: linear lon/lat → SVG (inherits `_project` from `species_maps.py`)
- Backdrop paths: all 39 WA counties, `class="county"` — fill `#f4f4f0`, stroke `#888` 0.5px
- Filled paths: contributed counties, `class="checklist-county"` — fill `#b0cfe8` opacity 0.5, stroke `#888` 0.5px
- Style block: embedded single `<style>` block (same `STYLE_CSS` constant reused by `collector_maps.py`)
- No occurrence dots (no `.occ` class paths on coverage maps)
- No legend, no choropleth gradients — binary fill only

### Ecoregion coverage map

- ViewBox: `"0 0 600 320"` (same as county map — same projection space)
- Projection: same `_project` linear projection from `species_maps.py`
- Polygon source: `ecoregions.geojson` from `EXPORT_DIR`, keyed by `NA_L3NAME` property
- Backdrop paths: all WA L3 ecoregion polygons (~9 distinct L3 names), `class="county"` — same fill as county backdrop
- Filled paths: contributed ecoregions, `class="checklist-county"` — same blue tint as county fill
- Style block: same `STYLE_CSS` constant — `.county` and `.checklist-county` semantics apply identically
- Binary fill only — no occurrence dots, no gradient

Note: reusing `.county` / `.checklist-county` class names for ecoregion shapes is
intentional. Class names are SVG-internal (SVG delivered as `<img>`) and do not
create semantic confusion in the HTML page DOM.
