<objective>
Wave A — the DATA LEG of the source→facets decomposition. **Fully replace** the overloaded
`source` enum on `marts/occurrences` (D-04) with two orthogonal materialized columns: `tier`
(`atlas` / `other`) and `record_type` (renamed per-arm vocabulary, `inat_obs` → `inat_expert`).
This is a dbt contract change (drop one column, add two → net +1, 38→39 columns), so the
**data-before-code release sequence** applies (D-04): this leg ships and publishes to S3 ALONE,
before the frontend wave (Plan 02) deploys.

Purpose: The arm→tier mapping is materialized once (in the five `int_combined` arm SELECTs, D-05)
so every downstream consumer — filter SQL, URL, symbology — reads a pre-computed column and never
recomputes the mapping. This is the foundation the frontend wave consumes.

Output: `int_combined.sql` (5 arms project tier+record_type, drop source), `occurrences.sql`
(j.tier, j.record_type), `schema.yml` (contract + two not_null where-clauses rewritten),
`collectors_export.py` (5 predicates in tier/record_type terms), `sqlite_export.py` (_GEO_COLS
index 6 source→tier). One-time `SKIP_INTEGRATION_GATE=1` nightly publishes to S3.
</objective>
