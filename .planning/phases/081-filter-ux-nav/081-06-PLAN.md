Close two gaps from `.planning/phases/081-filter-ux-nav/081-UAT.md`:

Root cause: `_includes/taxon-tree.njk` emits cross-route `<a href="/?taxon=…&taxonRank=…">` at species (line 20), genus (line 51), and family (line 101) ranks; `bee-taxon-nav._onClick` deliberately allows native navigation per a now-incorrect LINK-03 assumption. private _onClick = (e: MouseEvent): void => {
  const target = e.target as HTMLElement | null;
  if (!target) return;
  const li = target.closest('li[data-taxon]') as HTMLElement | null;
  if (!li) return;
  // Walks ancestor li[data-taxon] elements, building path[] and ranks[]
  // from outer→inner.
  const rank = ranks[ranks.length - 1] ?? 'species';
  this.dispatchEvent(new CustomEvent('taxon-selected', {
    bubbles: true, composed: true, detail: { path, rank },
  }));

From src/species/bee-species-filter.ts (existing `_setMonth`, line 127):
```typescript
private _setMonth(field: 'monthFrom' | 'monthTo', value: number): void {
  if (!Number.isFinite(value) || value < 1 || value > 12) return;
  this[field] = value;
  // WR-01 inversion guard
  if (this.monthFrom > this.monthTo) {
    if (field === 'monthFrom') this.monthTo = value;
    else this.monthFrom = value;
  }
  this._emit();
}
```
The `Number((e.target as HTMLInputElement).value)` cast in the change handler also works on `HTMLSelectElement.value` (a string of digits parses identically).
