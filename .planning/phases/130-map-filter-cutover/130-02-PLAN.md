<objective>
Wire the consumer side of the `taxon_id` cutover: build the lazy taxon cache and the
autocomplete index in `bee-atlas.ts` (off the `tablesReady` boot path), apply the D-03
label scheme and D-05 ordering, switch `bee-filter-controls.ts` tokens to the
`taxonId` shape, and rewrite `url-state.ts` to encode an integer `taxon=` with a
two-phase legacy back-compat decode.

The single most important data-source change is the autocomplete enumeration (D-01).
Runtime verification (Wave 0) proved the naive SQL EXISTS form is correct but ~10 s
(no index on `occurrences.taxon_id`); the efficient equivalent — enumerate the distinct
present occurrence `taxon_id`s, then union each one's `is_anthophila=1` ancestors via
`lineage_path` — produces the identical 666-taxon eligible set in ~3.5 ms and reuses
the cache build. Use the efficient form.

Purpose: Close MFILT-02 (autocomplete + disambiguation) and MFILT-03 (URL round-trip +
back-compat), and complete MFILT-01's filter-state wiring.
Output: working `taxon_id` autocomplete + filter + URL on the new contract; unit tests
green for labels, ordering, and URL round-trip/back-compat.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| URL params → FilterState | User-supplied `taxon=` value parsed at page load |
| FilterState → SQL | taxonId interpolated into buildFilterSQL (owned by Plan 01) |
| taxa/occurrences → worker | enumeration + cache queries run in wa-sqlite worker |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-130-LU | Tampering / Injection | legacy `taxon=<name>` URL back-compat path | mitigate | The legacy name string is resolved to an integer taxon_id via an in-memory `_taxonCache` lookup (exact-match on `name`/`rank`), NEVER interpolated into a SQL WHERE clause. Only the resolved integer `taxonId` reaches `buildFilterSQL`. If no cache match, the filter stays inactive — the raw string is discarded. |
| T-130-IV | Input validation | integer `taxon=` parse | mitigate | `parseInt` + `String(asInt) === taxonRaw` round-trip guard rejects non-canonical/float/overflow forms; only a clean integer becomes `taxonId`. |
| T-130-XSS | Information disclosure / XSS | pending-legacy name stored in state | accept | The legacy name is stored as filter state and used only for an equality cache lookup; it is never written to the DOM as HTML (no reflected XSS). Display labels come from the cache, not the URL string. |
| T-130-02 | Information disclosure | enumeration/cache queries | accept | Read-only public reference data (taxa/occurrences); no PII. |
</threat_model>
