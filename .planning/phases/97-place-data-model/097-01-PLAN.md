  <behavior>
    validate_places(toml_path: Path | str) -> None:
      - Reads toml_path with tomllib (stdlib, rb mode)
      - Raises ValueError("places.toml: place '{slug}': {reason}") for any of:
          a. Slug with characters outside [a-z0-9-] — reason: "slug contains invalid characters"
          b. Duplicate slug — reason: "duplicate slug"
          c. WKT that DuckDB cannot parse — reason: "invalid geometry WKT: {error}"
          d. Geometry coordinates outside WGS84 bounds (lon outside -180..180, lat outside -90..90)
             — reason: "geometry is not in WGS84 (coordinates out of range)"
          e. Any two places whose geometries ST_Intersects — reason: "polygon overlaps with place '{other_slug}'"
      - Returns None on success (no exception = valid)
      - Uses an in-memory DuckDB connection with spatial loaded (not a file-based DB)
      - All validation checks run before returning — collect all errors, raise on first

    validate_places_step() -> None:
      - Wrapper callable for run.py STEPS list (zero-argument)
      - Derives toml_path as Path(__file__).parent.parent / "content" / "places.toml"
      - Calls validate_places(toml_path)
      - Any ValueError raised by validate_places propagates naturally (run.py traceback handler catches it)
  </behavior>
  <action>
    Create data/places_validation.py. Python 3.14+; use tomllib (stdlib).

    Module-level imports: tomllib, re, duckdb, pathlib.Path.

    Function signature:
      def validate_places(toml_path: "Path | str") -> None:

    Implementation notes:

    1. SLUG REGEX — compile once: SLUG_RE = re.compile(r'^[a-z0-9-]+$')
       Check each place's slug. Raise ValueError for non-matching or duplicate (use a seen set).

    2. GEOMETRY VALIDITY + CRS — use DuckDB in-memory with spatial:
       ```
       con = duckdb.connect(":memory:")
       con.execute("LOAD spatial")
       ```
       For each place, validate geometry:
         a. Parse: run `SELECT ST_GeomFromText(?)` with (wkt_string,). If DuckDB raises an
            exception or returns NULL, the WKT is invalid — raise ValueError.
         b. WGS84 bounds check: use
            `SELECT ST_XMin(g), ST_XMax(g), ST_YMin(g), ST_YMax(g)
             FROM (SELECT ST_GeomFromText(?) AS g)`
            Assert xmin >= -180, xmax <= 180, ymin >= -90, ymax <= 90.
            This is the required CRS validation because WKT has no embedded CRS metadata.

    3. OVERLAP CHECK — after validating all individual geometries, check all pairs:
       Load all geometries into a temporary DuckDB table:
         `CREATE TABLE places_geom (slug VARCHAR, geom GEOMETRY)`
         INSERT each valid geometry.
       Then run:
         `SELECT a.slug, b.slug
          FROM places_geom a, places_geom b
          WHERE a.slug < b.slug AND ST_Intersects(a.geom, b.geom)`
       Each row is an overlap pair — raise ValueError for the first overlap found.
       ST_Intersects returns true if geometries share any point (including boundary contact).

    Callable wrapper for run.py:
      ```python
      def validate_places_step() -> None:
          toml_path = Path(__file__).parent.parent / "content" / "places.toml"
          validate_places(toml_path)
      ```

    DO NOT add shapely. DO NOT open a file-based DuckDB. Use duckdb.connect(":memory:").
    DO NOT use `INSTALL spatial` in the module (spatial is already installed in the pipeline
    environment); use `LOAD spatial` only.
