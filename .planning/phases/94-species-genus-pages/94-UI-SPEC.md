# Phase 94 — UI Design Contract

> Visual and interaction contract for Phase 94: Species & Genus Pages.
> Generated by gsd-ui-researcher.

---

## Scope Note

This phase introduces two new Eleventy-generated static page types:

1. **Species page** — `/species/{Genus}/{specificEpithet}/` — photo, per-species SVG
   occurrence map, seasonality bar chart, metadata.
2. **Genus page** — `/species/{Genus}/` — species listing table with swatches + counts,
   multi-color genus SVG occurrence map.

These pages extend the existing v3.2 species card pattern (`bee-species-card`,
`seasonality-viz`, `species.css`) rather than replacing it. All design tokens inherit
from `src/index.css`. No new component library or design system is introduced.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px (1rem) | 400 | 1.5 |
| Small / attribution / badge | 12px (0.75rem) | 400 | 1.5 |
| Section label / hint | 13.6px (0.85rem) | 400 | 1.5 |
| Page heading (h1 — species scientific name) | 24px (1.5rem) | 600 | 1.2 |

Scientific names (species and genus) MUST be rendered in `<em>` (italic) when appearing
as the primary `<h1>` of a taxon page and in table cells in the genus listing. The
author / authority suffix (if any) is roman (non-italic). Example:
`<h1><em>Agapostemon femoratus</em></h1>`.

Source: body/label from `src/index.css`; h1 size inferred from `bee-header.ts` h1 at
1.2rem — species pages use a slightly larger 1.5rem to give the species name appropriate
hierarchy as a standalone page.

## Layout Contract

### Species Page Layout

Single-column on mobile; on desktop (≥768px), a two-column layout mirrors the existing
species card pattern in `species.css`:

```
┌──────────────────────────────────────────┐
│ <bee-header>                             │
├──────────────────────────────────────────┤
│ [breadcrumb]  Halictidae / Agapostemon / │
│               femoratus                  │
├──────────────────────────────────────────┤
│ <h1><em>Agapostemon femoratus</em></h1>  │
├────────────────────┬─────────────────────┤
│  Photo (4:3 hero)  │  SVG Map (15:8)     │
│  max-height: 360px │  max-width: 600px   │
│                    │  width: 100%        │
├────────────────────┴─────────────────────┤
│  attribution line (0.75rem, muted)       │
├──────────────────────────────────────────┤
│  Seasonality chart (240×94 SVG)         │
├──────────────────────────────────────────┤
│  Metadata row: N records · N counties   │
│  · N ecoregions · first–last dates       │
├──────────────────────────────────────────┤
│  "View N occurrences on the atlas →"    │
└──────────────────────────────────────────┘
```

- **Mobile (<768px):** full-width single column; photo above map above seasonality.
- **Desktop (≥768px):** photo and map side-by-side in a 2-column grid
  (`grid-template-columns: 1fr 1fr`), max-width 1100px, centered.
- Page `max-width: 1100px; margin: 0 auto; padding: 1rem;`
- Photo: `aspect-ratio: 4/3; max-height: 360px; width: 100%; object-fit: contain; background: #f5f5f5;`
- SVG Map: `aspect-ratio: 15/8; width: 100%; max-width: 600px;` (existing viewBox 600×320)
- No photo and no map available: fallback states per Copywriting Contract below.

### Genus Page Layout

Single-column on mobile; on desktop, map and species list side-by-side:

```
┌──────────────────────────────────────────┐
│ <bee-header>                             │
├──────────────────────────────────────────┤
│ [breadcrumb]  Halictidae / Agapostemon   │
├──────────────────────────────────────────┤
│ <h1><em>Agapostemon</em></h1>            │
│ N species · N records                    │
├────────────────────┬─────────────────────┤
│  Genus SVG Map     │  Species list        │
│  (15:8 aspect,     │  (scrollable table) │
│  max-width: 600px) │                     │
└────────────────────┴─────────────────────┘
```

- **Mobile (<768px):** map above species list, full-width.
- **Desktop (≥768px):** `grid-template-columns: 1fr 1fr`, max-width 1100px, centered.
- Genus SVG Map: `aspect-ratio: 15/8; width: 100%;`

Species list on genus page:

```
┌──────────────────────────────────────────────────────┐
│ ● [swatch]  Agapostemon femoratus          91 records │
│ ● [swatch]  Agapostemon texanus            14 records │
│ ...                                                   │
└──────────────────────────────────────────────────────┘
```

- Rendered as `<ul>` with no bullets; each `<li>` contains:
  - 12×12px color swatch square (inline-block, border-radius: 2px) — HSL color matching
    the SVG map dot color for that species, assigned in same alphabetical sort order as
    Phase 93 D-01 and D-02.
  - 4px gap, then `<a href="/species/{Genus}/{epithet}/">` containing `<em>{scientificName}</em>`.
  - Right-aligned: `{occurrence_count} records` in `0.85rem` muted color.
- Sorted alphabetically by `canonical_name` (same order as Phase 93 D-01/D-02 color
  assignment — ensures swatch color matches SVG dot color).
- Species with `occurrence_count === 0` are listed but the swatch is rendered in `#ccc`
  (no occurrence color assigned by Phase 93) and link is still present.

## Interaction Contract

### Breadcrumb Navigation

Both page types render a static breadcrumb above the `<h1>`. No JavaScript required.

Format:
```
Halictidae / <a href="/species/Agapostemon/">Agapostemon</a> / femoratus
```

- Family name: plain text (non-linked in Phase 94; future Phase 96 adds the index).
- Genus name: links to the genus page `/species/{Genus}/`.
- Specific epithet (species page only): plain text (current page).
- Separator: ` / ` with 4px padding on each side.
- Font: 0.85rem, `--text-muted` (#666).

Genus page breadcrumb:
```
Halictidae / Agapostemon
```
Both segments are plain text (family links to nothing yet; genus is the current page).

### "View N occurrences on the atlas" Link

On species pages, one link at the bottom of the page navigates to the main atlas map
pre-filtered to that species. Exact format:

```
View {occurrence_count} occurrences on the atlas →
```

URL: `/?taxon={scientificName | urlencode}&taxonRank=species`

This is the existing pattern from `_pages/species.njk` `.spa-link`. Reuse without change.

### Photo Click / Zoom

No lightbox or zoom in this phase. Photo `<img>` is a plain img element, not interactive.

### Color Swatch (Genus Page)

The swatch square on the genus species list is a non-interactive `<span>` with
`display: inline-block; width: 12px; height: 12px; background: {hexColor};
border-radius: 2px; vertical-align: middle;`. No tooltip or label beyond the species
name that follows it.

### No JavaScript on Static Pages

Both page templates use plain Nunjucks + static HTML. `<bee-header>` is the only web
component included (same as the default layout). `<seasonality-viz>` may be included on
species pages as a static server-rendered fallback or as an enhanced LitElement (see
Seasonality section). No other JS is required.

## Seasonality Visualization

Reuse the existing `<seasonality-viz>` LitElement from `src/species/seasonality-viz.ts`.

On the new static species pages, `month_histogram` is available directly from
`species.json` at build time — no runtime fetch required.

**Contract:**
- Render `<seasonality-viz></seasonality-viz>` in the Nunjucks template.
- Pass `month_histogram` as a `data-` attribute or inline script on the element:
  ```html
  <seasonality-viz id="sviz"></seasonality-viz>
  <script>
    document.getElementById('sviz').data = {{ sp.month_histogram | dump | safe }};
  </script>
  ```
- If `occurrence_count < 5`, the component renders its existing text fallback
  (`{N} record(s), {month range}`). No special handling needed in the template.
- Chart dimensions: 240×94px viewBox (240 wide + 14px axis label strip = 94 total height
  per existing component). Full width of its container on mobile; intrinsic size on
  desktop.

Source: `src/species/seasonality-viz.ts` — thresholds, colors, and dimensions are locked
by existing implementation.

## SVG Map Dimensions

Both species and genus SVG maps use the same viewBox as Phase 92/93 output:

| Property | Value |
|----------|-------|
| viewBox | `0 0 600 320` |
| Aspect ratio | 15:8 |
| CSS display | `width: 100%; max-width: 600px; aspect-ratio: 15/8;` |
| Alt text — species | `"Occurrence map for {scientificName}"` |
| Alt text — genus | `"Occurrence map for genus {Genus}"` |

Map path on disk:
- Species: `/data/species-maps/{Genus}/{specificEpithet}.svg`
- Genus: `/data/species-maps/genus/{Genus}.svg`

Only render the `<img>` element when `occurrence_count > 0` for species maps. For genus
maps, always render (genus has at least one species with records if it appears in
species.json).

## Photo Display

| State | Rendering |
|-------|-----------|
| Photo available | `<img loading="lazy" src="{url}" alt="{caption or scientificName}">` followed by `<p class="attribution">{attribution}</p>` |
| No photo in TOML | Gray placeholder `<div class="photo-placeholder">No photo available</div>` with `background: #f5f5f5; aspect-ratio: 4/3; max-height: 360px; display: flex; align-items: center; justify-content: center; color: #767676; font-style: italic; font-size: 0.85rem;` |
| Multiple photos | Display only the first photo (`photos[0]`) in Phase 94. No gallery. |

Photo lookup key in `content/species-photos.toml` uses `scientificName` (the
`[species."Agapostemon femoratus"]` key format established in Phase 92).

## Copywriting Contract

| Element | Copy |
|---------|------|
| Page title (species) | `{scientificName} — BeeAtlas` |
| Page title (genus) | `{Genus} — BeeAtlas` |
| Species h1 | `{scientificName}` (in `<em>`) |
| Genus h1 | `{Genus}` (in `<em>`) |
| Genus subheading | `{speciesCount} species · {totalOccurrences} records` |
| Species metadata row | `{occurrence_count} records · {county_count} counties · {ecoregion_count} ecoregions` |
| Atlas deep-link | `View {occurrence_count} occurrences on the atlas →` |
| No photo fallback | `No photo available` |
| No map (zero occurrences) | Omit the `<img>` element entirely — no placeholder text |
| Empty genus (0 species with records) | `No occurrence records for this genus yet.` |
| Seasonality fallback (<5 records) | Handled by existing `seasonality-viz` component: `{N} record(s){, month range}` |
| Record count badge (species list) | `{occurrence_count} records` |

No destructive actions exist in this phase. No confirmation dialogs.

No new error states: these are static pages generated at build time. If a species SVG
map file is missing (occurrence_count === 0), the `<img>` is simply not rendered.
