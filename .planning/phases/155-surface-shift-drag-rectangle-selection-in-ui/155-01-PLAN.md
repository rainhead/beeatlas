<objective>
Make the existing shift-drag bounding-box gesture discoverable on desktop by adding a persistent, static hint line below the "where" input in the sidebar filters section. This is a purely additive, presentation-only change — no behavior change to the gesture, bounds-filter, near-me, or URL contract (D-01).

Purpose: Today, holding Shift and dragging on the map draws a rectangle that filters occurrences (bounds-filter), but nothing in the UI tells users this exists. This phase surfaces it for desktop users while keeping the affordance hidden on touch devices (where the gesture cannot fire). It respects the no-new-UI-pattern memory rule by reusing the existing `.hint` class, and the pure-presenter invariant by introducing zero reactive state.

Output:
- `src/bee-pane.ts`: a `<p class="hint hint--desktop-only">Shift-drag on map to set bounds</p>` element + a `.hint--desktop-only` CSS rule gated by `@media (hover: hover) and (pointer: fine)`.
- `src/tests/bee-sidebar.test.ts`: a new source-text `describe` block asserting the copy, the `.hint` class, and the media query.
</objective>
