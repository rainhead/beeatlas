| URL â†’ _onPopState | Browser history API delivers URL state back into application |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-39-03-01 | Tampering | _onViewChanged event detail | accept | Detail is `'map'` or `'table'` set by bee-sidebar's own code. The value cannot be externally injected since Lit CustomEvents from child components are not accessible from outside the shadow DOM boundary. No user-supplied string reaches this handler. |
| T-39-03-02 | Tampering | _onPopState URL restoration | mitigate | `parsed.ui?.viewMode ?? 'map'` uses the whitelist-validated result from `parseParams`. parseParams ternary ensures only `'table'` produces non-default; any injected value becomes `'map'`. Defense-in-depth: even if parseParams were bypassed, viewMode only affects layout (which component renders), not data access or authentication. |
| T-39-03-03 | Denial of Service | bee-map remount on tableâ†’map toggle | accept | Remount re-initializes OpenLayers. The `_isRestoringFromHistory` guard prevents URL corruption from initialization view-moved events. Acceptable cost for D-04 compliance (bee-map absent from DOM in table view). |
</threat_model>
