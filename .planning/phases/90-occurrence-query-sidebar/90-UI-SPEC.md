## Interaction Contract

### Selection-to-Sidebar Flow

| Step | Action | Visual Result |
|------|--------|---------------|
| 1 | User releases shift-drag on map | Rectangle removed instantly (Phase 89 contract) |
| 2 | `selection-drawn` event fires; prior sidebar state cleared synchronously | Any previously open sidebar closes; map returns to unselected state |
| 3 | SQLite bounds query runs asynchronously | No loading indicator — query is sub-100ms on WA dataset (~14k rows) |
| 4a | Query returns ≥1 occurrence | Sidebar opens with matched occurrences in `bee-occurrence-detail` |
| 4b | Query returns 0 occurrences | Sidebar stays closed; no error, no toast, no visual feedback |

**No loading spinner or skeleton:** The SQLite query runs in under 100ms on the full WA occurrence set. A spinner would flash for too short a time to be useful and would add visual noise. Omit it entirely.

**No empty-result feedback:** SEL-05 explicitly requires that the sidebar not open on an empty result. No toast, snackbar, or status message is shown. The gesture simply completes silently.

### Sidebar Title

The sidebar header reads **"Selected specimens"** — this is the existing `.sidebar-title` copy in `bee-sidebar.ts` (line 115). No change is needed. The title applies equally to cluster-click and rectangle-selection results.

### Sidebar Header

Existing `.sidebar-header` row: title on left, `×` close button on right. The close button emits a `close` event; `bee-atlas` sets `_sidebarOpen = false`. No changes to this structure in Phase 90.

### Prior State Clearing

When `_onSelectionDrawn` fires, the following must be cleared **synchronously** before the async query begins:

- `_selectedOccurrences = null` — clears any prior cluster-click or rectangle result
- `_selectedOccIds = null` — clears halo overlay
- `_selectedCluster = null` — clears cluster center/radius
- `_sidebarOpen = false` — closes sidebar visually while query is in flight

This prevents stale sidebar content from persisting when a new rectangle is drawn over an empty area.

Source: RESEARCH.md Pitfall 3; `_onOccurrenceClick` structural analog in `src/bee-atlas.ts`.

### Focus Management

No focus management changes. The close button in the sidebar header is already keyboard-accessible (`<button>` element with `aria-label="Close detail panel"`). No new interactive elements are introduced.
