<objective>
Replace the current "one pushState per settled gesture" viewport→history write
with **session-coalesced** history (D-01/D-02): an entire pan/zoom exploration
session produces exactly ONE browser history entry. A session resets — so the
next viewport move starts a fresh entry — whenever any non-viewport state change
occurs (filter/selection/boundary/pane/source toggle), and after a popstate
navigation.

Purpose: stop deliberate map exploration from polluting the back button (the
viewport pushState is the only pushState in the app today, so back navigates
only between camera positions). After this change the back stack reads as a
sequence of meaningful states, with the live URL still always reflecting the
current viewport.

Output: modified viewport/reset logic in `src/bee-atlas.ts` and tests in
`src/tests/bee-atlas.test.ts`. NO change to what the URL encodes
(`buildParams`/`parseParams` untouched), NO change to `<bee-map>` (it stays a
pure presenter), NO new capability.
</objective>
