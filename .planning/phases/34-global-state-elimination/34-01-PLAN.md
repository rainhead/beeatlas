<objective>
Eliminate module-level mutable state from filter.ts and make style.ts stateless.

Purpose: filter.ts currently exports `filterState`, `visibleEcdysisIds`, `visibleSampleIds`, and `setVisibleIds` as module-level mutable singletons. style.ts imports `visibleEcdysisIds` and `visibleSampleIds` directly. This creates hidden coupling and makes both modules impossible to test or import without side effects. This plan removes the mutable exports from filter.ts and converts style.ts functions into factories that accept getter callbacks.

Output: filter.ts exports only the FilterState interface and pure functions. style.ts exports factory functions. bee-map.ts owns filterState and visibleIds as class properties, wiring them into style factories via closures.
</objective>
