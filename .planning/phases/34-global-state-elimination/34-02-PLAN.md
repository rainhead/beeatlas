<objective>
Move all module-level OL objects in bee-map.ts into BeeMap class properties (STATE-02) and eliminate module-level source/layer/eager-load from region-layer.ts (STATE-03).

Purpose: bee-map.ts currently instantiates `specimenSource`, `clusterSource`, `specimenLayer`, `sampleSource`, `sampleLayer`, and `dataErrorHandler` at module scope. region-layer.ts instantiates `countySource`, `ecoregionSource`, `regionLayer` and eagerly calls `loadFeatures()` at import time. These side effects prevent clean imports and testing. Moving them into BeeMap makes the module importable without side effects.

Output: bee-map.ts has zero module-level OL objects. region-layer.ts exports only immutable style constants and the `makeRegionStyleFn` factory. All OL object construction happens in BeeMap's class body or `firstUpdated`.
</objective>
