<objective>
Add a nullable `reason` column to the `note_revisions` ledger table (D-09) so a
curator's takedown/restore can record an optional free-text reason. This is the
schema substrate the Phase-180 curator routes (Plan 02) write into.

Purpose: D-09 requires an optional, empty-allowed reason stored on the ledger
row. The column must be nullable forever — this is NOT the three-step
add→backfill→tighten dance of migration 0003 (that was only needed because
`body_html` became NOT NULL on a populated table). A single
`batch_alter_table.add_column(nullable=True)` is sufficient.

Output: `NoteRevision.reason` ORM field, Alembic migration 0004, and its
apply/no-downgrade tests. The live maderas SQLite file is migrated by the
operator in Plan 05 (not here — this plan is writable and verifiable locally).
</objective>
