## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| build → static HTML | Eleventy renders trusted build-time data (species.json) into static HTML at SSG time; output served via CloudFront. No runtime user input crosses any boundary on these pages. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-95-04 | Tampering | species.json input → tribe template | accept | Build-time data is pipeline-controlled (data/dbt/ + species_export.py via maderas nightly). No untrusted authoring path. |
| T-95-05 | Information Disclosure | rendered tribe HTML | accept | Pages contain only public taxonomic and aggregate occurrence data already exposed via the atlas map and existing genus pages. |
| T-95-06 | Spoofing (URL collision) | `/species/tribe/{TribeName}/` namespace | mitigate | Confirmed in RESEARCH (URL-04 row): no genus is named "tribe" or "Tribe" in species.json, so the `tribe/` path segment is a safe namespace. If a future pipeline run ever introduces such a genus name, the static build would emit two pages at `/species/tribe/index.html` (the inserted genus page would collide with the tribe-listing index, which we do not emit) — at worst a build error, not a runtime exploit. |
</threat_model>
