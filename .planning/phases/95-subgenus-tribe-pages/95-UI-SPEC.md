# Phase 95 — UI Design Contract

> Visual and interaction contract for Phase 95: Subgenus & Tribe Pages.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

---

## Scope Note

This phase introduces two new Eleventy-generated static page types that directly
extend the genus page pattern established in Phase 94:

1. **Subgenus page** — `/species/{Genus}/{Subgenus}/` — species listing with swatches
   and counts, multi-color subgenus SVG occurrence map. Breadcrumb includes genus link.
2. **Tribe page** — `/species/tribe/{TribeName}/` — genus listing with links, multi-color
   tribe SVG occurrence map. No species-level swatches (tribe groups by genus).

These pages are parallel to `_pages/genus.njk`. They share `src/styles/taxon-pages.css`
and inherit all design tokens from `src/index.css`. No new CSS file is needed.

The key URL disambiguation rule (REQUIREMENTS.md URL-03): subgenus URLs have the
same 2-segment shape as species URLs (`/species/{Genus}/{Name}/`), but the subgenus
name is **capitalized** (`Melandrena`) while species specific epithets are **lowercase**
(`milwaukeensis`). This is a build-time structural distinction — no runtime disambiguation
logic is required.

## Typography

Same as Phase 94 — no new type roles introduced.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px (1rem) | 400 | 1.5 |
| Small / attribution / badge | 12px (0.75rem) | 400 | 1.5 |
| Section label / hint / metadata | 13.6px (0.85rem) | 400 | 1.5 |
| Page heading (h1) | 24px (1.5rem) | 600 | 1.2 |

Scientific names (subgenus, genus within tribe listings) MUST be rendered in `<em>`
(italic) when appearing as the page `<h1>`. Genus names in the tribe species list are
rendered in `<em>` per the existing taxon convention. Tribe names are NOT italic (tribe
names are not Latinized binomials — they are collective group names).

Source: Phase 94 UI-SPEC typography section, `src/index.css`.

| Dominant (60%) | `#fafafa` (`--surface`) | Page background |
| Secondary (30%) | `#f5f5f5` (`--surface-subtle`) | Map region backdrop (in SVG) |
| Accent (10%) | `#2c7a2c` (`--accent`) | Not used in this phase |
| Text — body | `#213547` (`--text-body`) | All body copy |
| Text — muted | `#666` (`--text-muted`) | Record count badges, metadata lines |
| Text — hint | `#767676` (`--text-hint`) | Empty state copy |
| Border | `#ddd` (`--border`) | List row dividers |

| Link hover | `#535bf2` (`--link-hover`) | Link hover state |

Accent (`--accent` green) is reserved for: active filter state on the main atlas map.

Per-species colors on subgenus and tribe SVG maps use the same HSL assignment as
Phase 93 D-01 (`hue = i * 360 / n`, saturation 70%, lightness 50%), computed within
the subgenus or tribe group respectively. These colors appear as swatches in the
subgenus species listing. Tribe pages list genera (not species), so no swatches appear
in the genus list on tribe pages.

Source: Phase 94 UI-SPEC color section, `src/index.css`.

## Layout Contract

### Subgenus Page Layout

Identical structure to the genus page (`_pages/genus.njk`). Single-column on mobile;
two-column on desktop.

```
┌──────────────────────────────────────────┐
│ <bee-header>                             │
├──────────────────────────────────────────┤
│ [breadcrumb]  Halictidae /               │
│               Andrena / Melandrena       │
├──────────────────────────────────────────┤
│ <h1><em>Melandrena</em></h1>             │
│ N species · N records                    │
├────────────────────┬─────────────────────┤
│  Subgenus SVG Map  │  Species list        │
│  (15:8 aspect,     │  (scrollable)        │
│  max-width: 600px) │                     │
└────────────────────┴─────────────────────┘
```

- **Mobile (<768px):** map stacked above species list, full-width.
- **Desktop (≥768px):** `grid-template-columns: 1fr 1fr; gap: 1.5rem;`, max-width 1100px
  centered, `align-items: start`.
- Subgenus SVG Map: `aspect-ratio: 15/8; width: 100%; max-width: 600px;`
- Uses `.taxon-page` and `.media-grid` CSS classes from `taxon-pages.css` — no new CSS.

Species list on subgenus page:

```
┌────────────────────────────────────────────────────┐
│ [swatch]  Andrena melandrena             32 records │
│ [swatch]  Andrena milwaukeensis          14 records │
│ ...                                                 │
└────────────────────────────────────────────────────┘
```

- Rendered as `<ul class="species-list">` (existing class from `taxon-pages.css`).
- Each `<li>`: 12×12px color swatch (`<span class="swatch">`), `<a>` with `<em>`, right-aligned count.
- Link target: `/species/{Genus}/{specificEpithet}/` (to species detail page — SUBG-03).
- Sorted alphabetically by `canonical_name` (matches Phase 93 D-01/D-02 color assignment).

### Tribe Page Layout

Replaces the species list with a genus list. No species-level swatches.

```
┌──────────────────────────────────────────┐
│ <bee-header>                             │
├──────────────────────────────────────────┤
│ [breadcrumb]  Halictidae / Andrenini     │
├──────────────────────────────────────────┤
│ <h1>Andrenini</h1>                       │
│ N genera · N records                     │
├────────────────────┬─────────────────────┤
│  Tribe SVG Map     │  Genera list         │
│  (15:8 aspect,     │  (scrollable)        │
│  max-width: 600px) │                     │
└────────────────────┴─────────────────────┘
```

- **Mobile (<768px):** map stacked above genus list, full-width.
- **Desktop (≥768px):** same `media-grid` two-column layout as genus page.
- Tribe SVG Map: `aspect-ratio: 15/8; width: 100%; max-width: 600px;`
- Uses `.taxon-page` and `.media-grid` CSS classes — no new CSS.

Genus list on tribe page:

```
┌────────────────────────────────────────────────┐
│  Andrena                          423 records   │
│  Calliopsis                        18 records   │
│  ...                                           │
└────────────────────────────────────────────────┘
```

- Rendered as `<ul class="species-list">` (reuse existing class — same visual pattern).
- Each `<li>`: NO color swatch. `<a href="/species/{Genus}/">` containing `<em>{Genus}</em>`,
  right-aligned total record count for that genus within the tribe.
- Link target: `/species/{Genus}/` (genus page — TRIBE-03).
- Sorted alphabetically by genus name.

**No swatch on tribe genus list.** The SVG map uses per-species colors but the HTML list
groups by genus — species-to-swatch mapping would require one swatch per species which
bloats the list. Instead, the multi-color map serves as the visual key; the genus list
is name + count only.

## Interaction Contract

### Breadcrumb Navigation

Both page types render a static breadcrumb above the `<h1>`. No JavaScript required.

Subgenus page format:
```
{Family} / <a href="/species/{Genus}/">{Genus}</a> / {Subgenus}
```
- Family: plain text (non-linked — same as genus page).
- Genus: links to the genus page.
- Subgenus: plain text (current page).

Tribe page format:
```
{Family} / {TribeName}
```
- Family: plain text.
- Tribe name: plain text (current page).
- Note: Tribes span multiple genera that may belong to different families in edge cases.
  Use the most common family for the tribe, or omit family from tribe breadcrumb if
  the tribe is not cleanly within one family. Simple implementation: use the family
  of the first genus in the tribe (alphabetical).

Format: `.breadcrumb` class, 0.85rem, `--text-muted` (#666). Separator `.sep` spans
with 4px padding each side.

### "View on atlas" Link

Not included on subgenus or tribe pages in Phase 95. Filtering the main atlas to a
subgenus or tribe requires multi-value taxon filtering not yet supported. Omit the link.

### No JavaScript on Static Pages

Both templates use plain Nunjucks + static HTML. `<bee-header>` is the only web
component (via `default.njk`). No `<seasonality-viz>` on group pages — only individual
species pages carry the seasonality chart.

## SVG Map Dimensions and Paths

Same viewBox as all Phase 92/93/94 maps.

| Property | Value |
|----------|-------|
| viewBox | `0 0 600 320` |
| Aspect ratio | 15:8 |
| CSS display | `aspect-ratio: 15/8; width: 100%; max-width: 600px;` |

Map paths on disk:
- Subgenus: `/data/species-maps/subgenus/{Genus}/{Subgenus}.svg`
  (e.g. `/data/species-maps/subgenus/Andrena/Melandrena.svg`)
- Tribe: `/data/species-maps/tribe/{TribeName}.svg`
  (e.g. `/data/species-maps/tribe/Andrenini.svg`)

Alt text:
- Subgenus: `"Occurrence map for subgenus {Subgenus} ({Genus})"`
- Tribe: `"Occurrence map for tribe {TribeName}"`

Always render the map `<img>` (subgenus and tribe groups always have at least one
species with occurrences if they appear in the data — groups with zero occurrences are
not generated by `species_maps.py`).

Source: Phase 93 pipeline output structure confirmed by `ls public/data/species-maps/`.

## Data Requirements

The `_data/species.js` module must be extended to expose two new lists for Eleventy
pagination:

**`species.subgenusList`** — array of subgenus group objects:
```js
{
  genus: string,         // parent genus (e.g. "Andrena")
  subgenus: string,      // subgenus name (e.g. "Melandrena")
  family: string,
  subfamily: string,
  tribe: string,
  species: Array<{       // species in this subgenus with occurrence_count > 0
    ...sp,               // all species.json fields
    hexColor: string,    // HSL color matching species_maps.py subgenus output
  }>,
  speciesCount: number,
  totalOccurrences: number,
}
```

**`species.tribeList`** — array of tribe group objects:
```js
{
  tribe: string,         // tribe name (e.g. "Andrenini")
  family: string,        // family of the tribe (from first genus alphabetically)
  genera: Array<{        // distinct genera in this tribe with occurrence_count > 0
    genus: string,
    occurrence_count: number,  // total across all species in this genus + tribe
  }>,
  generaCount: number,
  totalOccurrences: number,
}
```

Color assignment for subgenus species list swatches must use the same HSL formula as
`_data/species.js` `genusList` but scoped to the subgenus group. The color index `i`
is assigned over all species in the subgenus sorted by `canonical_name` — matching the
Python `_group_colors` input for that subgenus SVG.
