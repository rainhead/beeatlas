<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Database values -> filesystem paths | Filter values from DuckDB become part of filenames via _slugify |
| Generated XML -> static file serving | Feed XML served as static files on S3/CloudFront |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-43-01 | Tampering | _slugify -> filename | mitigate | _slugify strips all non-[a-z0-9-] chars, preventing path traversal (../) and special characters in filenames. Test test_slugify verifies this. |
| T-43-02 | Information Disclosure | Atom feed content | accept | Feed content is derived from public occurrence/identification data already served via the map UI. No PII beyond collector names which are already public. |
| T-43-03 | Denial of Service | Large number of variant files | accept | Bounded by distinct values in WA geographies (~39 counties, ~10 ecoregions) and active collectors/genera. Thousands of files at worst, all small XML. |
</threat_model>

<anti_patterns_to_avoid>
- DO NOT copy the skip-on-empty guard from write_determinations_feed into variant writers. D-01 requires always writing variant feed files.
- DO NOT use `WHERE o.county = ?` or `WHERE o.ecoregion_l3 = ?` — those columns do not exist on ecdysis_data.occurrences. Use spatial joins.
- DO NOT enumerate counties/ecoregions from the 90-day window. Enumerate from geographies tables to ensure all known regions get a feed file (D-01 intent).
- DO NOT forget `LOAD spatial;` in main() before variant queries. The Phase 42 main() does not load it.
- DO NOT open a new DB connection inside variant writers. Pass the connection from main().
- DO NOT forget coordinate NULL guards in county/ecoregion spatial queries. Without them, CAST of empty string to DOUBLE will fail.
- DO NOT write index.json before all variant feeds complete. Collect metadata in memory, write index.json last.
</anti_patterns_to_avoid>
