## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| /app page → SW (postMessage) | Page scripts (including potentially a malicious tab sharing origin) can post messages to the registered SW. The SW must gate `self.skipWaiting()` behind a known message shape. |
| CloudFront/S3 → SW (NetworkFirst /data/manifest.json) | Network responses cross the SW boundary into the new `data-manifest` cache. CacheableResponsePlugin restricts to status 200. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-150-01 | Spoofing | `self.addEventListener('message', ...)` in `src/sw.ts` | mitigate | Gate on the literal `event.data?.type === 'SKIP_WAITING'` constant; no other message shape advances the SW lifecycle. Build-output assertion `_site/app/sw.js calls skipWaiting only inside a message handler (D-16)` verifies the gate survives the compile. Severity: low — a same-origin page script could forge the message, but the worst case is that the SW activates one update cycle earlier than the user chose; no data corruption (D-04 maxEntries:1 + atomic cache.put preserves DB integrity), and the `<bee-atlas>` banner from Plan 04 surfaces the activation visibly. |
| T-150-03 | Information Disclosure | NetworkFirst route on `/data/manifest.json` | accept | `manifest.json` is already public over CloudFront with no auth; NetworkFirst with 3 s timeout falls back to cache; CacheableResponsePlugin restricts caching to status 200 so 403/5xx responses are not poisoned into the cache. Severity: informational — no PII; intent is freshness UX, not access control. |
| T-150-SC | Tampering | npm install workbox-window classification | mitigate | `workbox-window@7.4.1` is already installed; RESEARCH.md `## Package Legitimacy Audit` approved as VERIFIED at `node_modules/workbox-window/package.json` (author Google's Web DevRel, ~10M/wk downloads). No new install; only a `package.json` classification move (devDependencies → dependencies). No `[ASSUMED]`/`[SUS]` packages introduced — no slopcheck checkpoint required. |
