# Phase 150 — UI Design Contract

> Visual and interaction contract for the cache-health & freshness surfaces that go into the BeeAtlas `/app` PWA. Generated by gsd-ui-researcher from upstream artifacts; verified by gsd-ui-checker.

This phase introduces **no new top-level custom element** (CONTEXT D-20). All surfaces live inside `<bee-header>` (state-receiving presenter) and `<bee-atlas>` (state owner). Visual decisions extend existing tokens in `src/index.css` and the existing `.offline-pill` style block in `src/bee-header.ts`. Mobile-first; the `/app` route is for volunteer collectors in the field.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — manual CSS scoped to LitElement shadow roots |
| Preset | not applicable |
| Component library | Lit 3.3.3 web components (no third-party UI kit) |
| Icon library | Inline SVG (existing 24×24 stroke-1.5 outline glyphs in `<bee-header>`); status glyphs in this phase are **Unicode-only** (`✓`, `✕`) — no new icon family |
| Font | `system-ui, Avenir, Helvetica, Arial, sans-serif` (inherited from `:root` in `src/index.css`) |

**Rationale (no shadcn):** the BeeAtlas frontend has no React or design-system framework — it is hand-rolled Lit components with CSS variables. The orchestrator already passed UI-SPEC for Phase 149's `.offline-pill`; this phase extends the same pattern. Adding shadcn would be a milestone-scope decision, not a Phase-150 decision.

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline gaps between adjacent pills; pill border thickness gaps |
| sm | 8px | Popover internal row spacing; banner inline gap between text and dismiss button |
| md | 16px | Popover internal padding; banner internal vertical padding |
| lg | 24px | Popover max-width breathing on tablet+ |
| xl | 32px | (reserved — not used by this phase) |

**Exceptions (mandatory, not preferences):**

- **44px** minimum tap target for `.ready-pill` (button), banner tap-body, banner `✕` button, popover `✕` button — matches the existing `.icon-btn { min-width: 44px; min-height: 44px; }` rule in `src/bee-header.ts:46-47`. Required for mobile field use; do not shrink.
- **Pill internal padding `0.2rem 0.6rem`** (~3.2px / 9.6px) — kept identical to the existing `.offline-pill` rule in `bee-header.ts:81` so the new `.ready-pill` is visually consistent. This is the one place the 4-multiple rule is broken on purpose, for parity with the locked-in 149 surface.
- **Bottom banner bottom-offset `16px`** from viewport bottom (or `env(safe-area-inset-bottom)` plus 16px on iOS standalone, to clear the home-indicator).
- **Popover offset from pill `4px`** below the pill bottom edge.

## Typography

The `/app` route inherits `:root { font-family: system-ui, Avenir, Helvetica, Arial, sans-serif; line-height: 1.5; font-weight: 400 }` from `src/index.css:34-36`. This phase declares exactly **four sizes** and **two weights**.

| Role | Size | Weight | Line Height | Used For |
|------|------|--------|-------------|----------|
| Caption | 12px (`0.75rem`) | 400 | 1.4 | `.offline-pill` text, `.ready-pill` text, `.freshness-caption` text, popover meta lines |
| Body | 14px (`0.875rem`) | 400 | 1.5 | `.cache-popover` primary lines ("X MB stored on this device", "Data as of …") |
| Subhead | 16px (`1rem`) | 600 | 1.4 | Popover header label ("Offline cache" or equivalent), `.update-banner` body text |
| Title | ~19px (`1.2rem`) | 400 | 1.2 | Existing `<h1>BeeAtlas` — unchanged; declared so the freshness caption beneath it lands at the right scale |

**Notes:**
- Caption 12px reuses the literal `0.75rem` from `.offline-pill` to keep both pills visually equivalent.
- Two weights only: **400** (default body / captions) and **600** (subheads). No 500, no italics, no 700.
- Mobile minimum readable size: 12px is acceptable for captions on a chrome surface that is **decorative information** (state pill, "as of …"). Primary actionable text in the banner is 16px so it is comfortably tappable-while-reading.

## Color

Existing palette is in `src/index.css:1-32`. This phase introduces **no new top-level color tokens** but adds **two derived per-surface tokens** (constants colocated with the components that use them) so the executor doesn't reinvent values.

| Role | Value | Source | Usage |
|------|-------|--------|-------|
| Dominant (60%) | `var(--surface)` `#fafafa` | existing `:root` | Map area, pane background, popover background |
| Secondary (30%) | `var(--header-bg)` `rgb(8, 13, 38)` | existing `:root` | Header chrome where the pills + freshness caption live |
| Accent (10%) | `var(--accent)` `#2c7a2c` | existing `:root` | **Reserved-for list below** |
| Destructive | `var(--error)` `#b00` | existing `:root` | NOT used by this phase (no destructive actions) |

**Accent (`#2c7a2c`) is reserved EXCLUSIVELY for, in this phase:**
1. The `✓` glyph in the `.ready-pill` "ready" state (and its mirror in the popover header line) — signals an honest successful cache state.
2. The `.update-banner` tap-body background fill (or left-border accent stripe — see Banner section). Signals "this is an actionable affordance the user is invited to take."

Accent is NOT applied to:
- The priming-state ("Caching… 47%") pill — that state uses neutral chrome-on-chrome (see derived tokens below); accent on an in-progress state would mis-signal "done."
- The `.freshness-caption` — pure informational text, no accent.
- The popover background, borders, body text, or storage-estimate row.
- The `✕` dismiss buttons (neutral chrome opacity, matches existing `.icon-btn` pattern).

**Derived per-surface tokens (colocated in `bee-header.ts` `static styles`, not promoted to `:root`):**

- Pill text `#ffffff` on `rgba(255,255,255,0.2)` over `--header-bg #080d26` — effective foreground/background ≈ 11:1; WCAG AAA. Verified by the existing `.offline-pill` shipping.

- Banner body `#ffffff` on `var(--header-bg) #080d26` — 19:1; WCAG AAA.
- Accent `✓` `#2c7a2c` on `rgba(255,255,255,0.2)` over `--header-bg` — effective ≈ 5:1; WCAG AA. Use the glyph alongside the word "Offline-ready" so contrast is not the sole signal.

## State Diagrams

### `.ready-pill` — three states

| State | Trigger | Text | Visual | Click |
|-------|---------|------|--------|-------|
| **A. Priming online** | `_primeProgress != null` AND `navigator.onLine === true` AND not all assets cached | `Caching… 47%` | Pill chrome (white-20%/border-40%); inline thin determinate bar (2px tall, white-45%) filling left-to-right under the text; bar width = `received / total * 100%` | Opens popover (detail view confirms state) |
| **B. Priming + offline mid-prime** | `_primeProgress != null` AND `navigator.onLine === false` | `Finish on WiFi` | Same pill chrome; NO progress bar (would mislead — fetch is stalled); `.offline-pill` is also visible to its left, providing context | Opens popover |
| **C. Ready** | `_cacheState.ready === true` | `✓ Offline-ready` | Same pill chrome; the `✓` glyph in `var(--accent)` `#2c7a2c`; text in white; NO progress bar | Opens popover |

**Hidden state:** if `_cacheState == null` (e.g., before first cache probe completes on cold cold-start), render NOTHING. Pill mounts when there is honest signal (quiet-UI posture per CONTEXT.md).

**Percentage rounding:** Floor to integer; clamp to `[0, 99]` until ready (never show 100% before state C — that would be a misleading transitional flash).

**Text-vs-progress motion:** Caption text stays anchored; only the inline bar animates. No layout reflow during prime (the pill width does not change as the percentage ticks up — reserve width for the longest priming string "Caching… 99%" so chrome doesn't jiggle).

### `.cache-popover` — open/closed

| State | Trigger | Behavior |
|-------|---------|----------|
| **Closed** | `_popoverOpen === false` | Not in the DOM |
| **Open** | `_popoverOpen === true` after pill click | Mounted absolutely-positioned, focus moves to first focusable element inside (the `✕` button), `aria-expanded="true"` on the pill, dismiss listeners (click-outside, ESC) attached |
| **Closing** | User clicks outside / presses ESC / taps `✕` | Listeners removed; `_popoverOpen = false`; focus returns to `.ready-pill` |

### `.update-banner` — visibility

| State | Trigger | Behavior |
|-------|---------|----------|
| **Absent** | `_updateAvailable === false` (initial) | Not in DOM |
| **Visible** | `sw-update-available` CustomEvent fired by `sw-registration.ts` → `<bee-atlas>` sets `_updateAvailable = true` | Mount with slide-up motion (see Motion section); banner does NOT block underlying UI (`pointer-events` constrained to the banner itself; map/pane remain interactive) |
| **Dismissed (session)** | User taps the `✕` button | `_updateAvailable = false`. Per D-15, no persistence — banner re-appears on next page load if SW is still waiting |
| **Acting** | User taps the banner body (not the `✕`) | Calls `window.__wb.messageSkipWaiting()`, then `window.location.reload()`. Banner unmounts as part of reload. |

### Composite states (5 explicit visuals to lock for the planner)

| # | State Name | `.offline-pill` | `.ready-pill` | `.cache-popover` | `.update-banner` |
|---|-----------|-----------------|---------------|------------------|------------------|
| 1 | **Priming online** | hidden | A (Caching… N%) | closed | absent |
| 2 | **Offline mid-prime** | visible ("Offline") | B (Finish on WiFi) | closed | absent |
| 3 | **Ready (online or offline)** | visible iff offline | C (✓ Offline-ready) | closed | absent |
| 4 | **Update available (ready + waiting SW)** | hidden (assume online) | C (✓ Offline-ready) | closed | visible at bottom |
| 5 | **Popover open (any pill state)** | unchanged | any of A/B/C | open under the pill | unchanged |

## Copywriting Contract

All English-only (i18n out of scope). Verb + noun where action is implied; honest description otherwise. No exclamation marks, no editorializing.

### `.ready-pill` text

| State | Exact copy |
|-------|------------|
| A — priming online | `Caching… 47%` (the integer is dynamic; the word, ellipsis, space, percent sign are literal). Ellipsis is the single `…` character (U+2026), not three dots. |
| B — offline mid-prime | `Finish on WiFi` |
| C — ready | `✓ Offline-ready` (single space between glyph and word) |

### `.freshness-caption` text

Format per CONTEXT D-09, driven by `formatFreshness(generatedAt)` in `src/manifest.ts`:

| Δ from `generated_at` | Exact copy |
|-----------------------|------------|
| < 1 day | `Today` |
| 1 day | `Yesterday` |
| 2–6 days | `3 days ago` (numeral varies; word "ago" literal, via `Intl.RelativeTimeFormat({numeric:'always'})`) |
| ≥ 7 days, < 1 year | `Data as of Jun 15, 2026` (month abbr + day + year via `Intl.DateTimeFormat({year:'numeric', month:'short', day:'numeric'})`) |
| ≥ 1 year | `Data as of Mar 2026` (month abbr + year only) |
| Unparseable (dev sentinel `"local"`) | render nothing; `console.warn` per D-12 |

**Note:** the relative forms (`Today`, `Yesterday`, `N days ago`) deliberately OMIT the `Data as of` prefix — adding it reads as redundant for a relative phrase. Absolute forms include the prefix because "Jun 15, 2026" alone is ambiguous (date of what?).

### `.cache-popover` content (top-to-bottom)

| Slot | Exact copy / pattern | Rule |
|------|----------------------|------|
| Header | `Offline cache` (Subhead 16px/600) | Always shown; provides context for the cluster |
| Dismiss button | `✕` (U+2715) glyph only, `aria-label="Close"` | Always shown; top-right corner |
| Row 1 — Ready status | Mirrors pill text in larger form: `Caching 12.3 MB of 28.4 MB` (state A — note MB not %; popover gives the real numbers), `Finish on WiFi to complete cache` (state B), `✓ Offline-ready` (state C, `✓` in accent) | Always present |
| Row 2 — Freshness | `Data as of Jun 15, 2026` (or relative form) | Hidden if `_freshnessLabel === null` |
| Row 3 — Storage | `23.4 MB stored on this device` | Hidden if `navigator.storage?.estimate` undefined (D-19) |
| Row 3 sub-line — Quota | `of 47 MB available` (Caption 12px, `var(--text-hint)`) — only when quota is non-null AND < 200 MB (D-18) | Hidden otherwise |
| Row 4 — Update affordance (passive) | `App update available — tap to reload` (acts as a duplicate button with same behavior as `.update-banner` tap-body) | Hidden when `_updateAvailable === false` |

### `.update-banner` text

| Slot | Exact copy |
|------|------------|
| Body (tap-target) | `A data update is available — tap to reload` |
| Dismiss button | `✕` (U+2715) glyph only, `aria-label="Dismiss update for this session"` |

### Other copy elements

| Element | Copy / behavior |
|---------|-----------------|
| Primary CTA (this phase) | Banner body tap → reload. CTA label is the banner body text itself. There is no button-styled "Reload" because the entire banner is the tap-target (mobile pattern; reduces visual weight). |
| Secondary action | Banner `✕` → dismiss for session. |
| Empty state | **No empty state.** Surfaces are quiet-UI: they appear only when there is something honest to say. If there is no `_cacheState`, the `.ready-pill` does not render. If there is no `_freshnessLabel`, the caption does not render. The `<bee-header>` layout must gracefully reflow when these mount/unmount (already does — flex with `gap: 4px`). |
| Error state | **No error state surface in this phase.** Per CONTEXT D-09 (149), `QuotaExceededError` is console-warn only — Workbox's `purgeOnQuotaError` handles the partial-write cleanup, and a failed cache write naturally leaves `_cacheState.ready === false`, which surfaces honestly as pill state A or B. Network errors during prime do not show a toast — the `online` event-driven re-prime is silent, again per the 149 contract. |
| Destructive confirmation | **No destructive actions in this phase.** Reloading to apply an SW update is non-destructive (no user data on the device beyond the cached read-only artifacts). No confirmation required. |


**Internal layout:** vertical flex, `gap: 8px` between rows. Header row is a horizontal flex (`justify-content: space-between`) holding the `Offline cache` subhead and the `✕` button.

**Dismiss interactions:**
- Click outside the popover → close. Implementation: `document.addEventListener('click', handler)` registered in `_togglePopover` when opening, removed when closing; handler checks `event.composedPath()` for the popover root.
- ESC keypress → close. Implementation: `document.addEventListener('keydown')` listening for `event.key === 'Escape'`.
- `✕` button click → close. Standard onClick handler.

**Pattern note:** explicitly NOT using the HTML5 `popover` attribute / `[popover]` element nor the CSS Anchor Positioning API. Safari coverage is incomplete as of 2026-06; the manual absolutely-positioned div is the correct, durable pattern for this codebase's existing Lit shadow-DOM chrome.

## Motion

Mobile-friendly, short, and **always respects `prefers-reduced-motion`**.

| Animation | Duration | Easing | Respects `prefers-reduced-motion` |
|-----------|----------|--------|-----------------------------------|
| Pill text change (A→B→C transitions) | 0 (no animation) | n/a | n/a — text swap is instantaneous, matches existing `.offline-pill` mount/unmount which has no transition |
| Inline progress-bar fill width | 200ms `ease-out` on `width` only | `ease-out` | If reduced: `transition: none` — bar snaps to new width on each progress tick |
| `.cache-popover` mount/unmount | 120ms `ease-out` opacity 0→1 + `translateY(-4px)`→`translateY(0)` | `ease-out` | If reduced: opacity instant, no translate |
| `.update-banner` mount | 200ms `ease-out` `translateY(100%)`→`translateY(0)` + opacity 0→1 | `ease-out` | If reduced: opacity instant, no translate |
| `.update-banner` dismiss | 120ms `ease-in` `translateY(0)`→`translateY(100%)` + opacity 1→0 | `ease-in` | If reduced: instant unmount |
