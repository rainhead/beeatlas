<objective>
Land the two data-layer foundations the whole phase builds on: (1) the ONE shared
Python markdown→HTML render+sanitize helper (D-04/D-06), and (2) the forward-only
Alembic migration that adds `body_html` and recasts `notes.author_id` to an integer
FK on `users.id` (D-05/D-08). Both must exist before the write API (179-02) and the
harvest (179-03) can consume them.

Purpose: Establish exactly one server-side renderer (never shipped to the browser)
and the schema shape that stores pre-sanitized HTML + durable authorship — so Phase
180 XSS hardening (MOD-03) hardens rather than rescues, and ownership checks key off
a real FK.
Output: `data/notes_store/render.py`, migration `0003`, updated `models.py`, new
`nh3`+`markdown-it-py` deps, and their tests.
</objective>
