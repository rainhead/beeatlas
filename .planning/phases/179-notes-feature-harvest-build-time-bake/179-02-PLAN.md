<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → /api/notes (write) | untrusted body_md + note_id + Origin cross here; require_author + ownership + render gate them |
| browser → /api/notes (read) | untrusted `species` param + optional session cookie cross here; server scopes to approved-only |
| client identity → author_id | client-supplied author is untrusted; author_id is always server-derived from g.identity |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-179-AUTHZ | Elevation of Privilege | PATCH/DELETE /api/notes/<id> | mitigate | `abort(403)` when `note.author_id != g.identity["uid"]`; identity from require_session's g.identity, never from the request body (D-08) |
| T-179-IDOR | Elevation of Privilege | note_id path param | mitigate | ownership check runs after the load; a guessed/enumerated id belonging to another author yields 403, not mutation |
| T-179-LEAK | Information Disclosure | GET /api/notes | mitigate | server-side `status == 'approved'` filter on every read; body_md only for the authenticated owner; pending/removed never serialized |
| T-179-XSS | Tampering | POST/PATCH body_md | mitigate | body_html = render_note_markdown(body_md) on every write (179-01 sanitizer); raw body kept only as edit source, never served as HTML |
| T-179-CSRF | Spoofing/Tampering | note-mutation verbs | mitigate | inherited from require_author: Origin allow-list check on POST/PATCH/DELETE + SameSite=Strict session cookie (no new CSRF surface) |
| T-179-GATE | Elevation | writes before launch/allowlist | accept→mitigate | inherited: require_author 503s when WRITES_ENABLED off and 403s on fresh-allowlist non-author (D-04/D-05) |
</threat_model>
