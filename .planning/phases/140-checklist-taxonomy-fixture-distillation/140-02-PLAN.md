<objective>
Rewrite the two checklist/taxonomy test files to consume the Plan-01 seams and fixtures, eliminating the two dominant per-test parse costs in the build-time tier.

Two deliverables:
1. **test_checklist_pipeline.py rewrite** (TFIXTURE-01 / D-01..D-05, D-08, D-09) — replace the per-test, file-DB, reload-every-test `checklist_db` usage in the ~20 fast-tier tests with a single module-scoped, in-memory, shared-connection `checklist_sample_db` fixture that calls `load_checklist(con=con)` ONCE against the 8-row sample. 2. **test_resolve_checklist_names.py wiring** (TFIXTURE-02 / D-06, D-07) — add one `monkeypatch.setattr(resolve_checklist_names, "TAXA_PATH", str(FIXTURES_DIR / "taxa_subset.csv.gz"))` to the existing function-scoped `checklist_resolver_db` fixture so all `refresh=True` tests read the 2-row fixture gz instead of the 39 MB real file; the fast tier then passes with the real file absent.

Purpose: This is the payoff phase — the checklist file stops reparsing 50k rows ~20× and the resolver stops parsing 39 MB ~5×. Coverage is preserved, not weakened: every rewritten count assertion maps to a known sample row.

Output: 2 modified test files. No production-code changes (those landed in Plan 01).
</objective>
