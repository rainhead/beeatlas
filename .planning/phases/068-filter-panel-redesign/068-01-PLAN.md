**render() method** — collapsed state shows trigger button; expanded state shows panel with four sections:

The trigger button:
- Shows a magnifying-glass SVG icon (16×16, viewBox="0 0 16 16", filled path: circle at cx=6 cy=6 r=4.5 + line from (9.5,9.5) to (13,13))
- Shows the count: `${this.summary?.totalSpecimens ?? '…'}` after the icon
- Gets class `filter-btn${isFilterActive(this.filterState) ? ' active' : ''}`
- On click calls `this._togglePanel()`

The expanded panel (shown when `this._open === true`) is an absolutely-positioned `.filter-panel` div with four section headers followed by `<bee-filter-controls>`:

Section headers use inline SVG icons (16×16, stroke-based, stroke-width=1.5, fill=none):
- **What** (magnifying glass / taxon): same icon as trigger, or a leaf/bug icon. Use a tag/label icon: M2 2h12v8H2z path (rectangle with a notch) — or simply reuse the search circle. Recommended: a simple tag shape `<path d="M1 3a1 1 0 0 1 1-1h8l4 5-4 5H2a1 1 0 0 1-1-1V3z"/>` label icon.
- **Who** (collector): person silhouette `<path d="M8 7a3 3 0 1 0 0-6 3 3 0 0 0 0 6zm-5 6a5 5 0 0 1 10 0"/>` 
- **Where** (geography): map pin `<path d="M8 1a5 5 0 0 1 5 5c0 3.5-5 9-5 9S3 9.5 3 6a5 5 0 0 1 5-5z"/><circle cx="8" cy="6" r="1.5"/>`
- **When** (time): calendar `<rect x="2" y="3" width="12" height="11" rx="1"/><path d="M2 7h12M5 1v4M11 1v4"/>`

Place ALL four section headers as visual labels above `<bee-filter-controls>`. Since `bee-filter-controls` renders as a single component managing all filter types internally, the section headers are decorative labels above the single `<bee-filter-controls>` block — they communicate what's filterable but don't partition the filter inputs (which are already partitioned internally as taxon search, elevation, etc.).
