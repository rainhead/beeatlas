<objective>
Make all species with `occurrence_count > 0` visible in `species.json`, the species tree, and the static site by deriving `specific_epithet` from the canonical_name token when the species is not on the WA checklist, and capitalizing the off-checklist `scientificName` fallback in the same SQL model. The fix is a single dbt intermediate model edit; all downstream Python (species_export.py, species_maps.py), Eleventy (_data/species.js, species-detail.njk), and nightly.sh S3 upload paths already handle non-null specific_epithet correctly and do not need to change.

Purpose: 65 two-token off-checklist species currently have `occurrence_count > 0` but `specific_epithet = NULL`, so they are excluded from species pages, SVGs, and the species tree. After this fix they appear automatically through the existing pipeline plumbing.

Output: Updated int_species_universe.sql (2 COALESCE expressions), 2 new pytest regression tests, and verified post-build artifacts (sandbox/species.parquet rows, sandbox/species-maps SVG count, one example static page).
</objective>
