## Decisions Made

- `render_as_batch=True` is set globally in `env.py` (not per-migration). This means all future `alembic revision --autogenerate` calls automatically inherit SQLite batch mode — the critical setting for DROP COLUMN / ADD CONSTRAINT in future migrations.
- `downgrade()` raises `NotImplementedError` in both the mako template (every future migration inherits the guard) AND the initial migration (explicit for the most critical schema boundary). Both layers = belt-and-suspenders against Pitfall 4.
- `env.py` offline mode raises `NotImplementedError` — only online migration against the live SQLite file is supported. No SQL dump path exists.
- `test_run_py_never_migrates` inspects `run.py` as plain text via `Path.read_text()` — never imports it — because importing `run.py` pulls in the entire 25-step pipeline with all its side-effectful module-level globals.
- `path_separator=os` added to `alembic.ini` and to the test's programmatic `Config` to eliminate the Alembic >=1.13 deprecation warning about legacy comma/space path splitting.
