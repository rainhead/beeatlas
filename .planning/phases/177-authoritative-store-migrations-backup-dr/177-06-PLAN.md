<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Live WAL store → snapshot artifact | A raw copy of the .db without WAL sidecars yields a torn/inconsistent snapshot |
| Snapshot artifact → S3 backup bucket | Snapshot must reach the isolated bucket via the pipeline profile (PutObject only) |
| Backup existence → recoverability | A backup that cannot restore is worse than none (silent failure) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-177-05 | Information Disclosure / Integrity | Torn/inconsistent snapshot | mitigate | `Connection.backup()` + `?mode=ro`; `test_no_raw_copy` forbids `shutil.copy`/`os.system` (Pitfall 1) |
| T-177-06 | Repudiation / Integrity | Backup that silently fails to restore | mitigate | `test_restore_roundtrip` asserts row count + `alembic_version` parity; the demonstrated drill (plan 177-07) is the phase-exit gate before any public write (D-12) |
| T-177-04b | Tampering | Backup pushed to wrong/unversioned location | mitigate | Uploads only to `NOTES_BACKUP_BUCKET` (the versioned isolated bucket); pipeline profile has no DeleteObject (plan 177-02) |
</threat_model>
