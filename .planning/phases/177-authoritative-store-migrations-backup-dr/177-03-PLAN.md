<objective>
Define the authoritative store's schema (STORE-01, D-05, D-06, D-08): SQLAlchemy 2.0 `Note` and `NoteRevision` models, plus a `make_engine()` factory that sets WAL mode + foreign_keys + synchronous pragmas on every connect (Pattern 2). The schema is shaped for moderation + attribution from day one so Phase 180 is not a retrofit: notes carry `author_id`, `status`, and audit timestamps; `note_revisions` is an append-only soft-delete/edit ledger. Multiple author-owned notes per species (D-06) means NO unique constraint on `canonical_name` — an index only.

Purpose: this is the contract every other store plan builds against (migrations, seed, backup, and the Phase-179 read path). Defining the ORM + engine first prevents downstream scavenger hunts.
Output: `notes_store/models.py`, `notes_store/db.py`, and `test_notes_store_schema.py`.

WAL mode (set here) is what enables the Phase-179 nightly harvest to open the DB read-only while the app writes (D-16) — but this plan does NOT implement the harvest; it only enables it via the pragma.
</objective>
