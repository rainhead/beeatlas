<objective>
Deliver the store's supporting surface (STORE-01): the committed roles allowlist (D-07), the seed script (D-04), and the minimal FastAPI app skeleton (D-01/D-02).

- **Roles allowlist** (`data/roles_allowlist.toml` + `roles.py`): a git-tracked TOML mapping iNat login → role (`author` = can create notes, `curator` = can take down any note). Git history IS the audit trail, so the schema needs no `roles` table this phase (D-07). Only `author_id` on notes references identity.
- **Seed script** (`seed.py`): inserts sample notes so the store is populatable without any write UI (this phase opens no write endpoint — that is Phase 178).
- **App skeleton** (`notes_app/main.py`): a single `GET /health` FastAPI route behind `root_path=/notes-api`, proving the maderas app layer exists and is Apache-proxy-shaped. Authenticated write/OAuth endpoints are explicitly OUT of scope (Phase 178).

Purpose: STORE-01 requires role/allowlist affordances and a seedable store; D-01/D-02 require the small Python app to exist.
Output: allowlist TOML + roles.py + seed.py + main.py + two test files.

Do NOT add note-CRUD, auth, OAuth, CSRF, or any write endpoint. Do NOT add a roles DB table (deferred — D-07). Do NOT create Apache/systemd config files (Phase 178 owns full deploy config; the runbook in plan 177-06 documents the run command).
</objective>
