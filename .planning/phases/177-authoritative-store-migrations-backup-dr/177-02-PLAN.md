<objective>
Add the dedicated authoritative-backup S3 bucket to the CDK `BeeAtlasStack` as a surgical edit (D-13, D-14), establishing the STORE-04 IAM/bucket isolation boundary. The boundary is *structural*: the GitHub OIDC deployer role is never granted anything on this bucket (it cannot even name it), and the pipeline user gets only PutObject + GetObject (never DeleteObject) so S3 Versioning is the sole recovery layer.

Purpose: STORE-04 requires that a normal green nightly (which uses the deployer role for site sync and the pipeline user for `s3 cp` to `data/`/`db/`/`raw/`) can never reach or overwrite authoritative backups. This plan builds the AWS-side half of that guarantee.
Output: `AuthoritativeBackupBucket` construct + a second `pipelineUser.addToPolicy` grant + a `BackupBucketName` CfnOutput.

Do NOT `cdk destroy`, do NOT modify the existing siteBucket, deployerRole, or the existing pipelineUser policy. The actual `cdk deploy` (which mutates AWS) is an operator step in plan 177-07 — this plan produces and locally-synthesizes the code only.
</objective>
