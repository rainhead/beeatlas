# Plan 50-01: Export Join & Schema Gate

## Objective

Add `specimen_observation_id` (nullable BIGINT) to `ecdysis.parquet`. The column links each ecdysis specimen to its WABA iNaturalist observation (if any) by matching the specimen's `field_number` against the catalog number stored in the WABA OFV (field_id=18116). Multiple WABA observers may photograph the same specimen, so deduplication is required (use `MIN(id)` to pick one observation per specimen).

## Context

- Phase 49 created `data/waba_pipeline.py` which writes WABA observations into `inaturalist_waba_data` schema (DuckDB tables: `observations`, `observations__ofvs`, `observations__observation_projects`)
- `data/export.py` already has a `export_ecdysis_parquet()` function with a complex CTE chain; the new `waba_link` CTE plugs in at the end
- `scripts/validate-schema.mjs` is the authoritative schema gate; pytest mirrors it
- The fixture DB in `data/tests/conftest.py` needs `inaturalist_waba_data` schema/tables for tests to pass

## Tasks

### Task 1: Add waba_link CTE and specimen_observation_id column to export.py

In `data/export.py`, modify `export_ecdysis_parquet()`:

1. Add a `waba_link` CTE before the final SELECT that:
   ```sql
   waba_link AS (
       SELECT
           CAST(regexp_extract(ofv.value, '[A-Z]+-(\d+)$', 1) AS INTEGER) AS catalog_suffix,
           MIN(waba.id) AS specimen_observation_id
       FROM inaturalist_waba_data.observations waba
       JOIN inaturalist_waba_data.observations__ofvs ofv
           ON ofv._dlt_root_id = waba._dlt_id
           AND ofv.field_id = 18116
           AND ofv.value != ''
       GROUP BY catalog_suffix
   )
   ```
   
   **Important**: The join to ecdysis uses the numeric suffix of `field_number` (e.g., 'TC-001' → 1, 'WABA-5594569' → 5594569). Extract with `regexp_extract(o.field_number, '\d+$', 0)::INTEGER`.

   Actually, re-examine the approach: the WABA OFV field_id=18116 stores the Ecdysis catalog number (e.g., 'ecdysis:5594569'). The ecdysis `field_number` field stores a collector number like 'TC-001'. These likely don't match by suffix.

   **Correct approach**: Check the PROJECT.md note: "join WABA observations to specimens via catalog number suffix". The WABA OFV value for field_id=18116 is the Ecdysis specimen catalog number/ID. The ecdysis `id` field (occurrence_id's numeric part) is the join key.

   More likely: WABA OFV field_id=18116 stores the ecdysis specimen `id` (e.g., '5594569'). Join on `CAST(ofv.value AS BIGINT) = CAST(o.id AS BIGINT)`.

   Use this join:
   ```sql
   waba_link AS (
       SELECT
           CAST(ofv.value AS BIGINT) AS ecdysis_id,
           MIN(waba.id) AS specimen_observation_id
       FROM inaturalist_waba_data.observations waba
       JOIN inaturalist_waba_data.observations__ofvs ofv
           ON ofv._dlt_root_id = waba._dlt_id
           AND ofv.field_id = 18116
           AND ofv.value != ''
       GROUP BY ecdysis_id
   )
   ```

2. Add LEFT JOIN to waba_link in the final SELECT:
   ```sql
   LEFT JOIN waba_link wl ON wl.ecdysis_id = CAST(o.id AS BIGINT)
   ```

3. Add `wl.specimen_observation_id` to the SELECT column list (nullable BIGINT — NULL for specimens without WABA observations).

4. Update the docstring and the print statement in `export_ecdysis_parquet()`.

**Verification inside the function**: After writing parquet, run:
```python

SELECT COUNT(*) FROM read_parquet('{out}') WHERE specimen_observation_id IS NOT NULL
""").fetchone()
print(f"  specimen_observation_id non-null: {row[0]}")
```
