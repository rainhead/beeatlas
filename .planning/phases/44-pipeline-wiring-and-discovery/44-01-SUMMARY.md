# Phase 44 Plan 01: Pipeline Wiring and Autodiscovery Summary

**One-liner:** Wired feed generation into nightly.sh via run.py delegation and added Atom autodiscovery tag to index.html.

## What Was Built

nightly.sh now delegates all pipeline execution to `run.py` (replacing 5-step inline Python heredoc), adding the missing steps: geographies, anti-entropy, and feeds. A single `aws s3 sync` call after the existing parquet/geojson uploads pushes all feed XML files to `s3://BUCKET/data/feeds/`. The `frontend/index.html` head section gained a standards-compliant `<link rel="alternate" type="application/atom+xml">` tag pointing to `/data/feeds/determinations.xml`, enabling feed reader autodiscovery without a manual URL.

## Decisions Made

| Decision | Rationale |
|----------|-----------|
| `s3 sync` for feeds directory | ~200+ feed XML files; sync only uploads changed files, far more efficient than individual cp calls |
| CloudFront invalidation path unchanged | Existing `/data/*` wildcard already covers `/data/feeds/*` — confirmed in D-05 |
| Autodiscovery href as absolute path `/data/feeds/determinations.xml` | Matches CloudFront serving path; no domain in href avoids CORS issues if path changes |
