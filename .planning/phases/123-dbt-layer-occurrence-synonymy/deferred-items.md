## Pre-existing Test Failures (not caused by Phase 123)

### test_sqlite_export.py — fixture parquet schema mismatch
- **Failing tests:** test_creates_occurrences_table, test_main_uses_sandbox_and_export_dir,
  test_row_count_matches, test_column_names_match, test_overwrites_existing_db
- **Root cause:** `sqlite_export.py` queries for `ecdysis_id` in its geo_blob step, but the
  test fixture parquet (`PARQUET_COLUMNS = ["lat", "lon", "scientificName", "year"]`) was
  not updated when that column was added to the query. Pre-existing before Phase 123.
- **Fix needed:** Update test fixture to include all columns queried by sqlite_export.py
  (ecdysis_id, observation_id, specimen_observation_id, genus, family, source).

### test_dbt_diff.py — species.json / seasonality.json stale sandbox output
- **Failing tests:** test_species_json_matches, test_seasonality_json_matches
- **Root cause:** sandbox/species.json and sandbox/seasonality.json dated May 25 (pre-Phase 123),
  but public/data versions differ. These JSON files are generated by a Python post-step
  (species_export.py) that was not re-run during Phase 123 execution. The parquet files
  are current; the JSON files are stale.
- **Fix needed:** Run the full nightly pipeline or the species_export.py post-step to regenerate
  sandbox JSON outputs. Then re-run test_dbt_diff.py to confirm parity.
