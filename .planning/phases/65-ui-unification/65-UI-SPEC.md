## Component Inventory

### `<bee-occurrence-detail>` — NEW (replaces `<bee-specimen-detail>` and `<bee-sample-detail>`)

Receives: flat array of raw occurrence row objects (OL feature properties).

Render logic:
1. **Specimen-backed rows** (`ecdysis_id` non-null): render existing grouped-by-sample display — `.sample` block with `.sample-header` (month + year), `.sample-meta` (recordedBy · fieldNumber), optional elevation `.sample-meta`, and `.species-list` with ecdysis + iNat links. Copy styles verbatim from `bee-specimen-detail`.
2. **Sample-only rows** (`ecdysis_id` null): render compact `.sample-entry` block — date (formatted via `_formatSampleDate`), observer, specimen count, optional elevation, iNat link. Style using same `.sample` padding and `.sample-meta` text rules from `bee-sample-detail`.
3. **Mixed cluster**: specimen groups first, then a `<hr class="separator">` (1px solid `var(--border-subtle)`, no margin), then sample-only entries.
4. **Null-omit rule**: omit any field that is null — no placeholder dashes.

CSS class reuse plan:
- `.sample` — padding block (carry from specimen detail)
- `.sample-header` — month/year or date heading (carry from specimen detail)
- `.sample-meta` — secondary metadata text (carry from both)
- `.species-list` — italic species list (carry from specimen detail)
- `.quality-badge`, `.quality-badge.research`, `.quality-badge.needs_id`, `.quality-badge.casual` — carry from specimen detail
- `.inat-missing` — carry from specimen detail
- `.no-determination` — carry from specimen detail
- `.sample-entry` — NEW class for sample-only row block (same padding as `.sample`, no species list)

### `<bee-table>` — MODIFIED

Unified `OCCURRENCE_COLUMN_DEFS` replaces `SPECIMEN_COLUMN_DEFS` / `SAMPLE_COLUMN_DEFS`.

Columns (in order):

| Key | Label | Data Field | Min Width | Behavior |
|-----|-------|-----------|-----------|---------|
| date | Date | `date` | 100px | Sortable; sort indicator shown when active |
| species | Species | `scientificName` | 180px | null shows "No Determination" in `.cell-null` italic |
| collector | Collector | `recordedBy` | 150px | null shows blank |
| observer | Observer | `observer` | 150px | null shows blank (specimen-backed rows) |
| county | County | `county` | 110px | — |
| ecoregion | Ecoregion | `ecoregion_l3` | 130px | — |
| elevation | Elev (m) | `elevation_m` | 80px | null shows blank |
| fieldNumber | Field # | `fieldNumber` | 80px | null shows blank |
| modified | Modified | `modified` | 100px | Sortable |
| photo | Photo | `specimen_observation_id` | 60px | Camera icon link when non-null; blank otherwise |

Sort applies to `date` column (unified across both row types, ordered date desc).

### `<bee-header>` — MODIFIED

Remove "Specimens" and "Samples" layer tab buttons and the `layerMode` property. Retain:
- "BeeAtlas" h1 title
- Map/Table view toggle icon buttons (unchanged)
- GitHub link (unchanged)
- Hamburger menu for small screens (remove layer tabs from hamburger items too)

After removal, the `.inline-tabs` container and `.hamburger-items` will contain only the disabled "Species" and "Plants" placeholder buttons (or those too may be removed — Claude's discretion).

### `<bee-sidebar>` — MODIFIED

Replace conditional `<bee-specimen-detail>` / `<bee-sample-detail>` branches with a single `<bee-occurrence-detail .occurrences=${this.occurrences}>` when `this.occurrences !== null`. Empty state copy unchanged.

## Interaction Contracts

### Sidebar detail panel

- `bee-sidebar` shows `<bee-occurrence-detail>` when any occurrence is selected (specimen-backed or sample-only)
- Single `occurrences` property (array of OccurrenceRow objects) replaces `samples` + `selectedSampleEvent`
- Close button behavior unchanged
- Scroll behavior unchanged (`:host { overflow-y: auto }`)

### Map click

- Clicking a specimen-backed dot: sidebar shows occurrence detail with specimen groups
- Clicking a sample-only dot: sidebar shows occurrence detail with compact sample entry
- Clicking a cluster: sidebar shows all occurrences in cluster (specimen groups first, then sample-only)
- Filter highlight: sample-only dots are now correctly highlighted when filter is active (via `visibleIds` unification — no visual change needed, just correctness fix)

### Table sorting

- `date` column is sortable (desc default)
- `modified` column is sortable
- All other columns: not sortable
- Sort indicator (▼) shown on active sort column header only
- Sort applies across all occurrence types uniformly

### URL state

- `layerMode` key removed from url-state params
- All other URL params unchanged
