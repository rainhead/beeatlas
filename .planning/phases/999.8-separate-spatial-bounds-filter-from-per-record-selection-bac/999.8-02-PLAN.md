<objective>
Migrate the URL contract so the spatial bounding box serializes under a NEW `bbox=` param (D-02) and the legacy `sel=` param is read-only for back-compat (D-03). The `{ type: 'bounds' }` variant is removed from `SelectionState`, so selection now means only individual-record ids/cluster. Type-level foundation: nothing in `bee-atlas.ts` changes yet — only `url-state.ts` and its tests.

Purpose: Bounds is a filter, so its URL param must live with the filter params, not the selection param. `buildParams` only ever writes `bbox=`; `parseParams` reads both `bbox=` (canonical) and legacy `sel=` (existing shared links), mapping both into `result.filter.bounds`. Old `?sel=…` links keep working and silently migrate to `bbox=` on the next URL write.

Output: `bbox=` write in `buildParams`, `bbox=`+legacy `sel=` readers in `parseParams` (both → `result.filter.bounds`), narrowed `SelectionState` union, and updated `url-state.test.ts`.
</objective>
