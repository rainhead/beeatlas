## Interaction Contract

### Boundary Mode Toggle — Places addition

- Boundary mode dropdown in `bee-map` extends from 3 options to 4: Off / Counties / Ecoregions / Places
- Button label when `boundaryMode === 'places'`: "Places"
- Active item styling: weight 600, `color: var(--accent)` — same as other active modes
- Dropdown remains a simple `<div class="region-menu">` with `<button>` items — no new widget type

### Place Polygon Click

- Cursor: pointer only when `boundaryMode === 'places'` and the fill layer is visible
- Click precedence: occurrence dot wins (queried first); place polygon fires only when no dot is hit (CONTEXT.md D-02, D-03)
- On click: emits `place-selected` CustomEvent with `{ detail: slug }` — consistent with `boundary-mode-changed` event pattern
- No tooltip or hover label on place polygons (CONTEXT.md Claude's Discretion — place label layer deferred)

### Place Filter Chip

- Appears in `bee-filter-panel` Where section when `filterState.selectedPlace !== null`
- Chip label: display name from `places.json` lookup; falls back to the slug if name not resolved (CONTEXT.md Specific Ideas)
- Chip remove (×): emits `filter-changed` with `selectedPlace: null` — same `_emitFilter()` dispatch pattern as county/ecoregion removal
- Chip renders alongside county and ecoregion chips in the same `.chips` container

### Place Filter Persistence Across Boundary Modes

- Filter chip remains active (occurrences ghosted) when user switches away from Places mode (CONTEXT.md D-04)
- Switching TO Places mode when place filter is active: selected polygon highlighted immediately via `setFeatureState` (CONTEXT.md D-05)

### URL Deep-Link

- `/?place={slug}` sets `filterState.selectedPlace = slug` AND `boundaryMode = 'places'` on load (CONTEXT.md D-01, D-09)
- No visual loading state needed — restored atomically during `_init` before first render

## Copywriting Contract

| Element | Copy |
|---------|------|
| Boundary mode label (collapsed) | "Places" (when active); "Regions" when off — existing pattern unchanged |
| Boundary mode menu item | "Places" |
| Place filter chip label | "{Place Display Name}" (e.g. "Ebey's Landing") with × remove button |
| Place chip aria-label | "Remove {Place Display Name}" |
| Place polygon click aria | No tooltip copy — click is visual only |
| Empty state (no occurrences in place) | Reuse existing empty occurrence copy — no place-specific empty state |
| Error state (places.geojson fails to load) | No user-visible error — layer silently absent; consistent with existing map data error handling |
| Destructive actions | None in this phase |

Primary CTA for this phase: "Places" (boundary mode menu item — the entry point for the interaction).
