<objective>
Build the data substrate for the per-collector event feed (STREAM-01/02/03): a new
export module that reads the existing mart parquet plus the raw Ecdysis
`identifications` table, emits a reverse-chronological Collected + Identified event
stream per collector, resolves each event's species name to a rank-aware taxon-page
slug, and pre-chunks events into static sub-pages.

Purpose: the collector page is JS-free (CONTEXT invariant / Phase 172 criterion 5),
so all event aggregation, sorting, rank-aware linking, and 2D-pagination chunking
must happen pipeline-side. This plan produces the JSON the frontend (Plan 02) renders.

Output:
- `data/collectors_events_export.py` — the new export step (mirrors `collectors_export.py`
  structure + `feeds.py` identifications join).
- `public/data/collectors.json` — extended in place with `first_page_events`,
  `total_event_pages`, `total_event_count` (existing 10 keys preserved untouched).
- `public/data/collector_event_pages.json` — flat `{login, page_num, total_pages, events}`
  array for pages 2+ (committed; ~24 MB build artifact per RESEARCH Q2 — print the real size).
- Wave 0 test scaffolds (pytest golden fixture + vitest artifact-shape assertions).

NOTE on the ~24 MB committed artifact (planner decision, per CONTEXT "where event data
lives = planner's call"): the collector sub-pages are generated by the deploy.yml Eleventy
build from committed JSON — exactly as Phase 169 committed `collectors.json` "so CI builds
without running the pipeline" (`.gitignore` line 145). Nightly does NOT push collector data
to S3, so committing is the only architecture-consistent path that keeps this phase
self-contained (no nightly/S3/operator step). The low commit cadence of `collectors.json`
(2 commits since Phase 169) bounds git growth. A future S3-pull is a documented option but
out of scope here.

Locked decisions implemented (data substrate): D-EVENT-01 (two event kinds only; cataloguing = continuous row, no fake event — STREAM-02 structural), D-EVENT-02 (waba_specimen → Collected/awaiting-ID, no Identified event, no iNat pull), D-IDSRC (Identified timestamped by `identifications.modified`), D-FEED-01 (flat stream, one entry per event), D-FEED-02 (full current+superseded re-ID history), D-SORT (reverse-chron by best-available timestamp), D-PAGE-01 (pre-chunk into static sub-pages), D-CARD-01 (determiner name), D-CARD-02 (rank-aware taxon slug), D-CARD-03 (no specimen/place context), D-EMPTY (zero-event collectors).
</objective>
