# Plan 071-01: Install Mapbox GL JS and rewrite bee-map.ts

## Goal

Replace OpenLayers with Mapbox GL JS v3 in bee-map.ts. After this plan, the basemap renders, occurrences load from SQLite as a clustered GeoJSON source with recency coloring, view state syncs with URL, and filtering/selection work.

## Context

### Current OL architecture (bee-map.ts, ~460 lines)

- `OpenLayersMap` with `View` (EPSG:3857), `TileLayer` + `StadiaMaps` for basemap
- `OccurrenceSource` (custom VectorSource) loads all rows from SQLite → OL Features
- `Cluster` source wraps OccurrenceSource (distance: 20, minDistance: 5)
- `VectorLayer` renders clusters via `makeClusterStyleFn` (imperative OL Styles)
- `regionLayer` (VectorLayer) for county/ecoregion boundaries → **deferred to Plan 071-02**
- Click handler uses `getFeatures(pixel)` for hit detection → **deferred to Phase 72**
- `moveend` event → emits `view-moved` custom event
- `updated()` repaints layers when `visibleIds`, `selectedOccIds`, `boundaryMode`, `viewState`, `panTo` change

### Mapbox equivalents

| OL concept | Mapbox equivalent |
|-----------|-------------------|
| `ol/Map` + `View` | `mapboxgl.Map` (center/zoom in constructor) |
| `TileLayer` + `StadiaMaps` | Built-in — Mapbox style URL (`mapbox://styles/mapbox/outdoors-v12`) |
| `OccurrenceSource` | `map.addSource('occurrences', { type: 'geojson', data, cluster: true })` |
| `Cluster` (distance: 20) | `clusterRadius: 20` on the GeoJSON source |
| `makeClusterStyleFn` | Three Mapbox layers: clusters (circle), cluster-count (symbol), unclustered (circle) |
| `visibleIds` filter | Mapbox `filter` expression on layers |
| `selectedOccIds` highlight | Separate "selected" layer with filter, or `feature-state` |
| `regionLayer` | Deferred to 071-02 |
| `getFeatures(pixel)` click | Deferred to Phase 72 |

### Task 2: Rewrite features.ts — GeoJSON data conversion

**Files:** `frontend/src/features.ts`

**Action:**
Replace `OccurrenceSource` (OL VectorSource subclass) with a function that:
1. Awaits `tablesReady`
2. Queries `SELECT * FROM occurrences` via wa-sqlite
3. Returns a GeoJSON `FeatureCollection` with each row as a Feature
4. Feature `id` follows existing convention: `ecdysis:<id>` or `inat:<observation_id>`
5. All occurrence columns are set as feature properties
6. Also computes and returns `summary` (DataSummary) and `taxaOptions` (TaxonOption[]) from specimen features — moving `computeSummary` and `buildTaxaOptions` out of bee-map.ts into features.ts where they belong

**Verify:** TypeScript compiles. The exported function signature is clear.

### Task 3: Rewrite style.ts — extract recency logic, remove OL styles

**Files:** `frontend/src/style.ts`

**Action:**
1. Keep `RECENCY_COLORS`, `recencyTier()`, and `hexWithOpacity()` — these are framework-agnostic
2. Remove all OL imports (`Feature`, `Circle`, `Fill`, `Stroke`, `Style`, `Text`)
3. Remove `makeClusterStyleFn` and `styleCache` — Mapbox styling is declarative

**Verify:** No OL imports remain in style.ts. TypeScript compiles.

### Task 4: Rewrite bee-map.ts — Mapbox map + occurrence layers

**Files:** `frontend/src/bee-map.ts`

**Action:**
This is the core rewrite. The new bee-map.ts must:

**4a. Imports and setup:**
- Remove all `ol/` imports
- Import `mapboxgl` from `mapbox-gl`
- Import `mapbox-gl/dist/mapbox-gl.css?raw` for shadow DOM styling (same pattern as OL CSS)
- Import `loadOccurrenceGeoJSON` from rewritten `features.ts`
- Import `RECENCY_COLORS` from `style.ts`

**4b. Map initialization (firstUpdated):**
```
mapboxgl.accessToken = import.meta.env.VITE_MAPBOX_TOKEN ?? '<production-public-token>';
this.map = new mapboxgl.Map({
  container: this.mapElement,
  style: 'mapbox://styles/mapbox/outdoors-v12',
  center: [this.viewState?.lon ?? DEFAULT_LON, this.viewState?.lat ?? DEFAULT_LAT],
  zoom: this.viewState?.zoom ?? DEFAULT_ZOOM,
});
```

**4c. Data loading (on map 'load' event):**
1. Call `loadOccurrenceGeoJSON()` — returns FeatureCollection + summary + taxaOptions
2. `map.addSource('occurrences', { type: 'geojson', data: geojson, cluster: true, clusterRadius: 20, clusterMinPoints: 2 })`
3. Add cluster properties to aggregate recency tier: `clusterProperties: { freshCount: ['+', ['case', ['==', ['get', 'recencyTier'], 'fresh'], 1, 0]], thisYearCount: ... }`
4. Add three layers:
   - `clusters` — circle layer filtered to `['has', 'point_count']`, paint by aggregated recency
   - `cluster-count` — symbol layer with `point_count` text
   - `unclustered-point` — circle layer filtered to `['!', ['has', 'point_count']]`, paint by `recencyTier` property
5. Emit `data-loaded` event with summary + taxaOptions

**4d. Filtering (visibleIds):**
When `visibleIds` changes:
- If null (no filter): remove filter from all occurrence layers
- If Set: apply `['in', ['get', 'occId'], ['literal', [...ids]]]` filter on unclustered layer
- For clusters: update source data to only include visible features, or use `filter` on source
- Emit `filtered-summary-computed`

**4e. Selection (selectedOccIds):**
- Add a `selected-point` layer (yellow ring) filtered to selected IDs
- Update filter when `selectedOccIds` changes

**4f. View sync:**
- `map.on('moveend', ...)` → emit `view-moved` with center (lng/lat) and zoom
- `updated()`: when `viewState` changes, call `map.jumpTo()`; when `panTo` changes, call `map.flyTo()`

**4g. Preserved unchanged:**
- Region menu UI (pure Lit template — no OL dependency)
- `_emit()` helper
- All `@property` declarations (same interface to bee-atlas)
- `disconnectedCallback` pattern

**Deferred to Phase 72:**
- Click handlers (occurrence, region, empty)
- Region boundary layers
- `countySource`/`ecoregionSource` integration

Stubbed for Phase 72: click handler emits `map-click-empty` for all clicks; boundary mode changes are no-ops.

**Verify:** Basemap renders. Occurrences display as colored clusters. Zooming/panning updates URL. Filter panel hides/shows points.

## Risks

- **Mapbox token management**: Public tokens are URL-restricted; need to set up restriction in Mapbox dashboard for beeatlas.net
- **Cluster aggregation**: Mapbox cluster properties use MapReduce-style aggregation — recency "best tier" requires counting per-tier then picking the best in paint expressions
- **Filter performance at 50K+**: Mapbox `['in', id, list]` filter may be slow with large ID sets. Alternative: update source data directly. Benchmark during implementation.

2. Basemap renders with Mapbox outdoors style
3. Occurrences cluster with recency colors (fresh=green, thisYear=orange, older=gray)
4. Ghosted styling for filtered-out points
5. Yellow ring on selected points
6. View state syncs to URL on pan/zoom
7. `visibleIds` filtering hides/shows points correctly
8. Tests pass (updated where needed)
