<objective>
Wire the manifest and iOS install meta into the `/app` page head — and ONLY that page.
Adds `<link rel="manifest">`, the `apple-mobile-web-app-*` standalone meta, the
`apple-touch-icon` link, and a belt-and-suspenders `theme-color` meta to
`_pages/app/index.html`. The root `_pages/index.html` is deliberately untouched.

Purpose: Chrome's installability check requires a linked manifest (PWA-01); iOS Safari
standalone behavior and the home-screen icon come ONLY from these legacy `apple-*` meta
tags, not the manifest (PWA-02). Linking from `/app` only preserves the Phase 147
no-PWA-on-`/` guarantee (D-04).

Output: modified `_pages/app/index.html` head; the Plan 01 build-output link assertions
turn green.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser ↔ /app HTML head | The browser parses the manifest `<link>` and apple-* meta from the served `/app/index.html` (HTTPS via CloudFront). These are static, declarative head tags — no script, no untrusted input. |
| /app ↔ / isolation | The no-PWA-on-/ guarantee (D-04): the manifest/iOS meta must remain absent from `/` so the root site never advertises installability. Enforced by editing the app-only head and asserted negatively in build-output.test.ts. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-151-03 | Spoofing/Elevation | manifest link leaking onto / | mitigate | Edit `_pages/app/index.html` only (it has its own full <head>, not a shared layout); Plan 01's build-output test negatively asserts `_site/index.html` has no `rel="manifest"`. Prevents `/` from advertising the PWA prematurely (graduation is FUT-05). |
| T-151-04 | Tampering | apple-* meta / theme-color values | accept | Static, public, non-sensitive declarative head tags served over HTTPS. No high-severity risk. |
| T-151-SC | Tampering | npm/pip/cargo installs | mitigate | No package installs in this plan (template-edit only). No supply-chain surface. |
</threat_model>
