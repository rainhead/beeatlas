## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| OS ↔ manifest/icons over HTTPS | The browser/OS fetches `/app/manifest.webmanifest` and icon PNGs from CloudFront over HTTPS at install/manifest-read time. No untrusted input crosses here — the assets are static, committed, and read-only. |
| (no server runtime) | Static hosting only; there is no server, no datastore write, and no user-supplied input in this plan. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-151-01 | Tampering | manifest/icons in transit | accept | Served over HTTPS via CloudFront (TLS integrity); assets are static, public, non-sensitive (a public bee atlas). No high-severity risk. |
| T-151-02 | Information Disclosure | manifest contents | accept | Manifest exposes only public app metadata (name, colors, icon URLs) — no secrets, no PII. Intended to be public. |
| T-151-SC | Tampering | npm/pip/cargo installs | mitigate | No package installs in this plan or phase (RESEARCH §Package Legitimacy Audit: N/A — HTML/static-asset/component-only). `rsvg-convert` is a pre-installed Homebrew binary, not an npm dependency; the gen script is not build-wired. No supply-chain surface introduced. |
</threat_model>
