### Caveats

**Criterion 3 — git size reclaim:** The phase context estimated ~150 MB of history bloat to reclaim from the filter-repo purge. The actual reclaim was approximately 1 MB (git reported `36M → 35M` on the .git directory). This discrepancy is explained by packfile delta-compression: git had already stored the repeated daily JSON blobs as deltas rather than full copies. The history-cleanliness intent of criterion 3 is fully met — both files have 0 revisions in `git log --all`, the allowlist is gone, and no future re-commit can reintroduce them. The byte-count estimate in the phase motivation was inaccurate, but the correctness goal was achieved.
