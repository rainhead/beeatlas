<objective>
Extend the static SVG pipeline to render county-fill polygons for checklist counties on every species page that has a checklist record, while preserving the existing occurrence-dot rendering for WABA species. After this plan: every species with `on_checklist = true` (~565 species) has a `{slug}.svg` file in `public/data/species-maps/`, county fills appear underneath occurrence dots, and the three pytest cases from Plan 01 are green.

Purpose: Per D-04 through D-07, county fills must be on the same SVG as occurrence dots (not a separate file), and per Pitfall #4 the SVG document order determines z-order — fills must be written first. Per Pitfall #5, the current `WHERE occurrence_count > 0` filter excludes checklist-only species and must be extended.

Output: Augmented SVGs for ~565 species; Plan 01 pytest cases for county fills are green; the existing `_write_species_svg` callers all pass the new arguments.
</objective>
