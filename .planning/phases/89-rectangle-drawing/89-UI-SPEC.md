### Phase 89 — New: Selection Rectangle Color

The selection rectangle is an ephemeral gesture indicator, not a selection state indicator. It must be visually distinct from:
- The yellow selection ring/halo (`#f1c40f`) used for selected occurrence points and cluster halos — that is a "selected state" color
- The green accent (`#2c7a2c`) used for active filter state

**Prescribed selection rectangle colors:**

| Property | Value | Rationale |
|----------|-------|-----------|
| Border color | `#3887be` | Mapbox official example color; mid-blue reads clearly on all basemap tile backgrounds; contrasts with yellow halo and green accent |
| Border width | 2px solid | Crisp, visible at any drag size; matches Mapbox official example |
| Fill | `rgba(56, 135, 190, 0.1)` | 10% opacity of border color; confirms drag area without occluding map features |
| pointer-events | none | Overlay must never intercept mouse events during the gesture |

Accent reserved for: active filter button state, active region menu item. The selection rectangle blue (`#3887be`) is NOT the project accent — it is used exclusively for the `.selection-box` gesture overlay.

## Interaction Contract

### Gesture States

| State | Cursor | Visual |
|-------|--------|--------|
| Default map (no shift held) | Mapbox default (`grab` / `grabbing` on drag) | No rectangle |
| Shift held, no drag started | `crosshair` | No rectangle; cursor signals intent |
| Shift held, dragging (>0px) | `crosshair` | `.selection-box` div visible, tracks cursor |
| Sub-threshold release (<5px drag) | Returns to default | Rectangle removed instantly; no event emitted |
| Valid release (>=5px drag) | Returns to default | Rectangle removed instantly; `selection-drawn` event emitted |

**Cursor:** Set `cursor: crosshair` on `map.getCanvasContainer()` when shift is detected on `mousedown`; restore to `''` (Mapbox default) in `_rectFinish()`. Do NOT set crosshair on the canvas at all times — only during the active gesture.

### Appearance and Disappearance

**Appearance:** Instant — the `.selection-box` div is created on first `mousemove` after shift+mousedown. No CSS transition or animation. The rectangle grows in real time as the mouse moves.

**Disappearance:** Instant on `mouseup` — `this._rectBox.remove()` with no fade or transition. The gesture is ephemeral; any visual lingering would imply the selection is still being drawn.

### Z-index

The `.selection-box` div is appended to `map.getCanvasContainer()` (inside the shadow DOM, inside the Mapbox canvas container). It must appear above the Mapbox canvas but below any floating UI controls (filter button, region button). Use `position: absolute; top: 0; left: 0` — no explicit `z-index` value needed because the div is stacked after the canvas in DOM order within its container.

### Shadow DOM Scoping

`.selection-box` CSS MUST be declared inside `BeeMap.static styles` (the `css\`` tagged template). The div is appended to a shadow DOM element; outer document styles do not pierce in.
