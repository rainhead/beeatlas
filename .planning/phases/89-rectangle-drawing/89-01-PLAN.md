<objective>
Implement the shift-drag rectangle gesture on the Mapbox canvas. The user holds shift and drags; a `.selection-box` div tracks the cursor in real time; releasing the drag removes the div and emits a `selection-drawn` CustomEvent carrying geographic bounds. `bee-atlas` stores the bounds in `_selectionBounds` reactive state (Phase 90 will consume them for the SQLite query). Mapbox's default BoxZoomHandler is disabled so the gesture is captured by the custom handler instead.

Purpose: This is the foundation phase of v3.5 Selection Rectangle — it ships the user-visible gesture and the event surface that Phases 90 (occurrence query + sidebar) and 91 (URL state) build on.

Output:
- `src/bee-map.ts` — gesture handlers, `.selection-box` CSS, boxZoom disable, disconnectedCallback cleanup
- `src/bee-atlas.ts` — `_selectionBounds` state, event binding, handler stub
- `src/tests/bee-atlas.test.ts` — SEL-01 and SEL-02 static-grep test blocks
</objective>

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-89-01 | Tampering | `_onSelectionDrawn` event detail | accept | Event originates from same-origin shadow-DOM component; only consumer in Phase 89 is `_selectionBounds` state assignment (no DB/network use yet). Phase 90 will introduce SQL query — bounds will be passed as parameterized values into the existing `queryVisibleIds` / `buildFilterSQL` infrastructure that already parameterizes lat/lon filters; no new injection surface in this phase. |
| T-89-02 | Denial of Service | rapid shift-drag stress | accept | Rectangle div is a single dynamically managed element; rAF coalescing not required at this stage because mousemove is naturally throttled by the browser to display refresh.
