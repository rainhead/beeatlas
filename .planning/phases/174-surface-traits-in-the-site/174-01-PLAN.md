<objective>
Emit the Phase 173 `species_traits` mart as an external parquet and merge its 11
trait fields into `species.json` at build time, keyed on `canonical_name`, using
RESEARCH Path B (Python-side merge — NOT a dbt JOIN). The dbt `species` mart
contract, `SPECIES_COLUMNS`, and the pyarrow schema stay byte-unchanged; trait
fields reach `species.json` only, via `_jsonify_rows()` serializing all dict keys.

This is the data leg of the phase (D-01, D-02, D-03). It must land before the
frontend waves consume the new `species.json` shape. Because the JSON shape
changes, the first post-deploy nightly trips the byte-stability integration gate —
the one-time `SKIP_INTEGRATION_GATE=1` transition nightly is captured here as an
operator checkpoint per memory `project_occurrences_contract_release_sequence`.

Purpose: deliver trait data over the established `species.json` fetch-at-build path
(TRAIT-UI-05) with no new manifest key, no new deploy.yml fetch line, and no
committed `public/data/` artifact.
Output: `species_traits.parquet` (sandbox), 11 trait fields in `species.json`,
extended Python test coverage, and a captured operator transition note.
</objective>
