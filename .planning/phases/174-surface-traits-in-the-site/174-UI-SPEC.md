# Phase 174 — UI Design Contract

> Visual and interaction contract for the species-trait surfacing work.
> Generated by gsd-ui-researcher. All tokens extracted from the existing codebase;
> no new design system introduced.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — custom CSS + CSS custom properties |
| Preset | not applicable |
| Component library | none |
| Icon library | none — Unicode glyphs only where needed |
| Font | system-ui, Avenir, Helvetica, Arial, sans-serif (index.css :root) |

This is an additive styling task on an established Eleventy/Nunjucks site. All tokens
come from `src/index.css` and `src/styles/taxon-pages.css`. Do not introduce new
type families, icon fonts, or component libraries.

## Spacing Scale

The existing site uses `rem` units. The scale below documents observed values from
`src/styles/taxon-pages.css` mapped to their px equivalents. All are multiples of 4.

| Token | rem | px | Usage |
|-------|-----|----|-------|
| xs | 0.25rem | 4px | Swatch margin-right, list-item padding-y |
| sm | 0.5rem | 8px | Gap between flex items (`.species-list li gap: 0.5rem`), compact margin |
| md | 1rem | 16px | Default padding (`.taxon-page { padding: 1rem }`) |
| lg | 1.5rem | 24px | Family-section margin-bottom, tree indentation per depth level |

Exceptions:
- Badge padding: `0.1rem × 0.35rem` (off-grid intentionally — pill inset must stay
  visually compact; increasing to on-grid `0.25rem × 0.5rem` would make badges bulky
  and crowd the dense species-index row).
- Traits DL column-gap: `0.75rem` (12px) — between dt and dd; not on the xs/sm/md
  steps because the closest on-grid value (sm=8px) is too tight for readable labeling.
- Traits DL row-gap: `0.25rem` (4px = xs) — tight vertical rhythm within the trait list.

### Section heading

The detail-page traits block heading:
- Element: `<h2 class="traits-heading">Traits</h2>`
- Text: "Traits" (exactly; not "Ecological Traits" or "Species Traits")

### Host-bee link copy

Host bee entries in `<dd>` under "Host bees":
- Linked species: `<a href="/species/{slug}/index.html"><em>Andrena accepta</em></a>`
- Linked genus: `<a href="/species/{Genus}/index.html"><em>Andrena</em></a>`
- Unresolved (plain text): `<em>Lasioglossum tegulare</em>` — NO link
- Multiple hosts: comma-separated with a space after each comma

### Empty and error states

- **All traits absent**: the entire `<section class="traits">` is omitted; no "No trait
  data" message is rendered. D-01 / TRAIT-UI-01 require silent omission.
- **host_bees absent on a cleptoparasitic species**: the "Host bees" row is omitted;
  the section still renders with whatever other traits are present.
- **Host-bee resolution fallback**: unresolved host names render as plain italic text,
  not as an error. No error message.
- **No CTA, no destructive action**: this phase adds read-only informational content.

## Layout Contract

### Detail Page: Traits Section Placement

File: `_pages/species-detail.njk`

Insert the `<section class="traits">` block between the checklist attribution
paragraph and the first `.taxon-action` link, in this order:

```
<seasonality-viz> + inline script
<p class="metadata">...
<p class="checklist-attribution">...  ← conditional, may be absent
[NEW] <section class="traits">...     ← INSERT HERE
<a class="taxon-action">View records →</a>  ← conditional
<a class="taxon-action">View on iNaturalist →</a>  ← conditional
```

The entire `<section class="traits">` is wrapped in a Nunjucks conditional:

```nunjucks
{%- set hasSociality  = sp.sociality -%}
{%- set hasDiet       = sp.diet_breadth -%}
{%- set hasNesting    = sp.nesting -%}
{%- set hasNative     = sp.native_status -%}
{%- set hasHostBees   = sp.resolvedHostBees and sp.resolvedHostBees.length > 0 -%}
{%- if hasSociality or hasDiet or hasNesting or hasNative or hasHostBees -%}
<section class="traits">
  ...
</section>
{%- endif -%}
```

### Detail Page: traits-dl Grid Layout

The `<dl class="traits-dl">` renders as a two-column CSS grid:
- Column 1 (`<dt>`): `max-content` width — label column, caps-small uppercase style
- Column 2 (`<dd>`): `1fr` — value column, body text

Each `<dt>` carries:
1. `tabindex="0"` — makes it keyboard-focusable so the `title=` provenance tooltip
   is reachable without a pointer device
2. `title="Source: ..."` — the full provenance string
3. No additional attributes

Each `<dt>`/`<dd>` pair is conditionally rendered with individual Nunjucks guards.
Order: Sociality → Diet → Nesting → Native status → Host bees.

The "Host bees" row is only present on cleptoparasitic species (i.e. when
`sp.resolvedHostBees` is non-empty). No guard is needed beyond the
`hasHostBees` condition already in the outer Nunjucks if.

### Index Page: Badge Placement in Leaf Row

File: `_pages/species.njk`, `renderNode` macro, `node.rank === "species"` branch.

Current species leaf structure:
```html
<li data-rank="species" data-name="{{ node.scientificName | lower }}">
  <a class="node-name" href="..."><em>...</em></a>
  <span class="node-counts">...</span>
  <a class="node-map" ...>Map</a>
</li>
```

After this change:
```html
<li data-rank="species" data-name="{{ node.scientificName | lower }}">
  <a class="node-name" href="..."><em>...</em></a>
  [sociality badge — conditional]
  [specialist badge — conditional]
  <span class="node-counts">...</span>
  <a class="node-map" ...>Map</a>
</li>
```

Badges sit between `.node-name` and `.node-counts`. The `data-name` attribute is
untouched — it keys only on `node.scientificName | lower`, which is set in the
existing `data-name` attribute and is not affected by badge spans.

The flex container (`.species-index .tree-node > summary` / `.species-list li`) has
`align-items: baseline`, so badge spans align to text baseline. Badges use
`flex: 0 0 auto` to remain non-stretching.

### Genus/Subgenus Pages: Badge Placement in Species List Row

Files: `_pages/genus.njk`, `_pages/subgenus.njk`

Current species `<li>` structure (in `.species-list`):
```html
<li>
  <span class="swatch" ...></span>
  <a href="..."><em>{{ sp.scientificName }}</em></a>
  <span class="count">...</span>
</li>
```

After this change:
```html
<li>
  <span class="swatch" ...></span>
  <a href="..."><em>{{ sp.scientificName }}</em></a>
  [sociality badge — conditional]
  [specialist badge — conditional]
  <span class="count">...</span>
</li>
```

The `.species-list li` has `display: flex; align-items: baseline; gap: 0.5rem` from
`taxon-pages.css`. Badge spans with `flex: 0 0 auto` fit naturally.

The same pattern applies in ALL three species-list contexts inside genus.njk:
1. `{%- for sp in sg.species -%}` (subgenus-grouped list)
2. `{%- for sp in genus.ungroupedSpecies -%}` (trailing flat list)
3. `{%- for sp in genus.species -%}` (D-05 no-subgenera flat list)

### Tribe pages: no badge

`_pages/tribe.njk` iterates genus rows, not species rows. No badge insertion is
needed or possible. D-07's "tribe path" means threading trait fields through the
`_data/species.js` data layer only; the tribe template does not need modification.
(Confirmed by direct inspection: `tribe.njk` has no `{%- for sp in ... -%}` species
loop.)

## Accessibility Contract

### Non-color badge differentiation

The text content of each badge ("Social", "Solitary", "Clepto", "Specialist") is
the PRIMARY differentiator. Color is a SECONDARY signal. Specifically:
- All badges have the same base gray border/text (no color difference between
  Social/Solitary/Clepto).
- Specialist badge adds a green border + green text as a secondary signal.
- Because the text alone uniquely identifies each badge type, the design is
  distinguishable without color.

### Keyboard accessibility for provenance

Each element carrying `title=` provenance must also be reachable via keyboard Tab
so screen-reader users and keyboard navigators can access the tooltip text.

Implementation:
- `<dt tabindex="0">` in `.traits-dl` on the detail page.
- `<span class="node-badge" tabindex="0">` on every badge in the index and
  genus/subgenus pages.
- Both carry `cursor: help` to signal the tooltip to pointer users.
- Both carry `:focus-visible` rings at `2px solid var(--accent)` with `outline-offset: 2px`.

### Filter interaction safety

The client-side species-index text filter (`species-index.ts`) keys on the `data-name`
attribute of `<li data-rank="species">` elements. Badge spans are sibling elements
of the `<a class="node-name">`, NOT children of it. The filter matches against
`data-name="{{ node.scientificName | lower }}"` only. Badge spans carry no `data-name`
and no text that the filter reads. No interference.

## Interaction Contract

### Provenance tooltip (title=)

Mechanism: native browser `title=` attribute only. Zero JS. No tooltip library.
This is a constraint from D-08; do not propose or implement a JS tooltip.

Behavior:
- Hover over any `<dt>` in `.traits-dl` → browser shows native tooltip (white box,
  OS-rendered) with the provenance string.
- Tab to any `<dt>` or `.node-badge` and the element receives focus-visible ring.
  Screen readers will announce the `title=` as part of the accessible name/description
  (browser-dependent, but `title` is announced by most screen readers on focusable
  elements).

There is no JavaScript interaction in this phase. No click handlers, no expand/collapse,
no filter changes.

### Host-bee links

- Link to species page: `href="/species/{genus}/{epithet}/index.html"`
- Link to genus page: `href="/species/{Genus}/index.html"`
- Unresolved: plain `<em>` text, no `href`
- All three cases resolved at build time in `_data/species.js`, never at runtime.
