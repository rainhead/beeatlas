<objective>
Render the threaded trait data on the templates: a "Traits" definition-list section
on the species detail page (D-04, D-05, D-08, D-09) and compact sociality + Specialist
badges on every species row of the index tree, genus pages, and subgenus pages
(D-06, D-07). `tribe.njk` is intentionally EXCLUDED — it iterates genus rows, not
species rows (RESEARCH §Open Questions 1; UI-SPEC §"Tribe pages: no badge").

All copy is fixed by UI-SPEC: friendly labels (sociality "Parasitic" → "Cleptoparasitic",
diet "specialist" + host_plant_family → "Specialist (Family)", "generalist" → "Generalist"),
exact `<dt>` terms, exact `title=` provenance strings per `*_source` enum, and short
badge text ("Social"/"Solitary"/"Clepto"/"Specialist"). Absent traits are silently
omitted; "Generalist" gets no badge.

Provenance is a native `title=` attribute on a `tabindex="0"` element (zero JS, D-08).
Nunjucks autoescape (Eleventy default, ON) escapes all interpolated trait/host-bee
strings in HTML, `title=`, and `href=` contexts — no `| safe` is applied to any trait
value.

Purpose: satisfy TRAIT-UI-01/02/03/04 — visible, scannable, provenance-bearing traits.
Output: the four modified templates and a human UAT sign-off.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| species.json strings → HTML | Trait labels, `host_plant_family`, and host-bee names cross into HTML body text, `title=` attributes, and `href=` attributes at template-render time. Source data is the trusted curated Phase 174-01 pipeline; the templates are the render boundary. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-174-04 | Injection (XSS, stored-from-seed) | trait/host-bee strings in HTML body + `title=` | mitigate | Nunjucks autoescape is ON (Eleventy default; `eleventy.config.js` does not override the library). Every `{{ … }}` interpolation of a trait/host-bee value is HTML/attribute-escaped. Acceptance criteria assert NO `| safe` is applied to any trait/host-bee value (only the unrelated `month_histogram | dump | safe` exists). |
| T-174-05 | Injection (href breakout) | host-bee `<a href>` construction | mitigate | `href`s interpolate ONLY `hb.slug` (path-safe `domain.slugify` output) or `hb.genusName` (set only when the name matched a known atlas genus in 174-02 — a clean single token). Unresolved names are typed `'text'` and rendered as escaped `<em>` with no `href`. Arbitrary `host_bees` input cannot reach an attribute breakout; autoescape additionally escapes attribute quotes. |
| T-174-06 | Injection (title= attribute) | provenance `title=` strings | mitigate | `title=` values are template-literal constants selected from the fixed `*_source` enum; the only dynamic part (`host_plant_family` in the diet `title=`) is autoescaped. No raw user input is concatenated unescaped into `title=`. |
| T-174-SC | Tampering | npm/pip/cargo installs | accept (N/A) | No package installs this phase (RESEARCH Package Legitimacy Audit: none). |

All dispositions are low-severity (trusted curated public-domain seed data + static hosting, no
runtime input), but documented per ASVS L1 reflected/stored-content review.
</threat_model>
