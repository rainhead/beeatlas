## Page Structure

Source of truth: `_pages/genus.njk` + `src/styles/taxon-pages.css`.

```
<article class="taxon-page species-index">
  <h1>Species</h1>
  <input type="search" id="species-filter" placeholder="Filter genera and species…" autocomplete="off">
  {%- for family, familyGroup in species.flat | groupby("family") -%}
  <section class="family-section" data-family="{{ family }}">
    <h2>{{ family }}</h2>
    {%- for genus, genusGroup in familyGroup | groupby("genus") -%}
    <div class="genus-row" data-genus="{{ genus }}">
      <a href="/species/{{ genus }}/"><em>{{ genus }}</em></a>
      <ul class="species-list">
        {%- for sp in genusGroup -%}
        <li data-name="{{ sp.scientificName | lower }}">
          <a href="/species/{{ sp.slug }}/"><em>{{ sp.scientificName }}</em></a>
          <span class="count">{{ sp.occurrence_count }} records</span>
        </li>
        {%- endfor -%}
      </ul>
    </div>
    {%- endfor -%}
  </section>
  {%- endfor -%}
</article>
<script type="module" src="/src/entries/species-index.ts"></script>
```

Notes:
- `.species-index` is a new modifier class that can add index-specific rules without conflicting with `.taxon-page` on genus/species pages.
- `data-family`, `data-genus`, `data-name` attributes are the filter JS hook points — no id required.
- Permalink preserves `/species/index.html` (from current `_pages/species.njk`).

## Interaction Contract

### Filter input (IDX-02)

**Element:** `<input type="search" id="species-filter">` rendered by the Nunjucks template; no custom element.

**JS entry:** A new thin module `src/entries/species-index.ts` wires the input. Pattern mirrors `taxon-page.ts`: import CSS, register nothing heavy. No Lit coordinator.

**Behavior:**
1. On `input` event, read `input.value.trim().toLowerCase()`.
2. For each `.family-section`:
   - For each `.genus-row` inside it:
     - For each `li` inside it: toggle `hidden` if `li.dataset.name` does not include the query.
     - Toggle `.genus-row`'s `hidden` if genus name (lowercased) does not include query AND all child `li` are hidden.
   - Toggle `.family-section`'s `hidden` if all child `.genus-row` elements are hidden.
3. When query is empty string, remove all `hidden` attributes (restore full list).
4. No debounce required for the expected list size (~400 species). If measured input lag >16ms on mid-range Android, add 50ms debounce.

**Accessibility:**
- Input has `aria-label="Filter genera and species"` (or visible `<label for="species-filter">`).
- Hidden rows use the HTML `hidden` attribute (not `display:none` via class) so assistive technology also skips them.
- Input `type="search"` provides native clear button on supporting browsers.

**Empty filter state:** When all sections are hidden (query matches nothing), show a `<p id="filter-empty" hidden>No genera or species match "<span id="filter-query"></span>".</p>` element. JS toggles its `hidden` based on whether any `.family-section` is visible.
