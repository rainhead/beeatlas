      - For each `.family-section`: iterate child `.genus-row` elements; for each row, iterate `li[data-name]` and set `li.hidden = !match` where `match = !query || li.dataset.name.includes(query) || row.dataset.genus.toLowerCase().includes(query)`
      - Row is hidden when all its `<li>` are hidden; family section is hidden when all its rows are hidden
      - Empty-state `<p id="filter-empty">` is shown when query is non-empty AND no family section is visible
      - The `<span id="filter-query">` text content is updated to `input.value.trim()` for the empty-state message
    - When query is empty string, the OR-short-circuit `!query` makes every li match → all rows + sections become visible, empty-state stays hidden
    - No debounce (UI-SPEC notes ~400 species is well under any threshold; can be added later if measured lag)
    - Module imports ONLY: `'../index.css'`, `'../styles/taxon-pages.css'`, `'../bee-header.ts'` — no Lit, no SPA modules
