<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| `OccurrenceRow.image_url` → `<img src>` | External URL from iNaturalist pipeline reaches the DOM as an image source |
| `OccurrenceRow.obs_url` → `<a href>` | External URL reaches the DOM as a link href |
| `OccurrenceRow.user_login` / `floralHost` / `scientificName` → DOM text | iNat-sourced strings rendered as text content via Lit interpolation |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-119-08 | Information disclosure | Image rendering of non-CC content | mitigate | License check `row.license.toUpperCase().startsWith('CC')` gates the `<img>` render path; non-CC rows never produce an `<img>` tag. Pipeline already constrains license to known iNat values |
| T-119-09 | Tampering | `obs_url` could in principle be `javascript:` URL | mitigate | Pipeline source is iNaturalist's CSV export which only emits `https://www.inaturalist.org/observations/{id}` URLs; the value reaches the DOM through Lit `${row.obs_url}` interpolation in an `href` attribute. Lit does not auto-strip javascript: schemes — accept this risk because the trust boundary is the pipeline (not user input). The `target="_blank" rel="noopener"` mitigates window.opener cross-origin access |
| T-119-10 | XSS | iNat-sourced text rendered as DOM content | mitigate | Lit `html` tagged template escapes all interpolated strings; no `unsafeHTML` usage in `_renderInatObs` |
| T-119-SC | Tampering | npm/pip/cargo installs | mitigate | No package installs in this plan |
</threat_model>
