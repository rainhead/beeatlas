### Map Point Colors — Source Discrimination (MAP-01)

Three occurrence source arms must be visually distinct on the Mapbox GL map. These are Mapbox layer paint properties, not CSS custom properties.

| Source | `source` value | Point color | Rationale |
|--------|---------------|-------------|-----------|
| Ecdysis specimens | `'ecdysis'` | Recency-tiered grays: `#c8cccd` (thisYear), `#7f8c8d` (lastYear/earlier) — **unchanged** | Existing RECENCY_COLORS; no change |
| WABA samples | `'waba_sample'` | `#c8cccd` / `#7f8c8d` recency-tiered — **unchanged** | Existing merged occurrences layer; WABA samples already mixed in |
| iNat expert obs | `'inat_obs'` | `#e8a020` (amber/gold) | Distinct from grays, visible against Mapbox outdoors basemap, not red (no alarm connotation), not blue (reserved for selection). Approved palette ref: place boundary amber `rgba(220, 130, 30, 0.85)` already in codebase — use a slightly brighter on-map read: `#e8a020`. |

iNat obs point geometry: unclustered circle, `circle-radius: 6`, `circle-stroke-width: 1`, `circle-stroke-color: '#ffffff'` — matches existing `unclustered-point` style dimensions.

iNat obs cluster: same recency-tiered cluster layer can be shared if source contains all arms, OR a separate `inat-obs` source+layer set may be added. Either way, clusters containing any iNat obs should display with the amber color when no recency-tiered ecdysis/waba points are present; otherwise fall through to the existing recency color expression. Implementation detail left to executor — this spec mandates only that `inat_obs` points are visually amber and distinguishable from gray ecdysis/waba points.

Selection ring color: `#f1c40f` (yellow) — **unchanged**, applies to all sources.
Ghost (filtered-out) points: `#aaaaaa` at 0.2 opacity — **unchanged**, applies to all sources.

## Source Toggle Controls — Filter Panel (MAP-02)

Location: within the existing `bee-pane` filter panel, added as a new "Sources" row in the filter section. Follow the existing `filter-row` + `row-icon` + `input-group` pattern.

Control type: three independent checkboxes (one per source arm). All three default to checked (all sources visible). Unchecking a source hides its points immediately.

Labels:
- "Ecdysis specimens" (source = `'ecdysis'`)
- "WABA samples" (source = `'waba_sample'`)
- "iNat expert obs" (source = `'inat_obs'`)

Visual style: `<label>` + `<input type="checkbox">` elements inline, no custom styling beyond existing token use. Checked state uses browser default + `accent-color: var(--accent)` on the input for the green brand tick.

Row icon: a simple layers or filter icon (inline SVG, 16×16, matching the existing `row-icon` opacity 0.6 pattern).

## URL State — Source Filter (MAP-03)

New URL parameter: `src`

Encoding: comma-separated list of **hidden** (unchecked) source arms. Absence of `src` param means all sources visible (default state).

Examples:
- All visible: `src` param absent
- Only Ecdysis hidden: `src=ecdysis`
- Ecdysis and WABA hidden: `src=ecdysis,waba_sample`
- All hidden: `src=ecdysis,waba_sample,inat_obs`

Implementation follows existing `buildParams` / `parseParams` pattern in `src/url-state.ts`. The `UiState` interface gains optional `hiddenSources?: Set<'ecdysis' | 'waba_sample' | 'inat_obs'>`.

## Occurrence Detail View — iNat Expert Obs (DET-01)

When a clicked occurrence has `source === 'inat_obs'`, `bee-occurrence-detail` renders a dedicated card using the existing `.panel-content.sample-dot-detail` layout pattern (same as `_renderSampleOnly` / `_renderProvisional`).

Required fields and rendering:

| Field | Data column | Render |
|-------|-------------|--------|
| Observed date | `date` | `formatRomanDate(row.date)` — same as existing, `.event-date` class (serif, weight 600) |
| Observer login | `user_login` | `.event-observer` class (0.8rem, `var(--text-muted)`) |
| Floral host | `floral_host` | Italic `<em>` in `.event-host` class; omit element entirely when null |
| Image | `image_url` + `license` | Show only when `license` is CC-licensed (any `CC-BY*` or `CC0`); render as `<img>` 100% width, max-height 200px, object-fit cover, border-radius 4px. Omit when null or non-CC license. |
| iNat link | `obs_url` | `<a href="${obs_url}" target="_blank" rel="noopener">View on iNaturalist</a>` — `.event-inat` class (0.85rem) |

CC-license check: `license` value starts with `CC` (case-insensitive). Licenses in the pipeline include values like `CC BY`, `CC BY-NC`, `CC0`, `CC BY-SA` etc. A non-CC value (e.g., null, `ARR`, `C`) means no image shown.

Image alt text: `"Photo of {scientificName} by {user_login} on iNaturalist"`.

Empty/null states:
- If `user_login` is null: omit observer line.
- If `floral_host` is null: omit host line entirely (no "no host" placeholder — cleaner than existing behavior; iNat obs are not WABA samples, host is optional metadata).
- If `obs_url` is null: omit link (should not occur in practice; non-null enforced by pipeline).

## Interaction Contracts

### Source toggle behavior
- Toggling a source checkbox dispatches `checklist-layer-changed`-style event upward (or extends `filter-changed`) so `bee-atlas` can update `_hiddenSources` state and push URL.
- `bee-map` receives the hidden sources set as a property and applies a Mapbox filter expression excluding features whose `source` property is in the hidden set.
- Response is synchronous (same frame): no async query needed — filtering is done via Mapbox `setFilter`.

### iNat obs click behavior
- iNat obs points participate in the existing `map-click-occurrence` / `_handlePointClick` flow. No new interaction pattern.
- `bee-occurrence-detail` receives the clicked `OccurrenceRow` (which will have `source === 'inat_obs'`). The existing `render()` dispatch must be extended to check `source` and route to a new `_renderInatObs(row)` method.
- Existing `isSpecimenBacked` / `isProvisional` predicates already exclude iNat obs; they fall through to a `nonSpecimen` rendering branch. The executor must add a third condition for `source === 'inat_obs'` alongside the existing `isProvisional` check.

### URL restore
- On page load, `parseParams` reads `src` param and restores `hiddenSources`.
- `bee-atlas.firstUpdated` passes `hiddenSources` to `bee-pane` (for checkbox init) and to `bee-map` (for filter application).
- `bee-atlas._buildCurrentParams` includes `src` in URL output when any source is hidden.
