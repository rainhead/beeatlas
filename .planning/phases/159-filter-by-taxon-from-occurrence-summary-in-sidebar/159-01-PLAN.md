<objective>
Add a one-click taxon-filter entry point inside the sidebar occurrence list
(`src/bee-occurrence-detail.ts`). Clicking a taxon name applies the existing
taxon filter to the map, saving the filter-panel round-trip. The external record
link that previously wrapped the taxon name is demoted to a small icon link so
every external destination stays reachable.

This is a new *entry point* into the existing filter — not new filter machinery.
The click builds a `FilterChangedEvent` from the current `filterState` (swapping
only `taxonId` + `taxonDisplayName`) and dispatches it upward with
`bubbles: true, composed: true`; `<bee-atlas>`'s existing `@filter-changed`
listener applies it through the same path the filter panel already uses.

Purpose: implements ROADMAP Phase 159 goal + CONTEXT decisions D-01..D-08. No
SQL changes, no new FilterState fields, no map-layer changes, no bee-atlas
changes.
Output: modified `bee-occurrence-detail.ts` (+ one-line `bee-pane.ts` thread) and
expanded source-text tests in two test files.
</objective>
