<objective>
Implement the server-side iNaturalist OAuth2 authorization-code exchange WITH PKCE (D-01/D-02) as
pure, unit-testable functions in `api/oauth.py`, using plain `requests` (already a dependency) — no
OAuth client library. The browser only ever carries the one-time `code`; the client_secret + PKCE
verifier stay server-side; after fetching identity from `/v1/users/me` the iNat access token and
24h JWT are discarded (D-03) and BeeAtlas mints its own session (in 178-05/178-06).

Purpose: WRITE-02 — server-derived identity, never trusting client-supplied identity, no secret in
the client bundle, no iNat token persisted.
Output: `api/oauth.py` (PKCE + exchange + identity) + fully mocked tests. No routes here (178-06 wires them).

Live-verification note (D-02): implement per the RESEARCH.md Pattern-1 example, which was
grounded against the live iNat endpoints and the official `inaturalistjs` client. Do NOT read
`~/dev/inaturalist/`. Carry the plain (no-PKCE) exchange as a one-line fallback if live testing in
178-08 shows Doorkeeper rejects `code_challenge` for a confidential client.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → /auth/callback | untrusted one-time `code` + `state` cross here |
| API → iNaturalist (Doorkeeper) | server-to-server; client_secret + PKCE verifier used here, never exposed |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-178-05 | Spoofing | OAuth callback identity-linking (CSRF-on-callback) | mitigate | `state` generated at /auth/login, verified exact-match on callback (wired in 178-06); authorize_url includes state |
| T-178-06 | Tampering / Spoofing | open redirect via attacker redirect_uri | mitigate | redirect_uri pinned to `https://api.beeatlas.net/auth/callback`; Doorkeeper rejects any non-registered value |
| T-178-07 | Information Disclosure | iNat access token / JWT leakage | mitigate | tokens are function-local only; fetch_identity returns identity dict only; no persistence/logging (D-03) |
| T-178-08 | Elevation of Privilege | over-broad OAuth scope | accept | iNat has no true identity-only scope (Pitfall 1) — mitigated behaviorally: never call write endpoints, discard token; documented so WRITE-03 "minimal scope" is not read as unmet |
</threat_model>
