<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| static bundle (public) → users' browsers | the bundle ships to everyone; must contain no secret/token |
| beeatlas.net JS → api.beeatlas.net | cross-origin credentialed fetch; cookie is HttpOnly (JS cannot read it) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-178-19 | Information Disclosure | secret/token in the client bundle | mitigate | auth-client holds only the public API base; grep gate asserts no `client_secret` in src/; the live network/localStorage proof is the 178-09 UAT |
| T-178-20 | Information Disclosure | session cookie readable by page JS | mitigate | cookie is HttpOnly (set server-side in 178-05/06); the UI reads identity only via the whoami endpoint, never the cookie |
| T-178-21 | Denial of Service | API down blocks page render | accept | fetchWhoami is non-blocking and resolves to `{authenticated:false}` on error; read path stays static |
</threat_model>
