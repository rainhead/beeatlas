<objective>
Build the security core: the app session mechanism (D-04), the users upsert (D-08), and the write
authorization gate (D-05/D-07/D-09 + CSRF). All are pure/store-touching modules with no HTTP routes
(178-06 wires them). This is the auditable, decoupled seam every authenticated request passes through.

- `api/session.py` — `itsdangerous.URLSafeTimedSerializer` signed cookie (NOT PyJWT — avoids the
  alg-confusion class), `HttpOnly`+`Secure`+`SameSite=Strict`, host-only, `max_age` expiry.
- `api/users.py` — `upsert_user(login, inat_user_id)` mints the internal id on first login, returns
  the existing internal id thereafter, writing to the 178-02 `users` table.
- `api/auth.py` — `require_session` / `require_author` decorators: verify the cookie, RE-READ the
  committed allowlist every request (revocation, D-05), derive author server-side (reject forged
  client-supplied author, D-07), enforce a server-side Origin allow-list (authoritative CSRF gate),
  and refuse writes unless the WRITE-04 launch gate is enabled.

Purpose: WRITE-02 (own session), WRITE-03 (allowlist authz + CSRF/origin), WRITE-04 (launch gate).
Output: three modules + their tests. No routes.

Serving note (D-04/D-17): the stateless signed cookie needs no server-side session store — originally
motivated by mod_fcgid's ephemeral workers, it remains the right call under Waitress (no per-process
session affinity, trivially restartable). This plan is otherwise unaffected by the serving pivot.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser (untrusted) → write endpoint | ambient cookie + request body cross here; author/role must be server-derived |
| committed allowlist TOML → authz decision | git-audited source of truth, re-read per write for revocation |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-178-09 | Spoofing | forged-author write (client-supplied author_id) | mitigate | author derived from the signed session on flask.g; request body author fields ignored (D-07) |
| T-178-10 | Tampering / EoP | algorithm-confusion / alg:none on the session token | mitigate | itsdangerous single fixed HMAC scheme (no alg field), never PyJWT |
| T-178-11 | Spoofing / Tampering | CSRF forged cross-site POST with ambient cookie | mitigate | server-side Origin allow-list (authoritative) + SameSite=Strict cookie (defense-in-depth) |
| T-178-12 | EoP | stale authorization after allowlist removal | mitigate | per-request allowlist recheck (D-05); the cookie's baked role is never used for the authz decision |
| T-178-13 | Information Disclosure | session cookie theft via XSS | mitigate | HttpOnly + Secure + host-only cookie (from 178-05 Task 1 constants) |
| T-178-14 | EoP | writes opened before backup proven | mitigate | require_author refuses writes (503) unless `writes_enabled` launch gate is set (WRITE-04) |
</threat_model>
