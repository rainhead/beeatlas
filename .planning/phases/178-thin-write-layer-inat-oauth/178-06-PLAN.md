  <behavior>
    - `GET /auth/login`: mint a fresh `state` + PKCE pair, stash `{state, verifier, return_to}` in a
      short-lived signed flow cookie (session serializer with a distinct salt/name), and 302 to
      `authorize_url(...)`. `return_to` is validated to an allowed origin (default https://beeatlas.net).
    - `GET /auth/callback`: reject (400) if the returned `state` does not exact-match the flow
      cookie; else `exchange_code` (with the stored verifier) → `fetch_identity` → discard tokens →
      `upsert_user(login, inat_id)` → read role from the committed allowlist → `mint_cookie` →
      Set-Cookie with COOKIE_KWARGS (HttpOnly/Secure/SameSite=Strict/host-only) → 302 to the
      validated `return_to` on beeatlas.net.
    - `GET /auth/whoami`: anonymous-friendly — returns 200 JSON `{authenticated: false}` with no/invalid
      cookie, or `{authenticated: true, login, role, is_author}` with a valid session. CORS-enabled.
    - `POST /auth/logout`: Origin-checked; clears the session cookie (empty value, immediate expiry); 200.
    - `POST /api/write-check`: `@require_author` no-op returning the server-derived identity
      `{uid, login, role}`; explicitly ignores any body author field. This is the WRITE-03 test target.
    - CORS: flask-cors restricted to ALLOWED_ORIGINS (https://beeatlas.net, https://www.beeatlas.net)
      with `supports_credentials=True`; never `*` with credentials.
    - `require_real_secrets()` is called inside the OAuth routes (not at import) so a missing secret
      fails the request loudly but never breaks import/tests.
  </behavior>
