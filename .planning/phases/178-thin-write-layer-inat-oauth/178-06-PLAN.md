<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| beeatlas.net JS → api.beeatlas.net | cross-origin, same-site; CORS + credentials cookie cross here |
| Apache (TLS term) → Waitress on loopback | reverse-proxy hop; Waitress is reachable ONLY via Apache, and trusts exactly one X-Forwarded hop |
| unauthenticated internet → any route | Waitress serves these behind Apache; error responses must never leak a traceback |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-178-15 | Information Disclosure | stack-trace leak on unhandled exception (Waitress serving) | mitigate | `app.debug=False` + a generic error handler returning no traceback (Pitfall 3 restated for Waitress; Waitress renders no cgitb page) |
| T-178-16 | Information Disclosure | CORS misconfig letting arbitrary origins read authed responses | mitigate | flask-cors explicit origin allow-list + supports_credentials; never wildcard+credentials (Pattern 3) |
| T-178-17 | Spoofing | OAuth callback state-fixation | mitigate | flow cookie stores `state`; callback rejects any mismatch before code exchange |
| T-178-18 | Tampering | open redirect via `return_to` | mitigate | `return_to` validated against ALLOWED_ORIGINS before the post-login 302 |
| T-178-25 | Spoofing / Tampering | forged X-Forwarded-* / direct-to-Waitress bypass | mitigate | ProxyFix trusts exactly ONE hop (x_for/x_proto/x_host=1); serve.py binds Waitress to 127.0.0.1 only (178-06 Task 2), unreachable except through Apache (D-17) |
| T-178-SC | Tampering | pip installs | mitigate | reference the 178-01 legitimacy gate (waitress + flask-cors); no new installs in this plan |
</threat_model>
