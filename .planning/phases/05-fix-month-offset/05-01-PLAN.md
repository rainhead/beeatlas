<objective>
Remove the `+ 1` month offset bug from parquet.ts so all specimen months are correct throughout the app.

Purpose: The Parquet file stores DarwinCore 1-indexed months (1–12). An earlier commit added `+ 1` when months were 0-indexed, but the pipeline was later refactored to emit 1-indexed values. The `+ 1` was never removed, causing feature months to be 2–13. All month-dependent subsystems (filter checkboxes, sidebar date display, recency tiers) are affected. January/December data is unreachable.

Output: `frontend/src/parquet.ts` with the single-line fix applied; human verification confirming correct behavior for January and December specimens.
</objective>
