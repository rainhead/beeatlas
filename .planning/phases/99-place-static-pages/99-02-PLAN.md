<!-- Concrete contracts the four new files must satisfy. -->

Eleventy data cascade (from _data/species.js convention):
- A file at `_data/places.js` exporting `export default { placesArray }` makes the `places` global available in all Nunjucks templates with field accessor `places.placesArray` (the global name is the file basename, the property name is the export key)
- The module is evaluated once per build (and on every HMR cycle in dev); reading parquet here is forbidden per Pitfall #8

places.json field contract (confirmed from disk):
- Array of `{ slug, name, land_owner, specimen_count, sample_count }` objects
- slug values are `[a-z0-9-]+` (validated by Phase 97); current seed slugs: rattlesnake-ledge, tiger-mountain
- specimen_count is 0 for both seed entries — the SVG `<img>` must be guarded

Phase 98 pipeline output (canonical SVG path):
- `public/data/place-maps/{slug}.svg` (SINGULAR "map", no trailing s)
- Templates reference these as `/data/place-maps/{slug}.svg` after Vite publicDir copy
- NOTE: CONTEXT.md D-07 says "places-maps/" (plural). That is a typo. Use the canonical singular path. The Plan-01 build-output test asserts neither "place-maps" nor "places-maps" appears when specimen_count=0, so the typo cannot reintroduce itself silently.

Default layout (`_layouts/default.njk`):
- Wraps page content in `<bee-header>` + `<main>{{ content | safe }}</main>`
- Does NOT inject extra `<link>` or `<script>` into `<head>` for child templates — CSS must be referenced from inside the template content

base.njk (`_layouts/base.njk`):
- Provides `<head>` with `<title>{{ title or "BeeAtlas" }}</title>` — the index template's static `title` and the detail template's `eleventyComputed.title` both populate this

Vite MPA behavior:
- Scans the entire generated HTML for `<link rel="stylesheet" href="...">` and `<script type="module" src="...">` tags regardless of where they appear in the document
- A `<link rel="stylesheet" href="/src/styles/places.css">` placed at the top of template content is rewritten to a hashed `/assets/places-{hash}.css` URL in the built output
- No JS entry is required for CSS-only processing — D-09 compliance is preserved
