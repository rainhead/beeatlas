## Typography

All sizes derived from `src/index.css` base (`font-size: 16px browser default`, `line-height: 1.5`, `font-weight: 400`). The project uses unitless rem/em relative to browser default 16px.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px (1rem) | 400 | 1.5 |
| Label / metadata | 13.6px (0.85rem) | 400 | 1.5 |
| Attribution / small | 12px (0.75rem) | 400 | 1.5 |
| Heading (h1) | browser default h1 (~2rem = 32px) | 700 (browser default bold) | 1.2 |

Source: taxon-pages.css uses `font-size: 0.85rem` for `.metadata`, `.breadcrumb`, `.species-list .count`; `font-size: 0.75rem` for `.attribution`. No additional weights are declared — system-ui is used at 400 and 700 only.

## Page Inventory

### Page 1: `/places.html` — Index

**Template:** `_pages/places.njk`
**Layout:** `default.njk`
**CSS:** `src/styles/places.css` (new, imported in template head via link tag following taxon-pages.css import pattern)

**Structure** (following species.njk, per D-04):

```
<article class="places-page places-index">
  <h1>Places</h1>
  <ul class="places-list">
    <li> [for each place]
      <a href="/places/{{ place.slug }}.html">{{ place.name }}</a>
      <span class="owner">{{ place.land_owner }}</span>
      <span class="count">{{ place.specimen_count }} specimens</span>
    </li>
  </ul>
</article>
```

- List items use `display: flex; align-items: baseline; gap: 0.5rem` matching `.species-list li` pattern
- Place name is the link text; owner and count are right-aligned metadata spans
- No JS — no filter input (D-09; unlike species index which has a search input, places list is small enough to not need client-side filtering)
- No grouping (places are not taxonomically nested; flat list in order from places.json)
- `<title>`: `Places — BeeAtlas` (static, not computed)

### Page 2: `/places/{slug}.html` — Detail

**Template:** `_pages/place-detail.njk`
**Layout:** `default.njk`
**CSS:** `src/styles/places.css`

**Structure** (following species-detail.njk, per D-05):

```
<article class="places-page place-detail">
  <h1>{{ place.name }}</h1>
  <div class="media-grid">
    [if place.specimen_count > 0]
      <img loading="lazy"
           src="/data/places-maps/{{ place.slug }}.svg"
           alt="Occurrence map for {{ place.name }}">
    [endif]
  </div>
  <p class="metadata">
    {{ place.specimen_count }} specimens · {{ place.land_owner }}
  </p>
  <a href="/?place={{ place.slug }}">View occurrences on the atlas →</a>
</article>
```

- No photo section (D-05: places have no photos)
- No breadcrumb (places are not hierarchically nested)
- SVG map: only rendered when `place.specimen_count > 0` (D-07)
- SVG map aspect ratio: 15:8 (600×320 viewBox, matching species-maps) — confirmed by D-07 referencing species_maps.py pattern
- `media-grid` is a single-cell grid when only the SVG is present (no photo column); at `min-width: 768px` the grid activates but collapses gracefully with one column if only SVG present
- `<title>` computed per place using `eleventyComputed`: `"{{ place.name }} — BeeAtlas"` (following genus.njk pattern, per CONTEXT.md Claude's Discretion)

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA (deep-link) | "View occurrences on the atlas →" |
| Index page heading | "Places" |
| Detail page heading | `{{ place.name }}` (data-driven, not copywritten) |
| Metadata line | `{{ place.specimen_count }} specimens · {{ place.land_owner }}` |
| Empty state (no SVG map) | No message shown — SVG block is silently omitted when `specimen_count == 0` (mirrors species-detail.njk behavior) |
| Error state | Not applicable — fully static Nunjucks; no runtime errors visible to user |
| Destructive confirmation | Not applicable — no destructive actions in this phase |

The deep-link copy "View occurrences on the atlas →" is locked from D-06. The trailing `→` (U+2192 rightward arrow) is consistent with species-detail.njk's deep-link pattern.

## Interaction Contract

This phase is fully static — no JavaScript, no interactive elements beyond standard anchor links.

| Interaction | Behavior |
|-------------|----------|
| Click place name on index | Navigate to `/places/{slug}.html` (standard anchor) |
| Click "View occurrences on the atlas →" | Navigate to `/?place={slug}` (standard anchor; Phase 100 activates the filter param) |
| Hover link | Color transitions from `--link: #646cff` to `--link-hover: #535bf2` (CSS `a:hover` rule in index.css) |

No loading states, no client-side state, no event handlers.

## Accessibility Contract

- `<img>` for SVG map has `alt="Occurrence map for {{ place.name }}"` — descriptive and unique per page
- `<h1>` is present on both pages
- `<article>` wraps page content (semantic landmark)
- Link contrast: `--link: #646cff` on `--surface: #fafafa` — 3.03:1 (passes WCAG AA for large text / UI components; acceptable for nav links per existing site convention)
- Metadata text `--text-muted: #666` on `--surface: #fafafa` — 5.74:1 (passes WCAG AA)
- No keyboard traps — static HTML only
