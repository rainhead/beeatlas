## Detail Card: `_renderChecklist`

Source: decisions D-05 through D-09 (all locked).

### Field Order and Rendering Rules

The `_renderChecklist(row: OccurrenceRow)` method wraps a `<div class="panel-content sample-dot-detail">` — identical container to `_renderInatObs` and `_renderSampleOnly`.

Field rendering order (top to bottom):

1. **Taxon name line** — CSS class `inat-id-label`
   - If `row.taxon_id` resolves in `taxonCache` AND `row.verbatim_name` differs from the resolved name:
     `<em>{accepted}</em> <span class="hint">(det. as {verbatim})</span>`
   - If `row.taxon_id` resolves in `taxonCache` AND names match (or no verbatim):
     `<em>{accepted}</em>` alone
   - If `row.taxon_id` is null AND `row.verbatim_name` is non-null:
     `<em>{verbatim}</em>` alone
   - If both null:
     `<span class="hint">No determination</span>`

2. **Collector line** — CSS class `event-observer`
   - `row.recordedBy` if non-null
   - Omit line entirely if null (do not show "unknown collector")

3. **Date line** — CSS class `event-date`, font-family Times New Roman/Georgia
   - Call extended `formatRomanDate(row.date)` — see Typography extension below
   - Renders `15 VI 2019` for full date (`YYYY-MM-DD`, length 10)
   - Renders `2019` for year-only date (`YYYY`, length 4)
   - Renders `VI 2019` for month-precision date (`YYYY-MM`, length 7)
   - Falls back to raw `row.date` string if none of the above match

4. **Locality line** — CSS class `event-host`
   - `row.locality` if non-null and non-empty
   - Omit line entirely if null — no placeholder, no dash

5. **Collapsed-count line** — CSS class `event-count`
   - Render `Represents ${row.collapsed_count} collapsed records` if `row.collapsed_count > 1`
   - Omit entirely if `collapsed_count` is 1, 0, or null

6. **Attribution line** — CSS class `hint` (muted, italic)
   - Always render: `Bartholomew et al. 2024`
   - Renders at bottom; no link (D-06, deferred)

### Edge / Null States

| Condition | Behavior |
|-----------|----------|
| `taxon_id` null, `verbatim_name` null | Show `<span class="hint">No determination</span>` |
| `verbatim_name === canonical_name` | Show accepted name only — no `(det. as …)` annotation |
| `recordedBy` null | Omit collector line — no "unknown" placeholder |
| `locality` null or empty string | Omit locality line completely |
| `collapsed_count` is 1 or null | Omit collapsed-count line |
| `date` is null or empty | Show nothing for date field (omit line) |

### `formatRomanDate` Extension

Extend the existing function in `bee-occurrence-detail.ts` to handle three precision levels:

```typescript
function formatRomanDate(dateStr: string | null): string {
  if (!dateStr) return '';
  if (dateStr.length === 10) {
    // YYYY-MM-DD full precision
    const d = new Date(dateStr + 'T00:00:00');
    if (isNaN(d.getTime())) return dateStr;
    return `${d.getDate()} ${ROMAN_MONTHS[d.getMonth()]} ${d.getFullYear()}`;
  }
  if (dateStr.length === 7) {
    // YYYY-MM month precision
    const [year, month] = dateStr.split('-').map(Number);
    return `${ROMAN_MONTHS[month - 1]} ${year}`;
  }
  if (dateStr.length === 4) {
    // YYYY year-only
    return dateStr;
  }
  return dateStr; // fallback: render as-is
}
```

This extension is backward-compatible — existing callers pass 10-character strings and still get full-precision output. The null guard (`if (!dateStr) return ''`) should also be added to make the function safe for checklist rows where date may be null.
