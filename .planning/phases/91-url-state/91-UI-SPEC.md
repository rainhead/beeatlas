## Scope Note

This phase has minimal visual surface. No new components are introduced. The design contract
covers three interaction states that must be specified precisely for the executor:

1. The sidebar loading state while `_restoreBoundsSelection` awaits `tablesReady` + query
2. The URL param appearance after a rectangle selection
3. The URL param removal on dismiss

All other design tokens (spacing, typography, color, copywriting for non-bounds states) are
inherited from the existing project design system and are not redeclared here.

## Interaction Contract

### 1. URL Param Format (SEL-06)

After a rectangle selection, the URL gains exactly one new parameter:

```
sel=<west>,<south>,<east>,<north>
```

- All four values are WGS84 decimal degrees, formatted to exactly **4 decimal places** (e.g. `sel=-122.3456,47.1234,-122.1234,47.5678`).
- The `sel=` param coexists with all existing filter params (`taxon`, `yr0`, `yr1`, `months`, `counties`, `ecor`, `collectors`, `elev_min`, `elev_max`, `bm`, `x`, `y`, `z`).
- The `o=` param is NOT emitted alongside `sel=`. When `_selectionBounds` is active and `_sidebarOpen` is true, `buildParams` emits `sel=` instead of `o=`.
- `replaceState` is called immediately; `pushState` fires after the existing 500ms debounce (same pattern as map view changes).

Source: CONTEXT.md D-01, D-02; `src/bee-atlas.ts` lines 494–511 (`_pushUrlState`).

### 2. Sidebar Loading State on Restore

When the page loads with a `sel=` param present, the sidebar opens **before** the async query completes:

- `_sidebarOpen = true` is set synchronously at page-load time (same pattern as `_restoreClusterSelection` in `src/bee-atlas.ts:896`).
- While `_selectedOccurrences` is `null`, `bee-sidebar` renders the existing hint paragraph:
  ```
  Click a point on the map to see details.
  ```
  (This is the existing null-occurrences rendering at `src/bee-sidebar.ts:120`.)
- Once `queryOccurrencesByBounds` resolves, `_selectedOccurrences` is assigned and the sidebar re-renders with occurrence rows.
- **No new spinner, skeleton, or progress indicator is introduced.** The existing null-occurrences hint copy serves as the implicit loading placeholder.
- **No change to this copy is required** — its use during an async restore is acceptable given the ephemeral display duration (< 1s typical on warm SQLite).

Source: CONTEXT.md Claude's Discretion note on `_restoreBoundsSelection`; `src/bee-sidebar.ts:118–121`.

### 3. Dismiss / URL Param Removal (SEL-07)

The `sel=` param is cleared from the URL whenever the sidebar is dismissed. Dismiss paths are:

| Path | Trigger | URL Effect |
|------|---------|------------|
| Close button click | `_onClose` | `sel=` removed, `_selectionBounds = null` → `_pushUrlState()` |
| Empty map click | `_onMapClickEmpty` | `sel=` removed, `_selectionBounds = null` → `_pushUrlState()` |
| Filter change | `_onFilterChanged` | `sel=` removed, `_selectionBounds = null` → `_pushUrlState()` |

No visual confirmation or animation is required on dismiss. The URL updates silently (existing `replaceState` behavior).

Source: CONTEXT.md D-03.

### 4. Validation: Silent Drop on Malformed `sel=`

If `sel=` is present but does not parse to 4 finite floats within valid ranges (`west/east ∈ [-180, 180]`, `south/north ∈ [-90, 90]`, `south < north`), the param is silently ignored:

- Sidebar does not open.
- No error message is shown to the user.
- The URL param is left in the address bar (not forcibly removed).

Source: CONTEXT.md Claude's Discretion note on `parseParams` bounds validation.
