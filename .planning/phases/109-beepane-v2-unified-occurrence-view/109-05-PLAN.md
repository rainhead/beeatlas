<task type="auto">
  <name>Task 1: Fix viewport overflow — max-height in bee-atlas.ts and overflow/scroll in bee-pane.ts</name>
  <files>src/bee-atlas.ts, src/bee-pane.ts</files>
  <action>
Two coordinated CSS-only changes that together constrain the list pane to the available vertical space.

In src/bee-atlas.ts, add `max-height: calc(100% - 1em)` to the `.content.pane-list bee-pane` rule (currently lines 100-103). After the fix the rule reads:

  .content.pane-list bee-pane {
    bottom: 0.5em;
    width: 25rem;
    max-height: calc(100% - 1em);
  }

The `calc(100% - 1em)` accounts for the 0.5em top offset of bee-pane plus 0.5em breathing room at the bottom.

In src/bee-pane.ts, make two changes to `static styles`:

1. Add `overflow: hidden` to the `:host` block (lines 113-121). The `:host` block already uses `display: flex; flex-direction: column` so adding `overflow: hidden` clips any child that overflows.

2. The scrollable region between `.sidebar-header` and `.list-pager` is rendered by `_renderListContent()` as a sequence of sibling elements: `.filter-panel`, `.divider`, optional `.selection-banner`, then the occurrence list (`bee-occurrence-detail`, loading div, or hint div). There is no wrapper div around these middle elements.

Add a new CSS rule targeting this scroll area. Rather than restructuring the template, wrap the scrollable middle section by adding a `<div class="list-scroll">` element in `_renderListContent()` around the `.filter-panel`, `.divider`, selection banner, and the occurrence/loading/hint content (everything between the `.sidebar-header` and the `.list-pager`). Then add to static styles:

  .list-scroll {
    flex: 1;
    min-height: 0;
    overflow-y: auto;
  }

The `.list-pager` must remain outside `.list-scroll` so it stays fixed at the bottom. The resulting flex column in `:host` is: `.sidebar-header` (fixed height) → `.list-scroll` (flex: 1, scrolls) → `.list-pager` (fixed height, conditional).

<task type="auto">
  <name>Task 2: Fix pane auto-close on filter-changed — guard collapse in _onFilterChanged</name>
  <files>src/bee-atlas.ts</files>
  <action>
In `_onFilterChanged` (around line 810 in src/bee-atlas.ts), the line:

  this._paneState = 'collapsed';

fires unconditionally on every `filter-changed` event. When the pane is already in list state and the user picks a taxon from the inline autocomplete, the pane collapses — the opposite of the intended behavior.

Change that single line to a guarded assignment:

  if (this._paneState !== 'list') this._paneState = 'collapsed';

This preserves the existing behavior for all other states (collapsed stays collapsed, table collapses) while keeping the pane open when the filter change originates from within the open list pane.

The `_selectedOccIds = null`, `_selectedCluster = null`, and `_selectionBounds = null` lines immediately above must remain unconditional — clearing selection on filter change is correct in all states.
