## Unsatisfied Requirements (Blockers)

### INFRA-03: GitHub Actions deploy never verified

**Assigned to:** Phase 2 (02-infrastructure)
**Status in REQUIREMENTS.md:** `[ ] Pending` — explicitly not satisfied

Plan 02-02 was a two-task plan. Task 1 (write `deploy.yml`) completed. Task 2 was a human checkpoint requiring:
1. Run `cdk bootstrap` + `cdk deploy`
2. Set GitHub repository variables: `AWS_DEPLOYER_ROLE_ARN`, `S3_BUCKET_NAME`, `CF_DISTRIBUTION_ID`
3. Push to main and verify build + deploy succeeds
4. Visit CloudFront URL and verify site loads

**This checkpoint was never completed.** The SUMMARY itself says "partial — awaiting checkpoint". The `requirements-completed: [INFRA-03]` frontmatter in 02-02-SUMMARY was written prematurely.

**Additional complication:** `deploy.yml` uses `vars.AWS_DEPLOYER_ROLE_ARN` (GitHub Actions **Variables**), but the SUMMARY setup instructions say to set "secrets". These are different GitHub settings tabs. An operator following the SUMMARY would create Secrets, which would cause `vars.*` references to expand to empty strings and the deploy job would fail silently.

---

### FILTER-02: Month filter broken by +1 offset in parquet.ts

**Assigned to:** Phase 4 (04-filtering)
**Status in REQUIREMENTS.md:** `[x]` — incorrectly marked complete

**Root cause:** `/Users/rainhead/dev/beeatlas/frontend/src/parquet.ts` line 34:
```ts
month: Number(obj.month) + 1,
```

Commit `99aba65` ("convert 0-indexed parquet months to 1-indexed") added `+1` when the Parquet was writing 0-indexed months. A subsequent pipeline refactor removed the prefix aliasing and the Parquet now emits DarwinCore 1-indexed months (1–12). The `+1` was never removed.

**Confirmed impact:**
- Parquet month values: 1–12 (verified; min=1, max=12, no zeroes)
- Feature month values after `+1`: 2–13
- Month filter checkboxes (1–12): January checkbox matches February specimens; December checkbox matches November specimens; January and December specimens unreachable
- Sidebar `_formatMonth` uses `new Date(year, month - 1)`: January data displays as "February"; December data (feature month=13) displays as "January" of year+1
- `recencyTier` in `style.ts`: specimens appear 1 month more recent than they are (months 1–11); December resolves correctly due to `temporal-polyfill` constraining month=13 to December

**Why human verification missed it:** Washington bee collection season is roughly April–September. The 04-05 verifier tested with specimens from those months where the offset (e.g., April checkbox matches May data) is visually indistinguishable. January and December have near-zero specimens in the dataset.

**Fix:** Remove `+ 1` at `frontend/src/parquet.ts` line 34:
```ts
month: Number(obj.month),   // DarwinCore months are 1-indexed
```
