## Detailed Findings by Severity

### P1 ‚ÄĒ Major (fix before release)

**[P1] Insufficient contrast: `.hint` / `.event-count` (#888 on white)**
- **Location:** `bee-sidebar.ts` CSS ‚ÄĒ `.hint { color: #888 }`, `.event-count { color: #888 }`
- **Category:** Accessibility
- **Impact:** #888 on #ffffff = 3.95:1. Fails WCAG AA (requires 4.5:1 for normal text). The `.hint` class is used for the primary loading/empty state, and `.event-count` appears in every recent event row ‚ÄĒ both are high-visibility elements.
- **WCAG:** 1.4.3 Contrast (Minimum) ‚ÄĒ Level AA
- **Recommendation:** Use #767676 minimum (4.54:1) or darken to #767676. The `.toggle-btn` inactive state (`color: #888`) has the same failure.
- **Suggested command:** `/colorize`

**[P1] Insufficient contrast: `.inat-missing` (#aaa on white)**
- **Location:** `bee-sidebar.ts` CSS ‚ÄĒ `.inat-missing { color: #aaa }`
- **Category:** Accessibility
- **Impact:** #aaaaaa on #ffffff = 2.33:1. Severe WCAG AA failure. Every specimen without an iNat link shows "iNat: ‚ÄĒ" in this color.
- **WCAG:** 1.4.3 Contrast (Minimum) ‚ÄĒ Level AA
- **Recommendation:** Use #767676 minimum, or de-emphasize via a different technique (smaller size or icon) without relying on light gray.
- **Suggested command:** `/colorize`

**[P1] Toggle buttons missing `aria-pressed`**
- **Location:** `bee-sidebar.ts:662-672` (`_renderToggle`) and `bee-sidebar.ts:584-598` (`_renderBoundaryToggle`)
- **Category:** Accessibility
- **Impact:** Screen reader users have no way to know which layer (Specimens/Samples) or boundary mode (Off/Counties/Ecoregions) is currently active. The visual indicator (green underline) is invisible to AT.
- **WCAG:** 4.1.2 Name, Role, Value ‚ÄĒ Level A
- **Recommendation:** Add `aria-pressed=${this.layerMode === 'specimens'}` / `aria-pressed=${this.boundaryMode === 'counties'}` etc. to each toggle button.
- **Suggested command:** `/harden`

**[P1] Filter inputs labeled by placeholder only**
- **Location:** `bee-sidebar.ts:740-753` (taxon), `:755-772` (year from/to), `:629-651` (county/ecoregion)
- **Category:** Accessibility
- **Impact:** Once a user fills a year input, the placeholder "From year" disappears. Screen reader users have no label. `<input type="number" placeholder="From year">` with no associated `<label>` or `aria-label` is a WCAG failure.
- **WCAG:** 1.3.1 Info and Relationships, 4.1.2 Name, Role, Value ‚ÄĒ Level A
- **Recommendation:** Add `aria-label="From year"` / `aria-label="To year"` / `aria-label="Filter by taxon"` / `aria-label="Filter by county"` / `aria-label="Filter by ecoregion"` to the respective inputs.
- **Suggested command:** `/harden`

**[P1] Chip remove buttons are 24√ó24px**
- **Location:** `bee-sidebar.ts` CSS ‚ÄĒ `.chip-remove { min-width: 24px; min-height: 24px }`
- **Category:** Responsive Design
- **Impact:** WCAG 2.5.5 requires 44√ó44px for touch targets. These remove buttons are the only way to deselect regions from the filter on mobile.
- **WCAG:** 2.5.5 Target Size ‚ÄĒ Level AAA (but 2.5.8 at AA requires 24px minimum ‚ÄĒ which this meets at the boundary, zero margin)
- **Recommendation:** Increase to `min-width: 44px; min-height: 44px`, or use negative margins to expand the touch area with `padding`.
- **Suggested command:** `/adapt`

**[P1] Toggle buttons under 44px height**
- **Location:** `bee-sidebar.ts` CSS ‚ÄĒ `.toggle-btn { padding: 0.6rem 1rem }`. At 1rem = 16px: 0.6√ó2 + 0.9 font ‚Čą 29px.
- **Category:** Responsive Design
- **Impact:** Primary navigation controls (Specimens/Samples, boundary mode) are below the touch target threshold on mobile where this app stacks vertically.
- **Recommendation:** Increase to `padding: 0.85rem 1rem` (‚Čą44px total).
- **Suggested command:** `/adapt`

**[P1] No CSS custom properties ‚ÄĒ all colors hardcoded**
- **Location:** `bee-sidebar.ts` (26 hardcoded color values), `bee-map.ts` CSS (5 values), `index.css` (4 values)
- **Category:** Theming
- **Impact:** Changing any color requires touching multiple files. The border color `#ddd`/`#eee`/`#ccc` appears 9+ times ‚ÄĒ a single border-color change touches all of them separately. Also blocks any future dark mode or theming.
- **Recommendation:** Extract to CSS custom properties on `:root` / `:host`: `--border: #ddd`, `--border-subtle: #eee`, `--text-muted: #666`, `--text-hint: #767676`, `--surface-subtle: #f5f5f5`.
- **Suggested command:** `/extract`

**[P1] `100vh` on body breaks on mobile browsers**
- **Location:** `index.css:58` ‚ÄĒ `body { height: 100vh }`
- **Category:** Responsive Design
- **Impact:** On iOS Safari and Chrome for Android, `100vh` includes the address bar, causing the layout to be taller than the visible viewport. Content gets clipped behind the browser chrome.
- **Recommendation:** Use `height: 100dvh` (dynamic viewport height), with `height: 100vh` as a fallback for older browsers.
- **Suggested command:** `/adapt`

### P2 ‚ÄĒ Minor (fix in next pass)

**[P2] `√ó` taxon clear button has no accessible name**
- **Location:** `bee-sidebar.ts:749` ‚ÄĒ `<button class="taxon-clear-btn" @click=${this._clearTaxon} title="Clear taxon filter">&#x2715;</button>`
- **Category:** Accessibility
- **Impact:** `title` is not a reliable accessible name source. Screen readers may announce "√ó" or nothing. The button acts on the taxon filter, so missing labeling leaves AT users without context.
- **Recommendation:** Add `aria-label="Clear taxon filter"`.
- **Suggested command:** `/harden`

**[P2] Back button `‚Üź Back` announces arrow character**
- **Location:** `bee-sidebar.ts:860` ‚ÄĒ `<button class="back-btn" ...>&#8592; Back</button>`
- **Category:** Accessibility
- **Impact:** Screen reader reads "left arrow Back" ‚ÄĒ the arrow character is announced literally. Minor annoyance.
- **Recommendation:** Wrap the arrow in `<span aria-hidden="true">&#8592;</span>` so only "Back" is announced.
- **Suggested command:** `/harden`

**[P2] Date sort creates O(n log n) Date objects**
- **Location:** `bee-map.ts:459-462`
- **Category:** Performance
- **Impact:** Each comparison call creates two `new Date()` calls. For 1000+ sample features, this generates thousands of short-lived objects per sort. Minor GC pressure.
- **Recommendation:** Pre-compute timestamps: `features.map(f => ({ f, t: new Date(f.get('date')).valueOf() })).sort((a,b) => b.t - a.t).map(x => x.f)`
- **Suggested command:** `/harden`

**[P2] OL CSS loaded via CDN `<link>` inside shadow DOM render template**
- **Location:** `bee-map.ts:623` ‚ÄĒ `<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/ol@v10.8.0/ol.css">`
- **Category:** Performance
- **Impact:** External network dependency for correct map rendering. If CDN is slow/unavailable, map controls render without styles. Also version-pinned independently from the npm `ol` package, risking drift.
- **Recommendation:** Import OL CSS via Vite: `import 'ol/ol.css'` in `bee-map.ts`, which bundles it locally.
- **Suggested command:** `/harden`

**[P2] No sidebar landmark role**
- **Location:** `bee-sidebar.ts` ‚ÄĒ `<bee-sidebar>` has no landmark. Inside, there are no `<nav>`, `<main>`, `<aside>` or equivalent ARIA roles.
- **Category:** Accessibility
- **Impact:** Screen reader users navigating by landmarks cannot jump to the filter panel or map area.
- **Recommendation:** Add `role="complementary" aria-label="Map filters and specimen detail"` to `<bee-sidebar>` host, and `role="main"` to the map container.
- **Suggested command:** `/harden`

**[P2] `color: #888` on `#f5f5f5` ‚ÄĒ verify contrast for uppercase headings**
- **Location:** `bee-sidebar.ts` CSS ‚ÄĒ `.event-date-heading`
- **Category:** Accessibility
- **Impact:** `color: #555` on `#f5f5f5` passes at 6.9:1. However, the uppercase letter-spacing text at `0.8rem` may qualify as large text (3:1 threshold) ‚ÄĒ flagging for explicit verification.
- **Recommendation:** Verify against WCAG large text exception (18pt / 14pt bold). If not large text, `#555` passes at 6.9:1 anyway.
- **Suggested command:** `/colorize`

### P3 ‚ÄĒ Polish

**[P3] System font stack with no custom typography**
- **Location:** `index.css:2`, `bee-sidebar.ts:119` ‚ÄĒ `system-ui, Avenir, Helvetica, Arial, sans-serif`
- **Category:** Anti-Patterns
- **Impact:** No real user impact. Typography is generic but functional. The sidebar separately declares `font-family: system-ui, sans-serif` ‚ÄĒ inconsistent fallback list.
- **Recommendation:** Consolidate to a single font declaration; consider a lightweight web font for heading hierarchy.
- **Suggested command:** `/typeset`

**[P3] Pure white background**
- **Location:** `index.css:8` ‚ÄĒ `background-color: #ffffff`
- **Category:** Anti-Patterns
- **Impact:** Purely cosmetic. Pure white is a tell of un-designed interfaces.
- **Recommendation:** Tint to `#fafafa` or a very subtle warm/cool off-white aligned to the navy header.
- **Suggested command:** `/colorize`

**[P3] `a { text-decoration: inherit }` hides link affordance globally**
- **Location:** `index.css:47`
- **Category:** Accessibility
- **Impact:** Links in running text lose their underline. Low-vision users who adjust color may lose the link cue entirely.
- **Recommendation:** Scope this rule to contexts where it's intentional (header, nav) rather than applying globally.
- **Suggested command:** `/clarify`

**[P3] Dead CSS: `align-content: stretch` on body**
- **Location:** `index.css:55` ‚ÄĒ `body { display: flex; align-content: stretch }`
- **Category:** Performance (dead code)
- **Impact:** None. `align-content` only applies to multi-line flex containers; body is single-column.
- **Recommendation:** Remove.
- **Suggested command:** `/distill`

## Patterns & Systemic Issues

1. **No CSS custom property system** ‚ÄĒ 30+ hardcoded hex values across `bee-sidebar.ts`, `bee-map.ts`, `index.css`. Any color change requires coordinated multi-file edits. Blocks future dark mode or theming.

2. **Form inputs consistently unlabeled** ‚ÄĒ All 6 filter inputs use placeholder as the only label. This is a pattern applied uniformly, not a one-off mistake.

3. **Touch targets consistently undersized** ‚ÄĒ Chip removes (24px), toggle buttons (~29px), action buttons (~25px). Every button in the sidebar falls short of 44px.

---

## Positive Findings

- **Responsive breakpoint is thoughtful**: `max-aspect-ratio: 1` (not a pixel breakpoint) correctly identifies portrait orientation. `50svh` map height uses the right unit for mobile browsers.
- **Region chip remove buttons** have proper `aria-label="Remove ${name}"` ‚ÄĒ good pattern to extend.
- **GitHub icon** has `aria-hidden="true"` with an `aria-label` on the link ‚ÄĒ correctly implemented.
- **`<html lang="en">`** present ‚ÄĒ screen readers will use correct language.
- **Loading and error states** are clearly handled with full overlays.
- **URL state persistence** is robust and correctly handles browser back/forward.
- **`rel="noopener"`** on all external links.
- **DuckDB WASM in a worker thread** keeps the main thread unblocked during data operations.
