# Requirements: Washington Bee Atlas v4.5

**Defined:** 2026-05-29
**Core Value:** Tighten learning cycles for volunteer collectors — surface existing data in ways difficult to achieve without the site; convey liveness and togetherness among participants.

## v4.5 Requirements

### Pre-Work / Contract Cleanup (PWK)

- [ ] **PWK-01**: `test_dbt_diff.py` docstring updated to reflect actual 36-column `occurrences` contract (not stale "30 cols")
- [ ] **PWK-02**: `resolve_taxon_ids.py` extended to cover iNat ARM 3 canonical names not yet in `canonical_to_taxon_id`; count of newly-resolved names documented
- [ ] **PWK-03**: Inactive taxon IDs in current `canonical_to_taxon_id` enumerated and count documented to scope Phase 128 work

### Species Visibility (SPV)

- [x] **SPV-01**: All species with `occurrence_count > 0` appear in `species.json` and the species tree with a valid `specific_epithet`, regardless of WA checklist membership
- [x] **SPV-02**: Each newly-visible species has a static page at `/species/{Genus}/{specificEpithet}/`
- [x] **SPV-03**: Each newly-visible species has an SVG occurrence map generated by `species_maps.py` and uploaded to S3

### Taxon IDs (TID)

- [x] **TID-01**: `species.parquet` includes a non-null `taxon_id` INTEGER column identifying the iNat taxon for every species row
- [ ] **TID-02**: `occurrences.parquet` includes a non-null `taxon_id` INTEGER column for every **identified** occurrence row — each occurrence carries the taxon_id of its finest identified rank (species → genus → subgenus/tribe/family). Truly-unidentified specimens (no taxonomic rank in any field) legitimately remain NULL. *(Re-scoped 2026-06-01: original "every occurrence row" was impossible — ~21k ecdysis specimens have no identification. Genus-rank backfill delegated to Phase 128.)*
- [x] **TID-03**: Species pages link to `https://www.inaturalist.org/taxa/{taxon_id}`

### Inactive Taxon Remapping (ITR)

- [x] **ITR-01**: Pipeline detects canonical names whose resolved iNat taxon ID is inactive; automatically generates remapping entries for those with a known current synonym (via iNat API `current_synonymous_taxon_ids`)
- [x] **ITR-02**: Canonical names with inactive taxon IDs and no resolvable current synonym are written to `data/inactive_unresolved.csv` for manual triage
- [x] **ITR-03**: Automated remappings are applied via the existing synonym JOIN mechanism (`int_combined`, `stg_checklist__species`) — same path as `occurrence_synonyms.csv`
- [x] **ITR-04**: Manual entries in `occurrence_synonyms.csv` take precedence over automated remappings when the same source name appears in both

## Future Requirements

### MPTT / Nested Set

- **MPTT-01**: `ancestors.parquet` exposes ancestor taxon ID chain per species, enabling "all occurrences of any Halictidae" queries without live tree traversal
- **MPTT-02**: Frontend filter supports selecting all occurrences of a tribe, genus, or other supertaxon

## Out of Scope

| Feature | Reason |
|---------|--------|
| DwC-A zip archive as taxonomy source | Disqualified in v4.0: lacks subfamily/tribe/ancestry columns; taxa.csv.gz covers all requirements |
| Frontend subtaxon queries (tribe/genus filter) | Deferred to next milestone (MPTT future); data layer prep only this milestone |
| Ancestor chain (`ancestors.parquet`) | Deferred — user excluded from v4.5 scope |
| `taxon_status` column in occurrences.parquet | Inactive unresolved names go to report file; avoids schema contract expansion |
| Redirecting old species URLs when names change | Complex frontend routing; out of scope for data pipeline milestone |
