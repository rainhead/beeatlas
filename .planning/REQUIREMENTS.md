# Requirements: BeeAtlas v3.2 Species Tab

**Defined:** 2026-05-02
**Core Value:** Tighten learning cycles for volunteer collectors — help them move from "I collected this thing" toward "I know what this thing is and where/when to expect more like it" by surfacing existing occurrence data per species.

## v3.2 Requirements

Each maps to one phase in `ROADMAP.md`. Categories follow the six phases identified in `.planning/research/SUMMARY.md`.

### CHECK — WA State Checklist Ingestion

- [ ] **CHECK-01**: WA bee checklist committed at `data/checklists/wa_bee_checklist.tsv` with provenance recorded in `data/checklists/README.md` (Bartholomew et al. 2024, JHR 97; supplement format documented and dated)
- [ ] **CHECK-02**: `data/checklist_pipeline.py::load_checklist()` reads the CSV and writes `checklist_data.species` table to DuckDB via `CREATE OR REPLACE` (full refresh; no incremental cursor)
- [ ] **CHECK-03**: Checklist table schema: `scientificName`, `family`, `subfamily`, `tribe`, `genus`, `subgenus`, `specific_epithet`, `status` (verified | likely-to-occur — v3.2 populates only `verified`; `likely-to-occur` reserved for v3.3+ when a curated "expected but not yet found" set is introduced), `source_citation`, `notes`
- [ ] **CHECK-04**: `data/run.py` STEPS list includes `("checklist", load_checklist)` between `anti-entropy` and `export`
- [ ] **CHECK-05**: Reconciliation step strips authority strings and subgenus parens from checklist names; consults `data/checklist_synonyms.csv` for known divergences; writes unmatched names to `data/checklist_unmatched.csv` for expert review
- [ ] **CHECK-06**: `canonical_name` derived consistently for both checklist and occurrence rows (lowercase, single-spaced, authority stripped) and used as the join key

### TAX — Taxonomy Gap-Fill

- [ ] **TAX-01**: `data/inaturalist_pipeline.py::enrich_taxon_lineage` extended to walk full ancestor chain and write `inaturalist_data.taxon_lineage_extended(taxon_id, family, subfamily, tribe, genus, subgenus)`
- [ ] **TAX-02**: Tribe / subfamily / subgenus precedence in species export: checklist value first, iNat ancestor value second, NULL otherwise (`COALESCE(checklist, inat)`)
- [ ] **TAX-03**: Subgenus level renders only when populated; "(no subgenus)" placeholder collapses cleanly in the nav tree
- [ ] **TAX-04**: pytest covers known name-disagreement cases (e.g. `Lasioglossum (Dialictus) zonulum` ↔ `Lasioglossum zonulum`)

### LIN — Lineage Coverage Expansion (Phase 77)

- [x] **LIN-01**: `data/resolve_taxon_ids.py::resolve_taxon_ids` queries iNat taxon-search API for every canonical_name in the FULL OUTER union (`checklist_data.species UNION ecdysis_data.occurrences`) that does not yet have a `taxon_id`; results persisted to a bridge table (e.g. `inaturalist_data.canonical_to_taxon_id`)
- [x] **LIN-02**: Resolution is rate-limited to ≤1 req/sec; honors iNat 429/5xx with retry/backoff (mirror Phase 76 pattern in `data/inaturalist_pipeline.py::enrich_taxon_lineage_extended`)
- [x] **LIN-03**: Bridge table is the cache — re-running the pipeline twice in a row produces zero new iNat API calls; a `--refresh-lineage` flag (or equivalent config knob) forces re-resolution
- [x] **LIN-04**: Unresolved names (404 / ambiguous / API error) are written to `data/lineage_unresolved.csv` with `(canonical_name, reason, attempted_at)`; downstream phases can read this list to surface "lineage unknown" cards rather than silent NULLs
- [x] **LIN-05**: After this phase ships, ≥95% of species in the FULL OUTER union have a non-NULL `family` via `taxon_lineage_extended` LEFT JOIN; pytest fixture asserts the threshold against a representative slice

### AGG — Per-Species Aggregations

- [ ] **AGG-01**: `data/species_export.py::export_species_parquet` joins `ecdysis_data.occurrences` (FULL OUTER) with `checklist_data.species` and (LEFT) with `taxon_lineage_extended`; output written to `public/data/species.parquet`
- [ ] **AGG-02**: `species.parquet` columns: `scientificName`, `canonical_name`, `family`, `subfamily`, `tribe`, `genus`, `subgenus`, `specific_epithet`, `on_checklist BOOL`, `status`, `occurrence_count INT`, `specimen_count INT`, `provisional_count INT`, `first_occurrence_date DATE`, `last_occurrence_date DATE`, `month_histogram INT[12]`, `county_count INT`, `ecoregion_count INT`, `slug`
- [ ] **AGG-03**: Slug generated by reusing `data/feeds.py::_slugify` (path-traversal-safe per v2.1) so SVG filenames, URL slugs, and the `slug` column agree
- [ ] **AGG-04**: `public/data/species.json` (flat array of species rows) emitted alongside `species.parquet` for build-time consumption by Eleventy `_data/species.js`
- [ ] **AGG-05**: `public/data/seasonality.json` emitted with per-species × per-county × per-ecoregion-l3 monthly histograms (~6 MB total budget); enables filter-driven seasonality lookup without in-browser KDE
- [ ] **AGG-06**: `scripts/validate-schema.mjs` extended to verify `species.parquet` column set and `species.json` top-level shape (mirrors existing schema gate)
- [ ] **AGG-07**: pytest covers `species_export.py`: synthetic fixture with checklist-only species, occurrence-only species, and matched species; assert FULL OUTER produces correct card counts and zero-record species render

### MAP — Static SVG Occurrence Maps

- [ ] **MAP-01**: `data/species_maps.py::generate_species_maps` writes one `public/data/species-maps/<slug>.svg` per species with `occurrence_count > 0`
- [ ] **MAP-02**: Each SVG uses fixed viewBox (`0 0 600 320`) and inline fill/stroke styling so it renders correctly via plain `<img src=".svg">`
- [ ] **MAP-03**: WA county polygons drawn from `geographies.us_counties` (`state_fips = '53'`) via `ST_AsGeoJSON(ST_SimplifyPreserveTopology(geom, 0.005))`; occurrence points drawn as `<circle>` with consistent radius and fill-opacity
- [ ] **MAP-04**: Coordinates outside the WA viewBox are clipped (no off-canvas points); a non-zero null/clipped count is logged but does not fail the run
- [ ] **MAP-05**: `data/run.py` STEPS list includes `("species-maps", generate_species_maps)` after `species-export` and before `feeds`
- [ ] **MAP-06**: pytest covers SVG well-formedness (parses as XML, contains expected `<circle>` count for fixture data, viewBox matches spec)

### PHOTO — Photo Manifest

- [x] **PHOTO-01**: `content/species-photos.toml` schema: `[species."<scientificName>"]` table with optional `description` (1–3 sentence ID-helpful note) and `[[photos]]` array of `observation_id`, `photo_id`, `url`, `caption`, `attribution`, `license`, `ordering` (and optional `view`, `navigational`)
- [x] **PHOTO-02**: `license` field required for every photo; allowed values `{cc0, cc-by, cc-by-nc, cc-by-sa, cc-by-nc-sa}`; `null`, missing, or `all-rights-reserved` rejected
- [x] **PHOTO-03**: `attribution` field required for non-CC0 photos; rendered verbatim adjacent to the photo (never `innerHTML`)
- [x] **PHOTO-04**: Photo URL stored in TOML at fill time (resolved from iNat API), not constructed at render time; renderer reads URL verbatim
- [x] **PHOTO-05**: `scripts/validate-species.mjs` parses TOML, cross-references `species.json`, validates required fields and license whitelist, exits nonzero on errors; warns (not fails) for unknown species names (allows transitional state)
- [x] **PHOTO-06**: `npm run build` chain runs `validate-species` after `validate-schema` and before `eleventy`
- [x] **PHOTO-07**: `scripts/seed-species-photos.mjs` (one-shot helper, NOT in CI) populates a starter manifest from iNat top-voted research-grade photos at ≤1 req/sec
- [x] **PHOTO-08**: pytest fixture seeds bad licenses and missing attribution; asserts validation rejects them

### PAGE — Page Scaffolding & Architecture

- [ ] **PAGE-01**: `_pages/species.njk` declares `layout: default.njk` and `permalink: /species/index.html`; renders one `<bee-species-card>` per species in light DOM (server-rendered) with named slots for photos, map, and description
- [ ] **PAGE-02**: `_data/species.js` reads `public/data/species.json` (NOT parquet — preserves HMR), builds hierarchical tree (family → subfamily → tribe → genus → subgenus → species), and exports `{ tree, flat, byScientificName }`
- [ ] **PAGE-03**: `_data/photos.js` reads `content/species-photos.toml` via `@iarna/toml`, sorts photos by `ordering`, exposes `Record<scientificName, { description, photos[] }>`
- [ ] **PAGE-04**: `src/entries/species.ts` Vite side-effect entry imports `bee-header` plus `src/species/*` Lit components; referenced via `<script type="module" src="/src/entries/species.ts">` in the page template
- [ ] **PAGE-05**: `<bee-species-page>` is the species-page coordinator owning `_activeTaxonPath`, `_geoFilter`, `_seasonFilter`, and URL state; mirrors the `<bee-atlas>` ARCH-03 invariant
- [ ] **PAGE-06**: `<bee-taxon-nav>`, `<bee-species-grid>`, `<bee-species-card>`, `<bee-species-filter>`, `<seasonality-viz>` are pure presenters — receive state via `@property`, emit `CustomEvent`s, never import from `bee-species-page.ts`
- [ ] **PAGE-07**: `<bee-species-card>` renders `loading="lazy"` on every photo and the SVG occurrence map; `content-visibility: auto` set on each card
- [ ] **PAGE-08**: ARCH-04 invariant test extends `src/tests/arch.test.ts` to assert `src/species/**.ts` does NOT import `mapbox-gl`, `wa-sqlite`, `../sqlite.ts`, `../filter.ts`, `../bee-map.ts`, or `../bee-atlas.ts`
- [ ] **PAGE-09**: `npm run build` produces a separate `species-*.js` chunk distinct from the SPA's `index-*.js`; verified by inspecting `_site/assets/`

### NAV — Hierarchical Taxon Navigation

- [ ] **NAV-01**: `<bee-taxon-nav>` renders a vertical left-rail tree with expand-on-click; root nodes are families, leaves are species cards
- [ ] **NAV-02**: Subgenus level renders only when at least one species under that genus has a populated subgenus value
- [ ] **NAV-03**: Selecting a node updates `_activeTaxonPath` on the coordinator and is reflected in URL state
- [ ] **NAV-04**: Filtered-out branches use mute-not-hide (opacity 0.35) rather than `display: none` so volunteers preserve orientation
- [ ] **NAV-05**: No-JS fallback: Eleventy server-renders the full tree as nested `<details>`/`<ul>` so the page is navigable without the Lit bundle (pnwmoths pattern)

### FILT — Page-Level Filter UX

- [ ] **FILT-01**: `<bee-species-filter>` renders county multi-select, ecoregion-l3 multi-select, and month range inputs sourced from `_data/species.js` option lists
- [ ] **FILT-02**: Filter state encoded in URL with disjoint params from the SPA: `?fam=`, `?subf=`, `?tribe=`, `?gen=`, `?subg=`, `?county=` (CSV), `?ecor=` (CSV), `?m0=`, `?m1=`
- [ ] **FILT-03**: `src/species/url-state.ts` round-trips all filter params; covered by Vitest tests separate from SPA's `src/url-state.ts`
- [ ] **FILT-04**: Cards with zero records under the active filter are muted (opacity 0.35), not hidden; per-card "N records" badge shows the filtered count
- [ ] **FILT-05**: Empty state ("No species match these filters. [Clear filters]") renders when zero cards have nonzero filtered counts
- [ ] **FILT-06**: Breadcrumb pill row above the cards reflects active filter (e.g. "Andrena · April–August · King County"); each pill dismissable
- [ ] **FILT-07**: "Clear filters" button resets all filter state and URL params

### VIZ — Seasonality Visualization

- [ ] **VIZ-01**: `<seasonality-viz>` renders an inline `<svg>` from a 12-element monthly histogram via Lit template; no chart-library runtime dep
- [ ] **VIZ-02**: When `n ≥ 5`, render monthly bars; when `n < 5`, render text fallback ("3 records, May–June")
- [ ] **VIZ-03**: X-axis shows abbreviated month labels (J F M A M J J A S O N D); winter/spring/summer/fall season-band background tint per BeeSearch reference
- [ ] **VIZ-04**: When the page-level geo filter is active, viz consumes pre-binned `seasonality.json` data for the selected county/ecoregion slice (no in-browser KDE)
- [ ] **VIZ-05**: Sample-size star annotation: `*` 20–49, `**` 50–99, `***` 100–999, `****` ≥1000 (BeeSearch convention)

### LINK — SPA Deep-Link Contract

- [ ] **LINK-01**: Per-card "View N occurrences →" button navigates to `/?taxon=<scientificName>&taxonRank=species` via a shared `buildSpaTaxonLink()` helper (NOT `/collection?taxon=...` — the seed example is wrong; verified at `src/url-state.ts:35-89`)
- [ ] **LINK-02**: Vitest round-trip test asserts `buildSpaTaxonLink('Andrena anograe')` produces a URL that the SPA's `parseParams` resolves to `{ taxonName: 'Andrena anograe', taxonRank: 'species' }`
- [ ] **LINK-03**: Genus/family-level deep-links from the nav rail use `taxonRank=genus`/`taxonRank=family` respectively
- [ ] **LINK-04**: SPA URL contract (`taxon` + `taxonRank` both required) documented as a stable interface in `src/url-state.ts` header comment

### PERF — Performance & Hardening

- [ ] **PERF-01**: Species-page Vite chunk size budget enforced in CI: `species-*.js` < 100 KB gzipped
- [ ] **PERF-02**: Lighthouse mobile LCP < 3 s on the largest subgenus page (Andrena, Lasioglossum, or Osmia depending on subgenus distribution)
- [ ] **PERF-03**: All `<img>` tags carry `loading="lazy"`; iNat photo URLs use `medium` (500px) size for hero by default; `square`/`small` available via `srcset` for thumb-strip use
- [ ] **PERF-04**: `scripts/check-photo-availability.mjs` HEADs each manifest photo URL on a weekly cron (NOT every build); writes failed URLs to `data/manifest_drift_report.json`
- [ ] **PERF-05**: Accessibility: every photo has `alt` text (caption or scientific name); nav tree has appropriate `role`/`aria` attributes; keyboard navigation works for tree expand/collapse and filter inputs
- [ ] **PERF-06**: UAT against the seed's stated use cases ("Which species of *Eucera* are present in this ecoregion?", "Which are most likely / frequently collected?") — passes with current data

## Out of Scope (v3.2 hard locks)

| Feature | Reason |
|---------|--------|
| Attribute filters (eye color, ID character, ease of photo ID) | Seed lock — deserves its own design milestone with attribute schema work |
| Species detail pages | Seed lock — cards are self-contained surfaces in v3.2 |
| Identification / dichotomous key UI | Out of scope per seed; deserves its own milestone |
| Build-time iNat photo fetch | Seed lock — causes flaky CI and rate-limit risk |
| Headless-browser SVG capture | Seed lock — adds CI complexity, uses Python `matplotlib` instead |
| Cross-table joins to floral hosts | Seed feature creep |
| Per-volunteer custom checklists | Cold-start risk per `notes/project-goals-liveness-community.md` |
| Photo upload UI on live site | Cold-start; community feature territory |
| Embedding the SPA map inside a card | Performance death; deep-link is the right affordance |
| Manual species-list editing bypassing the checklist | Diverges from authoritative source; introduces drift |
