## Integration Findings

### B-01 — BLOCKER: place-maps/*.svg not uploaded to S3/CDN

**Affects:** PPAGE-03, PPAGE-02
**Location:** `data/nightly.sh` — missing upload step after `species-maps`
**Detail:** `places_maps.py` writes per-place SVG maps to `$EXPORT_DIR/place-maps/{slug}.svg`. `nightly.sh` uploads `species-maps/` but has no corresponding step for `place-maps/`. When any occurrence's coordinates fall inside a seeded polygon, `specimen_count > 0`, and `place-detail.njk` renders `<img src="/data/place-maps/{slug}.svg">` — which will 404 in production because the file never reached the CDN.

**Currently latent:** Both seed polygons (rattlesnake-ledge, tiger-mountain) contain 0 matching occurrences at time of audit. The `specimen_count > 0` guard prevents the broken `<img>` tag from rendering today. Will manifest as soon as pipeline data populates.

**Fix:**
```bash
# In data/nightly.sh, after species-maps upload:
aws --profile "$AWS_PROFILE" s3 cp --recursive --no-progress \
  "$EXPORT_DIR/place-maps/" "s3://$BUCKET/data/place-maps/"
```
And add `/data/place-maps/*` to CloudFront invalidation paths.

### W-01 — WARNING: Mode switch away from Places doesn't clear selectedPlace

**Affects:** PMAP-01, PMAP-04
**Location:** `src/bee-atlas.ts` — `_onBoundaryModeChanged` (line ~1010)
**Detail:** When a user selects a place polygon (setting `selectedPlace='rattlesnake-ledge'`), then clicks the Counties mode button, `_boundaryMode` changes to `'counties'` but `_filterState.selectedPlace` is not cleared. `_pushUrlState` then emits both `bm=counties` and `place=rattlesnake-ledge`. The `placeImplied` logic in `parseParams` overrides `bm=counties` on the next URL round-trip, silently discarding the user's Counties mode selection. Additionally, the place filter SQL clause (`place_slug = 'rattlesnake-ledge'`) remains active while in Counties display mode, ghosting occurrences without visible indication.

---

### W-02 — WARNING: PLC-02 permit field validation not runtime-enforced

**Affects:** PLC-02
**Location:** `data/places_validation.py`
**Detail:** `validate_places()` checks slug format, duplicate slugs, WKT validity, WGS84 bounds, and polygon overlap — but does not validate that each `permits[]` record has `issuing_authority` (required) or `type` (required). A future `content/places.toml` entry with a malformed permit record will load silently without a pipeline error, violating PLC-02.

---

### W-03 — WARNING: run.py docstring lists stale pipeline order

**Affects:** None (documentation only)
**Location:** `data/run.py` — module docstring (line 9)
**Detail:** Docstring reads `places-validation -> dbt-build -> species-export -> species-maps -> feeds`. Actual STEPS omit `places-load`, `topology-postprocess`, `places-export`, and `places-maps`.
